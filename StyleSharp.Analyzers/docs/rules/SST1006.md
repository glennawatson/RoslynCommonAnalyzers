# SST1006 — Preprocessor keywords should not be preceded by a space

| Item | Value |
| --- | --- |
| Category | Spacing |
| Severity | Warning |
| Code fix | Yes (remove space) |

## Cause

A preprocessor keyword is separated from its leading `#` by whitespace (for example `# if`).

## Rule description

A preprocessor keyword immediately follows the `#` with no space, so directives read as a
single token (`#if`, `#region`, `#nullable`, `#pragma`). The rule applies to every directive
keyword, including `#endif`, `#else`, and `#endregion`.

```csharp
// 👎 reported
# if DEBUG

// 👍 keyword touches the hash
#if DEBUG
```

The StyleSharp counterpart of StyleCop's `SA1006`.

## How to fix violations

The code fix removes the whitespace between the `#` and the keyword.
