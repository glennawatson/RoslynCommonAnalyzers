# SST1113 — Comma should be on the same line as the previous parameter

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |

## Cause

A separating comma begins a line instead of following the previous parameter.

## Rule description

A comma follows the parameter it terminates, on the same line, so a leading comma never starts the next line.

```csharp
// 👎 reported
void M(int x
    , int y)

// 👍
void M(
    int x,
    int y)
```

The StyleSharp counterpart of StyleCop's `SA1113`.

## How to fix violations

Move the comma up onto the previous parameter's line.
