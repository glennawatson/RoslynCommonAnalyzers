# SST1203 — Constants should appear before fields

| Item | Value |
| --- | --- |
| Category | Ordering |
| Severity | Warning |
| Code fix | Yes (move) |

## Cause

A `const` field appears after a non-constant field of the same accessibility.

## Rule description

Constant fields appear before non-constant fields within the same accessibility
group.

```csharp
// 👎 reported — the constant follows a field
public class Widget
{
    private int _count;
    private const int Max = 10;
}

// 👍 constants first
public class Widget
{
    private const int Max = 10;
    private int _count;
}
```

The StyleSharp counterpart of StyleCop's `SA1203`.

## How to fix violations

Move the constant ahead of the non-constant fields. The code fix relocates it just
after the last member of an equal-or-earlier rank. The fix is only offered when the
file has no conditional (`#if`/`#elif`/`#else`/`#endif`) directives.
