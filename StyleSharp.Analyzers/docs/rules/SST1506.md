# SST1506 — Element documentation header should not be followed by a blank line

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | Yes (remove blank line) |

## Cause

A blank line separates an element's documentation header from the element it
documents.

## Rule description

A documentation header sits directly above the element it describes.

```csharp
// 👎 reported
/// <summary>Does the work.</summary>

public void M()
{
}

// 👍 header attached to the element
/// <summary>Does the work.</summary>
public void M()
{
}
```

The StyleSharp counterpart of StyleCop's `SA1506`.

## How to fix violations

The code fix removes the blank line(s) between the documentation header and the
element.
