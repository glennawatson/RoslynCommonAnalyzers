# SST1508 — Closing brace should not be preceded by a blank line

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | Yes (remove blank line) |

## Cause

A blank line immediately precedes a closing brace.

## Rule description

A closing brace follows the last statement of its block directly, not after a blank
line.

```csharp
// 👎 reported
private void M()
{
    Work();

}

// 👍 no trailing blank line
private void M()
{
    Work();
}
```

The StyleSharp counterpart of StyleCop's `SA1508`.

## How to fix violations

The code fix removes the blank line(s) directly before the closing brace.
