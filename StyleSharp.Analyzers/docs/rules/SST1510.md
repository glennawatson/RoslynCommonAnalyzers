# SST1510 — Chained statement block should not be preceded by a blank line

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | Yes (remove blank line) |

## Cause

A blank line precedes a chained `else`, `catch`, or `finally` keyword.

## Rule description

A chained clause follows the block it continues directly, with no blank line between
them.

```csharp
// 👎 reported
if (enabled)
{
}

else
{
}

// 👍 chained directly
if (enabled)
{
}
else
{
}
```

The StyleSharp counterpart of StyleCop's `SA1510`.

## How to fix violations

The code fix removes the blank line(s) before the chained keyword.
