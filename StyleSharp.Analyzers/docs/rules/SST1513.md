# SST1513 — Closing brace should be followed by a blank line

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | Yes (insert blank line) |

## Cause

The closing brace of an embedded statement block is directly followed by another
statement, with no blank line between them.

## Rule description

A statement block is separated from the statement that follows it by a blank line.
Closing braces that are followed by a chained keyword (`else`, `catch`, `finally`,
`while`), by another closing brace, by a statement terminator, or by end of file are
exempt. Element-body separation is governed by [SST1516](SST1516.md).

```csharp
// 👎 reported
if (enabled)
{
}
Work();

// 👍 blank line after the block
if (enabled)
{
}

Work();
```

The StyleSharp counterpart of StyleCop's `SA1513`.

## How to fix violations

The code fix inserts a blank line after the closing brace.
