# SST1700 — Extension blocks should not be empty

| Item | Value |
| --- | --- |
| Category | Extensions |
| Severity | Warning |
| Code fix | No |
| Applies to | C# 14 extension blocks |

## Cause

A C# 14 `extension` block declares no members.

## Rule description

An extension block exists to group extension members for a receiver type. An empty block contributes
nothing and is usually a leftover from editing.

```csharp
public static class StringExtensions
{
    // 👎 reported
    extension(string text)
    {
    }

    // 👍
    extension(string text)
    {
        public bool IsBlank => text.Trim().Length == 0;
    }
}
```

This rule has no StyleCop counterpart — it covers the new extension-member syntax introduced in
C# 14. On older compilers the syntax cannot occur, so nothing is reported.

## How to fix violations

Add members to the extension block, or remove it.
