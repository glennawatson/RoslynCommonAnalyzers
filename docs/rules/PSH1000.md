# PSH1000 — Anonymous functions without captures should be static

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | Allocations |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A lambda or anonymous method captures no state, but is not marked `static`.

```csharp
// 👎 reported
System.Func<int, int> square = x => x * x;

// 👍
System.Func<int, int> square = static x => x * x;
```

## Rule description

A capture-free anonymous function can already be cached as a single delegate
instance by the compiler, but nothing stops a later edit from silently capturing
a local, a parameter, or `this` — turning every call site into a closure
allocation. Marking it `static` (C# 9) makes the capture-free contract explicit:
any accidental capture becomes a compile error instead of a hidden allocation.

An anonymous function is reported only when **all** of the following hold:

- it is a lambda or a `delegate` anonymous method without the `static` modifier;
- the file is parsed as C# 9 or later (the modifier does not exist before that);
- it is not converted to `Expression<TDelegate>`, because static anonymous
  functions are illegal in expression trees; and
- flow analysis proves it captures nothing: no locals, no enclosing-method
  parameters, and no `this` (so no instance member access).

A lambda that only uses its **own** parameters is still capture-free and is
reported. A lambda that reads an instance field, calls an instance method, or
references any enclosing local or parameter is **not** reported.

## How to fix violations

Add the `static` modifier to the anonymous function. The provided code fix does
this.
