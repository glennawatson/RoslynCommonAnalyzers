# PSH1002 — Empty finalizers should be removed

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | Allocations |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

> Moved from StyleSharp.Analyzers, where it shipped as SST1434.

## Cause

A type declares a finalizer (`~T()`) whose body contains no statements.

```csharp
// 👎 reported
public class Connection
{
    ~Connection()
    {
    }
}

// 👍
public class Connection
{
}
```

## Rule description

Declaring any finalizer — even an empty one — registers every instance of the type
for finalization. Each instance then survives at least one extra garbage-collection
generation while it waits on the finalization queue, and the finalizer thread pays
for a call that does nothing. Deleting the empty finalizer removes that cost
without changing behavior.

Only a finalizer whose block body has zero statements is reported. A finalizer
containing statements — or an expression body — is never reported, whatever it
does.

## How to fix violations

Delete the finalizer. The provided code fix removes the declaration; Fix All
removes every empty finalizer in the document in one pass.
