# PSH1003 — 'in' parameters should use readonly structs

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | Allocations |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A parameter is passed by readonly reference (`in`, or the C# 12 `ref readonly`
pair) but its struct type is not declared `readonly`.

```csharp
public struct Point
{
    public int X;
    public int Y;

    public int Sum() => X + Y;
}

public static class Geometry
{
    // 👎 reported — Sum() defensively copies the whole struct first
    public static int Measure(in Point point) => point.Sum();
}

// 👍 a readonly struct is safe to access through an 'in' reference
public readonly struct Point
{
    public readonly int X;
    public readonly int Y;

    public int Sum() => X + Y;
}
```

## Rule description

`in` passes a struct by reference to avoid copying it — but the callee's reference
is readonly, and when the struct type is not `readonly` the compiler cannot prove a
member access leaves the value unmodified. To preserve the guarantee it emits a
hidden **defensive copy** of the entire struct before each method call or
non-readonly property access on the parameter, quietly costing more than the
by-value copy `in` was meant to save.

A parameter is reported only when **all** of the following hold:

- it is declared `in` (or `ref readonly`);
- its type is a named struct — primitives, enums, type parameters, and reference
  types are never reported; and
- the struct type is not declared `readonly`.

An `in` parameter of a `readonly struct` never defensively copies, so it is not
reported.

## How to fix violations

There is no code fix: the two correct changes each edit a different declaration,
and neither is safe to apply blindly.

- **Preferred — make the struct `readonly`** (or mark the members accessed
  through the parameter as `readonly` members). The compiler can then bind
  member accesses directly to the reference with no defensive copy. This is only
  possible when you own the struct.
- **Otherwise, drop `in` and pass the struct by value.** One up-front copy is
  cheaper than a defensive copy per member access. Note this changes call sites:
  any caller that explicitly writes `in x` at the argument must remove it.
