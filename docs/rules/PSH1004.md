# PSH1004 - Hoist constant array arguments

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | Allocations |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

An array made only from constant values is created directly inside an argument
list.

```csharp
Validate(new[] { "id", "name", "created" }); // PSH1004
```

## Rule description

An inline array expression allocates a new array every time the containing call is
executed. When every element is a compile-time constant, the array can usually be
stored once and reused by every call.

The rule reports explicit and implicit array creations used as invocation or
object-creation arguments. Empty arrays are handled by PSH1001 instead.

## How to fix violations

Move the constant array to a shared field and pass that field to the call.

```csharp
private static readonly string[] ValidationColumns = ["id", "name", "created"];

Validate(ValidationColumns);
```

Use a defensive copy instead when the callee mutates the array.

## When to suppress

Suppress only when the call intentionally requires a fresh mutable array and the
allocation is acceptable at that call site.
