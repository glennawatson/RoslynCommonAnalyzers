# PSH1005 — Structs should define equality members to avoid boxing comparisons

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | Allocations |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A public or internal struct neither overrides `Equals(object)` nor implements
`IEquatable<T>` of itself.

```csharp
// 👎 reported — comparisons fall back to ValueType.Equals, boxing both operands
public struct Point
{
    public int X;
    public int Y;
}

// 👍 equality members bind without boxing
public readonly record struct Point(int X, int Y);
```

## Rule description

A struct without its own equality members inherits `ValueType.Equals`, so every
`Equals` call **boxes both operands** to `object` — two heap allocations per
comparison — and, unless the runtime can prove the struct is bitwise-comparable,
falls back to reflecting over its fields. This bites silently wherever equality is
implied: `List<T>.Contains`, `Array.IndexOf`, dictionary and set lookups without a
custom comparer, and hand-written `a.Equals(b)` calls.

A struct is reported only when **all** of the following hold:

- it is `public` or `internal` — private nested helper structs are not worth the
  noise;
- it is not a `ref struct` (those cannot box) and not a record struct (those
  synthesize equality members); and
- it neither overrides `Equals(object)` nor implements `System.IEquatable<T>`
  constructed with the struct itself — implementing `IEquatable` of some *other*
  type does not help and is still reported.

`System.IEquatable<T>` is resolved once per compilation.

## How to fix violations

There is no code fix: the members' bodies depend on which fields participate in
equality, which only the author can decide.

- **Preferred — declare it a `record struct`** (ideally `readonly record struct`).
  The compiler synthesizes `IEquatable<T>`, `Equals(object)`, `GetHashCode`, and
  the `==`/`!=` operators over all fields, with no boxing.
- **Otherwise, add the members by hand:** implement `System.IEquatable<T>` with
  the struct itself as the type argument, and override `Equals(object)` and
  `GetHashCode` so all equality paths agree.

```csharp
public struct Point : System.IEquatable<Point>
{
    public int X;
    public int Y;

    public bool Equals(Point other) => X == other.X && Y == other.Y;

    public override bool Equals(object obj) => obj is Point other && Equals(other);

    public override int GetHashCode() => (X, Y).GetHashCode();
}
```
