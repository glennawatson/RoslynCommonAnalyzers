# PSH1006 - Use ConcurrentDictionary factory parameters

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | Allocations |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A `ConcurrentDictionary<TKey, TValue>` factory lambda captures the key variable
that was already passed to `GetOrAdd` or `AddOrUpdate`.

```csharp
return cache.GetOrAdd(key, _ => Create(key)); // PSH1006
```

## Rule description

`ConcurrentDictionary` passes the key into its value factories. Capturing the
outer key forces a closure allocation and keeps the delegate tied to the current
call. Using the lambda's own key parameter lets the delegate body avoid that
capture.

The rule reports `GetOrAdd` and `AddOrUpdate` calls where the first argument is a
key variable and a later factory lambda reads that same outer variable instead of
the factory parameter.

## How to fix violations

Use the key supplied by the factory lambda.

```csharp
return cache.GetOrAdd(key, static currentKey => Create(currentKey));
```

For factories that need extra values, prefer overloads that accept an additional
state argument when the target framework provides them.

## When to suppress

Suppress only when the outer variable is deliberately different from the key
value supplied by the dictionary factory.
