# PSH1007 - Pass large readonly structs by in reference

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | Allocations |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A large readonly struct is passed by value to a method, local function, delegate,
or lambda.

```csharp
static int Score(Snapshot snapshot) => snapshot.Count; // PSH1007
```

## Rule description

Copying a large readonly struct at every call can cost more than passing a
reference. An `in` parameter keeps the readonly contract while avoiding the full
value copy.

The rule is size-gated so small structs and known cheap framework structs stay
quiet. Project configuration can tune the size threshold and exclude additional
types when a by-value call is intentionally cheaper or required by an API shape.

## How to fix violations

Pass the value by `in` where the member is source-controlled and callers can be
updated.

```csharp
static int Score(in Snapshot snapshot) => snapshot.Count;
```

Do not change public API signatures casually. For externally consumed APIs,
weigh the binary/source compatibility impact before applying this rule.

## When to suppress

Suppress when the parameter belongs to a stable public API, when the struct is
known to be cheaper by value on the target runtime, or when calling conventions
make `in` unsuitable.
