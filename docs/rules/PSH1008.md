# PSH1008 — Remove SuppressFinalize calls for finalizer-free types

**Category:** Allocations
**Severity:** Warning
**Code fix:** Removes the statement (supports Fix All)

`GC.SuppressFinalize(this)` is called in a type that can never be registered
for finalization: a sealed class with no finalizer anywhere in its chain, or a
struct. SuppressFinalize only matters for objects the GC has queued for
finalization, so here the call is pure per-dispose overhead.

Unsealed classes are never reported — a derived type may add a finalizer and
rely on the base dispose path suppressing it, which is exactly what the
standard dispose pattern prescribes.

## Example

```csharp
public sealed class C : IDisposable
{
    public void Dispose()
    {
        GC.SuppressFinalize(this); // PSH1008: 'C' is sealed and has no finalizer, so this call does nothing
    }
}
```
