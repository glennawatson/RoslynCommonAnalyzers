# PSH1010 — Clear reference-typed arrays when returning them to the pool

**Category:** Allocations
**Severity:** Warning
**Code fix:** Adds `clearArray: true` (supports Fix All)

`ArrayPool<T>` keeps returned arrays indefinitely. When the elements are
reference types — or structs holding references — a non-cleared return pins
every referenced object graph in memory until the array is rented again and
overwritten. Passing `clearArray: true` releases those references at return
time.

Unmanaged element types (`byte`, `int`, unmanaged structs) are never reported
because nothing is kept alive. Unconstrained type parameters are skipped; a
`class`-constrained parameter is reported. A non-constant `clearArray`
argument is trusted.

## Example

```csharp
ArrayPool<string>.Shared.Return(rented);       // PSH1010: keeps every string reachable
ArrayPool<string>.Shared.Return(rented, true); // clean
ArrayPool<byte>.Shared.Return(rented);         // clean — unmanaged elements
```
