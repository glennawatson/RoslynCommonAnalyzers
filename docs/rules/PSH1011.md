# PSH1011 — Pass state to callbacks through the state-taking overload

**Category:** Allocations
**Severity:** Warning
**Code fix:** None (moving captures into a state argument reshapes the lambda)

A lambda that captures locals or `this` allocates a closure object and a
fresh delegate on every call. APIs that accept a callback-and-state pair —
`ContinueWith`, `CancellationToken.Register`/`UnsafeRegister`,
`ThreadPool.QueueUserWorkItem`, scheduler `Schedule<TState>` overloads —
exist precisely so a `static` lambda can receive the data through the state
argument and the delegate instance can be cached.

Reported only when the invoked member's own overload set has a same-name
sibling that adds a state parameter (typed `object` or a method-level type
parameter) whose callback can receive it, and only when the lambda really
captures something declared outside itself.

## Example

```csharp
task.ContinueWith(t => Log(label));                                // PSH1011
task.ContinueWith(static (t, state) => Log((string)state), label); // preferred
```
