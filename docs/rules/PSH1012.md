# PSH1012 — Compare type parameter values with EqualityComparer<T>.Default

**Category:** Allocations
**Severity:** Warning
**Code fix:** Rewrites the call to `EqualityComparer<T>.Default.Equals` (supports Fix All)

When a type parameter carries no `IEquatable<T>` constraint, `x.Equals(y)`
binds to `Object.Equals(object)` and boxes the argument on every value-type
instantiation; the static `object.Equals(x, y)` boxes both operands.
`EqualityComparer<T>.Default.Equals` compares without boxing and the JIT
devirtualizes it to the type's own equality.

Reference-constrained type parameters never box and are not reported.
Equatable-constrained type parameters bind to `IEquatable<T>.Equals`
directly and are also clean.

## Example

```csharp
bool Same<T>(T current, T next) => current.Equals(next);                             // PSH1012
bool Same<T>(T current, T next) => EqualityComparer<T>.Default.Equals(current, next); // fixed
```
