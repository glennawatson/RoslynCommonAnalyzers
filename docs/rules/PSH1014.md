# PSH1014 — Declare immutable structs as readonly

**Category:** Allocations
**Severity:** Warning
**Code fix:** Adds the `readonly` modifier (supports Fix All)

When a struct is not declared `readonly`, the compiler defensively copies it
before calling its members through `in` parameters, `ref readonly` locals,
and readonly fields — even if every field is already readonly, because the
declaration does not prove immutability. The `readonly` modifier removes
those hidden copies and lets `in` passing do what it promises.

The check is purely syntactic: every instance field must be readonly or
const, every auto-property must be get-only or init-only, and nothing may
reassign or ref-expose `this`. Partial declarations, fixed buffers,
field-like instance events, and record structs with a primary constructor
(whose synthesized properties are settable) are skipped. Requires C# 7.3.

## Example

```csharp
public struct Point            // PSH1014
{
    private readonly int _x;
    public int X => _x;
}

public readonly struct Point   // fixed
```
