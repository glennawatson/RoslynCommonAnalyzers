# PSH1015 — Avoid casting value types through object

**Category:** Allocations
**Severity:** Warning
**Code fix:** Removes the intermediate object cast (supports Fix All)

A cast to `object` followed by an immediate cast back to a value type, like
`(int)(object)dayOfWeek`, allocates a box that is discarded as soon as it is
unboxed. When a direct conversion between the two value types exists,
casting directly performs the conversion with no allocation.

Only concrete value types on both ends are reported. Casts written on type
parameters — `(int)(object)tValue` under a `typeof(T) == typeof(int)` guard —
are the well-known generic specialization pattern whose box the JIT already
elides, and stay clean. User-defined conversions are also skipped because a
direct cast would change semantics.

## Example

```csharp
int number = (int)(object)day; // PSH1015
int number = (int)day;         // fixed
```
