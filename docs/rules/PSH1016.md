# PSH1016 — Test enum flags with bitwise operators

**Category:** Allocations
**Severity:** Warning
**Code fix:** Rewrites the call to a bitwise flag test (supports Fix All)

`Enum.HasFlag` takes its argument as the abstract `System.Enum`, so on
runtimes without the JIT intrinsic (.NET Framework, and any netstandard2.0
library running there) every call boxes both the value and the flag and
dispatches through a virtual method. The bitwise form
`(value & flag) == flag` compiles to a register test on every runtime and is
never slower.

The rewrite repeats the flag operand on both sides of the comparison, so the
fix is only offered when re-evaluating it is safe — identifiers, member
access chains, literals, and bitwise-or combinations of those. A call whose
argument invokes a method keeps the diagnostic without a fix. A combined
argument is parenthesized so precedence is preserved: `value.HasFlag(A | B)`
becomes `(value & (A | B)) == (A | B)`. A directly negated call folds the
negation into the comparison: `!value.HasFlag(A)` becomes
`(value & A) != A`.

## Example

```csharp
if (options.HasFlag(Options.Cache)) { }              // PSH1016
if ((options & Options.Cache) == Options.Cache) { }  // fixed

var missing = !options.HasFlag(Options.Cache);             // PSH1016
var missing = (options & Options.Cache) != Options.Cache;  // fixed
```
