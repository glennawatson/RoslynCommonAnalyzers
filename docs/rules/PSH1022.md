# PSH1022 — Use EventArgs.Empty instead of allocating a new EventArgs

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | Allocations |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A parameterless `new EventArgs()` is constructed where the shared `EventArgs.Empty`
singleton would serve.

```csharp
// 👎 reported
Changed?.Invoke(this, new EventArgs());

// 👍
Changed?.Invoke(this, EventArgs.Empty);
```

## Rule description

`EventArgs` is the base type for event data, and the base type carries no state.
A parameterless `new EventArgs()` therefore produces an object that is
indistinguishable from every other parameterless `EventArgs` — and the runtime
already keeps one such object, `EventArgs.Empty`, for exactly this purpose.

An event raised on a hot path — a change notification, a tick, a progress update —
allocates one `EventArgs` per raise when it constructs its own. Reusing the shared
singleton makes the raise allocation-free, and reads more clearly as "no data
here" than a fresh empty object does.

## What is not reported

**A derived `EventArgs`.** A type that derives from `EventArgs` exists to carry
data, so constructing it is not redundant even when it is written without
arguments. The rule matches only a construction whose bound constructor's
containing type is `System.EventArgs` itself.

```csharp
// Not reported: a different type that may carry state.
Changed?.Invoke(this, new RenamedEventArgs());
```

**A construction with an object initializer or arguments.** Either is doing
something the shared singleton cannot, so neither shape is reported.

## How to fix violations

Take the code fix. The type is written back exactly as you wrote it — a
`new System.EventArgs()` becomes `System.EventArgs.Empty` — and a target-typed
`new()` becomes `EventArgs.Empty`, which the rule has already confirmed resolves
at that position. The replacement is an expression, so it fits wherever the
allocation did.

## Performance

The clean path costs one token check on each construction and binds nothing until
the parameterless `new EventArgs()` shape matches. The rule is gated once per
compilation on `System.EventArgs` exposing its static `Empty` field.
