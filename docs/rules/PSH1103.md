# PSH1103 — Prefer the collection's own count over enumerating

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | Collections |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A parameterless `System.Linq.Enumerable` `Count()` or `Any()` call on a receiver
whose static type already exposes a constant-time `Count` or `Length` property.

```csharp
// 👎 reported
var total = list.Count();
var hasItems = list.Any();
if (!items.Any())
{
    // ...
}

// 👍
var total = list.Count;
var hasItems = list.Count > 0;
if (items.Count == 0)
{
    // ...
}
```

## Rule description

`Enumerable.Count()` and `Enumerable.Any()` allocate an enumerator and dispatch
through `IEnumerable<T>` to answer a question the receiver can answer with a
single property read. When the receiver's static type has a constant-time count,
the property is both faster and clearer.

An invocation is reported only when **all** of the following hold:

- it is the member-access extension form with an empty argument list
  (`x.Count()` / `x.Any()`); the predicate overloads and the static
  `Enumerable.Count(x)` form are never reported;
- it binds to `System.Linq.Enumerable`; and
- the receiver's static type exposes the count in constant time: arrays and
  `string` (via `Length`), a public instance `int` `Count`/`Length` property on
  the type or one of its base types, or — for interface and type-parameter
  receivers — `Count` inherited from `ICollection<T>`/`IReadOnlyCollection<T>`.

A receiver typed as plain `IEnumerable<T>` is **not** reported, because counting
it genuinely requires enumeration. A type that exposes `Count` only through an
explicit interface implementation is also **not** reported, because the property
is not accessible on the static type and the rewrite would not compile.

## How to fix violations

Read the property instead of calling the extension. The provided code fix
rewrites `x.Count()` to `x.Count` (or `x.Length`), `x.Any()` to `x.Count > 0`,
and `!x.Any()` to `x.Count == 0`, parenthesizing the comparison when the
rewritten call sits inside a larger expression.
