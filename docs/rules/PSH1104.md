# PSH1104 — Use TryGetValue instead of ContainsKey followed by an indexer read

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | Collections |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A dictionary indexer read is guarded by `ContainsKey` on the same receiver and key,
so the key is hashed and located twice where `TryGetValue` needs one lookup.

```csharp
// 👎 reported
if (map.ContainsKey(key))
{
    Use(map[key]);
}

// 👍
if (map.TryGetValue(key, out var value))
{
    Use(value);
}
```

## Rule description

Two guard shapes are recognised:

- an `if` statement whose condition is exactly the `ContainsKey` invocation, or whose
  condition is a top-level `&&` chain with the `ContainsKey` invocation as its
  **leftmost** operand (`if (map.ContainsKey(key) && map[key] > 0)`); and
- a ternary whose condition is exactly the `ContainsKey` invocation
  (`map.ContainsKey(key) ? map[key] : fallback`).

A guard is reported only when **all** of the following hold, so merging the lookup
can never change behaviour:

- the receiver and key are **simple expressions** — an identifier, `this`, or a
  member-access chain of identifiers (the key may also be a literal) — so
  duplicating them into `TryGetValue` cannot repeat a side effect;
- every indexer use of the same receiver and key inside the guarded region (the
  rest of the condition and the guarded statement, or the ternary's true branch)
  is a **read**; any write — an assignment or compound-assignment target, `++`/`--`
  operand, `ref`/`out` argument, or deconstruction target — suppresses the rule;
- at least one guarded read exists; and
- `ContainsKey` binds to a one-parameter method whose receiver's static type (or a
  base type or implemented interface, such as `IDictionary<TKey, TValue>` or
  `IReadOnlyDictionary<TKey, TValue>`) exposes an accessible bool-returning
  `TryGetValue(key, out value)`.

A receiver without an accessible `TryGetValue` (for example a keyed wrapper that
only offers `ContainsKey` and an indexer) is **not** reported.

## How to fix violations

Call `TryGetValue` once. The provided code fix rewrites the guard to
`receiver.TryGetValue(key, out var value)` and replaces every guarded read of
`receiver[key]` with the out variable. The variable is named `value` unless an
identifier named `value` already appears anywhere in the enclosing member
declaration, in which case the deterministic fallback name `dictValue` is used.
