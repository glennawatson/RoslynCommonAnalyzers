# PSH1105 — Avoid double lookups on dictionaries and sets

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | Collections |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A membership guard (`ContainsKey`/`Contains`) protects a single mutating call that
already performs the same lookup and reports whether it acted, so the key is hashed
and located twice.

```csharp
// 👎 reported
if (set.Contains(item))
{
    set.Remove(item);
}

// 👍
set.Remove(item);
```

## Rule description

An `if` statement is reported when it has **no else clause**, its body is a
**single** expression-statement invocation (with or without a block), and the guard
and body form one of these pairings:

| Guard | Body | Semantic requirement | Fix |
| --- | --- | --- | --- |
| `dict.ContainsKey(k)` | `dict.Remove(k);` | `Remove` returns `bool` | `dict.Remove(k);` |
| `!dict.ContainsKey(k)` | `dict.Add(k, v);` | receiver's type exposes an accessible bool-returning `TryAdd(k, v)` | `dict.TryAdd(k, v);` |
| `!set.Contains(x)` | `set.Add(x);` | `Add` returns `bool` (`ISet<T>` semantics) | `set.Add(x);` |
| `set.Contains(x)` | `set.Remove(x);` | `Remove` returns `bool` | `set.Remove(x);` |

The guard's receiver and key must be **simple expressions** — an identifier,
`this`, or a member-access chain of identifiers (the key may also be a literal) —
and must be syntactically equivalent to the body's receiver and first argument, so
dropping the guard cannot change which lookup runs or repeat a side effect. The
value argument of the two-argument `Add` may be any expression: it is evaluated
exactly once both before and after the fix.

Pairings whose mutator does not report back stay silent: `List<T>.Add` returns
`void`, so `if (!list.Contains(x)) list.Add(x);` is **not** reported (removing the
guard there would change behaviour). Likewise the guarded two-argument `Add` is
only reported when the receiver's type actually exposes an accessible `TryAdd`.

## How to fix violations

Let the mutating call do the lookup. The provided code fix replaces the whole `if`
statement with the body call (keeping the statement's leading trivia), and rewrites
the guarded two-argument `Add` pairing to a single `TryAdd` call.
