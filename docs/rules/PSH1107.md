# PSH1107 - Filter sequences before sorting them

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | Collections |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

Sorting costs O(n log n); a `Where` applied after an `OrderBy` pays to order elements that are
immediately thrown away. Filtering first sorts only the survivors.

## Example

```csharp
return values.OrderBy(value => value).Where(value => value > 0);
```

Use:

```csharp
return values.Where(value => value > 0).OrderBy(value => value);
```

## Notes

The analyzer uses syntax-first matching and then verifies both calls resolve to `System.Linq.Enumerable`;
`IQueryable` chains are left alone because query providers translate and optimize the whole expression.
The code fix only swaps the `Where` with a single-key `OrderBy`/`OrderByDescending`. A `Where` that
follows a `ThenBy`/`ThenByDescending` (multi-key) chain is still reported, but moving the filter across
the whole chain is left as a manual edit.
