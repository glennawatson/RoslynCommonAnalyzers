# PSH1108 - Chain secondary sorts with ThenBy

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | Collections |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

An `OrderBy` or `OrderByDescending` applied to an already ordered sequence re-sorts everything and
discards the previous ordering. `ThenBy` and `ThenByDescending` refine the existing sort instead of
repeating the O(n log n) work.

## Example

```csharp
return values.OrderBy(value => value.Category).OrderBy(value => value.Name);
```

Use:

```csharp
return values.OrderBy(value => value.Category).ThenBy(value => value.Name);
```

## Notes

The analyzer uses syntax-first matching and then verifies both calls resolve to `System.Linq.Enumerable`;
`IQueryable` chains are left alone because query providers translate and optimize the whole expression.
The code fix renames the second sort to the refining operator with the same direction: `OrderBy`
becomes `ThenBy` and `OrderByDescending` becomes `ThenByDescending`.
