# PSH1110 — Use the collection's own predicate methods over LINQ

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | Collections |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A `FirstOrDefault`, `Any`, or `All` LINQ predicate call on a receiver whose type
ships the equivalent native predicate method: `List<T>` and `ImmutableList<T>`
(`Find`, `Exists`, `TrueForAll`) or a single-dimensional array (`Array.Find`,
`Array.Exists`, `Array.TrueForAll`).

```csharp
// 👎 reported
var first = list.FirstOrDefault(x => x > 1);
var any = list.Any(x => x > 1);
var all = values.All(x => x > 1); // values is int[]

// 👍
var first = list.Find(x => x > 1);
var any = list.Exists(x => x > 1);
var all = System.Array.TrueForAll(values, x => x > 1);
```

## Rule description

The `Enumerable` predicate extensions allocate an enumerator and dispatch every
element through `IEnumerable<T>`. `List<T>`, `ImmutableList<T>`, and arrays ship
purpose-built equivalents — `Find`, `Exists`, and `TrueForAll` — that walk the
backing storage directly without either cost.

An invocation is reported only when **all** of the following hold:

- it is the member-access form with exactly one argument, and that argument is a
  one-parameter lambda;
- it binds to `System.Linq.Enumerable`; and
- the receiver's static type is `List<T>`, `ImmutableList<T>`, or a
  single-dimensional array.

Receivers typed as `IEnumerable<T>` or another interface are **not** reported —
they have no native predicate methods. An `Any` call whose lambda is a pure
equality test on a Contains-bearing receiver is reported as
[PSH1111](PSH1111.md) instead, because a membership test needs no predicate at
all.

## How to fix violations

Call the native method. The provided code fix renames the invoked member for
`List<T>` and `ImmutableList<T>` (`FirstOrDefault` → `Find`, `Any` → `Exists`,
`All` → `TrueForAll`). For arrays it rewrites the call to the static helper —
`values.Any(x => x > 1)` becomes `System.Array.Exists(values, x => x > 1)` — but
only when the receiver is a simple expression (an identifier, `this`, or a
member-access chain over one); a receiver with its own evaluation, such as a
method call, is reported without a fix because the rewrite would move it into
argument position.
