# PSH1111 — Use Contains for membership tests

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | Collections |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

An `Enumerable.Any` call whose lambda is nothing but an equality test against
the element, on a receiver whose static type exposes an accessible instance
`bool Contains(T)`.

```csharp
// 👎 reported
var found = list.Any(x => x == target);
var member = hashSet.Any(x => target == x);

// 👍
var found = list.Contains(target);
var member = hashSet.Contains(target);
```

## Rule description

`Any` with an equality-only predicate allocates an enumerator and invokes a
delegate per element to answer a plain membership question. `Contains` scans the
backing storage directly, and on hash-based sets it is O(1) instead of a full
scan.

An invocation is reported only when **all** of the following hold:

- it is a member-access `Any` call with exactly one argument, and that argument
  is a one-parameter lambda with an expression body;
- the body is an equality (`param == expr` or `expr == param`) where one side is
  exactly the lambda parameter and the other side does not reference the
  parameter at all;
- the call binds to `System.Linq.Enumerable`; and
- the receiver's static type exposes an accessible instance `bool Contains(T)`
  for the element type — directly, on a base type, or via an implemented
  interface such as `ICollection<T>` (`List<T>`, `HashSet<T>`, `SortedSet<T>`,
  and other `ICollection<T>` implementations qualify).

A receiver typed as plain `IEnumerable<T>` is **not** reported, because it has
no `Contains` to call. An equality that mentions the parameter on both sides
(for example `x => x == x`) is not a membership test; on `List<T>` and array
receivers such calls fall through to [PSH1110](PSH1110.md).

This rule is checked before PSH1110, so an equality-only `Any` on `List<T>`
reports `Contains` rather than `Exists`.

## How to fix violations

Pass the compared value to `Contains` and drop the lambda. The provided code fix
rewrites `receiver.Any(x => x == value)` to `receiver.Contains(value)`.
