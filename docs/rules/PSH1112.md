# PSH1112 — Seed the collection through its constructor

**Category:** Collections
**Severity:** Warning
**Code fix:** Merges the bulk add into the initializer (supports Fix All)

Creating an empty `List<T>` or `HashSet<T>` and immediately bulk-adding a
source (`AddRange`, `UnionWith`) grows the backing store through the default
resize schedule. The seeding constructor sizes the store once from the
source's count and copies in a single pass.

Only the adjacent-statement shape is reported: the declaration must
immediately precede the bulk add, the creation must have no arguments and no
initializer, and the source must not mention the new collection. Creations
that already pass a capacity, comparer, or source are left alone.

The code fix emits a spread collection expression (`[.. source]`) when the
language is C# 12+, the declaration is explicitly typed (a spread has no
target type under `var`), and the `performancesharp.prefer_collection_expressions`
option (default `true`) is not disabled; otherwise it seeds through the
constructor.

## Example

```csharp
var list = new List<int>();
list.AddRange(source);          // PSH1112

// Fixed (var declaration):
var list = new List<int>(source);

// Fixed (explicit type, collection expressions preferred):
List<int> list = [.. source];
```

## Configuration

```ini
# Prefer new List<T>(source) over [.. source] in the code fix
performancesharp.prefer_collection_expressions = false
```
