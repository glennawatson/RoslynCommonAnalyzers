# PSH1113 — Sort naturally instead of ordering by the element itself

**Category:** Collections
**Severity:** Warning
**Code fix:** Rewrites to `Order()`/`OrderDescending()` (supports Fix All)

`OrderBy(x => x)` routes every comparison through a key-selector delegate
that returns the element unchanged. `Order()` and `OrderDescending()`
(.NET 7+) compare elements directly, skipping the delegate and the key
array.

The whole rule is gated on `Enumerable.Order` existing in the compilation,
so it costs nothing on older frameworks. A trailing comparer argument is
preserved by the fix. `Queryable` sources are skipped — rewriting expression
trees changes what a provider sees.

## Example

```csharp
values.OrderBy(x => x);           // PSH1113
values.Order();                   // fixed

values.OrderByDescending(x => x); // PSH1113
values.OrderDescending();         // fixed
```
