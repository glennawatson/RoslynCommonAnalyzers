# PSH1114 — Freeze static lookup collections that are never mutated

**Category:** Collections
**Severity:** Disabled by default (opt-in)
**Code fix:** Retypes the field and appends `ToFrozenDictionary`/`ToFrozenSet`

A private static readonly `Dictionary<K,V>` or `HashSet<T>` that is
initialized inline and only ever read can become a `FrozenDictionary`/
`FrozenSet` (.NET 8+), trading one-time construction cost for faster
lookups.

**Freezing is not free.** Construction is markedly slower than building a
regular dictionary, and only genuinely read-heavy tables win — which is why
the rule ships disabled. Opt in per project where lookup tables are hot:

```ini
dotnet_diagnostic.PSH1114.severity = warning
```

The usage scan is a whitelist: reads through `ContainsKey`, `TryGetValue`,
`Contains`, `Count`, an element read, or a foreach keep the field eligible.
Any other mention — mutation, escaping as an argument or return value,
`Keys`/`Values` access — keeps it clean, and partial types are skipped
entirely. The fix carries an explicit comparer argument through so lookup
semantics never change silently, and is only offered where the comparer is
visible (an object-creation initializer).

## Example

```csharp
private static readonly Dictionary<string, int> Lookup = new Dictionary<string, int>
{
    ["one"] = 1,
}; // PSH1114 (when opted in)

private static readonly FrozenDictionary<string, int> Lookup = new Dictionary<string, int>
{
    ["one"] = 1,
}.ToFrozenDictionary(); // fixed
```
