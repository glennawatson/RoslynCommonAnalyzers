# PSH1115 — Insert-if-absent should probe the dictionary once

**Category:** Collections
**Severity:** Warning
**Code fix:** Rewrites the ContainsKey shape to `TryAdd` (supports Fix All)

Guarding an indexer write with `ContainsKey`, or storing into the dictionary
after a failed `TryGetValue`, hashes and probes the key twice. `TryAdd`
inserts and answers in one probe. For the create-then-store shape,
`CollectionsMarshal.GetValueRefOrAddDefault` (.NET 6+) exposes the value
slot so a missing entry can be created and stored with a single lookup; that
shape is reported without a fix because the rewrite introduces a ref local
the surrounding code must adopt.

Note that `TryAdd` evaluates the value expression even when the key already
exists; this matches the framework guidance for the API.

## Example

```csharp
if (!map.ContainsKey(key)) { map[key] = value; } // PSH1115
map.TryAdd(key, value);                          // fixed

if (!map.TryGetValue(key, out var bucket))       // PSH1115 (advisory)
{
    bucket = new List<int>();
    map[key] = bucket;
}
```
