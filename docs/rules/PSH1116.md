# PSH1116 — Probe string-keyed collections with a span through GetAlternateLookup

**Category:** Collections
**Severity:** Warning
**Code fix:** None (the lookup handle is usually hoisted once per scope)

Allocating a string just to look it up — `span.ToString()` or
`new string(span)` as the key argument — throws the copy away right after
hashing. `GetAlternateLookup<ReadOnlySpan<char>>` (.NET 9+) probes
string-keyed dictionaries and sets with the span directly and allocates
nothing.

Reported only when the receiver's type actually exposes
`GetAlternateLookup`. The collection's comparer must support alternate
lookups — the ordinal defaults do; `GetAlternateLookup` throws for
comparers that do not, which is why this stays a report-only advisory.

## Example

```csharp
map.ContainsKey(name.ToString());               // PSH1116

var lookup = map.GetAlternateLookup<ReadOnlySpan<char>>();
lookup.ContainsKey(name);                       // preferred
```
