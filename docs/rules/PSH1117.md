# PSH1117 — Ask the collection whether it is empty

**Category:** Collections
**Severity:** Warning
**Code fix:** Rewrites the comparison to `IsEmpty` (supports Fix All)

Comparing `Count` or `Length` to zero answers emptiness indirectly. On some
collections counting is O(n) or synchronizes — `ConcurrentQueue` locks
segments to count, immutable collections may walk — while `IsEmpty` is a
cheap dedicated check. Reported only when the receiver actually exposes a
bool `IsEmpty` property, so `List<T>` and `string` stay clean.

Both operand orders and the equivalent literal forms are recognized:
`== 0`, `<= 0`, `< 1` map to `IsEmpty`; `!= 0`, `> 0`, `>= 1` map to
`!IsEmpty`.

## Example

```csharp
if (queue.Count == 0)   // PSH1117
if (queue.IsEmpty)      // fixed

if (span.Length > 0)    // PSH1117
if (!span.IsEmpty)      // fixed
```
