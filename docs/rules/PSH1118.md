# PSH1118 — Take the extreme element without sorting

**Category:** Collections
**Severity:** Warning
**Code fix:** Rewrites the chain to `Min()`/`Max()`/`MinBy(k)`/`MaxBy(k)` (supports Fix All)

`OrderBy(...).First()` sorts the entire sequence — an O(n log n) pass over
an allocated buffer of keys and elements — and then keeps exactly one of
them. `Min`, `Max`, `MinBy`, and `MaxBy` scan the sequence once, allocate
no buffer, and return the same element.

## Mapping

| Chain | Replacement |
| --- | --- |
| `OrderBy(k).First()` / `OrderBy(k).FirstOrDefault()` | `MinBy(k)` |
| `OrderBy(k).Last()` / `OrderBy(k).LastOrDefault()` | `MaxBy(k)` |
| `OrderByDescending(k).First()` / `OrderByDescending(k).FirstOrDefault()` | `MaxBy(k)` |
| `OrderByDescending(k).Last()` / `OrderByDescending(k).LastOrDefault()` | `MinBy(k)` |

When the key selector is the identity (`x => x`), the replacement is
`Min()`/`Max()` instead of `MinBy`/`MaxBy`, so the selector delegate
disappears entirely.

## Empty-sequence behavior

The two sides react differently to an empty sequence: `First`/`Last`
always throw, `FirstOrDefault`/`LastOrDefault` never do, while
`Min`/`Max`/`MinBy`/`MaxBy` throw for non-nullable value-typed elements
and return `default` for reference or nullable elements. The rule reports
only when the rewrite keeps the behavior identical:

- `First()`/`Last()` — reported only when the element type is a
  non-nullable value type (both sides throw on empty).
- `FirstOrDefault()`/`LastOrDefault()` — reported only when the element
  type is a reference or nullable type (both sides return `default` on
  empty).

Every other combination stays silent.

## Silent shapes

A `ThenBy`/`ThenByDescending` between the sort and the terminal is never
reported — the secondary sort still decides ties. Terminals carrying a
predicate (`First(x => ...)`) and sorts carrying a comparer
(`OrderBy(k, comparer)`) are also skipped, as are `Queryable` sources.
`MinBy`/`MaxBy` are suggested only where the API exists (.NET 6+); the
identity rewrite to `Min()`/`Max()` needs no gate.

## Example

```csharp
items.OrderBy(x => x.Key).First();   // PSH1118
items.MinBy(x => x.Key);             // fixed

values.OrderBy(x => x).First();      // PSH1118
values.Min();                        // fixed
```
