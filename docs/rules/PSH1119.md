# PSH1119 — Check for elements without counting them all

**Category:** Collections
**Severity:** Warning
**Code fix:** Rewrites the comparison to `Any()` / `!Any()` (supports Fix All)

`Enumerable.Count()` walks the entire sequence to produce a number that the
comparison immediately collapses to a yes or no; `Any()` stops at the first
element. When a `Count()` result is only compared against zero or one to ask
"is there anything?", the full count is wasted work — on an infinite or
expensive sequence it can be arbitrarily wasted work.

Both operand orders and the equivalent literal forms are recognized: `> 0`,
`>= 1`, `!= 0` map to `Any()`; `== 0`, `< 1`, `<= 0` map to `!Any()`. The
predicate overload carries its predicate over: `xs.Count(p) != 0` becomes
`xs.Any(p)`. Comparisons that need the real count (`> 5`, `>= 2`, `== 1`)
stay clean, as do instance `Count()` methods that do not bind to
`System.Linq.Enumerable`.

**Boundary with PSH1103:** for the parameterless form, a receiver whose static
type exposes an accessible constant-time `Count` or `Length` property is never
reported here — that receiver belongs to [PSH1103](PSH1103.md), which replaces
the Enumerable call with the property read. The two rules tile: `list.Count() > 0`
on a counted collection gets PSH1103, `xs.Count() > 0` on a plain
`IEnumerable<T>` gets PSH1119, and no comparison is reported by both. The
predicate form is reported on any receiver — `list.Count(p)` counts every
match no matter what the collection is, while `Any(p)` stops at the first.

## Example

```csharp
if (xs.Count() > 0)             // PSH1119
if (xs.Any())                   // fixed

if (xs.Count(x => x > 2) == 0)  // PSH1119
if (!xs.Any(x => x > 2))        // fixed
```
