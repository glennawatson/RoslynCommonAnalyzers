# PSH1126 — Ask whether an async sequence has elements without counting them

**Category:** Collections
**Severity:** Warning
**Code fix:** Rewrites the comparison to `AnyAsync()` (supports Fix All)

`await query.CountAsync() > 0` asks the provider to count the entire result set
— which, against a database, is a full scan — to learn something `AnyAsync()`
establishes from the first row and then stops.

## Mapping

| Comparison | Replacement |
| --- | --- |
| `await q.CountAsync() > 0` | `await q.AnyAsync()` |
| `await q.CountAsync() >= 1` | `await q.AnyAsync()` |
| `await q.CountAsync() != 0` | `await q.AnyAsync()` |
| `await q.CountAsync() == 0` | `!await q.AnyAsync()` |
| `await q.CountAsync() < 1` | `!await q.AnyAsync()` |
| `await q.CountAsync() <= 0` | `!await q.AnyAsync()` |

Both operand orders are recognized, so `0 < await q.CountAsync()` reads the
same as `await q.CountAsync() > 0`. Any arguments on the counting call — a
predicate, a cancellation token — are carried over unchanged.

## The sibling is proved, not guessed

`CountAsync` and `AnyAsync` are **provider extension methods, not framework
APIs**, so the replacement is never inferred from the name. The reported
`CountAsync` call is bound, and the `AnyAsync` candidates are taken from *its
own* containing type, reduced against the very receiver being counted, and then
required to:

- match the counting call's parameter types position for position, and
- return a boolean awaitable (`Task<bool>` or `ValueTask<bool>`).

The counting call itself must await to an `int`. A provider that can count but
has no `AnyAsync` fitting that receiver is therefore **never reported** — and
the rule works for any async query provider rather than one hard-coded vendor.

## Example

```csharp
if (await context.Orders.Where(o => o.IsOpen).CountAsync() > 0)   // PSH1126
if (await context.Orders.Where(o => o.IsOpen).AnyAsync())         // fixed

if (await context.Orders.CountAsync(token) == 0)                  // PSH1126
if (!await context.Orders.AnyAsync(token))                        // fixed
```

## Silent shapes

- **Comparisons that need the real count** — `> 1`, `>= 2`, `== 5`. Only the
  zero/one forms above ask an emptiness question.
- **A count that is not awaited inside the comparison itself.** The reported
  shape is `await q.CountAsync()` standing as one operand; a count already
  stored in a variable is left alone.
- **A provider without a matching `AnyAsync`** for that receiver.
