# PSH1202 — Append characters as char, not single-character strings

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | Strings |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A single-character plain string literal is passed to `StringBuilder.Append` or
`StringBuilder.Insert`.

```csharp
// 👎 reported
builder.Append("x");
builder.Insert(0, ",");

// 👍
builder.Append('x');
builder.Insert(0, ',');
```

## Rule description

`StringBuilder.Append(string)` and `Insert(int, string)` go through string length
and copy machinery for a one-character argument; the `Append(char)` and
`Insert(int, char)` overloads write the character directly into the buffer. The
overloads behave identically for a single character, so the rewrite is purely a
performance win.

A call is reported when the receiver is a `System.Text.StringBuilder` and the
argument is a plain single-character string literal. Verbatim, raw, and
interpolated strings are not reported, keeping the rule to arguments a char
literal can replace verbatim. `AppendLine` is not reported — it has no char
overload.

The `StringBuilder` type and its `Append(char)`/`Insert(int, char)` overloads are
probed once per compilation, so the rule costs nothing where they are missing.

## How to fix violations

Pass the character instead of a single-character string. The provided code fix
swaps the literal, escaping the char as needed.
