# PSH1203 — Let StringBuilder do the formatting work

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | Strings |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A `StringBuilder.Append` argument allocates an intermediate string the builder
could produce itself: a `string.Format(...)` result, a parameterless
`ToString()` result, or a `Substring(...)` slice.

```csharp
// 👎 reported
builder.Append(string.Format("{0:N2}", value));
builder.Append(value.ToString());
builder.Append(text.Substring(2, 3));
builder.Append(text.Substring(2));

// 👍
builder.AppendFormat("{0:N2}", value);
builder.Append(value);
builder.Append(text, 2, 3);
builder.Append(text, 2, text.Length - 2);
```

## Rule description

Each reported shape hands `StringBuilder` a throwaway string when an overload
exists that writes the same characters straight into the buffer:

- **`Append(string.Format(args...))`** — `AppendFormat` takes the same argument
  list (including the `IFormatProvider`-first forms) and formats directly into
  the buffer. The fix carries the argument list over verbatim.
- **`Append(x.ToString())`** — when `x` is a `bool`, `char`, or numeric
  primitive, the typed `Append` overload formats the value directly; when `x`
  is already a string the `ToString()` call is an identity conversion and is
  simply dropped. `ToString` calls with any arguments are not reported — a
  format or culture argument changes what is appended.
- **`Append(s.Substring(i))` / `Append(s.Substring(i, n))`** —
  `Append(string, int, int)` copies the requested slice without materializing
  it; the single-argument form becomes `Append(s, i, s.Length - i)`. Only
  simple receivers (an identifier, `this`, or a member-access chain over those)
  are reported, because the fix repeats the receiver and must not duplicate
  work or side effects.

A call is reported only when the receiver binds to `System.Text.StringBuilder`
and the invocation is the one-argument `Append(string)` overload. `AppendLine`
is never reported — it has no format overload. The `StringBuilder` type and the
overloads each shape rewrites to are probed once per compilation, so the rule
costs nothing where they are missing.

## How to fix violations

Let the builder do the work: call `AppendFormat`, pass the value to the typed
`Append` overload, or use `Append(string, int, int)` for slices. The provided
code fix performs the rewrite for each reported shape.
