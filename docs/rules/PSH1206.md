# PSH1206 — Do not build strings by concatenation in loops

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | Strings |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A string declared outside a loop is accumulated by concatenation inside it.

```csharp
// 👎 reported
var result = string.Empty;
for (var i = 0; i < count; i++)
{
    result += i;
}
```

## Rule description

Strings are immutable: every `result += item` (or `result = result + item`)
copies the entire accumulated value into a fresh allocation before appending,
so the loop does quadratic work and allocates on every iteration. A
`StringBuilder` appends into a growable buffer and materializes the string
once.

An assignment is reported when **all** of the following hold:

- the target is an identifier or a simple member access, and the assignment is
  `target += ...` or `target = target + ...` (the leftmost operand of the
  concatenation chain is the target itself);
- the assignment sits inside a `for`, `foreach`, `while`, or `do` statement.
  The ancestor walk stops at lambda, anonymous-method, local-function, and
  member boundaries, so a lambda inside a loop is its own scope — but a loop
  inside the lambda still counts;
- the target's type is `string`; and
- the target is declared outside the loop. Fields and parameters always count
  as outside; a local declared inside the loop starts fresh each iteration and
  is not reported.

## How to fix violations

There is no automated fix: introducing a `StringBuilder` is a multi-statement
redesign (declare the builder, rewrite each append, materialize the result
where the string was consumed). Rewrite manually:

```csharp
// before
var csv = string.Empty;
foreach (var item in items)
{
    csv += item + ",";
}

// after
var builder = new System.Text.StringBuilder();
foreach (var item in items)
{
    builder.Append(item).Append(',');
}

var csv = builder.ToString();
```
