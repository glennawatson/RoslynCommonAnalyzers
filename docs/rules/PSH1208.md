# PSH1208 — Encode constant strings with u8 literals

**Category:** Strings
**Severity:** Warning
**Code fix:** Rewrites the call to a u8 literal (supports Fix All)

`Encoding.UTF8.GetBytes` on a constant string re-encodes and heap-allocates
the same bytes on every call. A u8 literal (C# 11+) is encoded once at
compile time and read straight from the assembly's data section. When the
call's result is consumed as `ReadOnlySpan<byte>` the literal stands alone
and the fix removes the allocation entirely; a `byte[]` consumer gets
`"..."u8.ToArray()`, which still skips the runtime encode.

`Encoding.ASCII` receivers are only reported when the constant is pure
ASCII, since the ASCII encoder's replacement fallback would otherwise
produce different bytes. Constants with unpaired surrogates cannot be u8
literals and stay clean.

## Example

```csharp
ReadOnlySpan<byte> prefix = Encoding.UTF8.GetBytes("v1:"); // PSH1208
ReadOnlySpan<byte> prefix = "v1:"u8;                       // fixed
```
