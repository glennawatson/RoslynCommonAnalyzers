# PSH1211 — Pass values directly instead of ToString results

**Category:** Strings
**Severity:** Warning
**Code fix:** Removes the `ToString` call (supports Fix All)

Calling `ToString()` to feed an API that can take the value directly
allocates an intermediate string. Overloads that accept the value —
`Console.Write(int)`, `TextWriter.Write(long)` — and interpolated string
holes format without the throwaway copy; value types with span formatting
write straight into the destination buffer.

An argument is reported when the same method group has an overload identical
except that the slot accepts the receiver's type. Interpolation holes are
reported only when the interpolated string converts to a plain `string`, so
`FormattableString` and custom-handler culture behavior stay untouched.
`StringBuilder` appends are left to PSH1203, and single-hole wrappers like
`$"{x.ToString()}"` are left to PSH1205.

## Example

```csharp
Console.Write(value.ToString());   // PSH1211
Console.Write(value);              // fixed

var text = $"{count.ToString()} items"; // PSH1211
var text = $"{count} items";            // fixed
```
