# PSH1214 — Append the parts, not a concatenated whole

**Category:** Strings
**Severity:** Warning
**Code fix:** Splits the concatenation into chained Append calls (supports Fix All)

`Append(a + b)` concatenates first, allocating an intermediate string whose
characters are immediately copied into the builder and thrown away. Chaining
`Append(a).Append(b)` writes each part straight into the buffer.

Reported when the receiver binds to `System.Text.StringBuilder`, the call is
the one-argument `Append(string)` or `AppendLine(string)` overload, and the
argument is the built-in string concatenation. All-constant concatenations
(`"a" + "b"`) are folded by the compiler and stay silent, as do user-defined
`+` operators. The fix unrolls the left-nested chain in source order — every
part but the last becomes `Append`, and the last keeps the original method.
Only the string-concatenation spine is unrolled: an arithmetic prefix such as
`i + j + text` keeps its addition together as one operand.

## Example

```csharp
builder.Append(name + ":");       // PSH1214
builder.AppendLine(a + b + c);    // PSH1214

builder.Append(name).Append(":");             // fixed
builder.Append(a).Append(b).AppendLine(c);    // fixed
```

## Related rules

This rule reports only `+` concatenations. A single-character string argument
(`Append("x")`) belongs to PSH1202, and an argument that allocates through a
nested call — `Append(string.Format(...))`, `Append(x.ToString())`,
`Append(s.Substring(...))` — belongs to PSH1203.
