# PSH1215 — Concatenate when there is no separator

**Category:** Strings
**Severity:** Warning
**Code fix:** Renames the call to `string.Concat` and removes the separator argument (supports Fix All)

`string.Join` exists to insert a separator between elements. When the separator
is the empty string — the literal `""` or `string.Empty` — there is nothing to
insert, yet the call still runs the separator-insertion bookkeeping between
every element. `string.Concat` copies the pieces straight through.

A call is reported when it binds to a static `string.Join` overload that takes
a string separator and one values argument — `Join(string, params object[])`,
`Join(string, params string[])` (including the expanded params call form),
`Join(string, IEnumerable<string>)`, or `Join<T>(string, IEnumerable<T>)` — and
the matching `string.Concat` overload exists on the target framework.
`Join(string, string[], int, int)` has no `Concat` equivalent and is never
reported, and char-separator overloads cannot have an empty separator.

## Example

```csharp
var slug = string.Join("", parts);                // PSH1215
var slug = string.Concat(parts);                  // fixed

var key = string.Join(string.Empty, a, b, c);     // PSH1215
var key = string.Concat(a, b, c);                 // fixed
```
