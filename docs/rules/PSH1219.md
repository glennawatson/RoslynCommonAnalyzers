# PSH1219 — Ask whether a string is blank without trimming it

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | Strings |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A string is trimmed only to ask whether what is left is empty.

```csharp
// 👎 reported
if (text.Trim().Length == 0)
{
    // ...
}

// 👍
if (string.IsNullOrWhiteSpace(text))
{
    // ...
}
```

## Rule description

`Trim()` allocates a whole new string and copies the non-white-space characters
into it — of which, on the path that actually matters, there are none — and then
the copy is measured and thrown away. `string.IsNullOrWhiteSpace` scans the
original in place, stops at the first character that is not white space, and
allocates nothing.

Three shapes are reported, along with their `!=` forms:

| Written | Rewritten |
| --- | --- |
| `text.Trim().Length == 0` | `string.IsNullOrWhiteSpace(text)` |
| `text.Trim().Length != 0` | `!string.IsNullOrWhiteSpace(text)` |
| `text.Trim() == ""` / `text.Trim() == string.Empty` | `string.IsNullOrWhiteSpace(text)` |
| `text.Trim() != ""` / `text.Trim() != string.Empty` | `!string.IsNullOrWhiteSpace(text)` |
| `string.IsNullOrEmpty(text.Trim())` | `string.IsNullOrWhiteSpace(text)` |

The `0 == text.Trim().Length` and `"" == text.Trim()` orderings are reported too.
A negated `!string.IsNullOrEmpty(text.Trim())` keeps its `!`, because only the
call itself is replaced.

## The rewrite changes what happens on null — for the better

This is a behavior change:

| `text` | `text.Trim().Length == 0` | `string.IsNullOrWhiteSpace(text)` |
| --- | --- | --- |
| `null` | **throws `NullReferenceException`** | `true` |
| `""` | `true` | `true` |
| `"   "` | `true` | `true` |
| `" a "` | `false` | `false` |

The two agree on every input the original could actually survive. They differ
only where the original **crashed**: `null.Trim()` throws, and
`string.IsNullOrWhiteSpace(null)` answers `true`. So the rewrite removes a latent
`NullReferenceException` and makes the blank test more forgiving.

That is almost always what the author wanted — the whole expression is asking
"is there anything in here?" and null is the emptiest possible answer. But if
your code was **relying** on the throw as an argument guard, this rule takes it
away. Guard the argument explicitly instead of leaning on a crash inside a
boolean test.

## What is not reported

**`Trim` with arguments.** `text.Trim(',')` and `text.Trim(trimChars)` strip
specific characters, which has nothing to do with white space:
`string.IsNullOrWhiteSpace` would answer a different question.

```csharp
// Not reported: this asks whether the string is nothing but commas.
if (text.Trim(',').Length == 0)
{
}
```

**A trimmed value used for anything else.** The `Trim` call must be the receiver
of the `Length` read, an operand of the comparison, or the sole argument of
`IsNullOrEmpty`. Once the copy is stored, something else may still read it, and
removing the call would remove the value:

```csharp
// Not reported: the trimmed copy is returned.
var trimmed = text.Trim();
return trimmed.Length == 0 ? "blank" : trimmed;
```

**A test that is not an emptiness test.** `text.Trim().Length > 0` is left alone;
only `== 0` and `!= 0` (and the empty-string comparisons) ask the question this
rule can answer in place.

**Expression trees.** A test inside a lambda converted to
`System.Linq.Expressions.Expression<TDelegate>` is not reported: a query provider
translates the tree it is handed, and swapping one call for another there changes
the query it emits rather than the work this process does.

## How to fix violations

Take the code fix, which replaces the whole test with
`string.IsNullOrWhiteSpace(text)`, negated where the original asked whether the
string was *not* blank.

## Performance

`string.IsNullOrWhiteSpace` is gated on existing in the compilation before the
rule registers anything at all. The clean path is syntax-only: an invocation that
is not an argument-free `.Trim()` is rejected on a name comparison, and the
semantic model is only consulted once the surrounding shape is already known to
be an emptiness test.
