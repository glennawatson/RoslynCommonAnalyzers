# PSH1223 â€” Parse a reused format string once

**Category:** Strings
**Severity:** Warning
**Code fix:** Hoists the format into a `static readonly CompositeFormat` field and points the call at it

The placeholders in `"Hello, {0}!"` are fixed the moment the code is compiled. Yet
`string.Format` re-scans the string for them on **every single call** â€” finding the
brace, reading the index, checking the alignment, every time. A `CompositeFormat`
holds the parsed result, so the scanning happens once, at type initialization, and
the call site reads a field.

```csharp
// PSH1223
public string Greet(string name) => string.Format("Hello, {0}!", name);

// fixed
private static readonly CompositeFormat GreetFormat = CompositeFormat.Parse("Hello, {0}!");

public string Greet(string name) => string.Format(CultureInfo.CurrentCulture, GreetFormat, name);
```

## The hoisting fix, and why it does what it does

**The field goes first in the type, and that is not cosmetic.** Static field
initializers run in declaration order. A `string.Format` call sitting in another
static field's initializer would read a *null* `CompositeFormat` if the new field
were appended after it. Inserting at the top means the parsed format is always
ready before anything in that type can use it.

**The current culture is named, not left implicit.** Every `CompositeFormat`
overload takes an `IFormatProvider`, and `string.Format(format, args)` did not â€”
it passes `null` internally, which means the current culture. So the rewrite says
`CultureInfo.CurrentCulture`, which is exactly what the call was already doing.
Passing a bare `null` instead would be worse than obscure: with a `CompositeFormat`
in the next slot it is genuinely **ambiguous** against the old
`Format(string, object, object)` overload, and the rule's speculative bind would
refuse it. A call that already supplies a provider keeps it.

**The format is written out as a literal**, unless it is a `const` field â€” which
binds just as well at field-initializer level as it did inside the method. A
`const` *local* does not (it is not in scope up there), so its value is re-emitted
as a literal. The analyzer only reports a format it could read as a compile-time
constant, so this is exact either way.

**A colliding field name is made unique** rather than shadowing an existing member,
and a call in a type with nowhere to put a field â€” an interface, or a body-less
positional record â€” is reported without a fix rather than fixed wrongly.

## The format string is validated before it is hoisted

`CompositeFormat.Parse` throws on a malformed format. From a static field
initializer it would throw at **type initialization**, wrapped in a
`TypeInitializationException`, possibly a long way from the call that used to throw
a plain `FormatException`. Moving an exception is not this rule's business, so a
format the rule cannot prove well-formed is left alone:

```csharp
string.Format("Hello, {0!", name);   // not reported â€” CompositeFormat.Parse would throw
```

The check is deliberately conservative. It accepts the ordinary
`{index[,alignment][:format]}` grammar and doubled braces as escapes, and refuses
anything it is not sure about â€” which costs at most a missed report, never a broken
fix. A constant with no placeholder at all is not reported either; there is nothing
to parse.

## Proving the API exists

`CompositeFormat` and the `string.Format` overloads that take one **arrived in
.NET 8**. Both are resolved from the analyzed compilation and the rule registers no
syntax action at all when either is missing â€” so on netstandard2.0, on .NET
Framework, and on every .NET before 8, the same `string.Format` call is simply left
alone.

Because the hoisted field does not exist yet, the rewritten call cannot be bound
against it directly. Instead a `default(System.Text.CompositeFormat)` expression
stands in for it: overload resolution sees a `CompositeFormat` in that slot, which
is exactly what it will see once the field is there. That is what proves the
rewrite is unambiguous before it is offered.

## Performance

The clean path is syntactic: an invocation whose member name is not `Format`, or
which carries fewer than two positional arguments, is rejected before anything is
bound.
