# PSH1226 — Iterate a string without copying it to a char array

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | Strings |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A `string.ToCharArray()` result is stored in a local whose every use is
iteration — a `foreach`, a `Length` read, or an index read.

```csharp
// 👎 reported
char[] chars = text.ToCharArray();
for (var i = 0; i < chars.Length; i++)
{
    Consume(chars[i]);
}

// 👍
string chars = text;
for (var i = 0; i < chars.Length; i++)
{
    Consume(chars[i]);
}
```

## Rule description

`ToCharArray()` allocates a `char[]` and copies the string's characters into it.
A string is already indexable and enumerable — `text[i]`, `text.Length`, and
`foreach (var c in text)` all work directly and allocate nothing — so when the
copied array is only ever read for iteration, the copy buys nothing.

The rule reports the **stored-array** shape: the copy is assigned to a local, and
that local is then only iterated. Dropping the `ToCharArray()` lets the code
iterate the string itself, and the fix retypes the local so it holds the string.

## What is not reported

The uses of the local are a whitelist — `foreach`, `Length`, an index read — and
anything else stands the rule down, so a genuine mutable-buffer use is never
rewritten into code that would not compile.

**A mutated array.** A `char[]` element is a writable location; a string element
is not.

```csharp
// Not reported: the array is written, so a string cannot stand in for it.
var chars = text.ToCharArray();
chars[0] = 'x';
```

**An array that escapes.** A local that is returned, reassigned, or passed as an
argument may be handed to something that genuinely wants an array.

```csharp
// Not reported: the callee takes a char[].
var chars = text.ToCharArray();
Consume(chars);
```

**A copy consumed directly, without a local.** `foreach (var c in text.ToCharArray())`
and `text.ToCharArray()[i]` hand the copy straight to the thing that consumes it,
with no local in between — a different, separately reported shape. The two never
both fire.

**A local whose declared type is neither `var` nor `char[]`.** Those are the two
the fix can retype to a string; any other declared type is left alone.

## How to fix violations

Take the code fix. It drops the `ToCharArray()` and retypes the local: a
`char[]` declaration becomes `string`, and a `var` declaration re-infers the
string. Every iteration use compiles unchanged on the string.

## Performance

The clean path is a token check on the call; nothing binds until a `ToCharArray`
is found initializing a local. Only then does the rule bind the call and scan the
local's references, so the reference walk stays off the keystroke path.
