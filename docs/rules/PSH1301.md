# PSH1301 - Use a single task directly

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | Concurrency |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

`Task.WhenAll` or `Task.WaitAll` is called with exactly one task.

```csharp
await Task.WhenAll(operation); // PSH1301
```

## Rule description

Wrapping one task in `WhenAll` or `WaitAll` creates the machinery for combining
multiple tasks even though there is nothing to combine. Awaiting or waiting the
task directly avoids the extra array and continuation work.

For `Task<T>`, the analyzer avoids reporting shapes that actually use the
`WhenAll` result array.

## How to fix violations

Await or wait the task itself.

```csharp
await operation;

operation.Wait();
```

Prefer `await` over blocking waits in asynchronous code.

## When to suppress

Suppress only when code generation or API normalization intentionally uses the
multi-task shape and the extra work is accepted.
