# PSH1302 — TaskCompletionSource should run continuations asynchronously

**Category:** Concurrency
**Severity:** Warning
**Code fix:** Supplies `TaskCreationOptions.RunContinuationsAsynchronously` (supports Fix All)

Completing a `TaskCompletionSource` without
`TaskCreationOptions.RunContinuationsAsynchronously` runs every waiter's
continuation inline on the completing thread. That can stall the completer
for an unpredictable time, invert priorities, or deadlock outright when the
completing code holds a lock the continuation needs. The flag queues
continuations to the scheduler instead.

A creation is reported when the bound constructor takes no
`TaskCreationOptions` at all, or when the options argument is a compile-time
constant that lacks the flag. Non-constant options are trusted — the flag
may arrive at runtime.

The fix appends the flag as a new argument, substitutes it for
`TaskCreationOptions.None`, or or-combines it with other constant options,
and spells the enum fully qualified when its simple name does not resolve at
the creation site.

## Example

```csharp
var tcs = new TaskCompletionSource<int>(); // PSH1302
var tcs = new TaskCompletionSource<int>(TaskCreationOptions.RunContinuationsAsynchronously); // fixed
```
