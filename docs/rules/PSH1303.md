# PSH1303 — Do not block an async method with Thread.Sleep

**Category:** Concurrency
**Severity:** Warning
**Code fix:** Rewrites to `await Task.Delay(...)` (supports Fix All)

`Thread.Sleep` inside an async method holds a thread-pool thread hostage for
the whole pause, defeating the scalability the async signature promises.
Awaiting `Task.Delay` frees the thread and resumes the method when the time
elapses.

The nearest enclosing function decides the context, so a synchronous local
function or lambda inside an async method stays clean. Both `Sleep`
overloads (milliseconds and `TimeSpan`) have a matching `Delay` overload,
so the fix carries the argument over unchanged.

## Example

```csharp
public async Task M()
{
    Thread.Sleep(100);      // PSH1303
    await Task.Delay(100);  // fixed
}
```
