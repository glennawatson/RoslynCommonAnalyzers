# PSH1304 — Use PeriodicTimer for periodic asynchronous work

**Category:** Concurrency
**Severity:** Warning
**Code fix:** None (the rewrite restructures the loop)

A loop paced by `await Task.Delay` allocates a fresh timer and task on every
iteration and drifts by the loop body's execution time. `PeriodicTimer`
(.NET 6+) reuses one timer, ticks on a fixed cadence, and stops cleanly
through its dispose or a cancellation token.

The whole rule is gated on `System.Threading.PeriodicTimer` existing in the
compilation. Only unconditional pacing in a `while`/`do` loop is reported —
the delay must be a direct child of the loop body. Retry backoff stays
clean: any identifier used in the delay argument that is written inside the
loop suppresses the report, and bounded `for` loops are skipped entirely.

## Example

```csharp
while (running)
{
    DoWork();
    await Task.Delay(1000); // PSH1304
}

// Preferred shape:
using var timer = new PeriodicTimer(TimeSpan.FromSeconds(1));
while (await timer.WaitForNextTickAsync(token))
{
    DoWork();
}
```
