# PSH1306 — Guard one-time execution with an interlocked latch

**Category:** Concurrency
**Severity:** Warning (disabled by default — opt-in)
**Code fix:** None (retyping the flag field touches every usage)

A bool once-guard written as a read, an early return, and a later assignment —
the classic `if (_disposed) return; ... _disposed = true;` — lets every
thread that passes the check before the first write run the protected code.
`Interlocked.Exchange` on an int latch admits exactly one caller for one
atomic instruction:

```csharp
private int _disposed;

public void Dispose()
{
    if (Interlocked.Exchange(ref _disposed, 1) != 0)
    {
        return;
    }

    // runs exactly once
}
```

Guards whose check and write both sit inside a lock are already serialized
and stay clean. Whether a given guard must be thread-safe at all is
contextual — plenty of single-threaded types write this shape deliberately —
so the rule is opt-in:

```editorconfig
dotnet_diagnostic.PSH1306.severity = warning
```
