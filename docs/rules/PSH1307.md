# PSH1307 — Access interlocked fields with Volatile

**Category:** Concurrency
**Severity:** Warning
**Code fix:** Wraps the access in `Volatile.Read`/`Volatile.Write` (supports Fix All)

A field updated through `Interlocked` participates in lock-free
synchronization, but the compiler and CPU may reorder or cache its plain
reads and writes, hiding updates from other threads. `Volatile.Read` and
`Volatile.Write` pair correctly with the interlocked updates.

Once any `Interlocked.*(ref field, ...)` call is found in a type, every
other plain read or write of that field is reported. Constructor and
initializer accesses (single-threaded construction), `ref`/`out` arguments,
`nameof` operands, and accesses inside `lock` statements are not reported.
Compound assignments and increments are reported without a fix — they need
an `Interlocked` read-modify-write, not just visibility.

## Example

```csharp
private int _count;

public void Add() => Interlocked.Increment(ref _count);

public int Count => _count;                    // PSH1307
public int Count => Volatile.Read(ref _count); // fixed
```
