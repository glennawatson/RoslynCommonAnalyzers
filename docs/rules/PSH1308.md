# PSH1308 — Return the completed task instead of Task.FromResult

**Category:** Concurrency
**Severity:** Warning
**Code fix:** Rewrites the call to `Task.CompletedTask` (supports Fix All)

`Task.FromResult` allocates a `Task<TResult>` to carry a value. When the
caller only ever sees the non-generic `Task` — a `Task`-returning method, a
`Task`-typed variable — that value can never be read, and
`Task.CompletedTask` returns the shared cached instance instead.

The conversion target decides: only calls implicitly converted to the
non-generic `Task` are reported, so awaited or `Task<T>`-typed results stay
clean. Suggested only where `CompletedTask` exists.

## Example

```csharp
public Task FlushAsync() => Task.FromResult(0);    // PSH1308
public Task FlushAsync() => Task.CompletedTask;    // fixed
```
