# PSH1309 — Register cancellation callbacks without flowing the execution context

**Category:** Concurrency
**Severity:** Warning (disabled by default — opt-in)
**Code fix:** Renames the call to `UnsafeRegister` (supports Fix All)

`CancellationToken.Register` captures the current `ExecutionContext` and
restores it inside the callback on every registration. `UnsafeRegister`
skips the capture, which removes allocation and restore cost on hot
registration paths.

Only the two-argument callback-and-state overloads are reported, because
only they have `UnsafeRegister` twins with identical signatures. A callback
that reads `AsyncLocal` state would observe different values without the
flowed context, so the rule is opt-in:

```editorconfig
dotnet_diagnostic.PSH1309.severity = warning
```

## Example

```csharp
token.Register(static state => ((IDisposable)state).Dispose(), resource);       // PSH1309
token.UnsafeRegister(static state => ((IDisposable)state).Dispose(), resource); // fixed
```
