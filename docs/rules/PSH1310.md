# PSH1310 — Dispose asynchronously in async code

**Category:** Concurrency
**Severity:** Warning
**Code fix:** Inserts the `await` keyword (`using` → `await using`, supports Fix All)

Disposing a resource that implements `IAsyncDisposable` with a synchronous
`using` runs its cleanup — often a flush or a close that performs real I/O —
as a blocking call, holding the current thread for the whole time. Inside an
async method the state machine already exists, so `await using` performs the
same cleanup while freeing the pool thread until it completes.

Only synchronous `using` statements and using declarations whose nearest
enclosing method, local function, or lambda is `async` are reported, so a
synchronous helper inside an async method stays clean. A statement declaring
several resources is reported only when every one is asynchronously
disposable. Suggested only where `System.IAsyncDisposable` exists and the
language version supports `await using` (C# 8+).

## Example

```csharp
public async Task SaveAsync(Stream destination, string line)
{
    using var writer = new StreamWriter(destination);    // PSH1310
    await writer.WriteLineAsync(line);
}

public async Task SaveAsync(Stream destination, string line)
{
    await using var writer = new StreamWriter(destination);    // fixed
    await writer.WriteLineAsync(line);
}
```
