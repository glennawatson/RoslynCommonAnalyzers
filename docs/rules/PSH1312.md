# PSH1312 ‚Äî Return a completed task, never null

**Category:** Concurrency
**Severity:** Warning
**Code fix:** Replaces the null with `Task.CompletedTask` or `Task.FromResult<T>(default)` (supports Fix All)

A `null` returned where the declared return type is `Task` or `Task<T>` turns
every `await` at the call site into a `NullReferenceException` and pushes
defensive null checks onto every caller. The runtime caches completed task
instances ‚Äî `Task.CompletedTask` is a shared singleton, and `Task.FromResult`
hands back cached tasks for default-like values ‚Äî so a real completed task
costs nothing over the null.

Only non-async methods, local functions, and property/indexer getters are
reported. In an `async` member `return null` produces a completed task
*carrying* a null result, which is a valid, different shape and stays clean.
A `default` `ValueTask` is already a completed task and is never reported.

In nullable-enabled code the `Task.FromResult<T>(default)` replacement may
need a nullability tweak when `T` is a reference type ‚Äî for example
`Task.FromResult<string?>(null)` ‚Äî or a real value instead of `default`.

## Example

```csharp
public Task FlushAsync() => null;                                     // PSH1312
public Task FlushAsync() => Task.CompletedTask;                       // fixed

public Task<string> FindAsync() => null;                              // PSH1312
public Task<string> FindAsync() => Task.FromResult<string>(default);  // fixed
```
