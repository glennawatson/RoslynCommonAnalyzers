# PSH1400 — Use the static HashData method for one-shot hashing

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | ApiSelection |
| Severity | Warning |
| Code fix | Yes (chained shape only) |
| Default | Enabled (where HashData exists) |

## Cause

A hash algorithm instance (`MD5`, `SHA1`, `SHA256`, `SHA384`, `SHA512`, or the
SHA-3 variants) is created only to hash a single buffer, where the static
`HashData` methods (introduced in .NET 5, SHA-3 in .NET 8) hash in one call.

```csharp
// 👎 reported (chained create-and-compute; the instance is never even disposed)
byte[] hash = SHA256.Create().ComputeHash(data);

// 👎 reported (using-scoped instance used for nothing but hashing)
using var sha = SHA256.Create();
byte[] hash = sha.ComputeHash(data);

// 👍
byte[] hash = SHA256.HashData(data);
```

## Rule description

Creating a `HashAlgorithm` instance to hash a single buffer allocates and
disposes the algorithm for no benefit; the static `HashData` methods hash in
one call without an instance. Two shapes are reported:

- **Chained**: `SHA256.Create().ComputeHash(data)` — an invocation of
  `ComputeHash` whose bound overload takes exactly one `byte[]` parameter, whose
  receiver is the static parameterless `Create()` of a gated algorithm type.
  The chained shape also never disposes the instance it creates.
- **Using-scoped single use**: a local declared `using var sha = SHA256.Create();`
  (or `using (var sha = SHA256.Create()) { ... }`) where **every** reference to
  the local inside the enclosing block is the receiver of a single-argument
  `ComputeHash(byte[])` call (one or more) — the instance is used for nothing
  else. A local that is also read, passed as an argument, or used with another
  overload (such as the three-argument `ComputeHash`) is **not** reported.

### Availability detection

This rule targets a **runtime API**, not a language feature, so it is gated on
each algorithm type exposing a static `HashData(byte[])` method in the
referenced framework (detected once per compilation via
`Compilation.GetTypeByMetadataName` and a member probe). On pre-.NET 5 targets
it reports nothing rather than suggesting a method that does not exist. No
target framework string is parsed. Custom `HashAlgorithm` subclasses with their
own `Create()` factories are never reported.

## How to fix violations

Call the algorithm's static `HashData` method with the buffer. The provided
code fix does this for the **chained** shape, reusing however the source
referred to the type (so `System.Security.Cryptography.SHA512.Create()...`
stays fully qualified).

The **using-scoped** shape has **no automated fix** — removing the declaration
and rewriting every call site is a larger transform, so replace the
`ComputeHash` calls with `HashData` and delete the `using` declaration
manually:

```csharp
// before
using var sha = SHA256.Create();
byte[] first = sha.ComputeHash(data);
byte[] second = sha.ComputeHash(more);

// after
byte[] first = SHA256.HashData(data);
byte[] second = SHA256.HashData(more);
```
