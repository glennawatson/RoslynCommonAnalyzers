# PSH1402 — Use const for compile-time constants

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | ApiSelection |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A non-public `static readonly` field of a const-capable type is initialized
with a compile-time constant.

```csharp
// 👎 reported
private static readonly int MaxRetries = 3;

// 👍
private const int MaxRetries = 3;

// 👍 not a compile-time constant
private static readonly object Gate = new object();
```

## Rule description

A `static readonly` field costs a field load on every use; a `const` folds the
value directly into each call site, so there is no field, no load, and no
static-constructor ordering concern. When the value is already a compile-time
constant, `const` is the same value for less work.

A field is reported only when **all** of the following hold:

- its modifiers contain both `static` and `readonly`;
- it is **not** `public` or `protected` (private, internal, or file-scoped —
  an omitted accessibility modifier counts as private);
- the declaration has a single declarator with an initializer;
- the field type admits `const` (`bool`, `char`, the integral types, `float`,
  `double`, `decimal`, `string`, or an enum type); and
- the initializer is a compile-time constant (`SemanticModel.GetConstantValue`
  succeeds) — literals, constant arithmetic, and references to other consts all
  qualify.

Multi-variable declarations (`static readonly int A = 1, B = 2;`) are skipped
to keep the rule's scope tight.

### Why public fields are excluded

`const` values are **baked into consuming assemblies** at compile time: a
consumer compiled against `const int MaxRetries = 3` keeps `3` even after you
ship an update that changes the value, until that consumer recompiles. For a
`public` (or `protected`) field that trade-off is a versioning decision the
author must make deliberately, so the rule never suggests it. Internal and
private constants only ever fold into the same compilation, where the value can
never go stale.

## How to fix violations

Declare the field `const`. The provided code fix replaces the
`static readonly` modifier pair with `const`, preserving trivia and any other
modifiers (`new` stays); Fix All converts every reported field in the document
in one pass.
