# PSH1402 — Use const for compile-time constants

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | ApiSelection |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A non-public `static readonly` field of a const-capable type is initialized
with a compile-time constant, or a local of a const-capable type is initialized
with a compile-time constant and never written again.

```csharp
// 👎 reported
private static readonly int MaxRetries = 3;

// 👍
private const int MaxRetries = 3;

// 👍 not a compile-time constant
private static readonly object Gate = new object();
```

```csharp
// 👎 reported — never written after initialization
int scale = 10;
return value * scale;

// 👍
const int scale = 10;
return value * scale;

// 👍 reassigned, cannot be const
int total = 10;
total += value;
return total;
```

## Rule description

A `static readonly` field costs a field load on every use; a `const` folds the
value directly into each call site, so there is no field, no load, and no
static-constructor ordering concern. A mutable local likewise occupies a slot
the JIT must treat as writable, while a `const` local is folded into every
expression that uses it. When the value is already a compile-time constant,
`const` is the same value for less work.

A **field** is reported only when **all** of the following hold:

- its modifiers contain both `static` and `readonly`;
- it is **not** `public` or `protected` (private, internal, or file-scoped —
  an omitted accessibility modifier counts as private);
- the declaration has a single declarator with an initializer;
- the field type admits `const` (`bool`, `char`, the integral types, `float`,
  `double`, `decimal`, `string`, or an enum type); and
- the initializer is a compile-time constant (`SemanticModel.GetConstantValue`
  succeeds) — literals, constant arithmetic, `nameof`, and references to other
  consts all qualify.

A **local** is reported only when **all** of the following hold:

- it is an ordinary declaration — not already `const`, not a `using`
  declaration, and not a `ref` local;
- the declaration has a single declarator with an initializer;
- the local's type (declared, or inferred for `var`) admits `const`, exactly as
  for fields — `null` for a `string` local qualifies;
- the initializer is a compile-time constant; and
- nothing in the enclosing scope writes the local after initialization: no
  reassignment (simple, compound, or deconstructing), no increment or
  decrement, no `ref`/`out`/`in` argument, no `ref` alias, and no address-of.
  The scan covers lambda and local-function bodies, so a mutated capture also
  disqualifies the local — a capture that only reads it does not.

Loop variables declared by `for` or `foreach` are never considered.
Multi-variable declarations (`static readonly int A = 1, B = 2;` or
`int a = 1, b = 2;`) are skipped to keep the rule's scope tight.

### Why public fields are excluded

`const` values are **baked into consuming assemblies** at compile time: a
consumer compiled against `const int MaxRetries = 3` keeps `3` even after you
ship an update that changes the value, until that consumer recompiles. For a
`public` (or `protected`) field that trade-off is a versioning decision the
author must make deliberately, so the rule never suggests it. Internal and
private constants only ever fold into the same compilation, where the value can
never go stale. A local never leaves its method, so no such trade-off exists.

## How to fix violations

Declare the field or local `const`. The provided code fix replaces a field's
`static readonly` modifier pair with `const`, preserving trivia and any other
modifiers (`new` stays). For a local it prepends `const`, spelling out the
inferred type when the declaration used `var` (`var x = 3` becomes
`const int x = 3`). Fix All converts every reported declaration in the
document in one pass.
