# PSH1404 - Get the assembly from a type

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | ApiSelection |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

Code calls `Assembly.GetExecutingAssembly()` from inside a type declaration.

```csharp
var assembly = Assembly.GetExecutingAssembly(); // PSH1404
```

## Rule description

`Assembly.GetExecutingAssembly()` must identify the executing assembly through
runtime call context. When the code is already inside a type, `typeof(T).Assembly`
names the same assembly directly and avoids that lookup work.

The code fix uses the nearest containing type. Top-level statements are not
rewritten automatically because there is no containing type to name.

## How to fix violations

Use `typeof(CurrentType).Assembly`.

```csharp
var assembly = typeof(CurrentType).Assembly;
```

## When to suppress

Suppress only when the call-stack behavior is intentionally required.
