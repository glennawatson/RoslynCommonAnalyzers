# PSH1405 - Use direct Environment APIs

| Item | Value |
| --- | --- |
| Package | PerformanceSharp.Analyzers |
| Category | ApiSelection |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

Code reaches process or thread identity through `Process` or `Thread` when a
direct `Environment` property is available.

```csharp
var id = Process.GetCurrentProcess().Id; // PSH1405
```

## Rule description

The direct `Environment` properties avoid creating or navigating richer process
and thread objects when the code only needs a simple value.

The rule reports these shapes when the replacement API exists in the target
framework:

- `Process.GetCurrentProcess().Id` -> `Environment.ProcessId`
- `Process.GetCurrentProcess().MainModule.FileName` -> `Environment.ProcessPath`
- `Thread.CurrentThread.ManagedThreadId` -> `Environment.CurrentManagedThreadId`

## How to fix violations

Use the direct property.

```csharp
var id = Environment.ProcessId;
var path = Environment.ProcessPath;
var threadId = Environment.CurrentManagedThreadId;
```

## When to suppress

Suppress only when code deliberately needs the richer `Process` or `Thread`
object for adjacent work and keeping the existing chain is clearer.
