# PSH1409 — Use the built-in throw helpers for argument guards

**Category:** ApiSelection
**Severity:** Warning
**Code fix:** Rewrites the guard to the helper call (supports Fix All)

A hand-written check-and-throw inlines exception construction into every
caller and enlarges methods past JIT inlining budgets. The framework throw
helpers keep the guard to a single call that stays on the hot path and move
the throw out of line.

The mapped shapes, each suggested only where its helper exists:

| Guard | Helper |
| --- | --- |
| `if (x is null) throw new ArgumentNullException(nameof(x));` | `ArgumentNullException.ThrowIfNull(x)` |
| `if (string.IsNullOrEmpty(x)) throw ...` | `ArgumentException.ThrowIfNullOrEmpty(x)` |
| `if (string.IsNullOrWhiteSpace(x)) throw ...` | `ArgumentException.ThrowIfNullOrWhiteSpace(x)` |
| `if (cond) throw new ObjectDisposedException(GetType().FullName);` | `ObjectDisposedException.ThrowIf(cond, this)` |
| `if (x < 0) throw new ArgumentOutOfRangeException(nameof(x));` | `ArgumentOutOfRangeException.ThrowIfNegative(x)` |
| `<`, `<=`, `>`, `>=`, `==`, `!=` against a bound | `ThrowIfLessThan`, `ThrowIfNegativeOrZero`, `ThrowIfGreaterThan`, ... |

The guard's parameter name must match the checked value, and the helpers'
standard messages replace hand-written ones.

A null guard on a `System.Threading.Lock` is left alone: `ThrowIfNull` takes
`object?`, and widening a `Lock` to `object` is a compile error (CS9216, the
would-be `Monitor`-based `lock` trap), so the suggestion could never be
applied.

## Helper aliases and older frameworks

Projects that multi-target down to net462 often alias helper names to an
internal polyfill on old frameworks and to the BCL type on new ones:

```xml
<ItemGroup Condition="$(TargetFramework.StartsWith('net4'))">
  <Using Include="MyLib.Internal.ArgumentExceptionHelper" Alias="ArgumentExceptionHelper" />
</ItemGroup>
<ItemGroup Condition="!$(TargetFramework.StartsWith('net4'))">
  <Using Include="System.ArgumentNullException" Alias="ArgumentExceptionHelper" />
</ItemGroup>
```

The rule probes the conventional alias names (`ArgumentExceptionHelper`,
`ArgumentNullExceptionHelper`, `ArgumentOutOfRangeExceptionHelper`,
`ObjectDisposedExceptionHelper`) first; when one resolves to a type exposing
the helper, the guard is reported on every target framework and the fix
emits the alias spelling, so the rewritten code compiles down to net462.

## Example

```csharp
if (name is null)                                  // PSH1409
{
    throw new ArgumentNullException(nameof(name));
}

ArgumentNullException.ThrowIfNull(name);           // fixed
```
