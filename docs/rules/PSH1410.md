# PSH1410 — Mark trivial forwarders for aggressive inlining

**Category:** ApiSelection
**Severity:** Warning (disabled by default — opt-in)
**Code fix:** Adds `[MethodImpl(MethodImplOptions.AggressiveInlining)]` (supports Fix All)

An expression-bodied method that only forwards to another member — a
delegation, member read, or constant — can still be skipped by the JIT's
IL-size inlining heuristics, leaving a call frame around a one-line body.
`MethodImplOptions.AggressiveInlining` makes the intent explicit.

Virtual, abstract, override, async, extern, partial, and interface members
are skipped, as is anything already carrying a `MethodImpl` attribute
(including `NoInlining`, which is an explicit decision the rule respects).
Blanket inlining attributes are an opinionated convention, so the rule is
opt-in:

```editorconfig
dotnet_diagnostic.PSH1410.severity = warning
```

## Example

```csharp
public int GetValue() => _value;   // PSH1410

[MethodImpl(MethodImplOptions.AggressiveInlining)]
public int GetValue() => _value;   // fixed
```
