# PSH1415 — Hold the concrete type when the concrete type is what you have

**Category:** ApiSelection
**Severity:** Warning
**Code fix:** Changes the declared type (supports Fix All)

A local or field typed as an interface dispatches through it: every call is
virtual, and none of it can be inlined. When the code assigns exactly one
concrete type and never needs the abstraction, declaring the concrete type
turns those calls direct and lets the JIT see through them.

## Example

```csharp
IList<int> items = new List<int>();     // PSH1415
List<int> items = new List<int>();      // fixed
```

## What the rule will touch

- **Locals and `private` fields only** — never a parameter, a return type, or a
  visible field. Those are contracts, and widening them is not a performance
  decision.
- **Every value the variable is ever given must be a `new` of the same concrete
  type.** A variable reassigned from a factory, or from a second
  implementation, is left alone.
- A private field in a type declared **across several files** is skipped: an
  assignment in another file cannot be seen from here.

## Silent shapes

Two things would silently break the rewrite, and both are excluded.

- **A `ref` or `out` argument.** The argument's type must match the parameter
  exactly, so narrowing the declaration would stop it binding.
- **A call that reaches a member the concrete type implements explicitly.**
  Such a member is only reachable through the interface, so narrowing would
  hide it.

That second test is made **per call, not per type**. `List<T>` hides
`ICollection<T>.IsReadOnly` behind an explicit implementation and is still a
perfectly good declaration for code that only ever calls `Add` and `Count` —
only a call that actually reaches the explicitly implemented member disqualifies
the variable.

```csharp
IList<int> items = new List<int>();   // PSH1415
items.Add(1);
_ = items.Count;                      // fine: List<T> implements both directly

IList<int> flags = new List<int>();   // clean
_ = flags.IsReadOnly;                 // List<T> implements this one explicitly
```

## Nothing dispatched, nothing to win

A candidate that never carries a call is left alone. The concrete type pays off
by turning a virtual call direct — so where there is no call, there is no
payoff, and reporting would be churn rather than a speed-up.

The archetype is a sentinel: a value that exists only to be compared by
reference and swapped in atomically. No member is ever invoked on it, so its
declared type costs nothing.

```csharp
private static readonly IDisposable DisposedSentinel = new Marker();   // clean

public bool IsDisposed => ReferenceEquals(Volatile.Read(ref _current), DisposedSentinel);
// ... Interlocked.Exchange(ref _current, DisposedSentinel);
```

The rule reports only where the symbol actually carries a dispatch — a member
access, an indexer, or a `foreach`. Comparing, passing, returning, and assigning
dispatch nothing.
