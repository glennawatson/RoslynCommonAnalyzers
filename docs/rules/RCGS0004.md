# RCGS0004 — Indexer declaration parameters should be on unique lines

| | |
| --- | --- |
| **Category** | Readability |
| **Severity** | Warning |
| **Code fix** | Yes — puts each parameter on its own line |
| **Applies to** | indexer declarations — `this[int a, int b]` |

## Cause

The parameters in a indexer declaration are split across lines such that some share a line while others are wrapped (a "jagged" layout), instead of all being on one line or each on its own line.

## Rule description

A indexer declaration should keep its parameters visually consistent: either everything on a single line, or one parameter per line. Mixed/jagged layouts hurt readability and produce noisy diffs when items are added or removed.

## How to fix violations

Put every parameter on its own line (the supplied code fix does this), or collapse the list back onto a single line.

### Example

```csharp
// 👎 Violates RCGS0004
int this[int a, int b,
    int c] => 0;

// 👍 OK — each parameter on its own line
int this[
    int a,
    int b,
    int c] => 0;

// 👍 OK — all on one line
int this[int a, int b, int c] => 0;
```

## When to suppress

Generally don't — this is purely a formatting/readability convention. If a particular file or generated code needs an exception, configure the severity in `.editorconfig` (`dotnet_diagnostic.RCGS0004.severity = none`) rather than scattering pragmas.
