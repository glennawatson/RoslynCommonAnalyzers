# SES1106 — Do not send HttpClient requests to a cleartext http URL

**Category:** Transport
**Severity:** Warning
**Code fix:** None — see [Why there is no code fix](#why-there-is-no-code-fix)

A request whose URL begins with `http://` travels the network unencrypted.
Anyone on the path — a shared Wi-Fi hop, a compromised router, a middlebox — can
read the full request and response and can silently alter either one. Credentials,
bearer tokens, cookies, and request bodies are exposed, and an attacker can swap
the response your code trusts. `https://` closes both holes: it encrypts the
traffic and authenticates the server.

The rule reports a hard-coded cleartext URL **literal** used at the call site of an
`System.Net.Http.HttpClient` request:

```csharp
using System;
using System.Net.Http;

// all reported (SES1106):
await client.GetAsync("http://example.com/api");                 // string overload
await client.PostAsync("http://example.com/p", content);         // string overload
await client.GetAsync(new Uri("http://example.com/api"));        // new Uri argument
client.BaseAddress = new Uri("http://example.com");              // BaseAddress assignment
```

Switch the scheme to `https` once you have confirmed the endpoint serves TLS:

```csharp
await client.GetAsync("https://example.com/api");
client.BaseAddress = new Uri("https://example.com");
```

## What is reported

The URL is flagged only in these local, high-precision shapes:

- a string literal passed as the request-URL argument of an HttpClient request
  method — `GetAsync`, `GetStringAsync`, `GetByteArrayAsync`, `GetStreamAsync`,
  `PostAsync`, `PutAsync`, `PatchAsync`, `DeleteAsync`, or `SendAsync`;
- a string literal inside a `new System.Uri(...)` that is itself the request-URL
  argument to one of those methods;
- a string literal inside a `new System.Uri(...)` assigned to
  `HttpClient.BaseAddress`.

The literal may be an ordinary, verbatim (`@"…"`), or raw (`"""…"""`) string, and
the `http` scheme is matched case-insensitively. The invoked method or assigned
property is **bound** to confirm its container is `HttpClient`; a same-named method
on an unrelated type is never reported.

Loopback hosts are treated as clean, because cleartext there stays on the machine
and is normal in local development:

- `localhost` and any `*.localhost` host;
- `127.0.0.1`;
- the IPv6 loopback `[::1]`.

```csharp
await client.GetAsync("http://localhost:5000/api");   // not reported
client.BaseAddress = new Uri("http://127.0.0.1");      // not reported
await client.GetAsync("http://api.localhost/data");    // not reported
```

## What is not reported

Only a literal supplied *at the call* is examined. A URL held in a variable, built
by string interpolation or concatenation, returned by a method, or read from
configuration is not flagged — tracking it would require whole-program data-flow
analysis, which this rule deliberately avoids to stay fast and free of false
positives.

```csharp
string url = "http://example.com";
await client.GetAsync(url);              // not reported — flows through a variable
await client.GetAsync($"http://{host}"); // not reported — interpolated, not a literal
```

## Why there is no code fix

There is no safe automatic rewrite. Flipping `http` to `https` only works when the
target actually serves TLS on that host and port; when it does not, the rewritten
call fails at runtime and the change silently breaks the caller. Whether the
endpoint supports HTTPS is knowledge the analyzer does not have, so the rule
reports and leaves the decision to the author.

## Proving the API exists

The rule resolves `System.Net.Http.HttpClient` in the analyzed compilation and
registers nothing when it is absent, so a project targeting a framework without the
type receives no diagnostic. Because the request-URL parameter and `BaseAddress`
are both `System.Uri`-typed, an object creation in those positions is necessarily a
`Uri`, so no separate probe of `System.Uri` is needed.

## Performance

The no-diagnostic path is syntactic. An invocation whose member name is not a
guarded request method, and an assignment whose target is not `BaseAddress`, are
rejected before the semantic model is touched. The scheme and host are parsed
directly from the literal text — a substring scan, with no `Uri` allocation on the
analyzer path — and the compilation is only bound once a cleartext, non-loopback
literal has already been found.
