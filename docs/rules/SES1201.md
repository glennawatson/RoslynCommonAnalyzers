# SES1201 — Do not hard-code secrets in source

| Item | Value |
| --- | --- |
| Category | Secrets |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A string literal holds what is almost certainly a real credential — an API key, an access token, a private
key, or a connection-string password.

```csharp
public sealed class Client
{
    private const string ApiKey = "sk-1A2b3C4d5E6f7G8h9I0jKlMnOpQr"; // SES1201
}
```

## Rule description

A credential written into source is committed to version control, copied into every build artifact, and
readable by anyone with the repository. The moment it lands it must be treated as compromised, and the only
remedy is to rotate it — so the analyzer flags it while it is still being typed.

Detection is deliberately high precision: the rule reports only shapes that are almost never anything but a
real secret, so a match rarely needs triage.

| Shape | What it matches |
| --- | --- |
| OpenAI-style key | `sk-` followed by at least 20 base62 characters |
| AWS access key id | `AKIA` followed by 16 upper-case alphanumerics |
| GitHub token | `ghp_` / `gho_` / `ghu_` / `ghs_` / `ghr_` followed by 36 base62 characters |
| Slack token | `xox` + a type letter (`b`/`a`/`p`/`r`/`s`) + `-` + a token body |
| Google API key | `AIza` followed by 35 url-safe characters |
| PEM private key | a `-----BEGIN … PRIVATE KEY-----` header (a public certificate is not a secret and is ignored) |
| Azure key | a `SharedAccessKey=` or `AccountKey=` marker followed by a base64 body |
| Connection-string password | a `Password=` or `Pwd=` value inside text that also carries a `Server=`, `Data Source=`, `Initial Catalog=`, or `Host=` marker |

The rule stays silent on the shapes that look like a secret but are not one:

- angle-bracket templates such as `<your-api-key>`;
- repeated fill characters such as `sk-xxxxxxxxxxxxxxxxxxxx` or a run of one character;
- low-entropy bodies with only a handful of distinct characters;
- spelled-out placeholder passwords such as `changeme` or `password`;
- strings shorter than each shape's minimum length.

Detection is purely syntactic — string literals exist on every target framework, so the rule needs no
semantic model, no framework probing, and no configuration, and it reports on every target framework. The
credential markers are matched case-sensitively in their canonical form; a lower-cased or reformatted variant
may not be recognised.

## How to fix violations

Move the value out of source and read it at run time from a location that is not committed:

```csharp
public sealed class Client
{
    private readonly string _apiKey;

    public Client(IConfiguration configuration)
        => _apiKey = configuration["OpenAI:ApiKey"]
            ?? throw new InvalidOperationException("OpenAI API key is not configured.");
}
```

User secrets during development, environment variables, and a managed secret store in production all keep the
credential out of the repository. Treat any secret that was ever committed as leaked, and rotate it.

## Performance

The clean path is a length screen and a first-character dispatch, so an ordinary string literal is rejected
after a handful of comparisons with no allocation. Only after the cheap screen passes does a hand-rolled
prefix and character-class scan confirm the shape; no regular expressions run on the analyzer hot path.
