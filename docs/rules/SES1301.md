# SES1301 — Do not build a process command line from non-constant string parts

**Category:** Injection
**Severity:** Warning
**Code fix:** None — see [Why there is no code fix](#why-there-is-no-code-fix)

A process command line built by string interpolation or concatenation hands the
launched program a single string that the operating system then re-parses into
arguments. Any special character that reaches that string changes the parse: a
space splits one intended value into two, and a quote or a shell metacharacter can
inject an extra argument or, on Windows, an entirely separate command. When one of
the composed parts is attacker-influenced, this is command injection.

`ProcessStartInfo.ArgumentList` closes the hole. It takes each argument as a
discrete string and applies the platform's quoting rules per element, so a value
that contains spaces or quotes is delivered to the child process unchanged, as a
single argument:

```csharp
using System.Diagnostics;

// reported (SES1301):
var psi = new ProcessStartInfo("git");
psi.Arguments = $"clone {userInput}";           // interpolation
psi.Arguments = "clone " + userInput;           // concatenation
Process.Start("git", "clone " + userInput);     // Process.Start(fileName, arguments)

// clean — each argument is escaped for you:
var safe = new ProcessStartInfo("git");
safe.ArgumentList.Add("clone");
safe.ArgumentList.Add(userInput);
```

## What is reported

The value is flagged only in these local, high-precision shapes, and only when it
is **not** a compile-time constant:

- an assignment (or object-initializer member) of `System.Diagnostics.ProcessStartInfo.Arguments`;
- the `arguments` string argument of a `System.Diagnostics.Process.Start(string fileName, string arguments)` call.

In both, the value must be a composition — an interpolated string with at least one
interpolation, or a `+` concatenation. The assigned property or invoked method is
**bound**: a same-named `Arguments` property on an unrelated type, and a `Start`
method that is not `System.Diagnostics.Process.Start(string, string)`, are never
reported. The collection overload `Process.Start(string, IEnumerable<string>)` is
already safe and is left alone.

## What is not reported

A fully constant value carries nothing injectable, so it is clean even when written
as a concatenation or interpolation:

```csharp
const string Repo = "https://example.com/repo.git";
psi.Arguments = "clone " + Repo;    // not reported — constant folds to one string
psi.Arguments = $"clone {Repo}";     // not reported — constant interpolation
psi.Arguments = "clone --depth 1";   // not reported — plain literal
```

A bare variable is also not reported. Deciding whether the value behind a variable
is trusted would require whole-program data-flow analysis, which this rule
deliberately avoids to stay fast and free of false positives — it targets the local
*composition* shape, which is the strong on-site signal that a command line is being
assembled from parts:

```csharp
psi.Arguments = args;   // not reported — no composition at this site
```

## Why there is no code fix

There is no safe mechanical rewrite. Splitting a formatted command-line string back
into discrete arguments requires knowing the intended argument boundaries — where
one argument ends and the next begins — which is exactly the information the string
form has thrown away. A quote or a space inside an interpolated value may be part of
one argument or may separate two, and only the author knows which. The rule reports
the site and leaves the split to a human.

## Proving the API exists

The rule resolves `System.Diagnostics.ProcessStartInfo` in the analyzed compilation
and registers nothing unless the type exposes an `ArgumentList` property. That
property is a .NET Core 2.1+ member and is absent on netstandard2.0 and .NET
Framework, where the suggested fix would not compile; on those targets the rule
stays silent so a project never receives a diagnostic it cannot act on. The
collection overload of `Process.Start` ships on the same frameworks, so the same
gate keeps both suggestions actionable.

## Performance

The no-diagnostic path is syntactic. An assignment whose target is not named
`Arguments`, and an invocation whose member is not `Start` or whose arguments slot
is not a composition, are rejected before the semantic model is touched. Binding
and the constant-value check run only after a composition assigned to a candidate
`Arguments` sink has already been found.
