# SES1406 ‚Äî Reflection must not reach non-public members to bypass their declared accessibility

**Category:** Serialization
**Severity:** Warning ‚Äî **disabled by default** (opt-in; see [Enabling the rule](#enabling-the-rule))
**Code fix:** None ‚Äî see [Why there is no code fix](#why-there-is-no-code-fix)

A member is declared `private` or `internal` so that only its own type or assembly
may touch it. That boundary is what lets the author rename, change, or delete the
member safely, and it is often the only thing keeping a field that holds a key, a
token, or an invariant out of reach. Passing `BindingFlags.NonPublic` to a reflection
member lookup reaches straight past that boundary and binds to whatever hidden member
the name resolves to, so code can read a private secret, mutate an internal invariant,
or call a method that was never meant to be public. It also silently welds the caller
to another type's internals: the reflected member is part of no contract, so a routine
refactor of the target breaks the caller at run time with no compiler warning.

```csharp
using System;
using System.Reflection;

public void Probe(Type t)
{
    // reported (SES1406): the flags include the NonPublic bit
    var m = t.GetMethod("secret", BindingFlags.NonPublic | BindingFlags.Instance);
    var f = t.GetField("_key", BindingFlags.NonPublic | BindingFlags.Instance);
    t.InvokeMember("Hidden", BindingFlags.NonPublic | BindingFlags.InvokeMethod, null, null, null);
}
```

Prefer a public API, an interface, or an `[InternalsVisibleTo]` contract that the
target type deliberately offers:

```csharp
// the target exposes what callers may use; no accessibility is bypassed
public void Probe(IThing thing) => thing.DoWork();
```

If a private member genuinely must be reached ‚Äî a test seam, or a serializer over a
type you do not own ‚Äî isolate that reflection in one place and treat the coupling as a
known, deliberate cost.

## What is reported

A call bound to one of `System.Type`'s member-lookup methods ‚Äî

`GetMethod`, `GetMethods`, `GetField`, `GetFields`, `GetProperty`, `GetProperties`,
`GetMember`, `GetMembers`, `GetConstructor`, `GetConstructors`, `GetEvent`,
`GetEvents`, or `InvokeMember`

‚Äî whose `System.Reflection.BindingFlags` argument is a **compile-time constant** that
includes the `NonPublic` bit. The bit is recognized whether it is written as the
literal `BindingFlags.NonPublic`, folded into an `|` expression such as
`BindingFlags.NonPublic | BindingFlags.Instance`, or referenced through a `const` local
or field whose constant value carries it. A `TypeInfo` receiver inherits these methods
from `System.Type`, so lookups on a `TypeInfo` are covered as well. The whole call is
reported.

These stay silent:

- flags that do not include `NonPublic` (for example `BindingFlags.Public | BindingFlags.Instance`);
- a lookup overload with no `BindingFlags` argument at all;
- a lookup on a type that is not the reflection `System.Type` / `TypeInfo`.

## Run-time flags are out of scope

The rule reports only when the flags value is a compile-time constant. A value
assembled at run time ‚Äî a non-`const` field, a parameter, a computed expression ‚Äî

```csharp
private static readonly BindingFlags Flags = BindingFlags.NonPublic | BindingFlags.Instance;

public void M(Type t) => t.GetMethod("secret", Flags);   // not reported: Flags is not a constant
```

is left alone, because deciding whether it carries the `NonPublic` bit would require
data-flow tracking, which this rule deliberately avoids to stay fast and free of false
positives.

## Enabling the rule

The rule is **off by default**: reaching non-public members through reflection is
legitimate in some tools, test harnesses, and frameworks, so it reports only as a
deliberate opt-in. Turn it on in `.editorconfig`:

```ini
[*.cs]
dotnet_diagnostic.SES1406.severity = warning
```

## Why there is no code fix

There is no safe automatic rewrite. Removing the `NonPublic` bit would change which
members the lookup can find and usually break the call; the right replacement ‚Äî a
public API, an interface, or an `[InternalsVisibleTo]` contract ‚Äî depends on what the
target type is willing to expose, which only the author knows. So the rule reports and
leaves the decision to the developer.

## Proving the markers exist

The rule resolves `System.Reflection.BindingFlags` and `System.Type` once per
compilation and matches against those symbols, so a look-alike type or flags enum of
the same shape in another namespace is never reported. When a marker is absent the
comparison simply never matches and the rule stays silent.

## Performance

The no-diagnostic path is syntactic. An invocation whose member is not one of the
lookup names, or which carries no arguments, is rejected before the semantic model is
touched; only a matching shape is bound to confirm the receiver type and inspect the
flags constant.
