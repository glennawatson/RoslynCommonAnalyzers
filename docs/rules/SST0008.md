# SST0008 — Attribute arguments should be on unique lines

| | |
| --- | --- |
| **Category** | Readability |
| **Severity** | Warning |
| **Code fix** | Yes — puts each argument on its own line |
| **Applies to** | attribute argument lists — `[Foo(a, b)]` |

## Cause

The arguments in a attribute argument list are split across lines such that some share a line while others are wrapped (a "jagged" layout), instead of all being on one line or each on its own line.

## Rule description

A attribute argument list should keep its arguments visually consistent: either everything on a single line, or one argument per line. Mixed/jagged layouts hurt readability and produce noisy diffs when items are added or removed.

## How to fix violations

Put every argument on its own line (the supplied code fix does this), or collapse the list back onto a single line.

### Example

```csharp
// 👎 Violates SST0008
[Foo(a, b,
    c)]
class C { }

// 👍 OK — each argument on its own line
[Foo(
    a,
    b,
    c)]
class C { }

// 👍 OK — all on one line
[Foo(a, b, c)]
class C { }
```

## When to suppress

Generally don't — this is purely a formatting/readability convention. If a particular file or generated code needs an exception, configure the severity in `.editorconfig` (`dotnet_diagnostic.SST0008.severity = none`) rather than scattering pragmas.
