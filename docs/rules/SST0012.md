# SST0012 — Class primary constructor parameters should be on unique lines

| | |
| --- | --- |
| **Category** | Readability |
| **Severity** | Warning |
| **Code fix** | Yes — puts each parameter on its own line |
| **Applies to** | `class Foo(int a, int b)` (C# 12) |

## Cause

The parameters in a class primary constructor are split across lines such that some share a line while others are wrapped (a "jagged" layout), instead of all being on one line or each on its own line.

## Rule description

A class primary constructor should keep its parameters visually consistent: either everything on a single line, or one parameter per line. Mixed/jagged layouts hurt readability and produce noisy diffs when items are added or removed.

## How to fix violations

Put every parameter on its own line (the supplied code fix does this), or collapse the list back onto a single line.

### Example

```csharp
// 👎 Violates SST0012
class Foo(int a, int b,
    int c);

// 👍 OK — each parameter on its own line
class Foo(
    int a,
    int b,
    int c);

// 👍 OK — all on one line
class Foo(int a, int b, int c);
```

## When to suppress

Generally don't — this is purely a formatting/readability convention. If a particular file or generated code needs an exception, configure the severity in `.editorconfig` (`dotnet_diagnostic.SST0012.severity = none`) rather than scattering pragmas.
