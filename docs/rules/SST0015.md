# SST0015 — Constructor initializer arguments should be on unique lines

| | |
| --- | --- |
| **Category** | Readability |
| **Severity** | Warning |
| **Code fix** | Yes — puts each argument on its own line |
| **Applies to** | `: base(a, b)` / `: this(a, b)` |

## Cause

The arguments in a constructor initializer are split across lines such that some share a line while others are wrapped (a "jagged" layout), instead of all being on one line or each on its own line.

## Rule description

A constructor initializer should keep its arguments visually consistent: either everything on a single line, or one argument per line. Mixed/jagged layouts hurt readability and produce noisy diffs when items are added or removed.

## How to fix violations

Put every argument on its own line (the supplied code fix does this), or collapse the list back onto a single line.

### Example

```csharp
// 👎 Violates SST0015
public Foo() : base(a, b,
    c) { }

// 👍 OK — each argument on its own line
public Foo() : base(
    a,
    b,
    c) { }

// 👍 OK — all on one line
public Foo() : base(a, b, c) { }
```

## When to suppress

Generally don't — this is purely a formatting/readability convention. If a particular file or generated code needs an exception, configure the severity in `.editorconfig` (`dotnet_diagnostic.SST0015.severity = none`) rather than scattering pragmas.
