# SST0021 — Type argument lists should be on unique lines

| | |
| --- | --- |
| **Category** | Readability |
| **Severity** | Warning |
| **Code fix** | Yes — puts each type argument on its own line |
| **Applies to** | generic type argument lists — `Foo<int, string>`, `M<int, string>()` |

## Cause

The type arguments in a generic type argument list are split across lines such that some share a line while others are wrapped (a "jagged" layout), instead of all being on one line or each on its own line.

## Rule description

A generic type argument list should keep its type arguments visually consistent: either everything on a single line, or one type argument per line. Mixed/jagged layouts hurt readability and produce noisy diffs when items are added or removed.

## How to fix violations

Put every type argument on its own line (the supplied code fix does this), or collapse the list back onto a single line.

### Example

```csharp
// 👎 Violates SST0021
Func<int, string,
    bool> f;

// 👍 OK — each type argument on its own line
Func<
    int,
    string,
    bool> f;

// 👍 OK — all on one line
Func<int, string, bool> f;
```

## When to suppress

Generally don't — this is purely a formatting/readability convention. If a particular file or generated code needs an exception, configure the severity in `.editorconfig` (`dotnet_diagnostic.SST0021.severity = none`) rather than scattering pragmas.
