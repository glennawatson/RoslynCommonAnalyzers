# SST0022 — Function pointer parameter lists should be on unique lines

| | |
| --- | --- |
| **Category** | Readability |
| **Severity** | Warning |
| **Code fix** | Yes — puts each parameter on its own line |
| **Applies to** | `delegate*<int, string, void>` |

## Cause

The parameters in a function pointer parameter list are split across lines such that some share a line while others are wrapped (a "jagged" layout), instead of all being on one line or each on its own line.

## Rule description

A function pointer parameter list should keep its parameters visually consistent: either everything on a single line, or one parameter per line. Mixed/jagged layouts hurt readability and produce noisy diffs when items are added or removed.

## How to fix violations

Put every parameter on its own line (the supplied code fix does this), or collapse the list back onto a single line.

### Example

```csharp
// 👎 Violates SST0022
delegate*<int, string,
    void> fp;

// 👍 OK — each parameter on its own line
delegate*<
    int,
    string,
    void> fp;

// 👍 OK — all on one line
delegate*<int, string, void> fp;
```

## When to suppress

Generally don't — this is purely a formatting/readability convention. If a particular file or generated code needs an exception, configure the severity in `.editorconfig` (`dotnet_diagnostic.SST0022.severity = none`) rather than scattering pragmas.
