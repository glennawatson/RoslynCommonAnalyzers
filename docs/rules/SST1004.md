# SST1004 — Documentation lines should begin with a single space

| Item | Value |
| --- | --- |
| Category | Spacing |
| Severity | Warning |
| Code fix | Yes (add space) |

## Cause

A documentation comment line's text starts immediately after the `///` with no separating space.

## Rule description

A documentation comment line begins with `/// ` so the documentation text is easy to read and
matches the rendered XML doc tooltips. The rule flags only the clear case where text abuts the
`///`; a line that already has a space, a tab, or is empty (`///` alone) is left untouched.
Plain comment markers (`////`, used to comment out code) are not documentation and are exempt.

```csharp
// 👎 reported
///<summary>Calculates the total.</summary>
public int Total() => 0;

// 👍 single leading space
/// <summary>Calculates the total.</summary>
public int Total() => 0;
```

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

The code fix inserts a space after the `///`.
