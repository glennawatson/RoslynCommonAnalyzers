# SST1010 — Opening square brackets should be spaced correctly

| Item | Value |
| --- | --- |
| Category | Spacing |
| Severity | Warning (disabled by default) |
| Code fix | Yes (add or remove space) |

## Cause

An element-access or array opening square bracket `[` is spaced incorrectly, or a
collection expression does not match the configured inner-padding style.

## Rule description

This rule is **disabled by default**. Enable it in `.editorconfig`:

```ini
[*.cs]
dotnet_diagnostic.SST1010.severity = warning
```

It is friendly with modern C# `[...]` literals — **collection expressions** and
**list patterns**. The *outer* space, where `[` opens a new expression or pattern
after an operator or keyword, is always allowed, so the rule never fights the
idiomatic `x = [1, 2]` or `x is [1, 2]` style:

```csharp
// 👍 outer space kept (no diagnostic)
int[] values = [1, 2, 3];
if (x is [1, 2]) { }

// 👎 reported — element access / array
var first = items [0];   // → items[0]
```

### Collection-expression inner spacing

The space *inside* a collection expression or list pattern is controlled by
[`stylesharp.collection_expression_spacing`](../CONFIGURATION.md#collection-expression-spacing):

- `none` (default) — tight `[1, 2]`. Inner padding is reported and removed.
- `space` — padded `[ 1, 2 ]`. Missing inner padding is reported and added.

```ini
[*.cs]
# Opt in to padded collection expressions.
stylesharp.collection_expression_spacing = space
```

The same option governs the closing bracket ([SST1011](SST1011.md)), so both sides
stay symmetric. The padded style needs SST1010 enabled to enforce the opening side.
Empty collections stay `[]` under either setting. Element-access and array brackets
are unaffected by the option and are always tight (`items[0]`).

Attribute brackets are governed by [SST1016](SST1016.md).

The StyleSharp counterpart of StyleCop's `SA1010`.

## How to fix violations

The code fix removes or inserts the space adjacent to the `[` to match the
configured style.
