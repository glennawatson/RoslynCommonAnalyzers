# SST1011 — Closing square brackets should be spaced correctly

| Item | Value |
| --- | --- |
| Category | Spacing |
| Severity | Warning |
| Code fix | Yes (add or remove space) |

## Cause

A closing square bracket `]` is preceded by a space, or a collection expression does
not match the configured inner-padding style.

## Rule description

```csharp
// 👎 reported
var first = items[0 ];

// 👍
var first = items[0];
```

### Collection-expression inner spacing

For **collection expressions** and **list patterns**, the space before `]` follows
[`stylesharp.collection_expression_spacing`](../CONFIGURATION.md#collection-expression-spacing),
the same option that governs the opening bracket ([SST1010](SST1010.md)):

- `none` (default) — tight `[1, 2]`. A space before `]` is reported and removed.
- `space` — padded `[ 1, 2 ]`. A missing space before `]` is reported and added.

Empty collections stay `[]` under either setting. Element-access and array brackets
are unaffected by the option and are always tight (`items[0]`).

Attribute brackets are governed by [SST1017](SST1017.md).

## How to fix violations

The code fix removes or inserts the space before the `]` to match the configured
style.
