# SST1012 — Opening braces should be spaced correctly

| Item | Value |
| --- | --- |
| Category | Spacing |
| Severity | Warning |
| Code fix | Yes (add space) |

## Cause

The opening brace of a single-line brace block is not followed by a space.

## Rule description

A non-empty single-line brace block (object/collection initializer, single-line accessor
list, single-line block) has a space after the opening brace. Multi-line braces are
governed by [SST1500](SST1500.md); empty `{}` and interpolation braces are exempt.

```csharp
// 👎 reported
var values = new[] {1, 2 };

// 👍
var values = new[] { 1, 2 };
```

The StyleSharp counterpart of the analyzer's `the rule` (the inner-spacing case).

## How to fix violations

The code fix inserts a space after the opening brace.
