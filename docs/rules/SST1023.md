# SST1023 — Dereference and address-of symbols should be spaced correctly

| Item | Value |
| --- | --- |
| Category | Spacing |
| Severity | Warning |
| Enabled by default | **No** (opt-in) |
| Code fix | Yes (remove space) |

## Cause

A unary dereference (`*`) or address-of (`&`) symbol is separated from its operand by a space.

## Rule description

A dereference or address-of symbol touches the operand it applies to, so `*ptr` and `&value`
read as a single unit rather than as a binary `*` or `&`.

This rule applies **only to unsafe pointer code** — the `*` and `&` it inspects are the operator
tokens of pointer-indirection and address-of expressions, never the binary multiply/bitwise-and
operators (which are governed by [SST1003](SST1003.md)) and never pointer-type declarations such
as `int* p`.

```csharp
// 👎 reported
int value = * ptr;
int* address = & value;

// 👍 the symbol touches its operand
int value = *ptr;
int* address = &value;
```

## Why it is opt-in

The rule is **disabled by default**. It is only meaningful in `unsafe` code, which most projects
never use, so it is left off to keep the default rule set focused. Enable it in `.editorconfig`
when your project works with pointers:

```ini
dotnet_diagnostic.SST1023.severity = warning
```

## How to fix violations

The code fix removes the whitespace between the symbol and its operand.
