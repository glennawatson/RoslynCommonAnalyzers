# SST1024 — Colons should be spaced correctly

| Item | Value |
| --- | --- |
| Category | Spacing |
| Severity | Warning |
| Code fix | Yes |

## Cause

A colon does not follow the spacing convention for its context.

## Rule description

The spacing depends on where the colon appears:

- **A space on each side** — base lists, constructor initializers, ternary expressions,
  and type-parameter constraints.
- **No space before, a space after** — labels (`case`, `default`, labelled statements),
  named arguments, and attribute targets (`[return: ...]`).

Interpolation format colons (`{x:F2}`) are not affected.

```csharp
// 👎 reported
class Widget: Control { }
Configure(name :"main");

// 👍
class Widget : Control { }
Configure(name: "main");
```

## How to fix violations

The code fix adjusts the spacing around the colon for its context.
