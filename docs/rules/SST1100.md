# SST1100 — Do not prefix calls with base unless a local override exists

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |

## Cause

A `base.` prefix is used to reach a member that the current type does not override.

## Rule description

`base.X` is only meaningful when the current type overrides `X` and you deliberately want the base
implementation. When the type does not override the member, `base.X` and `this.X` bind to the same
member, so the `base.` prefix is redundant and misleading.

```csharp
// 👎 reported — Derived does not override Help
class Derived : Base
{
    void Call() => base.Help();
}

// 👍 base. is meaningful — Derived overrides Run
class Derived : Base
{
    public override void Run() => base.Run();
}
```

## How to fix violations

Remove the redundant `base.` prefix.
