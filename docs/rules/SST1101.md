# SST1101 — Prefix local calls with this

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Enabled by default | **No** (opt-in) |
| Code fix | Yes (add `this.`) |

## Cause

An instance member (field, property, method, or event) is accessed as a bare identifier without a `this.` prefix.

## Rule description

Some teams prefer every instance-member access to carry a `this.` prefix so it is visibly distinct
from a local, parameter, or static member.

```csharp
// 👎 reported (when enabled)
private int M() => field;

// 👍
private int M() => this.field;
```

The rule only fires on a bare identifier that binds to a non-static member of the enclosing type; it
skips already-qualified references, statics, locals, parameters, `nameof`, named arguments, and
object-initializer member names.

## Why it is opt-in

The rule is **disabled by default**. Most .NET style guides — and this repository — deliberately
omit the `this.` prefix, so enabling it would fight the prevailing convention. Turn it on in
`.editorconfig` only if your team follows the `this.`-prefix style:

```ini
dotnet_diagnostic.SST1101.severity = warning
```

## How to fix violations

The code fix prepends `this.` to the reference.
