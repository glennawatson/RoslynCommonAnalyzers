# SST1102 — Query clause should follow the previous clause

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |

## Cause

A clause of a LINQ query expression is separated from the previous clause by a blank line.

## Rule description

Query clauses read as a single statement, so each clause begins on the previous clause's line or the
line directly after it — never after a blank line.

```csharp
// 👎 reported
var q = from x in items

        where x > 0
        select x;
```

## How to fix violations

Remove the blank line between the clauses.
