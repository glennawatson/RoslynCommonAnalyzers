# SST1103 — Query clauses should be on the same line or each on its own line

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |

## Cause

A query mixes layout — some clauses share a line while the query as a whole spans several lines.

## Rule description

A query is either written entirely on one line or with each clause on its own line; mixing the two
makes the clause structure hard to follow.

```csharp
// 👎 reported
var q = from x in items
        where x > 0 select x;

// 👍
var q = from x in items
        where x > 0
        select x;
```

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

Put every clause on its own line, or collapse the whole query onto one line.
