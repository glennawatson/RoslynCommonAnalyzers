# SST1104 — Query clause should begin on a new line after a multi-line clause

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |

## Cause

A query clause begins on the last line of a previous clause that spans multiple lines.

## Rule description

When a clause spans multiple lines, the next clause starts on a fresh line so the boundary between
them is clear.

```csharp
// 👎 reported
var q = from x in items
        where x > 0 &&
            x < 10 select x;
```

## How to fix violations

Move the following clause onto its own line.
