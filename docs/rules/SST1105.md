# SST1105 — Query clauses spanning multiple lines should begin on their own line

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |

## Cause

A query clause that itself spans multiple lines begins on the previous clause's line.

## Rule description

A multi-line clause starts on its own line so its extent is obvious and it is not visually merged
with the previous clause.

```csharp
// 👎 reported
var q = from x in items where x > 0 &&
            x < 10
        select x;
```

## How to fix violations

Move the multi-line clause onto its own line.
