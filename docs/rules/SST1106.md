# SST1106 — Code should not contain empty statements

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes (remove the statement) |

## Cause

An empty statement — a stray `;` with no expression — appears in the code.

## Rule description

An empty statement carries no meaning. It is usually a typo (a doubled semicolon) or a leftover
from deleted code, and it can hide bugs such as a loop body that was meant to follow.

```csharp
// 👎 reported
var x = GetValue();;

// 👍
var x = GetValue();
```

A labeled empty statement (`done: ;`) is left alone because the label is the meaningful part.

The StyleSharp counterpart of StyleCop's `SA1106`.

## How to fix violations

The code fix removes the empty statement. The fix is offered only when the semicolon stands in a
block, switch section, or top-level statement list; an empty statement used as a loop or `if` body
is reported but left for manual review, because deleting it would change control flow.
