# SST1107 — Code should not contain multiple statements on one line

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |

## Cause

A statement begins on the same line that a previous statement ended.

## Rule description

Each statement belongs on its own line so the control flow reads top to bottom and each line can
be set, stepped, or covered independently in a debugger.

```csharp
// 👎 reported
DoFirst(); DoSecond();

// 👍
DoFirst();
DoSecond();
```

The rule inspects the statements of blocks and switch sections. Empty statements are governed by
[SST1106](SST1106.md) and are skipped here to avoid a duplicate report.

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

Move the second and subsequent statements onto their own lines.
