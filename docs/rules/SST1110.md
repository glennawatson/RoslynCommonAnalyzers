# SST1110 — Opening parenthesis or bracket should be on the declaration line

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |

## Cause

An opening parenthesis or bracket of a parameter or argument list is not on the line of the name or keyword that precedes it.

## Rule description

The opening bracket stays attached to the method name (or `this`, `base`, type, etc.) so a reader sees at a glance where the list begins.

```csharp
// 👎 reported
void M
    (int x)

// 👍
void M(
    int x)
```

## How to fix violations

Move the opening bracket up onto the line of the preceding code.
