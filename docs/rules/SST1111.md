# SST1111 — Closing parenthesis or bracket should be on the line of the last parameter

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |

## Cause

The closing parenthesis or bracket of a non-empty list is on a line of its own rather than following the last parameter.

## Rule description

The closing bracket follows the last parameter on the same line, keeping the list compact.

```csharp
// 👎 reported
void M(
    int x
)

// 👍
void M(
    int x)
```

This rule pairs with the SST1150–SST1171 "parameters on unique lines" family, which keeps each
parameter on its own line while this rule keeps the closing bracket attached to the last one.

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

Move the closing bracket up onto the last parameter's line.
