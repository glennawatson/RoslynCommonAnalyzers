# SST1112 — Closing parenthesis should be on the line of the opening parenthesis when the list is empty

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |

## Cause

An empty parameter or argument list splits its parentheses across two lines.

## Rule description

An empty list keeps `()` together on one line; there is nothing to separate.

```csharp
// 👎 reported
void M(
)

// 👍
void M()
```

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

Place the closing parenthesis on the same line as the opening parenthesis.
