# SST1114 — Parameter list should follow the declaration

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |

## Cause

A blank line separates the opening parenthesis from the first parameter.

## Rule description

The first parameter begins on the line of the opening parenthesis or the line directly after it, with no blank line between.

```csharp
// 👎 reported
void M(

    int x)
```

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

Remove the blank line before the first parameter.
