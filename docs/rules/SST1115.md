# SST1115 — A parameter should follow the comma without a blank line

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |

## Cause

A blank line separates a parameter from the comma that precedes it.

## Rule description

Each parameter begins on the line of the preceding comma or the line directly after it, with no blank line between.

```csharp
// 👎 reported
void M(int x,

    int y)
```

## How to fix violations

Remove the blank line before the parameter.
