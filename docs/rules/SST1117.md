# SST1117 - Use configured instance-member qualification

Instance member access should match `stylesharp.instance_member_qualification`.
The default StyleSharp style is `omit_this`, so fields and members are read directly as `_field`
when that shorter name still binds to the same member. Projects that prefer explicit receiver
qualification can set the option to `require_this`; the same rule then reports bare instance members
and the code fix adds `this.`.

The analyzer keeps the semantic checks narrow. In `omit_this` mode it verifies that removing `this.`
does not change binding. In `require_this` mode it only reports bare identifiers that bind to
non-static fields, properties, ordinary methods, or events.

## Default style

```csharp
public sealed class Counter
{
    private readonly int _value;

    public int Value => this._value;
}
```

Use:

```csharp
public sealed class Counter
{
    private readonly int _value;

    public int Value => _value;
}
```

## Require `this.`

```ini
[*.cs]
stylesharp.instance_member_qualification = require_this
```

```csharp
public sealed class Counter
{
    private readonly int _value;

    public int Value => _value;
}
```

Use:

```csharp
public sealed class Counter
{
    private readonly int _value;

    public int Value => this._value;
}
```

## Notes

Shadowing is preserved. If a local, parameter, or local function would capture the shorter spelling,
the analyzer leaves `this.` in place. In `require_this` mode the analyzer skips statics, locals,
named arguments, `nameof(...)`, already-qualified expressions, and object-initializer member names.
