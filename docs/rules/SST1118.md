# SST1118 — A parameter should not span multiple lines

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Enabled by default | **No** (opt-in) |
| Code fix | No |

## Cause

A single parameter or argument spans more than one line.

## Rule description

Keeping each argument on one line makes a call easy to scan. Some expressions, however,
legitimately span multiple lines — lambdas, anonymous methods, object/collection/array initializers,
`switch` expressions, query expressions, and interpolated strings — and those are exempt.

```csharp
// 👎 reported
Add(1 +
    2);

// 👍 exempt — a multi-line callback
Run(() =>
{
    DoWork();
});
```

## Why it is opt-in

The rule is **disabled by default**. Distinguishing an argument that *should* be one line from one
that conventionally spans several is heuristic, so the rule is left off to avoid false positives;
enable it in `.editorconfig` when you want the stricter layout:

```ini
dotnet_diagnostic.SST1118.severity = warning
```

The StyleSharp counterpart of StyleCop's `SA1118`.

## How to fix violations

Collapse the parameter onto a single line, or extract it into a local variable.
