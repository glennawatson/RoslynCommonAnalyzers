# SST1120 — Comments should contain text

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes (remove the comment) |

## Cause

A single-line (`//`) or multi-line (`/* */`) comment has no text content.

## Rule description

An empty comment adds visual noise without documenting anything. It is usually a leftover from
editing. A comment should either say something or be removed.

```csharp
// 👎 reported (empty)
//
/*  */

// 👍 carries text
// returns the cached total
```

Comments made of extra slashes (`////`, used to comment out code) keep their content and are not
flagged.

## How to fix violations

The code fix removes the empty comment. When the comment is alone on its line the whole line is
removed; when it trails code only the comment and the whitespace before it are removed.
