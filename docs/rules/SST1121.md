# SST1121 — Use built-in type alias

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Enabled by default | **No** (opt-in) |
| Code fix | Yes (replace with the keyword) |

## Cause

A framework primitive type name (`Int32`, `System.String`, …) is used instead of its C# keyword
alias (`int`, `string`, …).

## Rule description

The keyword aliases are the conventional spelling for the built-in types and read more naturally in
C# code.

```csharp
// 👎 reported
private System.Int32 count;
private String name;

// 👍
private int count;
private string name;
```

Only the classic fifteen aliases are handled; `IntPtr`/`UIntPtr` are intentionally excluded because
`nint`/`nuint` are not an unconditional substitute.

## Why it is opt-in

The rule is **disabled by default** because many projects already enforce the same convention through
the compiler's built-in style settings. Enable it in `.editorconfig`:

```ini
dotnet_diagnostic.SST1121.severity = warning
```

## How to fix violations

The code fix replaces the framework type name with its keyword alias.
