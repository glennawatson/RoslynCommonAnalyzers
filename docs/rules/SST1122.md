# SST1122 — Use string.Empty for empty strings

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes (replace with string.Empty) |

## Cause

An empty string literal (`""`) is used where `string.Empty` would express the intent more clearly.

## Rule description

Writing `string.Empty` instead of `""` makes the empty-string intent explicit and avoids confusing
an empty literal with one that accidentally contains whitespace.

```csharp
// 👎 reported
var name = "";

// 👍
var name = string.Empty;
```

`string.Empty` is a static field, not a compile-time constant, so the rule **skips** positions that
require a constant — it would not compile there:

```csharp
private const string Key = "";          // not flagged (const)
[Display("")]                            // not flagged (attribute argument)
void M(string value = "") { }            // not flagged (default parameter)
case "": break;                          // not flagged (case label)
```

The StyleSharp counterpart of StyleCop's `SA1122`.

## How to fix violations

The code fix replaces the empty string literal with `string.Empty`.
