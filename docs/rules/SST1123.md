# SST1123 — Do not place regions within elements

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |

## Cause

A `#region` directive is placed inside the body of a code element (a method, accessor, or other block).

## Rule description

A region buried inside a method body hides part of the implementation and signals that the method
is doing too much. Extract a method instead of hiding code behind a region.

```csharp
// 👎 reported
void M()
{
    #region setup
    ...
    #endregion
}
```

A region at this position is also reported by [SST1124](SST1124.md) (which forbids regions
entirely); SST1123 specifically calls out the in-body case.

## How to fix violations

Remove the region, and extract a well-named method if the code needs grouping.
