# SST1124 — Do not use regions

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |

## Cause

A `#region` directive is used.

## Rule description

Regions hide code from a casual reader and let files grow unwieldy. A well-factored type with
focused members does not need them.

```csharp
// 👎 reported
#region Fields
private int field;
#endregion
```

## How to fix violations

Remove the `#region` and `#endregion` directives. If a type needs that much grouping, consider
splitting it into smaller types or `partial` files.
