# SST1127 — Generic type constraints should be on their own line

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |

## Cause

A generic type-constraint clause (`where T : ...`) shares its line with the declaration or another
constraint clause.

## Rule description

Placing each `where` clause on its own line below the declaration keeps long generic signatures
readable and makes each constraint easy to find.

```csharp
// 👎 reported
void M<T, U>() where T : class where U : new() { }

// 👍
void M<T, U>()
    where T : class
    where U : new()
{
}
```

## How to fix violations

Move each constraint clause onto its own line.
