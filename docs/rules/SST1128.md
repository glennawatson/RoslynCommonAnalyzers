# SST1128 — Constructor initializers should be on their own line

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |

## Cause

A constructor initializer (`: base(...)` or `: this(...)`) shares its line with the constructor signature.

## Rule description

Placing the initializer on its own line separates the call to the other constructor from the
parameter list and keeps it visible.

```csharp
// 👎 reported
public Derived(int x) : base(x) { }

// 👍
public Derived(int x)
    : base(x)
{
}
```

## How to fix violations

Move the constructor initializer onto its own line below the signature.
