# SST1129 — Do not use default value type constructor

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes (replace with `default(T)`) |

## Cause

A value type is created with a parameterless `new T()` call that produces the type's zero value.

## Rule description

`default(T)` states the intent — "the zero value" — more clearly than `new T()`, and is recognised
at a glance.

```csharp
// 👎 reported
var when = new DateTime();

// 👍
var when = default(DateTime);
```

A `struct` that declares its **own** parameterless constructor is not flagged, because that
constructor runs code and is not equivalent to `default`.

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

The code fix replaces the construction with `default(T)`.
