# SST1131 — Use readable conditions

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes (swap operands) |

## Cause

A comparison places the constant on the left and the variable on the right (a "yoda" condition).

## Rule description

Most readers parse a comparison as "variable relates to value", so `count == 0` reads faster than
`0 == count`.

```csharp
// 👎 reported
if (0 == count)
if (0 < count)

// 👍
if (count == 0)
if (count > 0)
```

The rule fires only when exactly one side is a literal constant, so the swap is unambiguous; a
literal-only comparison such as `1 == 2` is left alone.

The StyleSharp counterpart of StyleCop's `SA1131`.

## How to fix violations

The code fix swaps the operands and flips the relational operator (`<` becomes `>`, `<=` becomes
`>=`, and so on).
