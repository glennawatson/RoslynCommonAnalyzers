# SST1132 — Do not combine fields

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |

## Cause

More than one field is declared in a single statement (`private int x, y;`).

## Rule description

Declaring each field on its own keeps its modifiers, type, and initializer unambiguous, and keeps
diffs small when a field is added, removed, or given an initializer.

```csharp
// 👎 reported
private int x, y;

// 👍
private int x;
private int y;
```

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

Split the declaration so each field has its own statement.
