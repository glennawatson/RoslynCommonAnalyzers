# SST1132 — Do not combine declarations

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |

## Cause

More than one variable is declared in a single field or local statement (`private int x, y;` or
`int x = 0, y = 1;`).

## Rule description

Declaring each variable on its own keeps its modifiers, type, and initializer unambiguous, and keeps
diffs small when a variable is added, removed, or given an initializer.

```csharp
// 👎 reported
private int x, y;

void M()
{
    int a = 0, b = 1;   // reported
}

// 👍
private int x;
private int y;

void M()
{
    int a = 0;
    int b = 1;
}
```

Both a field declaration and a local declaration statement are reported. A `for` initializer is **not**:

```csharp
for (int i = 0, j = 10; i < j; i++, j--)   // NOT reported
{
}
```

A single declaration statement is the only way to declare more than one loop variable in a `for`
initializer, so its combined form is not a choice to be flagged. The same applies to the declaration in a
`using` or `fixed` statement — those declarations are not local declaration *statements*, so the rule never
sees them.

## How to fix violations

Split the declaration so each variable has its own statement. There is no code fix: a local's declarator
may need to move relative to later statements that read an earlier one, so the split is left to the author.
