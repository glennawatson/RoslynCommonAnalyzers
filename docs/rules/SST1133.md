# SST1133 — Do not combine attributes

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |

## Cause

Several attributes are combined in one bracket pair (`[A, B]`).

## Rule description

Writing each attribute in its own `[...]` keeps diffs small when attributes are added or removed and
lets each attribute carry its own comment.

```csharp
// 👎 reported
[Serializable, Obsolete]

// 👍
[Serializable]
[Obsolete]
```

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

Give each attribute its own bracket list, one per line.
