# SST1134 — Attributes should not share a line

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |

## Cause

A member's attribute list shares its line with another attribute list or with the element it decorates.

## Rule description

Each attribute on its own line keeps declarations scannable and the attribute set easy to edit.

```csharp
// 👎 reported
[Serializable] internal class C
[A] [B] void M()

// 👍
[Serializable]
internal class C
```

Attributes on parameters and accessors are inline by convention and are not inspected.

The StyleSharp counterpart of StyleCop's `SA1134`.

## How to fix violations

Place each attribute, and the decorated element, on its own line.
