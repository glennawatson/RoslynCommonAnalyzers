# SST1136 — Enum values should be on separate lines

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |

## Cause

An enum member begins on the same line a previous member ended.

## Rule description

One member per line lets the values read as a list and keeps diffs small when members are added or
reordered.

```csharp
// 👎 reported
enum Color { Red, Green, Blue }

// 👍
enum Color
{
    Red,
    Green,
    Blue,
}
```

## How to fix violations

Place each enum member on its own line.
