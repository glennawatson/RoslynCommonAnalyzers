# SST1137 — Elements should have the same indentation

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |

## Cause

A member or statement is indented to a different column from its sibling elements.

## Rule description

Siblings in a block, type body, namespace body, or enum body share one indentation level, so the
block structure is unambiguous and the code lines up.

```csharp
// 👎 reported
void M()
{
    var a = 1;
      var b = 2;
    var c = 3;
}

// 👍
void M()
{
    var a = 1;
    var b = 2;
    var c = 3;
}
```

The first sibling sets the reference indentation; any later sibling that begins its own line at a
different column is flagged. Elements that share a line with another (governed by
[SST1107](SST1107.md)/[SST1136](SST1136.md)) are not inspected.

The StyleSharp counterpart of StyleCop's `SA1137`.

## How to fix violations

Indent the element to match its siblings.
