# SST1138 — A free-standing block that declares nothing should be removed

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A brace-delimited block appears directly inside another block and declares nothing.

```csharp
public void M()
{
    { // SST1138
        Console.WriteLine("a");
        Console.WriteLine("b");
    }
}
```

## Rule description

Because the block introduces no scope of its own, it is identical to splicing its statements into the parent.
It is usually the leftover of a deleted `if`, `using`, `lock`, or `fixed` whose statements were kept while its
head was removed, and the stray braces now say nothing.

The rule reports only a block that declares nothing that a splice would leak or lose:

- A block that declares a **local, a local function, or a label** is real scoping and is left alone — its
  contents would change meaning if hoisted into the parent.
- A block whose only content is a **comment** is left alone: the comment documents an intentional placeholder.
- A method body, a loop body, and any block that is not the direct child of another block are not
  free-standing and are not considered here.

## How to fix violations

Splice the block's statements into the enclosing block and drop the braces.

```csharp
public void M()
{
    Console.WriteLine("a");
    Console.WriteLine("b");
}
```

The code fix does exactly that, moving the inner statements up one level and removing the redundant braces.

## Performance

The clean path is a parent-kind test — the block must sit directly inside another block — followed by a scan
of the statements for a declaration. No semantic model is consulted; a block that scopes a local is rejected on
syntax.
