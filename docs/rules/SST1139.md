# SST1139 — Use literal suffix notation instead of casting

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes (apply the suffix) |

## Cause

A numeric literal is cast to a wider or floating type where a literal suffix would express the type
directly.

## Rule description

A suffix such as `L`, `UL`, `U`, `F`, `D`, or `M` is shorter than a cast and makes clear the literal
is that type from the start, with no runtime-looking cast on a compile-time constant.

```csharp
// 👎 reported
long ticks = (long)1;
float ratio = (float)2;

// 👍
long ticks = 1L;
float ratio = 2F;
```

The rule applies only to a cast wrapping an untyped numeric literal, and only when the suffix is
valid for the literal's form (a floating literal is never given an integer suffix).

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

The code fix removes the cast and appends the matching literal suffix.
