# SST1140 - Start conditional operators on indented continuation lines

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `Sst1145ConditionalOperatorPlacementAnalyzer` |

## Cause

A conditional expression wraps at one of its `?` or `:` operators, but that
operator does not begin a continuation line one indent step deeper than the
owning expression.

```csharp
var value = condition ?
    whenTrue :
    whenFalse;
```

## Rule description

Wrapped conditional operators scan better when the branch markers line up. This
rule is stricter than `SST1145`: once either conditional operator participates in
line wrapping, both operators lead their branch lines at one additional
indentation level. Branch expressions may still wrap after the operator line when
the value expression needs more space.

```csharp
var value = condition
    ? whenTrue
    : whenFalse;
```

## How to fix violations

Move the `?` and `:` operators to branch-leading continuation lines. The code fix
rewrites both operators for the containing conditional expression when the gap
contains only whitespace.
