# SST1141 — Use tuple syntax

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A value tuple is written with the explicit `ValueTuple<...>` type instead of the
language tuple syntax `(T1, T2)`.

```csharp
// 👎 reported
public ValueTuple<int, string> M() => default;

// 👍
public (int, string) M() => default;
```

## Rule description

The C# tuple syntax `(T1, T2)` is the idiomatic way to write a value tuple type;
the underlying `System.ValueTuple<...>` type is an implementation detail. This is
the StyleSharp counterpart of the analyzer's `the rule`. Single-element `ValueTuple<T>`
and the eight-or-more-element (`TRest`) forms are not reported, since they have no
direct tuple-syntax equivalent.

## How to fix violations

Rewrite the type using tuple syntax. The provided code fix does this automatically.
