# SST1142 — Refer to tuple elements by name

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A named tuple element is accessed through its positional `ItemN` field instead of
its name.

```csharp
(int Count, int Total) data = Measure();

// 👎 reported
return data.Item1;

// 👍
return data.Count;
```

## Rule description

When a tuple element has a name, using that name instead of `Item1`/`Item2`/…
makes the code self-describing. Unnamed tuples (where the only available accessor
*is* `ItemN`) are not reported.

## How to fix violations

Replace the `ItemN` access with the element's name. The provided code fix does
this automatically.
