# SST1143 — Do not compare to a boolean literal

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A boolean expression is compared to the `true` or `false` literal.

```csharp
// 👎 reported
if (enabled == true) { }
if (enabled != false) { }
return ready == false;

// 👍
if (enabled) { }
if (enabled) { }
return !ready;
```

## Rule description

Comparing a boolean to `true` or `false` is always redundant — the expression is
already a boolean. Using it directly is shorter and clearer. This is a StyleSharp
original (no StyleCop counterpart).

## How to fix violations

Remove the comparison, negating the operand when the comparison was `== false` or
`!= true`. The provided code fix does this automatically and parenthesizes the
operand when negation requires it.
