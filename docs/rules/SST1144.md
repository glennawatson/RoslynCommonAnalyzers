# SST1144 — Combine case labels with an or-pattern

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Disabled (opt-in) |

## Cause

A switch section stacks several case labels that could be a single `or` pattern.

```csharp
// 👎 reported (when enabled)
switch (value)
{
    case 1:
    case 2:
        return Small;
}

// 👍
switch (value)
{
    case 1 or 2:
        return Small;
}
```

## Rule description

C# 9 `or` patterns let several alternatives share one `case` label, which reads
more directly than stacked labels. The rule only fires when **every** label in the
section can be merged — a `default` label or a label with a `when` guard stops it.
It is gated on C# 9, where `or` patterns were introduced.

**This rule is off by default** — stacked case labels are a common and perfectly
valid style. Enable it in `.editorconfig`:

```ini
[*.cs]
dotnet_diagnostic.SST1144.severity = warning
```

## How to fix violations

Combine the labels into one `case A or B:` label. The provided code fix does this
automatically.
