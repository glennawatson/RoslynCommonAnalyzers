# SST1147 — Do not nest conditional operators

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |
| Default | Disabled (opt-in) |

## Cause

A conditional expression is directly nested in another conditional expression.

```csharp
var text = value == 0 ? "zero" : value > 0 ? "positive" : "negative";
```

## Rule description

Nested `?:` expressions become difficult to scan. The rule is opt-in because shallow
conditional chains are a valid style choice.

## How to fix violations

Extract the nested expression into an `if` statement, local variable, or switch expression.
