# SST1148 — Commented-out code should be removed

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |
| Default | Disabled (opt-in) |

## Cause

A regular comment conservatively resembles a C# statement.

```csharp
// return result;
```

## Rule description

Source control preserves removed code, while comments should explain the current code.
The classifier is intentionally conservative and ignores documentation comments, file
headers, URLs, and `TODO` or `HACK` markers.

## How to fix violations

Delete the commented-out statement or replace it with explanatory prose.
