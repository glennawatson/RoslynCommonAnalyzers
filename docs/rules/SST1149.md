# SST1149 — Prefer the `is null` pattern for null checks

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A null check is written as `== null` or `!= null`.

```csharp
// 👎 reported
if (value == null) { }
if (value != null) { }

// 👍
if (value is null) { }
if (value is not null) { }
```

## Rule description

The pattern forms `is null` and `is not null` read directly as null checks and do
not depend on any overloaded `==` operator. They make the intent explicit and
avoid the ambiguity of equality syntax for reference-like types.

Null checks inside expression-tree lambdas are excluded because `is` patterns do
not compile there.

## How to fix violations

Rewrite `x == null` as `x is null`, and rewrite `x != null` as `x is not null`.
The provided code fix does this automatically and parenthesizes the operand when
needed to preserve the original meaning.
