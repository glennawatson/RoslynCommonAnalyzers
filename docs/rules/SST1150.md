# SST1150 - Constructor declaration parameters should be on unique lines

| Item | Value |
| --- | --- |
| Package | StyleSharp.Analyzers |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A constructor parameter list is split across lines with some parameters sharing a
line and others wrapped.

```csharp
public Bag(object a, object b,
    object c) { } // SST1150
```

## Rule description

Wrapped lists should use one item per line. Keeping some items together and
wrapping others makes the declaration harder to scan and creates noisy diffs when
parameters are added or removed.

## How to fix violations

Put every parameter on its own line, or keep the entire parameter list on one
line.

```csharp
public Bag(
    object a,
    object b,
    object c) { }
```

## When to suppress

Prefer configuring the rule for generated or intentionally formatted files over
adding local suppressions.
