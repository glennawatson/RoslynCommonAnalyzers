# SST1152 - Delegate declaration parameters should be on unique lines

| Item | Value |
| --- | --- |
| Package | StyleSharp.Analyzers |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A delegate declaration parameter list is split unevenly across lines.

```csharp
public delegate void Changed(object sender, EventArgs args,
    CancellationToken cancellationToken); // SST1152
```

## Rule description

Delegate signatures are API surface. When they wrap, each parameter should occupy
its own line so additions and removals are obvious in review.

## How to fix violations

Place each parameter on its own line.

```csharp
public delegate void Changed(
    object sender,
    EventArgs args,
    CancellationToken cancellationToken);
```

## When to suppress

Prefer configuration for generated delegate declarations rather than local
suppression comments.
