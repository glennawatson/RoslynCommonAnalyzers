# SST1153 - Indexer declaration parameters should be on unique lines

| Item | Value |
| --- | --- |
| Package | StyleSharp.Analyzers |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

An indexer parameter list wraps some parameters but leaves others on the same
line.

```csharp
public string this[int row, int column,
    int layer] => values[row, column, layer]; // SST1153
```

## Rule description

Multi-line indexer parameter lists should use one parameter per line to keep the
index shape readable.

## How to fix violations

Put each parameter on its own line.

```csharp
public string this[
    int row,
    int column,
    int layer] => values[row, column, layer];
```

## When to suppress

Use scoped configuration for generated indexers or code that follows an external
formatting contract.
