# SST1154 - Invocation arguments should be on unique lines

| Item | Value |
| --- | --- |
| Package | StyleSharp.Analyzers |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A method call has a multi-line argument list where some arguments share a line.

```csharp
Log(id, name,
    created); // SST1154
```

## Rule description

Wrapped invocation arguments should use one argument per line. This keeps call
sites easy to compare and avoids unrelated diff churn when an argument is added.

## How to fix violations

Put each argument on its own line, or collapse the whole call to one line.

```csharp
Log(
    id,
    name,
    created);
```

## When to suppress

Prefer rule configuration for generated call sites or source copied from another
formatter.
