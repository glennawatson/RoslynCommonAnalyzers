# SST1155 - Object creation arguments should be on unique lines

| Item | Value |
| --- | --- |
| Package | StyleSharp.Analyzers |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

An object creation argument list is split across lines inconsistently.

```csharp
var item = new Item(id, name,
    created); // SST1155
```

## Rule description

When constructor arguments wrap, each argument should be on its own line so the
created value is easy to inspect.

## How to fix violations

Use one argument per line or keep all arguments on one line.

```csharp
var item = new Item(
    id,
    name,
    created);
```

## When to suppress

Prefer scoped configuration for generated object creation code.
