# SST1160 - Record primary-constructor parameters should be on unique lines

| Item | Value |
| --- | --- |
| Package | StyleSharp.Analyzers |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A record primary-constructor parameter list wraps unevenly.

```csharp
public sealed record Item(string Id, string Name,
    DateTime Created); // SST1160
```

## Rule description

Record primary-constructor parameters often define public data shape. When that
list wraps, each parameter should be on its own line.

## How to fix violations

Use one parameter per line.

```csharp
public sealed record Item(
    string Id,
    string Name,
    DateTime Created);
```

## When to suppress

Prefer scoped configuration for generated records.
