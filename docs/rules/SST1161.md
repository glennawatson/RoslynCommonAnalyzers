# SST1161 - Class primary-constructor parameters should be on unique lines

| Item | Value |
| --- | --- |
| Package | StyleSharp.Analyzers |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A class primary-constructor parameter list is split unevenly across lines.

```csharp
public sealed class Service(IFoo foo, IBar bar,
    ILogger<Service> logger); // SST1161
```

## Rule description

Wrapped class primary-constructor parameters should use one parameter per line so
constructor dependencies are easy to scan.

## How to fix violations

Place each parameter on its own line.

```csharp
public sealed class Service(
    IFoo foo,
    IBar bar,
    ILogger<Service> logger);
```

## When to suppress

Prefer scoped configuration for generated classes or source following another
formatter.
