# SST1163 - Target-typed object creation arguments should be on unique lines

| Item | Value |
| --- | --- |
| Package | StyleSharp.Analyzers |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A target-typed `new` argument list wraps inconsistently.

```csharp
Item item = new(id, name,
    created); // SST1163
```

## Rule description

Target-typed object creation hides the constructed type at the call site, so a
consistent argument layout matters when the argument list wraps.

## How to fix violations

Place each argument on its own line, or keep the whole list on one line.

```csharp
Item item = new(
    id,
    name,
    created);
```

## When to suppress

Prefer scoped configuration for generated source.
