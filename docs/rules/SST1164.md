# SST1164 - Constructor initializer arguments should be on unique lines

| Item | Value |
| --- | --- |
| Package | StyleSharp.Analyzers |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A `this(...)` or `base(...)` constructor initializer argument list wraps
unevenly.

```csharp
public Item(string id, string name)
    : this(id, name,
        DateTime.UtcNow) // SST1164
{
}
```

## Rule description

Wrapped constructor initializer arguments should use one argument per line so
delegated construction remains easy to inspect.

## How to fix violations

Put each initializer argument on its own line.

```csharp
public Item(string id, string name)
    : this(
        id,
        name,
        DateTime.UtcNow)
{
}
```

## When to suppress

Prefer scoped configuration for generated constructors.
