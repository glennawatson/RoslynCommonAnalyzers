# SST1165 - Primary-constructor base arguments should be on unique lines

| Item | Value |
| --- | --- |
| Package | StyleSharp.Analyzers |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A base argument list on a primary-constructor declaration wraps unevenly.

```csharp
public sealed class Service(IFoo foo, IBar bar)
    : BaseService(foo, bar,
        "service"); // SST1165
```

## Rule description

When a primary constructor forwards arguments to a base type and that list wraps,
each argument should occupy its own line.

## How to fix violations

Place each base argument on its own line.

```csharp
public sealed class Service(IFoo foo, IBar bar)
    : BaseService(
        foo,
        bar,
        "service");
```

## When to suppress

Use scoped configuration for generated declarations.
