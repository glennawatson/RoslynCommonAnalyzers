# SST1168 - Conversion operator parameters should be on unique lines

| Item | Value |
| --- | --- |
| Package | StyleSharp.Analyzers |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A conversion operator parameter list uses an inconsistent multi-line layout.

```csharp
public static explicit operator Target(
    Source value) => new(value); // normally already compliant
```

## Rule description

Conversion operators only have one source parameter, so practical reports are
rare. The rule exists so parameter-list layout is handled consistently for every
C# declaration shape.

## How to fix violations

Keep the single parameter on the same line as the operator when the declaration
fits, or put it cleanly on its own line when the signature wraps.

## When to suppress

Prefer scoped configuration for generated declarations.
