# SST1169 - Type parameters should be on unique lines

| Item | Value |
| --- | --- |
| Package | StyleSharp.Analyzers |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A generic type parameter list wraps with some type parameters sharing a line.

```csharp
public sealed class Mapper<TSource, TDestination,
    TContext> { } // SST1169
```

## Rule description

Wrapped generic type parameter lists should use one type parameter per line so
generic shape changes are easy to review.

## How to fix violations

Put each type parameter on its own line.

```csharp
public sealed class Mapper<
    TSource,
    TDestination,
    TContext> { }
```

## When to suppress

Prefer scoped configuration for generated generic declarations.
