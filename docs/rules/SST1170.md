# SST1170 - Type arguments should be on unique lines

| Item | Value |
| --- | --- |
| Package | StyleSharp.Analyzers |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A generic type argument list wraps unevenly.

```csharp
Dictionary<string, List<(int Id, string Name)>,
    HashSet<string>> map; // SST1170
```

## Rule description

Wrapped type argument lists should keep one type argument on each line. This is
especially useful for nested generic types where commas are otherwise difficult
to follow.

## How to fix violations

Place each type argument on its own line, or keep the whole list on one line.

```csharp
Dictionary<
    string,
    List<(int Id, string Name)>,
    HashSet<string>> map;
```

## When to suppress

Prefer scoped configuration for generated or externally formatted code.
