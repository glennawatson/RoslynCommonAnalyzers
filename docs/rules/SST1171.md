# SST1171 - Function pointer parameters should be on unique lines

| Item | Value |
| --- | --- |
| Package | StyleSharp.Analyzers |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A function pointer parameter list wraps with multiple parameters sharing a line.

```csharp
delegate* unmanaged<int, int,
    void> callback; // SST1171
```

## Rule description

Function pointer signatures are dense. When their parameter list wraps, each
parameter should be on its own line so the calling convention and return type are
easier to distinguish from the argument types.

## How to fix violations

Put each function pointer parameter on its own line.

```csharp
delegate* unmanaged<
    int,
    int,
    void> callback;
```

## When to suppress

Prefer scoped configuration for generated interop declarations.
