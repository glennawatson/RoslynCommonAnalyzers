# SST1173 — Redundant anonymous-type member names should be omitted

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `ExpressionSimplificationAnalyzer` |

## Cause

An anonymous-type member is assigned from a member, variable, or parameter whose name
matches the explicit member name, so the name is restated needlessly.

```csharp
// 👎 reported
var a = new { X = point.X, count = count };

// 👍 (the names X and count are inferred)
var a = new { point.X, count };
```

## Rule description

Anonymous types infer a member name when the initializer is a simple name or a member
access — `new { obj.X }` produces a member named `X`. Writing `new { X = obj.X }` repeats
that inferred name. Omitting the explicit name keeps the projection compact while
producing the identical type.

A name is only redundant when the initializer is an identifier (`count`), a member access
(`obj.X`), or a member binding (`?.X`) whose final name equals the explicit name. When the
names differ (`new { Renamed = obj.X }`) or the initializer is a more complex expression
(`new { Sum = a + b }`) the name is required and is not reported.

The rule uses a purely syntactic, allocation-free implementation.

## How to fix violations

Delete the `Name =` prefix. The provided code fix does this automatically.
