# SST1175 — Unnecessary casts should be removed

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `ExpressionSimplificationAnalyzer` |

## Cause

A cast targets the type the operand already has, so the cast does nothing.

```csharp
// 👎 reported
int a = (int)anInt;
object o = (object)alreadyObject;

// 👍
int a = anInt;
object o = alreadyObject;
```

## Rule description

When the cast's target type matches the operand's own type — compared **including
nullability** — the conversion is an identity cast that only adds noise. Because the
types are equal (not merely related), removing the cast cannot change overload
resolution or binding.

To avoid false positives the comparison keeps nullability, so a cast that changes the
null-state (`(string)maybeNull`) is left in place, and `default` / `default(T)` operands
are never reported (they have no independent type). The semantic model is consulted only
after the cheap syntactic match on a `CastExpression`.

The `as` and `Cast<T>()`/`OfType<T>()` variants are out of scope for this implementation.

## How to fix violations

Delete the cast, keeping the operand. The provided code fix does this automatically.
