# SST1177 — Redundant base types should be removed

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `RedundantCodeAnalyzer` |

## Cause

A base list restates a type the compiler already implies — `object` as a class base, or
`int` as an enum's underlying type.

```csharp
// 👎 reported
public class C : object { }
public enum E : int { A }

// 👍
public class C { }
public enum E { A }
```

## Rule description

Every class and record implicitly derives from `object`, and every enum's default underlying
type is `int`, so listing them adds nothing. The first base-list entry is bound and reported
only when it is exactly `System.Object` (for a class or record) or `System.Int32` (for an
enum's underlying type); any other base type or underlying type is meaningful and left alone.

This is the clean, low-false-positive subset of the general redundant-base-type check. The
"interface already implied by another interface" case is intentionally out of scope.

## How to fix violations

Remove the redundant base type. The provided code fix deletes it, dropping the whole base
list when it was the only entry.
