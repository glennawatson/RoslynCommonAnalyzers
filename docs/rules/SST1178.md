# SST1178 — Redundant base constructor calls should be removed

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `RedundantCodeAnalyzer` |

## Cause

A constructor declares a `: base()` initializer that passes no arguments. The compiler already
emits a parameterless base-constructor call, so the initializer restates the default.

```csharp
// 👎 reported
public sealed class Widget : Control
{
    public Widget()
        : base()
    {
    }
}

// 👍
public sealed class Widget : Control
{
    public Widget()
    {
    }
}
```

## Rule description

Only a `: base()` initializer with an empty argument list is reported. A `: base(arg)` call
that forwards arguments, and a `: this(...)` initializer that chains to another constructor,
both carry meaning and are left alone. The check is purely syntactic.

## How to fix violations

Delete the `: base()` initializer. The provided code fix does this automatically.
