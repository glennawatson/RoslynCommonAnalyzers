# SST1179 — Redundant default switch sections should be removed

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `RedundantCodeAnalyzer` |

## Cause

A `default:` section in a `switch` contains only a `break;`. A switch with no `default`
section already falls through to the same place, so the explicit section adds nothing.

```csharp
// 👎 reported
switch (value)
{
    case 1:
        Handle();
        break;
    default:
        break;
}

// 👍
switch (value)
{
    case 1:
        Handle();
        break;
}
```

## Rule description

The section is only reported when its single label is `default:` and its only statement is
`break;`. A `default:` that runs real code, or a section that shares the label with `case`
labels (`case 1: default:`), is left alone, because removing it could change behavior. The
check is purely syntactic.

## How to fix violations

Delete the `default:` section. The provided code fix does this automatically.
