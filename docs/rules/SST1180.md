# SST1180 — Empty else clauses should be removed

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `RedundantCodeAnalyzer` |

## Cause

An `else` clause has an empty body — an empty block or a stray semicolon — so it does nothing.

```csharp
// 👎 reported
if (ready)
{
    Start();
}
else
{
}

// 👍
if (ready)
{
    Start();
}
```

## Rule description

The clause is reported when its body is an empty `{ }` block or a lone `;`. An `else if`
chain is never reported, because its body is the nested `if` statement, not an empty block.
The check is purely syntactic.

## How to fix violations

Delete the empty `else` clause. The provided code fix does this automatically.
