# SST1182 — Conditional expressions should not just return boolean literals

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `ExpressionSimplificationAnalyzer` |

## Cause

A conditional expression yields the boolean literals `true` and `false` for its branches, so
it just restates (or negates) its own condition.

```csharp
// 👎 reported
bool isPositive = value > 0 ? true : false;
bool isEmpty = list.Count == 0 ? false : true;  // wait — this negates

// 👍
bool isPositive = value > 0;
bool isEmpty = !(list.Count == 0);
```

## Rule description

`c ? true : false` is exactly `c`, and `c ? false : true` is `!c`. Only the `true`/`false`
pairing is reported; a conditional such as `c ? true : true` is a different (always-true)
smell and is left alone. The check is purely syntactic.

## How to fix violations

Use the condition directly. The provided code fix replaces the conditional with the condition
(or its negation when the branches are swapped).
