# SST1183 — Interpolated strings without interpolations should be plain strings

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `ExpressionSimplificationAnalyzer` |

## Cause

An interpolated string (`$"..."`) contains no `{...}` interpolations, so the `$` prefix buys
nothing — it is a constant string written the long way.

```csharp
// 👎 reported
var label = $"Ready";

// 👍
var label = "Ready";
```

## Rule description

The string is reported when none of its content is an interpolation. Escaped braces (`{{` and
`}}`) are still constant text, so they are reported too; the code fix preserves the resulting
literal value. An interpolated string with even one real hole is left alone. The check is
purely syntactic.

## How to fix violations

Remove the `$` prefix. The provided code fix rewrites the value as a plain string literal.
