# SST1184 — Verbatim string literals without escapes should be regular strings

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `ExpressionSimplificationAnalyzer` |

## Cause

A verbatim string (`@"..."`) contains no backslash, embedded quote, or line break, so it reads
exactly the same as a regular string literal — the `@` adds nothing.

```csharp
// 👎 reported
var name = @"Ready";

// 👍
var name = "Ready";
```

## Rule description

The literal is reported only when its text needs no verbatim quoting: it contains no `\`, no
`"`, and no newline or carriage return. A verbatim string holding a Windows path (`@"C:\temp"`)
or an embedded quote keeps the `@` and is left alone. Raw string literals and UTF-8 literals
are not affected. The check is purely syntactic.

## How to fix violations

Remove the `@` prefix. The provided code fix rewrites the literal as a regular string holding
the same text.
