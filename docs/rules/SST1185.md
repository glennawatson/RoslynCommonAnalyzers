# SST1185 — Use a compound assignment operator

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `ExpressionSimplificationAnalyzer` |

## Cause

An assignment recomputes its own target — `x = x op y` — where the compound assignment
`x op= y` says the same thing more directly.

```csharp
// 👎 reported
count = count + 1;
total = total * factor;

// 👍
count += 1;
total *= factor;
```

## Rule description

The assignment is reported when its right-hand side is a binary expression whose left operand
is syntactically identical to the assignment target, for the arithmetic, bitwise, and shift
operators that have a compound form (`+`, `-`, `*`, `/`, `%`, `&`, `|`, `^`, `<<`, `>>`).

To avoid changing behavior, the target must be side-effect-free — an identifier, `this`, or a
member-access chain of those. A target that involves an indexer or method call (where folding
would change how often it is evaluated) is left alone. The check is purely syntactic.

## How to fix violations

Use the compound assignment operator. The provided code fix rewrites `x = x op y` as `x op= y`.
