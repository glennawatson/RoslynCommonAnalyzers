# SST1186 — A literal should be on the right side of a comparison

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `ExpressionSimplificationAnalyzer` |

## Cause

A literal sits on the left of an equality comparison (`0 == count`). A comparison reads more
naturally with the variable first and the literal second.

```csharp
// 👎 reported
if (0 == count)
{
}

// 👍
if (count == 0)
{
}
```

## Rule description

The rule covers the equality operators `==` and `!=`. It reports a comparison whose left
operand is a non-`null` literal and whose right operand is not a literal. Two-literal
comparisons are left alone (a constant-folding concern, not readability), and `null`
comparisons are left to the `is null` rule ([SST1149](SST1149.md)). The check is purely
syntactic.

## How to fix violations

Place the literal on the right. The provided code fix swaps the operands, keeping the operator.
