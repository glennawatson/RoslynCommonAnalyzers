# SST1187 тАФ Assignments should not be chained

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |
| Analyzer | `ExpressionSimplificationAnalyzer` |

## Cause

An assignment is used as the value of another assignment (`a = b = c`), which hides one of the
writes inside the expression.

```csharp
// ЁЯСО reported
a = b = c;

// ЁЯСН
b = c;
a = b;
```

## Rule description

The outer assignment is reported when its right-hand side is itself an assignment. Writing each
assignment on its own statement makes both targets obvious and keeps the order of effects
explicit. The check is purely syntactic.

No automatic code fix is offered, because splitting the chain correctly depends on the
surrounding statement context.

## How to fix violations

Split the chained assignment into separate statements, assigning the shared value first.
