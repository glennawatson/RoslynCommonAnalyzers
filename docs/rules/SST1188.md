# SST1188 — Use the 'default' literal instead of 'default(T)'

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `ExpressionSimplificationAnalyzer` |

## Cause

A `default(T)` expression names its type where the target type is already known from context,
so the bare `default` literal is clearer.

```csharp
// 👎 reported
public int Count() => default(int);
int total = default(int);

// 👍
public int Count() => default;
int total = default;
```

## Rule description

The expression is reported only where the compiler supplies an unambiguous target type and the
bare literal keeps the same meaning: a `return`, an arrow body, an assignment value, or a
non-`var` initializer, and only when the inferred type equals the spelled-out type.

A `var x = default(int)` is left alone (bare `default` has no type there), as is a converting
context such as `object o = default(int)` (where `default` would produce a different value).
The shape is checked syntactically before the type is confirmed semantically.

## How to fix violations

Use the `default` literal. The provided code fix replaces `default(T)` with `default`.
