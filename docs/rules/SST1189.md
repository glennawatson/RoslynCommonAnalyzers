# SST1189 — Variables should not be self-assigned

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `ExpressionSimplificationAnalyzer` |

## Cause

An assignment copies a value onto itself (`x = x`), which has no effect.

```csharp
// 👎 reported
_value = _value;

// 👍 — remove it, or correct the intended target or source
```

## Rule description

A simple assignment is reported when its left and right sides are syntactically identical and the
target is side-effect-free (an identifier, `this`, or a member-access chain of those). It usually
signals a typo where a different field, parameter, or local was meant. The check is purely syntactic;
assignments involving an indexer or method call are left alone.

## How to fix violations

Remove the self-assignment, or correct the target or source. The provided code fix deletes the
statement.
