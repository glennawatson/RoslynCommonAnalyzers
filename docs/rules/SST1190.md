# SST1190 — Doubled negation operators should be removed

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `ExpressionSimplificationAnalyzer` |

## Cause

The same prefix-negation operator is applied twice (`!!x`, `~~x`), so the pair cancels out.

```csharp
// 👎 reported
bool ready = !!flag;

// 👍
bool ready = flag;
```

## Rule description

A run of the same prefix operator (`!` or `~`) is reported once, on the outermost operator. An even
number of operators cancels entirely; an odd number leaves a single operator. The check is purely
syntactic.

## How to fix violations

Remove the doubled operator. The provided code fix collapses the run, keeping one operator only when
the count is odd.
