# SST1191 — Long numeric literals should use digit separators

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |
| Default | Disabled (opt-in) |
| Analyzer | `LiteralFormattingAnalyzer` |

## Cause

A base-10 integer literal has many digits and no digit separators, so its magnitude is hard to read.

```csharp
// 👎 reported (opt-in)
const int Limit = 1000000;

// 👍
const int Limit = 1_000_000;
```

## Rule description

A plain base-10 integer literal of five or more digits with no `_` separators is reported. Hex,
binary, floating-point, and already-grouped literals are left alone, because each has its own grouping
convention. This rule is **disabled by default** — grouping style is a matter of taste. Enable it in
`.editorconfig`:

```ini
dotnet_diagnostic.SST1191.severity = warning
```

## How to fix violations

Add `_` digit separators in your preferred grouping (commonly every three digits).
