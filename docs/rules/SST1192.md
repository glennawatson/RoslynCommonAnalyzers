# SST1192 — Control characters in string literals should be escaped

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | No |
| Default | Disabled (opt-in) |
| Analyzer | `LiteralFormattingAnalyzer` |

## Cause

A string literal embeds a raw control character (such as a tab) verbatim, which is easy to misread or
alter by accident.

```csharp
// 👎 reported (opt-in) — a literal tab is embedded in the source between the words
var row = "name<TAB>value";

// 👍
var row = "name\tvalue";
```

## Rule description

A **regular** string literal whose source text contains a character below `U+0020` is reported. An
explicit escape sequence states the intent and survives reformatting.

Verbatim (`@"…"`) and raw (`"""…"""`) string literals are not reported: they hold their content
verbatim and cannot use escape sequences, so the newlines of a multi-line raw or verbatim string — and
any other control characters in them — are intentional, not a mistake. This rule is **disabled by
default**. Enable it in `.editorconfig`:

```ini
dotnet_diagnostic.SST1192.severity = warning
```

## How to fix violations

Replace the embedded control character with an explicit escape sequence (`\t`, `\v`, …).
