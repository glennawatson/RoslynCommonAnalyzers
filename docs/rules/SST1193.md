# SST1193 - Keep initial member values with construction

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `LanguageStyleAnalyzer` |

## Cause

A local object is created and then immediately has one of its members assigned.

```csharp
var person = new Person();
person.Name = "Ada";
```

## Rule description

Object initialization reads better when construction and initial member values are kept together.
This rule reports only the simple local pattern where an empty object creation is followed
immediately by a field or property assignment on that same local.

## How to fix violations

Move the first assignment into the initializer. The provided code fix performs that rewrite for the
first immediate member assignment.
