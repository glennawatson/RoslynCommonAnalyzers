# SST1194 - Keep initial collection values with construction

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `LanguageStyleAnalyzer` |

## Cause

A local collection is created and then immediately populated with an `Add` call.

```csharp
var values = new List<int>();
values.Add(1);
```

## Rule description

Collection initialization reads better when the initial contents are attached to the construction.
This rule reports empty local object creations followed immediately by an `Add` call on the same
local when the constructed type implements `System.Collections.IEnumerable`.

## How to fix violations

Move the first `Add` call into the initializer. The provided code fix handles single-argument
`Add` calls.
