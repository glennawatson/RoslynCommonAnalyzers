# SST1199 - Prefer compile-time type names

| Item | Value |
| --- | --- |
| Category | Readability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `LanguageStyleAnalyzer` |

## Cause

Code reads a non-generic type's runtime `Name`.

```csharp
return typeof(Person).Name;
```

## Rule description

`nameof(Person)` is compile-time only and tracks renames. The analyzer avoids generic type syntax
because `typeof(List<int>).Name` is not equivalent to `nameof(List)`.

## How to fix violations

Replace the runtime type name read with `nameof`.
