# SST1202 — Elements should be ordered by access

| Item | Value |
| --- | --- |
| Category | Ordering |
| Severity | Warning |
| Code fix | Yes (move) |

## Cause

Within a kind, a more accessible member appears after a less accessible one.

## Rule description

Members of the same kind appear from most to least accessible:

1. `public`
2. `internal`
3. `protected internal`
4. `protected`
5. `private protected`
6. `private`

```csharp
// 👎 reported — public follows private
public class Widget
{
    private void Helper() { }
    public void Run() { }
}

// 👍 public before private
public class Widget
{
    public void Run() { }
    private void Helper() { }
}
```

The StyleSharp counterpart of StyleCop's `SA1202`.

## How to fix violations

Move the member ahead of the less accessible members of its kind. The code fix
relocates it just after the last member of an equal-or-earlier rank. The fix is only
offered when the file has no conditional (`#if`/`#elif`/`#else`/`#endif`) directives.
