# SST1204 — Static members should appear before instance members

| Item | Value |
| --- | --- |
| Category | Ordering |
| Severity | Warning |
| Code fix | Yes (move) |

## Cause

A static member appears after an instance member of the same kind and accessibility.

## Rule description

Static members appear before instance members within the same kind and
accessibility group.

```csharp
// 👎 reported — the static method follows an instance method
public class Widget
{
    public void Run() { }
    public static Widget Create() => new();
}

// 👍 static first
public class Widget
{
    public static Widget Create() => new();
    public void Run() { }
}
```

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

Move the static member ahead of the instance members of its group. The code fix
relocates it just after the last member of an equal-or-earlier rank. The fix is only
offered when the file has no conditional (`#if`/`#elif`/`#else`/`#endif`) directives.
