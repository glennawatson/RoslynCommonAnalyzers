# SST1206 — Declaration keywords should follow the standard order

| Item | Value |
| --- | --- |
| Category | Ordering |
| Severity | Warning |
| Code fix | Yes (reorder modifiers) |

## Cause

The modifiers on a declaration do not follow the canonical order.

## Rule description

Modifiers appear in the conventional order: the access modifiers first, then `static`,
then the remaining keywords, with `partial` last — matching the repository's
`csharp_preferred_modifier_order`.

```csharp
// 👎 reported
static public int Count => 0;

// 👍 canonical order
public static int Count => 0;
```

## How to fix violations

The code fix reorders the modifiers into the canonical order.
