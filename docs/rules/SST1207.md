# SST1207 — Protected should come before internal

| Item | Value |
| --- | --- |
| Category | Ordering |
| Severity | Warning |
| Code fix | Yes (reorder modifiers) |

## Cause

In a combined access modifier, `internal` appears before `protected`.

## Rule description

When two access keywords are combined, they appear in canonical order: `protected
internal` (not `internal protected`) and `private protected`.

```csharp
// 👎 reported
internal protected int Value;

// 👍 canonical order
protected internal int Value;
```

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

The code fix reorders the access modifiers so `protected` precedes `internal`.
