# SST1210 — Using directives should be ordered alphabetically by namespace

| Item | Value |
| --- | --- |
| Category | Ordering |
| Severity | Warning |
| Code fix | Yes (sort using directives) |

## Cause

Regular using directives are not ordered alphabetically by namespace.

## Rule description

Within each group, regular using directives are ordered alphabetically by namespace
(System namespaces still sort first).

```csharp
// 👎 reported
using System.Text;
using System.Collections;

// 👍 alphabetical
using System.Collections;
using System.Text;
```

Conditional compilation directives (`#if`/`#elif`/`#else`/`#endif`) restart the
ordering: each branch is sorted on its own, and directives are never compared
across the boundary, so a directive that can only live inside one branch is not
flagged for being out of order relative to the unconditional usings around it.

```csharp
// 👍 not reported — the conditional block is ordered independently
using System.Threading;
#if NET
using System.Collections;
#endif
```

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

The code fix sorts the using directives into the canonical order. It is not offered
when the using block spans conditional directives, since the usings cannot be
reordered across `#if`/`#else` branches without scrambling the directives.
