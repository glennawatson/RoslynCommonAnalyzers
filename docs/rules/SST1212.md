# SST1212 — Property accessors should be ordered with get first

| Item | Value |
| --- | --- |
| Category | Ordering |
| Severity | Warning |
| Code fix | Yes (reorder accessors) |

## Cause

A property or indexer declares its `get` accessor after its `set`/`init` accessor.

## Rule description

A `get` accessor appears before the `set` or `init` accessor.

```csharp
// 👎 reported
public int X
{
    set => _x = value;
    get => _x;
}

// 👍 get first
public int X
{
    get => _x;
    set => _x = value;
}
```

## How to fix violations

The code fix reorders the accessors so the `get` accessor appears first.
