# SST1213 — Event accessors should be ordered with add first

| Item | Value |
| --- | --- |
| Category | Ordering |
| Severity | Warning |
| Code fix | Yes (reorder accessors) |

## Cause

An event declares its `add` accessor after its `remove` accessor.

## Rule description

An `add` accessor appears before the `remove` accessor.

```csharp
// 👎 reported
public event EventHandler E
{
    remove { /* ... */ }
    add { /* ... */ }
}

// 👍 add first
public event EventHandler E
{
    add { /* ... */ }
    remove { /* ... */ }
}
```

## How to fix violations

The code fix reorders the accessors so the `add` accessor appears first.
