# SST1214 — Readonly fields should appear before non-readonly fields

| Item | Value |
| --- | --- |
| Category | Ordering |
| Severity | Warning |
| Code fix | Yes (move) |

## Cause

A `readonly` field appears after a non-readonly field of the same accessibility.

## Rule description

Readonly fields appear before non-readonly fields within the same accessibility
group (after constants, which sort first — see [SST1203](SST1203.md)).

```csharp
// 👎 reported — the readonly field follows a mutable one
public class Widget
{
    private int _count;
    private readonly int _id;
}

// 👍 readonly first
public class Widget
{
    private readonly int _id;
    private int _count;
}
```

## How to fix violations

Move the readonly field ahead of the non-readonly fields. The code fix relocates it
just after the last member of an equal-or-earlier rank. The fix is only offered when
the file has no conditional (`#if`/`#elif`/`#else`/`#endif`) directives.
