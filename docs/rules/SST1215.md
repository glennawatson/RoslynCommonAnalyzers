# SST1215 — Instance readonly fields should appear before instance non-readonly fields

| Item | Value |
| --- | --- |
| Category | Ordering |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

Within a type's instance fields of the same accessibility, a `readonly` field
appears after a non-readonly field.

```csharp
// 👎 reported
private int _mutable;
private readonly int _value;

// 👍
private readonly int _value;
private int _mutable;
```

## Rule description

`readonly` fields are grouped before the mutable fields of the same accessibility,
so the immutable state of a type is easy to see at a glance. This rule is the
instance-field counterpart of [SST1214](SST1214.md) (which covers static fields).

## How to fix violations

Move the readonly field above the non-readonly fields. The shared member-ordering
code fix does this automatically (offered when the file has no `#if` directives).
