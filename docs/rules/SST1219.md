# SST1219 — A default section should appear last

| Item | Value |
| --- | --- |
| Category | Ordering |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A `switch` statement's `default` label sits ahead of one or more case sections.

```csharp
switch (x)
{
    default: // SST1219
        return 0;
    case 1:
        return 1;
}
```

## Rule description

A reader expects the fall-through case at the bottom, and putting it there is always safe. The language forbids
implicit fall-through between sections, and `goto case` / `goto default` target a label rather than a position,
so moving the `default` section cannot change which code runs. A `switch` expression enforces the same ordering
at compile time, where a discard arm that is not last is an error; this rule brings the statement form into
line.

A `switch` whose `default` is already last, or that has no `default` at all, is not reported.

## How to fix violations

Move the `default` section to the end of the `switch`.

```csharp
switch (x)
{
    case 1:
        return 1;
    default:
        return 0;
}
```

The code fix relocates the `default` section to the bottom, leaving the case sections in their existing order.

## Performance

Pure syntax: an indexed scan of the sections up to the second-to-last, with no semantic model. A `switch` with
a trailing `default`, or none at all, is rejected before any label is examined.
