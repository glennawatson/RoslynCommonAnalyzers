# SST1220 — Named arguments should match the parameter declaration order

| Item | Value |
| --- | --- |
| Category | Ordering |
| Severity | Info |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `Sst1220NamedArgumentOrderAnalyzer` |

## Cause

Every argument in a call is named, but the names are listed in a different order from the parameters.

```csharp
// 👎 reported
Copy(destination: target, source: origin);

// 👍
Copy(source: origin, destination: target);
```

## Rule description

When a call names all of its arguments, the compiler binds each one by name regardless of position, so
the source order is free to follow the signature. Listing the arguments in parameter-declaration order
lets a reader line the call up against the method it is calling.

The rule fires only when **every** argument is named — a call with even one positional argument already
fixes the order, so it is left alone. It applies to method invocations and object-creation expressions.

## How to fix violations

The fix reorders the named arguments to match the parameter declaration order, preserving the separators
and each slot's trivia. Because the arguments were already bound by name, the reordered call binds to the
same member.
