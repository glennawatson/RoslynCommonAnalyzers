# SST1302 — Interface names should begin with I

| Item | Value |
| --- | --- |
| Category | Naming |
| Severity | Warning |
| Code fix | Yes (renames the interface) |

## Cause

An interface is declared with a name that does not begin with the capital
letter `I`.

## Rule description

The .NET framework design guidelines require interface names to begin with the
capital letter `I`, distinguishing them from classes at a glance — for example
`ICustomer`, `IDisposable`, `IWidget`.

```csharp
// 👎 reported
public interface Widget { }

// 👍 prefixed with I
public interface IWidget { }
```

This is the StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

Rename the interface so it begins with `I`. The provided code fix performs the
rename across the whole solution, updating every reference.

## When to suppress

Suppression is not recommended. If a specific interface intentionally breaks the
convention (rare), configure the rule's severity in `.editorconfig` for that
scope rather than leaving the code inconsistent.
