# SST1304 — Non-private readonly fields should be PascalCase

| Item | Value |
| --- | --- |
| Category | Naming |
| Severity | Warning |
| Code fix | Yes (rename) |

## Cause

A non-private (`public`/`internal`/`protected`) readonly field does not begin
with an upper-case letter.

## Rule description

Externally visible readonly fields follow the .NET PascalCase convention.

```csharp
// 👎 reported
public readonly int maxValue;

// 👍 PascalCase
public readonly int MaxValue;
```

Static readonly fields are covered by [SST1311](SST1311.md); private fields use
`_camelCase` ([SST1309](SST1309.md)).

## How to fix violations

Rename the field to PascalCase. The code fix renames it across the solution.
