# SST1306 — Field names should begin with a lower-case letter

| Item | Value |
| --- | --- |
| Category | Naming |
| Severity | Warning (disabled by default) |
| Code fix | No |

## Cause

A field name begins with an upper-case letter.

## Rule description

This is the the analyzer field-naming style. It is **disabled by default** because it
conflicts with the .NET runtime `_camelCase` convention enforced by
[SST1309](SST1309.md). Enable it only if you prefer the the analyzer style (and disable
SST1309).

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

Rename the field to begin with a lower-case letter.
