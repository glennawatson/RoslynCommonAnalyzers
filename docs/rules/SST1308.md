# SST1308 — Field names should not be prefixed

| Item | Value |
| --- | --- |
| Category | Naming |
| Severity | Warning (disabled by default) |
| Code fix | No |

## Cause

A field name is prefixed with `m_`, `s_`, or `t_`.

## Rule description

This is the the analyzer field-naming style. It is **disabled by default** because the
repository's `.editorconfig` deliberately allows the `s_` prefix for static fields.
Enable it in `.editorconfig` if you want to forbid these prefixes.

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

Remove the prefix from the field name.
