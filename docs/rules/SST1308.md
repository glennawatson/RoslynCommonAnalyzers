# SST1308 — Field names should not be prefixed

| Item | Value |
| --- | --- |
| Category | Naming |
| Severity | Warning (disabled by default) |
| Code fix | No |

## Cause

A field name is prefixed with `m_`, `s_`, or `t_`.

## Rule description

This forbids `m_`, `s_`, and `t_` field-name prefixes. It is **disabled by default**
because the repository's `.editorconfig` deliberately allows the `s_` prefix for
static fields. Enable it in `.editorconfig` if you want to forbid these prefixes.

## How to fix violations

Remove the prefix from the field name.
