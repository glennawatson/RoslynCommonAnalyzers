# SST1309 — Private fields should be _camelCase

| Item | Value |
| --- | --- |
| Category | Naming |
| Severity | Warning |
| Code fix | Yes (rename) |

## Cause

A private (or `private protected`) field is not named using a leading underscore
followed by camelCase.

## Rule description

Private fields use the .NET **runtime** convention of a leading underscore
followed by camelCase — `_value`, `_logger`. This applies to private instance and
private static mutable fields (const and static-readonly fields are PascalCase;
see [SST1303](SST1303.md) and [SST1311](SST1311.md)).

```csharp
// 👎 reported
private int Value;
private readonly ILogger logger;

// 👍 _camelCase
private int _value;
private readonly ILogger _logger;
```

> **Note:** StyleSharp follows the dotnet/runtime style, which *requires* the
> leading underscore on private fields — the opposite of conventions that forbid a
> leading underscore.

## How to fix violations

Rename the field to `_camelCase`. The code fix renames it across the solution.
