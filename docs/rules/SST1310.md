# SST1310 — Field names should not contain underscores

| Item | Value |
| --- | --- |
| Category | Naming |
| Severity | Warning (disabled by default) |
| Code fix | No |

## Cause

A field name contains an underscore.

## Rule description

This is the StyleCop field-naming style. It is **disabled by default** because it
directly contradicts the .NET runtime `_camelCase` convention enforced by
[SST1309](SST1309.md). Enable it only if you prefer the StyleCop style (and disable
SST1309).

The StyleSharp counterpart of StyleCop's `SA1310`.

## How to fix violations

Rename the field to remove the underscore.
