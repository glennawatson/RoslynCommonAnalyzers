# SST1310 — Field names should not contain underscores

| Item | Value |
| --- | --- |
| Category | Naming |
| Severity | Warning (disabled by default) |
| Code fix | No |

## Cause

A field name contains an underscore.

## Rule description

This enforces a field-naming style that forbids underscores. It is **disabled by
default** because it directly contradicts the .NET runtime `_camelCase` convention
enforced by [SST1309](SST1309.md). Enable it only if you prefer the underscore-free
style (and disable SST1309).

## How to fix violations

Rename the field to remove the underscore.
