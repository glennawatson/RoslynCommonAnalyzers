# SST1311 — Static readonly fields should be PascalCase

| Item | Value |
| --- | --- |
| Category | Naming |
| Severity | Warning |
| Code fix | Yes (rename) |

## Cause

A `static readonly` field does not begin with an upper-case letter.

## Rule description

Static readonly fields follow the .NET PascalCase convention regardless of
accessibility (they behave like named constants).

```csharp
// 👎 reported
private static readonly char[] separators = { ',', ';' };

// 👍 PascalCase
private static readonly char[] Separators = { ',', ';' };
```

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

Rename the field to PascalCase. The code fix renames it across the solution.
