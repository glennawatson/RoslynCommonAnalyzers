# SST1312 — Local variables should be camelCase

| Item | Value |
| --- | --- |
| Category | Naming |
| Severity | Warning |
| Code fix | Yes (rename) |

## Cause

A local variable (including a `foreach` loop variable) does not begin with a
lower-case letter.

## Rule description

Local variables follow the .NET camelCase convention. `const` locals are exempt
(constants are PascalCase).

```csharp
// 👎 reported
int ItemCount = list.Count;
foreach (var Item in list) { }

// 👍 camelCase
int itemCount = list.Count;
foreach (var item in list) { }
```

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

Rename the variable to camelCase. The code fix renames it across the method.
