# SST1313 — Parameters should be camelCase

| Item | Value |
| --- | --- |
| Category | Naming |
| Severity | Warning |
| Code fix | Yes (rename) |

## Cause

A parameter does not begin with a lower-case letter.

## Rule description

Parameters follow the .NET camelCase convention.

```csharp
// 👎 reported
public void Configure(int MaxRetries) { }

// 👍 camelCase
public void Configure(int maxRetries) { }
```

Record positional parameters are **not** reported — they surface as PascalCase
properties and are governed by [SST1300](SST1300.md). The StyleSharp counterpart
of the analyzer's `the rule`.

## How to fix violations

Rename the parameter to camelCase. The code fix renames it across the solution.
