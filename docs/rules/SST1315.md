# SST1315 — Union member names should match the configured casing

| Item | Value |
| --- | --- |
| Category | Naming |
| Severity | Warning |
| Code fix | Yes (rename) |
| Default | PascalCase |

## Cause

A union type or one of its cases does not match the configured casing
convention (PascalCase by default).

## Rule description

Union types and their cases follow a casing convention — PascalCase by default.

```csharp
// 👎 reported (default PascalCase)
public sealed class circle : Shape { }

// 👍
public sealed class Circle : Shape { }
```

C# 15 union syntax is not yet exposed by Roslyn, so unions are detected
structurally by the `System.Runtime.CompilerServices.IUnion` marker interface
(the same approach used by the SourceDocParser library) rather than a
version-specific API. A type is treated as union-related when it implements the
marker (a union base) or its base type does (a union case). The rule is **gated**
on that marker being present in the compilation — it costs nothing when no
unions are in play, and lights up automatically once they are.

> **Overlap with SST1300:** because union cases are also types, a mis-cased case
> may also be reported by [SST1300](SST1300.md). Keep whichever rule you want
> enabled; SST1315 exists so union naming can be configured independently.

## Configuration

Set the casing in `.editorconfig`:

```ini
[*.cs]
# pascal_case (default) or camel_case
stylesharp.union_member_naming = pascal_case
# rule-specific override wins over the general key:
stylesharp.SST1315.union_member_naming = pascal_case
```

## How to fix violations

Rename the type to match the convention. The code fix renames it across the
solution.
