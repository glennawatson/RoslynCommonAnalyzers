# SST1316 — Tuple element names should match the configured casing

| Item | Value |
| --- | --- |
| Category | Naming |
| Severity | Warning |
| Code fix | No |
| Default | PascalCase |

## Cause

An explicitly named tuple element does not match the configured casing
convention (PascalCase by default).

## Rule description

Tuple element names follow a casing convention — PascalCase by default, matching
how the elements surface as properties.

```csharp
// 👎 reported (default PascalCase)
public (int count, string name) GetStats() => default;
var point = (x: 1, y: 2);

// 👍
public (int Count, string Name) GetStats() => default;
var point = (X: 1, Y: 2);
```

Inferred element names (taken from a variable, e.g. `(count, name)`) are **not**
reported — only names you write explicitly. This is the StyleSharp counterpart
of StyleCop's `SA1316`.

## Configuration

Set the casing in `.editorconfig`:

```ini
[*.cs]
# pascal_case (default) or camel_case
stylesharp.tuple_element_naming = pascal_case
# rule-specific override wins over the general key:
stylesharp.SST1316.tuple_element_naming = pascal_case
```

## How to fix violations

Rename the tuple element to match the convention. (No automatic code fix —
tuple element names are structural rather than a single renameable symbol.)
