# SST1317 — Asynchronous method names should end with 'Async'

| Item | Value |
| --- | --- |
| Category | Naming |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `MethodNamingAnalyzer` |

## Cause

A method returns a task but its name does not end with the `Async` suffix, so callers cannot
tell at a glance that the result is awaitable.

```csharp
// 👎 reported
public Task Load() => ...;

// 👍
public Task LoadAsync() => ...;
```

## Rule description

A method is reported when it returns `Task`, `Task<T>`, `ValueTask`, or `ValueTask<T>` (or is
declared `async`, excluding `async void`) and its name does not already end with `Async`.

Overrides, explicit and implicit interface implementations, and `Main` are skipped, because
their names are fixed by the base member or the runtime. The name and return type are checked
syntactically before the method symbol is bound, so the no-diagnostic path stays cheap.

## How to fix violations

Rename the method to end with `Async`. The shared rename code fix updates the declaration and
every reference.
