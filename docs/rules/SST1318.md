# SST1318 — Overriding parameter names should match the base declaration

| Item | Value |
| --- | --- |
| Category | Naming |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `MethodNamingAnalyzer` |

## Cause

A method overrides or implements another, but a parameter name differs from the base
declaration. Callers using named arguments, and documentation that references the parameter,
then disagree across the hierarchy.

```csharp
public abstract class Base
{
    public abstract void Process(int count);
}

// 👎 reported
public class Derived : Base
{
    public override void Process(int value)
    {
    }
}

// 👍
public class Derived : Base
{
    public override void Process(int count)
    {
    }
}
```

## Rule description

For an `override`, an explicit interface implementation, or an implicit interface
implementation, each parameter name is compared with the corresponding parameter on the matched
base member, and a mismatch is reported on the parameter. The candidate is recognised
syntactically (an `override` modifier, an explicit interface specifier, or membership in a type
with a base list) before the symbol is bound.

## How to fix violations

Rename the parameter to match the base declaration. The shared rename code fix updates the
declaration and every reference.
