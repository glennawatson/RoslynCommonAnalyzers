# SST1319 — Enumeration names should be PascalCase

| Item | Value |
| --- | --- |
| Category | Naming |
| Severity | Warning |
| Code fix | Yes (rename) |
| Default | Enabled |

## Cause

An enumeration's **type name** starts upper-case but is still not PascalCase — it holds an underscore,
or an acronym written all in capitals.

```csharp
public enum Log_Level { Debug, Info }   // SST1319
public enum HTTPStatus { Ok, NotFound } // SST1319
```

## Rule description

An enum's type name follows the .NET capitalization convention like any other type name:

- it begins with an upper-case letter;
- it holds no underscore;
- it capitalizes acronyms the way the .NET guidelines do — a **two-letter** acronym stays upper
  (`IOMode`, `MyIO`), and a **longer** one is written as a word (`HttpStatus`, not `HTTPStatus`).

```csharp
// 👎 reported
public enum Log_Level { }    // underscore
public enum HTTPStatus { }   // acronym longer than two letters
public enum MyENUM { }       // trailing acronym longer than two letters
public enum XYZ { }          // same, with nothing else in the name

// 👍 PascalCase
public enum LogLevel { }
public enum HttpStatus { }
public enum IOMode { }       // two-letter acronym stays upper
public enum XY { }
public enum Http2Client { }
```

### Scope

Only the enum's **type name** — the members are not this rule's business.

The **first character** of the name belongs to [SST1300](SST1300.md), which owns the upper-case start
of every type name, an enum's included. SST1319 picks up where it leaves off and reports only a name
that *already* starts upper-case, so a badly named enum is reported once and not twice:

| Name | Reported by |
| --- | --- |
| `log_level` | SST1300 (starts lower-case) |
| `Log_Level` | SST1319 (underscore) |
| `HTTPStatus` | SST1319 (long acronym) |

## How to fix violations

Rename the enum. The code fix renames it across the solution, and the name it offers already conforms:
underscores are dropped and the word they separated is capitalized, and an acronym longer than two
letters becomes a word.

| Reported | Code fix offers |
| --- | --- |
| `Log_Level` | `LogLevel` |
| `HTTPStatus` | `HttpStatus` |
| `MyENUM` | `MyEnum` |
| `XYZ` | `Xyz` |

## Replaces

the analyzer **the rule** ("Enumeration types should comply with a naming convention"). the analyzer checks the
regular expression `^([A-Z]{1,3}[a-z0-9]+)*([A-Z]{2})?$`; SST1319 encodes the same .NET convention
structurally rather than transcribing the pattern.

Every name the rule rejects is still reported — but a name that starts **lower-case** (`log_level`) comes
out as [SST1300](SST1300.md), not SST1319, because that half of the convention was already this
package's. When retargeting an `the rule` suppression, check which of the two the site reports.

One deliberate difference: the analyzer's regular expression also rejects a name whose **trailing** capital
stands alone, such as `A` or `MyA`. Those are valid PascalCase, and SST1319 stays quiet on them.
