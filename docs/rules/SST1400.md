# SST1400 — Access modifier should be declared

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | Yes (add modifier) |

## Cause

A type or member relies on its implicit access modifier instead of declaring one.

## Rule description

Declaring accessibility explicitly makes intent obvious and avoids surprises from
the language defaults. Elements where an access modifier is not permitted are
skipped: interface members, explicit interface implementations, static
constructors, and partial methods. The `file` modifier (C# 11) counts as an
explicit access modifier, and records / record structs are treated like any other
type.

```csharp
// 👎 reported — relies on the implicit default
class Widget
{
    int _count;
}

// 👍 explicit
internal class Widget
{
    private int _count;
}
```

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

The code fix inserts the implicit accessibility — `internal` for a top-level type,
`private` for everything else — leaving behaviour unchanged.
