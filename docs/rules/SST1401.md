# SST1401 — Fields should be private

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | No |

## Cause

A field is exposed with more than `private` accessibility.

## Rule description

Fields are an implementation detail; expose state through properties so it can be
validated, changed, or made thread-safe later. Constants and `static readonly`
fields (such as singleton instances) may be any accessibility, because they expose
an immutable value that cannot be reassigned.

```csharp
// 👎 reported
public int Count;

// 👍 private field behind a property
private int _count;
public int Count => _count;

// 👍 allowed — static readonly singleton
public static readonly Sequencer Instance = new();
```

The StyleSharp counterpart of StyleCop's `SA1401`.

## How to fix violations

Make the field private and expose it through a property, or — for a genuinely
shared value — declare it `const` or `static readonly`.
