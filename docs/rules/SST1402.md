# SST1402 — File should contain a single type

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | Yes (move type to its own file) |

## Cause

A file declares more than one top-level type.

## Rule description

One top-level type per file makes types easy to locate. Partial declarations of the
same type count once, and nested types are ignored.

```csharp
// 👎 reported — two top-level types in one file
internal class Widget { }
internal class Gadget { }
```

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

The code fix moves the flagged type into its own file named after the type. The new
file keeps the original namespace, using directives and file header, and the blank
lines left at the removal seams are tidied so neither file picks up a new layout
diagnostic. The original file's newline style and trailing-newline state are
preserved, and the new file is created once and linked into every target framework
(so a multi-targeted project does not get one copy per framework).

Generic types are named with the configured convention — `Widget{T}.cs` by default,
or `Widget`1.cs` when `stylesharp.file_naming_convention = metadata`. See
[CONFIGURATION](../CONFIGURATION.md).

"Fix all" is intentionally not offered: the batch fixer can only merge text edits
within existing documents, not create files, so each type is moved individually.
