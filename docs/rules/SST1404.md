# SST1404 — Code analysis suppression should have justification

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | No |

## Cause

A `[SuppressMessage]` attribute has no `Justification`, an empty one, or the IDE
placeholder `<Pending>`.

## Rule description

Every suppression records why the rule is being silenced so reviewers can judge it.

```csharp
// 👎 reported
[SuppressMessage("Category", "RULE0001")]

// 👍 justified
[SuppressMessage("Category", "RULE0001", Justification = "False positive on generated code.")]
```

The StyleSharp counterpart of StyleCop's `SA1404`.

## How to fix violations

Add a `Justification` that explains the suppression.
