# SST1407 — Arithmetic expressions should declare precedence

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | Yes (add parentheses) |

## Cause

Arithmetic or shift operators of different precedence are mixed without
parentheses.

## Rule description

Parentheses make the intended precedence explicit so readers do not have to recall
operator-precedence rules.

```csharp
// 👎 reported
var x = a + b * c;

// 👍 explicit
var x = a + (b * c);
```

## How to fix violations

The code fix wraps the higher-precedence sub-expression in parentheses.
