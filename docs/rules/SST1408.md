# SST1408 — Conditional expressions should declare precedence

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | Yes (add parentheses) |

## Cause

The `&&` and `||` operators are mixed without parentheses.

## Rule description

Parentheses make the intended grouping explicit.

```csharp
// 👎 reported
if (a || b && c)

// 👍 explicit
if (a || (b && c))
```

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

The code fix wraps the `&&` sub-expression in parentheses.
