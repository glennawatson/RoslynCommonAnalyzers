# SST1413 — Use a trailing comma in multi-line initializers

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | Yes (add trailing comma) |

## Cause

The last element of a multi-line object, collection, array initializer, anonymous
object, or enum is not followed by a trailing comma.

## Rule description

A trailing comma after the last element keeps later reordering and diffs clean.
Single-line initializers are exempt.

```csharp
// 👎 reported
int[] values =
{
    1,
    2
};

// 👍 trailing comma
int[] values =
{
    1,
    2,
};
```

The StyleSharp counterpart of StyleCop's `SA1413`.

## How to fix violations

The code fix adds a trailing comma after the last element.
