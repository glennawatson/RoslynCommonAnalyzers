# SST1414 — Tuple types in signatures should have element names

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A tuple type that appears in a member signature (return type, parameter, property,
field, or delegate) has one or more unnamed elements.

```csharp
// 👎 reported
public (int, string) Lookup() => default;

// 👍
public (int Id, string Name) Lookup() => default;
```

## Rule description

Naming the elements of a tuple in a signature lets callers use the names instead
of positional `Item1`/`Item2` access (see [SST1142](SST1142.md)), and documents
intent. Tuple types used locally — inside a method body — are a deliberately
separate case and are **not** reported. This is the StyleSharp counterpart of
the analyzer's `the rule`.

## How to fix violations

Add a name to each tuple element. (No automatic code fix — choosing the names is a
manual change.)
