# SST1415 — Use nameof for parameter references

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A string literal that names a parameter is passed to an `Argument…Exception`
constructor, where `nameof` would track renames.

```csharp
// 👎 reported
public void M(string value)
{
    if (value is null) throw new ArgumentNullException("value");
}

// 👍
public void M(string value)
{
    if (value is null) throw new ArgumentNullException(nameof(value));
}
```

## Rule description

`nameof(value)` produces the same string as `"value"` but is updated automatically
if the parameter is renamed, and is verified by the compiler. The rule is
intentionally narrow to avoid false positives: it only fires for a string literal
that is **both** an argument to an `Argument…Exception` constructor **and** equal
to the name of a parameter in scope (a method, constructor, local function,
indexer, or lambda parameter). A message string that merely contains the name is
not reported.

## How to fix violations

Replace the string literal with `nameof(parameter)`. The provided code fix does
this automatically.
