# SST1416 — Do not declare public members in a non-public type

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | Yes |
| Default | Disabled (opt-in) |

## Cause

A `public` member is declared in a type that is not externally visible (an
internal, private, or file-local type, or one nested in such a type).

```csharp
// 👎 reported (when enabled)
internal class C
{
    public void M() { }
}

// 👍
internal class C
{
    internal void M() { }
}
```

## Rule description

The effective accessibility of a member is capped by its containing type, so a
`public` member of an `internal` type is really only internal — the `public`
modifier overstates its reach. Declaring it `internal` makes the real
accessibility obvious.

The following are **not** reported, because they must stay public:

- members that implement an interface member; and
- operators and conversion operators (which the language requires to be public).

**This rule is off by default** — declaring members `public` inside an internal
type is a widespread, harmless habit, so adopting the stricter style is opt-in.
Enable it in `.editorconfig`:

```ini
[*.cs]
dotnet_diagnostic.SST1416.severity = warning
```

## How to fix violations

Change the member's `public` modifier to `internal`. The provided code fix does
this automatically.
