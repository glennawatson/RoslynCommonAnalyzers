# SST1417 — Namespace should match the folder structure

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | No |
| Default | Disabled (opt-in) |

## Cause

A top-level namespace does not mirror the file's folder path under the project
directory.

```text
Project root (RootNamespace = MyApp)
└── Models/
    └── Widget.cs
```

```csharp
// 👎 reported — file is in Models/, so the namespace should be MyApp.Models
namespace MyApp.Things;

// 👍
namespace MyApp.Models;
```

## Rule description

The expected namespace is the **root namespace** followed by each folder between
the project directory and the file — the same convention as the SDK's `IDE0130`.
Aligning namespaces with folders makes types easy to locate.

The project directory and root namespace are read from the compiler-provided
`build_property.ProjectDir` and `build_property.RootNamespace` options, so the rule
does nothing when those are unavailable (for example, outside an SDK-style build).
Folders that are not valid identifiers cause the file to be skipped rather than
misreported. Only top-level namespace declarations are checked.

**This rule is off by default** — folder/namespace alignment is a project choice,
and renaming a namespace is a cross-file refactor. Enable it in `.editorconfig`:

```ini
[*.cs]
dotnet_diagnostic.SST1417.severity = warning
# Override the root namespace if it differs from the MSBuild RootNamespace:
stylesharp.namespace_root = MyApp
```

## How to fix violations

Rename the namespace (and move the file, or adjust the folders) so the two match.
(No automatic code fix — changing a namespace is a cross-file refactor that the
IDE's "Sync namespace" command handles safely.)
