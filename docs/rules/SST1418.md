# SST1418 — Declare precedence when mixing the null-coalescing operator

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A binary expression is used as an operand of the `??` operator without
parentheses.

```csharp
// 👎 reported
var x = a ?? b + c;
var y = first + second ?? fallback;

// 👍
var x = a ?? (b + c);
var y = (first + second) ?? fallback;
```

## Rule description

The null-coalescing operator `??` has lower precedence than every arithmetic,
shift, comparison, and logical operator, so when it is mixed with one the reader
has to recall the precedence rules. Parenthesizing the binary operand makes the
grouping explicit. This extends the precedence family [SST1407](SST1407.md) /
[SST1408](SST1408.md) to `??` (StyleSharp original, inspired by StyleCop issue
#3890). Chained `a ?? b ?? c` is not reported.

## How to fix violations

Parenthesize the binary operand. The shared precedence code fix does this
automatically.
