# SST1421 — Write-only properties should not be used

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A property declares `set` or `init` without a getter.

```csharp
public string Password { set => Store(value); }
```

## Rule description

Write-only properties are surprising to callers. A method communicates a write-only
operation more clearly. Overrides and explicit interface implementations are excluded.

## How to fix violations

Add a getter or replace the property with a purpose-named method.
