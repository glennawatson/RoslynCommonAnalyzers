# SST1422 — Private fields used only as locals should be local variables

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | No |
| Default | Disabled (opt-in) |

## Cause

A private field is reset at the start of the only method that references it.

## Rule description

Such a field does not represent object state and can normally be local storage. The
analysis is conservative and excludes partial types, initialized fields, closures, and
fields referenced by more than one method.

## How to fix violations

Declare an appropriately initialized local variable in the method and remove the field.
