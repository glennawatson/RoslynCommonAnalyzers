# SST1423 — Switch statements should not have too many sections

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | No |
| Default | Enabled; maximum 30 sections |

## Cause

A switch statement contains more sections than the configured maximum.

## Rule description

Very large switches are often clearer as lookup tables, smaller dispatch methods, or
polymorphic implementations. Stacked labels count as one section.

## Configuration

```ini
stylesharp.max_switch_sections = 30
stylesharp.SST1423.max_switch_sections = 20
```

The rule-specific value takes precedence.

## How to fix violations

Split the switch or replace it with a more suitable dispatch abstraction.
