# SST1424 — Fields that are never reassigned should be readonly

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | Yes |
| Default | Disabled (opt-in) |

## Cause

A private instance field is assigned only by its initializer or instance constructors.

## Rule description

`readonly` records the intended immutability and prevents later reassignment. Partial
types, static fields, volatile fields, closure writes, and writes outside constructors
are excluded.

A non-readonly struct field that is mutated in place outside the constructor is also
excluded. Calling a non-readonly instance member on a `readonly` struct field runs against
a defensive copy, so the mutation would be silently discarded — making the field `readonly`
would be a behavior change, not a cleanup. The field is left alone when, outside the
constructor, it is the receiver of a non-readonly method, a property/indexer write or a
non-readonly getter, or is passed by `ref`/`out`. A `readonly struct` field, or one only
read through a `readonly` member, is still reported.

## How to fix violations

Add the `readonly` modifier. The provided code fix does this automatically.
