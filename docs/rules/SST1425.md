# SST1425 — Do not reassign captured primary-constructor parameters

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A class or struct primary-constructor parameter is reassigned, incremented,
decremented, or passed as `ref`/`out`.

```csharp
// 👎 reported
public class Counter(int count)
{
    public void Reset()
    {
        count = 0;
        count++;
    }
}

// 👍
public class Counter(int count)
{
    private int _count = count;

    public void Reset()
    {
        _count = 0;
    }
}
```

## Rule description

On classes and structs, primary-constructor parameters can be captured into
hidden mutable state. Reassigning one later does not mutate a normal local
parameter slot; it mutates that hidden captured field, which is surprising and
easy to misread.

This rule applies only to class and struct primary constructors. Record primary
constructor parameters are excluded because they represent init-only properties
with different semantics.

## How to fix violations

Move the mutable state into an explicit field or local variable, and stop
reassigning the primary-constructor parameter directly. (No automatic code fix
— the right replacement depends on the intended design.)
