# SST1426 — Use [SuppressMessage] instead of #pragma warning disable

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A `#pragma warning disable` directive silences an analyzer warning (for example
`SST1309`) where a scoped `[SuppressMessage]` attribute would be reviewable and
bounded to a single declaration.

```csharp
// 👎 reported — the disable leaks until the matching restore
#pragma warning disable SST1309
private int field;
#pragma warning restore SST1309

// 👍 scoped to the member it applies to
[System.Diagnostics.CodeAnalysis.SuppressMessage("Naming", "SST1309", Justification = "<Pending>")]
private int field;
```

## Rule description

A `#pragma warning disable` directive turns a rule off for every line until a
matching `restore` (or the end of the file). The suppression is easy to leave
open by accident, hides which declaration it was meant for, and carries no
explanation. A `[SuppressMessage]` attribute is attached to one declaration,
travels with it through refactors, and records a `Justification`.

Compiler warnings are exempt. A directive whose codes are **all** compiler
warnings — `CS####` such as `CS0168`, or a bare numeric code such as `0168` — is
not reported, because the C# compiler only honours `#pragma` for those and there
is no attribute equivalent. A directive that mixes a compiler code with an
analyzer code is still reported (the analyzer code could move), but the code fix
is not offered for it.

## How to fix violations

Replace the directive with a `[SuppressMessage]` attribute on the nearest
enclosing declaration. The provided code fix does this automatically: it adds one
`[SuppressMessage]` per suppressed code to the closest method, property, field,
or type, removes the `disable` directive and its matching `restore`, and seeds a
`Justification = "<Pending>"` for you to complete. The fix is offered only when
every code on the directive is an analyzer id; directives that also carry a
compiler (`CS`) code are left for you to handle by hand.
