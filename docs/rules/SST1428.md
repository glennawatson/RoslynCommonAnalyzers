# SST1428 — Abstract types should not declare public constructors

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `TypeDesignAnalyzer` |

## Cause

An abstract type declares a `public` constructor. An abstract type cannot be instantiated
directly, so only a derived type can ever call the constructor — `public` is misleading.

```csharp
// 👎 reported
public abstract class Shape
{
    public Shape()
    {
    }
}

// 👍
public abstract class Shape
{
    protected Shape()
    {
    }
}
```

## Rule description

A constructor is reported when it carries the `public` modifier and its containing type is
`abstract`. A `protected` (or private) constructor states clearly that only derived types call
it. Static constructors, which have no accessibility, are not affected.

`abstract` need only appear on one part of a partial type, so a constructor is reported even
when its own part omits the keyword:

```csharp
// 👎 reported: A is abstract, so this public constructor is unreachable
public abstract partial class A
{
}

public partial class A
{
    public A()
    {
    }
}
```

## How to fix violations

Change `public` to `protected`. The provided code fix does this automatically.
