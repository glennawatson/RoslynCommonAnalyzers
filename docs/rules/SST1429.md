# SST1429 — Empty catch clauses should not swallow the base exception

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |
| Analyzer | `ExceptionHandlingAnalyzer` |

## Cause

A `catch (Exception)` — or a bare `catch` — has an empty body, so it silently discards every
error, including ones the code is not prepared for.

```csharp
// 👎 reported
try
{
    Process();
}
catch (Exception)
{
}

// 👍
try
{
    Process();
}
catch (Exception ex)
{
    logger.LogError(ex);
    throw;
}
```

## Rule description

The clause is reported when its body has no statements and either it is a bare `catch { }` or
its declared type is the base `System.Exception`. A `when` filter, a narrower exception type,
or any statement in the body all mark the catch as deliberate and are left alone. The empty
body is checked syntactically; the caught type is confirmed against `System.Exception`
semantically.

No automatic code fix is offered, because the right handling depends on intent — log, rethrow,
or narrow the caught type.

## How to fix violations

Handle the exception, rethrow it, or catch a narrower type. If the exception genuinely must be
ignored, add a comment and a `when` filter or a narrower type that documents the intent.
