# SST1430 тАФ Rethrow with 'throw;' to preserve the stack trace

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `ExceptionHandlingAnalyzer` |

## Cause

A `catch` block re-throws the caught exception with `throw ex;`. That resets the exception's
stack trace to the rethrow location, hiding where the error actually came from.

```csharp
// ЁЯСО reported
try
{
    Process();
}
catch (Exception ex)
{
    Cleanup();
    throw ex;        // stack trace now starts here
}

// ЁЯСН
catch (Exception ex)
{
    Cleanup();
    throw;           // original stack trace preserved
}
```

## Rule description

A `throw ex;` is reported when `ex` is the variable of the directly enclosing `catch` clause,
so a bare `throw;` would rethrow the same exception while keeping its original trace. The rule
checks the thrown identifier against the catch variable syntactically and confirms they are the
same symbol semantically, so throwing a different exception (`throw new ...(ex)`) is left alone.
A `throw;` inside a lambda or local function тАФ where the bare form is not valid тАФ is not
reported.

## How to fix violations

Replace `throw ex;` with `throw;`. The provided code fix does this automatically.
