# SST1431 — Static members of a generic type should use a type parameter

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |
| Analyzer | `TypeDesignAnalyzer` |

## Cause

A static, externally visible member of a generic type has a signature that never mentions the
type's type parameters. Callers must pick an arbitrary type argument to reach it
(`Cache<int>.Clear()`), even though the argument is irrelevant.

```csharp
// 👎 reported
public class Cache<T>
{
    public static void Clear()      // 'T' appears nowhere in the signature
    {
    }
}

// 👍 — move the member to a non-generic type
public static class Cache
{
    public static void Clear()
    {
    }
}
```

## Rule description

The rule reports a `public`, `internal`, or `protected` static field, method, property, or
event of a generic type that references none of the enclosing type's type parameters
**anywhere** in its declaration — signature, attributes, initializer, accessor, or body.
Members that do use a type parameter, and private static helpers, are left alone.

A member counts as using a type parameter when it names one anywhere, **including inside a
closed generic type** such as `typeof(Owner<T>)`. This is what keeps the common dependency-
property pattern clean — a registration whose value type is unrelated to `T` but whose owner
argument is the closed self-type still references `T`:

```csharp
// 👍 not reported — the owner argument names TViewModel
public static readonly BindableProperty ViewModelProperty = BindableProperty.Create(
    nameof(ViewModel), typeof(object), typeof(ReactiveView<TViewModel>));
```

Each enclosing type parameter name (the member's own type and any outer generic type) is matched
against the identifiers across the whole member, behind the cheap static-member-of-generic-type gate.

### Property-system registrations

A static field or property whose **type** is a known property-system registration type is exempt
even when it never names a type parameter (for example a member that forwards to a nested holder).
The registration must be a static field by design, so relocating it is not an option. The built-in
list covers WPF/WinUI `DependencyProperty`, MAUI `BindableProperty`, and Avalonia
`AvaloniaProperty` / `StyledProperty<>` / `DirectProperty<,>` / `AttachedProperty<>`. Add framework
types of your own with the
[`stylesharp.SST1431.additional_per_owner_types`](../CONFIGURATION.md#sst1431) option.

No automatic code fix is offered, because the right move — relocating the member to a
non-generic type — depends on the surrounding design.

## How to fix violations

Reference a type parameter in the member's signature, or move the member to a non-generic
(possibly companion) type.
