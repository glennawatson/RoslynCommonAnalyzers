# SST1432 — Classes with only static members should be static

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | Yes |
| Default | Disabled (opt-in) |
| Analyzer | `TypeDesignAnalyzer` |

## Cause

A class declares only static members yet is not marked `static`, so it can still be
instantiated and inherited from for no reason.

```csharp
// 👎 reported
public class MathHelpers
{
    public static int Square(int x) => x * x;
}

// 👍
public static class MathHelpers
{
    public static int Square(int x) => x * x;
}
```

## Rule description

The class is reported when every member is static-compatible — a static member, a constant, a
nested type, or a static constructor — and the class has no base list, no primary constructor,
and is not already `static` or `abstract`. Marking it `static` forbids instantiation and
inheritance and documents the intent.

This rule is **disabled by default**. Marking an already-published, instantiable type `static`
is a binary-breaking change, and a type may be instantiated in code this rule cannot see, so it
is opt-in. Enable it in `.editorconfig`:

```ini
dotnet_diagnostic.SST1432.severity = warning
```

## How to fix violations

Mark the class `static`. The provided code fix adds the modifier.
