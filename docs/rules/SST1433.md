# SST1433 — Redundant constructors should be removed

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `EmptyCodeAnalyzer` |

## Cause

A type's only constructor is a public, parameterless, empty constructor — exactly the default
constructor the compiler would emit anyway.

```csharp
// 👎 reported
public sealed class Widget
{
    public Widget()
    {
    }
}

// 👍
public sealed class Widget
{
}
```

## Rule description

The constructor is reported only when it is the type's single constructor and is public, parameterless,
attribute-free, has an empty body, and has no `: base(...)`/`: this(...)` initializer. A private
constructor (which forbids instantiation), a parameterized constructor, or one of several constructors
all carry meaning and are left alone. The check is purely syntactic.

## How to fix violations

Delete the constructor. The provided code fix does this automatically.
