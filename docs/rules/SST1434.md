# SST1434 — Empty finalizers should be removed

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `EmptyCodeAnalyzer` |

## Cause

A finalizer has an empty body, so it does no cleanup yet still forces the runtime to track the object
on the finalization queue.

```csharp
// 👎 reported
public class Resource
{
    ~Resource()
    {
    }
}

// 👍
public class Resource
{
}
```

## Rule description

A destructor whose body is an empty block is reported. An empty finalizer delays collection and serves
no purpose. The check is purely syntactic.

## How to fix violations

Delete the finalizer. The provided code fix does this automatically.
