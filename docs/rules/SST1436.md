# SST1436 — Empty types should not be declared

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | No |
| Default | Disabled (opt-in) |
| Analyzer | `EmptyCodeAnalyzer` |

## Cause

A class, struct, or record declares no members.

```csharp
// 👎 reported (opt-in)
public class Settings
{
}
```

## Rule description

A class, struct, or record with no members, no base list, and no primary-constructor parameter list is
reported. An empty type is usually an oversight. This rule is **disabled by default**, because empty
types are sometimes deliberate — markers, placeholder DTOs, or extension points. Enable it in
`.editorconfig`:

```ini
dotnet_diagnostic.SST1436.severity = warning
```

## How to fix violations

Add the intended members, or delete the type.
