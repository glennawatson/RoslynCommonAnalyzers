# SST1437 — Empty interfaces should not be declared

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | No |
| Default | Disabled (opt-in) |
| Analyzer | `EmptyCodeAnalyzer` |

## Cause

An interface declares no members and inherits none, so it defines no contract.

```csharp
// 👎 reported (opt-in)
public interface IEntity
{
}
```

## Rule description

An interface with no members and no base interfaces is reported. An empty interface defines no contract
and is often better expressed with an attribute. This rule is **disabled by default**, because marker
interfaces are an accepted pattern. Enable it in `.editorconfig`:

```ini
dotnet_diagnostic.SST1437.severity = warning
```

## How to fix violations

Add the intended members, derive from the interfaces that form the contract, or replace the marker with
an attribute.
