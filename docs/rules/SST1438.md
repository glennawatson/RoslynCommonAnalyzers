# SST1438 — Methods should not be empty

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | No |
| Default | Disabled (opt-in) |
| Analyzer | `EmptyCodeAnalyzer` |

## Cause

A method has an empty body, which usually signals unfinished work.

```csharp
// 👎 reported (opt-in)
public void Flush()
{
}
```

## Rule description

A method with an empty block body is reported. Several legitimately-empty shapes are excluded:

- `override` and `virtual` methods (common hooks), and methods carrying attributes;
- `abstract`/`partial`/`extern` methods, which have no body at all;
- methods that **implement an interface member** — an empty no-op such as a null `IDisposable.Dispose`
  or a stub event handler is intentional;
- methods whose body contains a **comment** documenting why it is intentionally empty.

That last point is the escape hatch named in the message ("document why it is intentionally empty"): a
single `// …` inside the body satisfies the rule.

```csharp
public void Dispose()
{
    // Intentionally empty: nothing to release.
}
```

This rule is **disabled by default**. Enable it in `.editorconfig`:

```ini
dotnet_diagnostic.SST1438.severity = warning
```

## How to fix violations

Implement the method, or document why it is intentionally empty (for example with an explanatory
comment, or by throwing `NotSupportedException`).
