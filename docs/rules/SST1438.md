# SST1438 — Methods should not be empty

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | No |
| Default | Disabled (opt-in) |
| Analyzer | `EmptyCodeAnalyzer` |

## Cause

A method has an empty body, which usually signals unfinished work.

```csharp
// 👎 reported (opt-in)
public void Flush()
{
}
```

## Rule description

A method with an empty block body is reported. To avoid noise, `override` and `virtual` methods (common
legitimate hooks) and methods carrying attributes are skipped, as are `abstract`/`partial`/`extern`
methods that have no body at all. This rule is **disabled by default**. Enable it in `.editorconfig`:

```ini
dotnet_diagnostic.SST1438.severity = warning
```

## How to fix violations

Implement the method, or document why it is intentionally empty (for example with an explanatory
comment, or by throwing `NotSupportedException`).
