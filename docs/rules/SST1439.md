# SST1439 — Nested code blocks should not be left empty

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |
| Analyzer | `EmptyCodeAnalyzer` |

## Cause

A loop or guard statement has an empty embedded block, which usually means missing code or a stray
semicolon.

```csharp
// 👎 reported
while (TryAdvance())
{
}

// 👍 — do the work in the body, or restructure the loop
```

## Rule description

An empty block used as the body of a loop (`for`, `foreach`, `while`, `do`) or guard (`if` then-branch,
`lock`, `fixed`, `using`) is reported. An empty `else` is left to its dedicated rule
([SST1180](SST1180.md)), and method, constructor, and accessor bodies are not treated as nested blocks.
The check is purely syntactic.

No automatic code fix is offered, because the right resolution — fill the body, remove the statement, or
restructure — depends on intent.

## How to fix violations

Add the intended statements, or remove the empty loop or guard.
