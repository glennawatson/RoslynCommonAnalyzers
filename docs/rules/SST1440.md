# SST1440 - Remove private members with no local use

Private members are implementation details. When a private member has no reads, calls, or local references
inside its declaring type, it adds dead surface area that reviewers and maintainers still have to scan.

## Example

```csharp
private void Unused()
{
}
```

Remove the member when it has no local use.

## Notes

The analyzer scans one type at a time and only binds identifier names that match private candidates. Partial
types include references from generated declarations, so code-behind members used by generated component code are
kept. Members with attributes are skipped because reflection or serialization frameworks may depend on them.
