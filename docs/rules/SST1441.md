# SST1441 - Remove private fields that are never read

A private field that is assigned but never read does not represent observable object state. Keeping the field
and its writes makes the type look stateful while the value is actually dead.

## Example

```csharp
private int _value;

public void Reset()
{
    _value = 1;
}
```

Remove the field or the dead writes that feed it.

## Notes

The analyzer tracks simple reads and writes inside the declaring type. Attributed members are skipped to
avoid flagging fields consumed by external frameworks.
