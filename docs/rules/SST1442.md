# SST1442 - Functions should keep branching complexity low

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | No |
| Default | Enabled; maximum 10 branch points |
| Analyzer | `FunctionComplexityAnalyzer` |

## Cause

A method, constructor, operator, accessor, local function, or expression-bodied property contains more
direct branch points than the configured maximum.

## Rule description

Branching complexity is a lightweight count of the decisions a reader has to account for while reading a
function. The rule counts conditionals, loops, null-coalescing branches, conditional access, logical
operators, switch constructs, and pattern combinations.

Wide switch statements and switch expressions count as one branch point, not one point per case or arm.
They are often lookup-table-shaped code, so penalizing every limb produces noisy diagnostics without
matching the actual reading cost.

## Configuration

```ini
stylesharp.max_cyclomatic_complexity = 10
stylesharp.SST1442.max_cyclomatic_complexity = 12
```

The rule-specific value takes precedence.

## How to fix violations

Split unrelated decisions into helper methods, flatten nested conditions with guard clauses, or move
table-like dispatch into a switch or dictionary when that better expresses the logic.
