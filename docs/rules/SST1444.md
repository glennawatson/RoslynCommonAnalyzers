# SST1444 - Loops should be able to iterate again

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |
| Analyzer | `SingleIterationLoopAnalyzer` |

## Cause

A loop body contains an unconditional `break`, `continue`, `return`, or `throw` that prevents the loop from
naturally reaching a second iteration.

## Rule description

A loop that always jumps before it can iterate again usually hides straight-line code behind loop syntax.
The analyzer reports unconditional jumps in `for`, `foreach`, `while`, and `do` loops.

Jumps inside nested loops, lambdas, anonymous methods, and local functions are not treated as jumps from
the outer loop. A `break` that exits a nested `switch` is not treated as a loop exit.

## How to fix violations

Replace the loop with direct code, move the jump behind a real condition, or restructure the loop so at
least one path can continue to the next iteration.
