# SST1445 — Remove unnecessary using directives

**Category:** Maintainability
**Severity:** Warning
**Code fix:** Removes the using directive (supports Fix All)

A using directive imports a namespace, static type, or alias that nothing in
the file resolves through. Dead imports slow down reading, invite name
collisions, and hide which dependencies the file really has.

The scan marks each directive used the moment a bound symbol resolves through
it and stops as soon as every directive is accounted for, so files whose usings
are all needed pay almost nothing. Extension members — both classic
`this`-parameter methods and C# 14 `extension(...)` block members — query
syntax, foreach enumerators, deconstructions, collection-initializer `Add`s,
awaiter lookups, and XML doc `cref`s all count as usage. `global using`
directives are never reported, and when usage cannot be proven either way the
directive is treated as used — the rule under-reports rather than suggesting a
removal that would break the build.

## Example

```csharp
using System;
using System.Text;   // SST1445: The using directive for 'System.Text' is unnecessary

public class C
{
    public void M() => Console.WriteLine("x");
}
```
