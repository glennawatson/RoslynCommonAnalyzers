# SST1446 — Inheritance chains should stay shallow

**Category:** Maintainability
**Severity:** Warning
**Code fix:** None

A class sits deeper in an inheritance chain than the configured maximum
(default **5**). Every inheritance level adds state and behavior a reader must
load to understand the leaf type; deep chains are rigid and hard to change.
Prefer composition or flatter hierarchies.

By default only ancestors declared in the analyzed assembly count, so deriving
from a framework base like `Exception` is free. Set
`stylesharp.SST1446.count_external_types = true` to count every ancestor below
`object`.

## Example

```csharp
public class C0 { }
public class C1 : C0 { }
public class C2 : C1 { }
public class C3 : C2 { }
public class C4 : C3 { }
public class C5 : C4 { }

// SST1446: 'C6' is 6 levels deep in its inheritance chain, which exceeds the maximum of 5
public class C6 : C5 { }
```

## Configuration

```ini
# .editorconfig
[*.cs]
stylesharp.SST1446.max_inheritance_depth = 5
stylesharp.SST1446.count_external_types = false

# Applies without a rule id too
stylesharp.max_inheritance_depth = 5
```
