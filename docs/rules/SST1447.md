# SST1447 — Equality overrides should not delegate to object

**Category:** Maintainability
**Severity:** Warning
**Code fix:** None

An `Equals` or `GetHashCode` member calls `base.Equals(...)` or
`base.GetHashCode()` and that call binds to `object`'s implementation. Object's
implementation compares by reference identity, so the base call silently
reintroduces reference semantics inside an override whose purpose is value
semantics. Compare the members that define equality instead, or drop the
override if reference semantics are actually intended.

A base call that binds to a real base-class override is meaningful and never
reported.

## Example

```csharp
public class C
{
    // SST1447: 'Equals' calls the object implementation of 'Equals'
    public override bool Equals(object obj) => base.Equals(obj);

    public override int GetHashCode() => 42;
}
```
