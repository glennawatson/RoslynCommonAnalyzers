# SST1448 — Let the compiler supply caller-info arguments

**Category:** Maintainability
**Severity:** Warning
**Code fix:** Removes the argument when the removal cannot shift other arguments

An argument is passed explicitly to a parameter marked `[CallerMemberName]`,
`[CallerFilePath]`, or `[CallerLineNumber]`. Those parameters exist so the
compiler injects the real call site; passing a value defeats the mechanism and
usually reports the wrong caller.

Forwarding your own caller-info parameter onward is the intended propagation
pattern and is never reported.

## Example

```csharp
public void Log(string message, [CallerMemberName] string caller = "") { }

public void M()
{
    Log("text", "MyCaller"); // SST1448: Remove this argument so the compiler supplies the caller's member name
    Log("text");             // ✓ compiler supplies "M"
}

public void Outer(string message, [CallerMemberName] string caller = "")
    => Log(message, caller); // ✓ forwarding the original call site
```
