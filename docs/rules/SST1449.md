# SST1449 — Avoid writing directly to the console

**Category:** Maintainability
**Severity:** Warning
**Code fix:** None

Code calls `Console.Write` or `Console.WriteLine` directly. Console writes
bypass log levels, sinks, and redirection, and turn into noise or silently lost
output when the code runs without an attached console (services, tests, GUI
hosts). Route diagnostics through the application's logging abstraction.

For genuine console applications where direct output is the product, disable
the rule for that project:

```ini
[*.cs]
dotnet_diagnostic.SST1449.severity = none
```

## Example

```csharp
public void Process()
{
    Console.WriteLine("processed"); // SST1449: Replace this 'Console.WriteLine' call with the application's logging abstraction
    _logger.LogInformation("processed"); // ✓
}
```
