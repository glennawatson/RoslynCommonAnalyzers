# SST1450 — Store files as UTF-8 without a byte order mark

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning (disabled by default) |
| Code fix | No |

## Cause

A source file is stored as UTF-8 with a byte order mark.

## Rule description

A StyleSharp-original rule and the inverse of [SST1412](SST1412.md) — many .NET projects
prefer UTF-8 without a BOM. The two rules are mutually exclusive; enable at most one. Both
are off by default.

## How to fix violations

Save the file as UTF-8 without a byte order mark.
