# SST1451 ‚Äî Specify a DateTimeKind when creating a DateTime

**Category:** Maintainability
**Severity:** Warning
**Code fix:** None

A `DateTime` is constructed without stating its kind. The result is
`DateTimeKind.Unspecified`, and later conversions (`ToUniversalTime`,
`ToLocalTime`, serialization) silently guess whether the value was UTC or
local ‚Äî the classic source of off-by-timezone bugs. State the kind, or use
`DateTimeOffset` which carries its offset explicitly.

The parameterless constructor is not reported (it is just
`default(DateTime)`), and neither are overloads that take a `DateTimeKind`.

## Example

```csharp
var a = new DateTime(2024, 1, 1);                          // SST1451
var b = new DateTime(2024, 1, 1, 0, 0, 0, DateTimeKind.Utc); // ‚úì
var c = new DateTimeOffset(2024, 1, 1, 0, 0, 0, TimeSpan.Zero); // ‚úì
```
