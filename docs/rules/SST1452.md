# SST1452 — Remove unused type parameters

**Category:** Maintainability
**Severity:** Warning
**Code fix:** None (changing arity is a breaking change to make deliberately)

A generic type parameter is declared but nothing in the declaration references
it — not the signature, not the body, not another parameter's constraints. It
forces every caller to supply a meaningless type argument and usually marks a
refactoring leftover.

Declarations whose arity is not theirs to change are skipped: `partial` types
and methods, `override`/`virtual`/`abstract` members, and explicit interface
implementations. The check is a single syntactic token scan — no semantic
binding.

## Example

```csharp
public void M<T>(int value) { }   // SST1452: The type parameter 'T' is never used; remove it
public T Wrap<T>(T value) => value; // ✓
```
