# SST1453 - Remove unreachable statements

**Category:** Maintainability
**Severity:** Warning
**Code fix:** None

A statement after an unconditional exit in the same block cannot execute. Keeping
it in the source makes the reachable path harder to read and often means a
previous refactor left code behind.

The analyzer checks the simple block-local cases: statements after `return`,
`throw`, `break`, or `continue`. It does not try to prove arbitrary control-flow
facts across nested branches.

## Example

```csharp
void Save(bool enabled)
{
    if (!enabled)
    {
        return;
        LogDisabled(); // SST1453
    }
}
```

Remove the unreachable statement or move it before the exit if it should still
run.
