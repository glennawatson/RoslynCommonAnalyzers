# SST1453 - Remove unreachable statements

**Category:** Maintainability
**Severity:** Warning
**Code fix:** None

A statement after an unconditional exit in the same block cannot execute. Keeping
it in the source makes the reachable path harder to read and often means a
previous refactor left code behind.

The analyzer checks the simple block-local cases: statements after `return`,
`throw`, `break`, `continue`, or `goto`. It does not try to prove arbitrary
control-flow facts across nested branches.

## What is not reported

Two statement kinds after an exit are not unreachable, and the compiler raises no
CS0162 for them either:

- **Local functions**, which are hoisted rather than executed at their lexical
  position. Declaring them after the terminating `return` is idiomatic C#.
- **Labeled statements**, which a `goto` can enter. A label restores reachability
  for the statements that follow it.

```csharp
static int Sum(int[] values)
{
    var total = 0;
    foreach (var v in values)
    {
        total = Add(total, v);
    }

    return total;

    static int Add(int a, int b) => a + b; // not reported: hoisted
}
```

Code after a hoisted local function is still reported, because the local function
does not make it reachable.

## Example

```csharp
void Save(bool enabled)
{
    if (!enabled)
    {
        return;
        LogDisabled(); // SST1453
    }
}
```

Remove the unreachable statement or move it before the exit if it should still
run.
