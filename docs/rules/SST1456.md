# SST1456 - Avoid readonly fields of mutable structs

**Category:** Maintainability
**Severity:** Warning
**Code fix:** None

A readonly field of a mutable source-defined struct can produce defensive-copy
surprises when members are accessed. The field looks immutable, but the value's
type still exposes mutation.

The analyzer reports readonly fields whose struct type is defined in source and
is not itself readonly. External structs are skipped because their mutability
contract is harder to audit cheaply and safely.

## Example

```csharp
struct MutableCounter
{
    public int Value { get; set; }
}

sealed class Holder
{
    private readonly MutableCounter _counter; // SST1456
}
```

Make the struct readonly when it is intended to be immutable, or remove
`readonly` from the field so the mutability is explicit.
