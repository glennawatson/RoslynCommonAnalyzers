# SST1460 - Mark non-mutating struct members readonly

**Category:** Maintainability
**Severity:** Warning
**Code fix:** None

An instance member on a mutable struct can be marked `readonly` when its body
does not mutate state. That lets callers use it through readonly receivers
without a defensive copy and documents the member's non-mutating contract.

The analyzer deliberately under-reports. Calls, assignments, increment/decrement
operations, and `ref`/`out` arguments are treated as possible mutation.

## Example

```csharp
public struct Counter
{
    private int _value;
    public int Value() => _value; // SST1460
}
```

Use:

```csharp
public readonly int Value() => _value;
```
