# SST1461 - Remove unread private parameters

**Category:** Maintainability
**Severity:** Warning
**Code fix:** None

A parameter on a private method or local function that is never read is dead
local API surface. Removing it keeps call sites and implementation intent in
sync.

The analyzer is scoped to private methods and local functions. Public,
overriding, virtual, abstract, partial, attributed, and explicit-interface
members are skipped because their signatures can be consumed outside the local
body.

Conventional event handlers are also exempt: a method taking exactly
`(object sender, EventArgs e)` — where the first parameter is `object` or
`object?` and the second derives from `System.EventArgs` — keeps both parameters
even when one is unread, because the delegate the handler is subscribed to fixes
the signature and dropping a parameter would break the subscription. Callbacks
bound to other delegate shapes (for example a bare `object state` continuation)
are not detected, since that needs whole-program usage analysis; suppress those
locally or rename the parameter to `_`.

## Example

```csharp
private int M(int value) => 1; // SST1461
```

Use:

```csharp
private int M() => 1;
```
