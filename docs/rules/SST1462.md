# SST1462 - Remove suppressions for disabled diagnostics

**Category:** Maintainability
**Severity:** Warning
**Code fix:** None

A `SuppressMessage` attribute for a diagnostic configured to `none` or `silent`
cannot suppress an active report. Keeping it in source makes the suppression
history look current when the rule is already disabled by policy.

The analyzer reads the active compilation diagnostic options and analyzer config
scope for the suppressed diagnostic id.

## Example

```csharp
[SuppressMessage("Style", "SST9999:Disabled rule", Justification = "Reviewed.")] // SST1462 when SST9999 is disabled
public sealed class C;
```
