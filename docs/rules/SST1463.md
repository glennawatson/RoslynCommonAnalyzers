# SST1463 - Use nameof for symbol-name strings

**Category:** Maintainability
**Severity:** Warning
**Code fix:** None

A string literal passed to a name-shaped parameter should use `nameof` when it
matches an in-scope symbol. That keeps rename operations correct and avoids
drifting notification or metadata strings.

The analyzer only performs a symbol lookup after the literal is passed to a
parameter whose name contains `name`, which keeps ordinary message strings out of
scope.

## Example

```csharp
public int Count { get; set; }
void M() => Notify("Count"); // SST1463 when Notify takes propertyName
```

Use:

```csharp
void M() => Notify(nameof(Count));
```
