# SST1468 - Boolean logic should short-circuit

**Category:** Maintainability
**Severity:** Warning
**Code fix:** Replaces the operator with `&&` / `||` (supports Fix All)

The bitwise `&` and `|` operators evaluate both operands even when the left
operand already decides the result. On boolean operands the conditional `&&`
and `||` forms compute the same value and stop as soon as the answer is known,
which reads as the intended logic and avoids pointless evaluation.

The analyzer reports only when the right operand is side-effect-free —
identifiers, member-access chains, literals, logical negation, parenthesized
forms of those, and simple comparisons between them — so skipping its
evaluation cannot change behavior. A right operand that does work (an
invocation, element access, assignment, or similar) keeps the eager operator,
and lambdas converted to expression trees are left alone.

## Example

```csharp
bool CanRun(bool enabled, bool ready)
{
    return enabled & ready; // SST1468
}
```

Use:

```csharp
bool CanRun(bool enabled, bool ready)
{
    return enabled && ready;
}
```
