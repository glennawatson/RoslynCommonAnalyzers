# SST1470 - Remove a catch clause that only rethrows

**Category:** Maintainability
**Severity:** Warning
**Code fix:** Removes the catch clause; a bare try/catch is unwrapped entirely (supports Fix All)

A catch clause whose body is exactly `throw;` re-raises the exception it
caught without observing or changing it. When that clause declares no `when`
filter and is the last clause of its try statement, deleting it leaves the
exception propagating exactly as before, so the clause only adds noise around
the handlers that do work.

The analyzer reports only the **last** catch clause of a try statement. An
earlier rethrow-only clause is left alone, because removing it would let a
later, more general clause catch exceptions it previously never saw — a
behavior change. The catch declaration itself does not matter: `catch`,
`catch (Exception)`, and `catch (Exception ex)` are all reported, since a body
of exactly `throw;` cannot use a declared variable.

The code fix removes the reported clause. When other catch clauses remain, or
a `finally` clause exists, the rest of the try statement is kept as is; when
the clause was a bare try/catch's only handler, the whole try statement is
replaced by the statements of its try block.

## Example

```csharp
void M()
{
    try
    {
        Work();
    }
    catch (IOException)
    {
        Recover();
    }
    catch (Exception) // SST1470
    {
        throw;
    }
}
```

Use:

```csharp
void M()
{
    try
    {
        Work();
    }
    catch (IOException)
    {
        Recover();
    }
}
```
