# SST1471 — Magic numbers should be named constants

| Item | Value |
| --- | --- |
| Category | Maintainability |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A numeric literal appears in an expression without anything to explain what it means.

```csharp
public bool IsStale(TimeSpan age) => age > TimeSpan.FromMinutes(30);
```

## Rule description

A literal buried in an expression states a value but not its meaning, and the same value
repeated in two places cannot be changed in one. Naming it turns the value into
documentation and gives it a single home.

```csharp
private static readonly TimeSpan CacheLifetime = TimeSpan.FromMinutes(30);

public bool IsStale(TimeSpan age) => age > CacheLifetime;
```

A literal is **not** reported when it already carries its meaning:

- **Its value is allowed.** `-1`, `0` and `1` by default — not-found, empty and first.
  Values are matched numerically, so `0`, `0.0`, `0L` and `0x0` are all zero, and `-1` is
  compared as `-1` rather than as `1`.
- **A declaration names it.** A `const` or `readonly` field, a `const` local, an `enum`
  member, an attribute argument, or a default parameter value. These exempt their whole
  initializer, so `static readonly TimeSpan Retry = TimeSpan.FromSeconds(30);` is clean —
  it is the fix this rule asks for.
- **It is the entire initializer** of a plain field, local, or property: `var timeout = 500;`.
  A literal buried deeper is still reported, because `var task = Task.Delay(500);` names
  the task, not the delay.
- **It is a bit pattern** written as hex or binary: `0xFF00FF`, `0b1010`.
- **It is a shift distance:** `hash >> 16`, `flags <<= 13`.
- **It guards a cardinality:** compared against `Count`, `Length`, `LongLength` or `Rank`,
  as in `if (args.Count < 2)`.
- **It is in a `GetHashCode` body**, where mixing primes like `397` are idiom, not policy.
- **It is a named argument:** `new List<int>(capacity: 4)`.
- **It is a buffer length:** `new byte[16]`, `stackalloc char[32]`.
- **It is an argument to a positional BCL constructor:** `DateTime`, `DateTimeOffset`,
  `TimeSpan`, `Version` and `Guid` take meaning from argument position, so
  `new DateTime(2025, 1, 1)` is clean.

A literal inside a lambda or local function does not inherit an enclosing declaration's
name, so `static readonly Func<int> F = () => 500;` still reports `500`.

Static factory methods are **not** exempt. `TimeSpan.FromMinutes(30)` names the unit, but
the magnitude is still a policy value that deserves a name. Case labels and constant
patterns are not exempt either — `case 3:` is where magic numbers hide.

## How to fix violations

Introduce a named constant, or move the value to a `const`/`static readonly` field.

## Configuration

Override the allowed values with a comma-separated list. An unset, empty or unparsable
value falls back to the defaults, so a typo never turns every literal into a diagnostic.

```ini
# Rule-specific
stylesharp.SST1471.magic_number_allowed_values = -1, 0, 1, 2

# Project-wide
stylesharp.magic_number_allowed_values = -1, 0, 1
```
