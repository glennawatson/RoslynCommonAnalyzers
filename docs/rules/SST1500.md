# SST1500 — Braces for multi-line statements should not share a line

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | Yes (place brace on its own line) |

## Cause

In a construct that spans multiple lines, an opening or closing brace shares its
line with other code.

## Rule description

Each brace of a multi-line block sits on its own line so the block structure is
unambiguous (Allman style). Single-line constructs are exempt — they are governed
by [SST1501](SST1501.md) and [SST1502](SST1502.md).

```csharp
// 👎 reported — the opening brace shares the declaration line
private void M() {
    Work();
}

// 👍 each brace on its own line
private void M()
{
    Work();
}
```

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

The code fix moves the brace onto its own line, indented to match its owner. It is
skipped when a comment sits between the brace and the code it shares a line with.
