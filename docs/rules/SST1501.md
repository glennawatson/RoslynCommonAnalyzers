# SST1501 — Statement should not be on a single line

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | Yes (expand onto multiple lines) |

## Cause

An embedded statement block — the body of an `if`, loop, `using`, `lock`,
`try`/`catch`/`finally`, or a bare nested block — is collapsed onto a single line
with its content.

## Rule description

Each statement in a block is written on its own line. Element bodies are governed by
[SST1502](SST1502.md) and accessor consistency by [SST1504](SST1504.md).

```csharp
// 👎 reported
if (enabled) { Work(); }

// 👍 expanded
if (enabled)
{
    Work();
}
```

## How to fix violations

The code fix moves the opening brace, each statement, and the closing brace onto
their own lines. It is skipped when a comment shares a line with the code.
