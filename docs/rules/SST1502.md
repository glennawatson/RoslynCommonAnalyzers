# SST1502 — Element should not be on a single line

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | Yes for block bodies (expand onto multiple lines) |

## Cause

A non-empty element body — a method, constructor, destructor, operator, local
function, or a type that holds members — is collapsed onto a single line.

## Rule description

A non-empty element body spans multiple lines. Empty bodies (`{ }`) and
expression-bodied members are exempt; embedded statement blocks are governed by
[SST1501](SST1501.md).

```csharp
// 👎 reported
private void M() { Work(); }

// 👍 expanded
private void M()
{
    Work();
}
```

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

The code fix expands a single-line method or local-function body onto multiple lines.
Type and enum bodies (whose members are not plain statements) are reported but not
auto-fixed.
