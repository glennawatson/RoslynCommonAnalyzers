# SST1503 — Braces should not be omitted

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | Yes (add braces) |

## Cause

A control-flow statement (`if`, `else`, `for`, `foreach`, `while`, `do`, `using`,
`lock`, `fixed`) has a child statement that is not wrapped in braces.

## Rule description

The child statement of a control-flow statement is always wrapped in braces, even when
it is a single statement. An `else if` chain is not treated as an unbraced child.

This is the strict always-braces rule. The repository disables it by default in favour
of [SST1519](SST1519.md)/[SST1520](SST1520.md), but it is provided for
projects that prefer the single combined rule.

```csharp
// 👎 reported
if (enabled)
    Work();

// 👍 braces
if (enabled)
{
    Work();
}
```

## How to fix violations

The code fix wraps the child statement in braces.
