# SST1504 — All accessors should be single-line or multi-line

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | Yes (expand accessors onto multiple lines) |

## Cause

Within one property, indexer, or event, the block-bodied accessors mix single-line
and multi-line forms.

## Rule description

Either every block-bodied accessor is written on a single line, or every one spans
multiple lines. Auto (`get;`) and expression-bodied (`get => x;`) accessors are
ignored.

```csharp
// 👎 reported — get is single-line, set is multi-line
public int X
{
    get { return _x; }
    set
    {
        _x = value;
    }
}

// 👍 consistent
public int X
{
    get
    {
        return _x;
    }
    set
    {
        _x = value;
    }
}
```

## How to fix violations

The code fix expands each single-line accessor body onto multiple lines so all
accessors are consistently multi-line.
