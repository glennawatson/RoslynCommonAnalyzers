# SST1505 — Opening brace should not be followed by a blank line

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | Yes (remove blank line) |

## Cause

A blank line immediately follows an opening brace.

## Rule description

An opening brace is followed directly by the first statement of its block, not by a
blank line.

```csharp
// 👎 reported
private void M()
{

    Work();
}

// 👍 no leading blank line
private void M()
{
    Work();
}
```

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

The code fix removes the blank line(s) directly after the opening brace.
