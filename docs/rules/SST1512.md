# SST1512 — Single-line comment should not be followed by a blank line

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | Yes (remove blank line) |

## Cause

A blank line follows a standalone single-line comment.

## Rule description

A single-line comment sits directly above the code it describes. Consecutive comment
lines form one block; the check applies to the last line of the block.

```csharp
// 👎 reported
// describes the work

Work();

// 👍 comment hugs its code
// describes the work
Work();
```

## How to fix violations

The code fix removes the blank line(s) after the comment.

## Performance

This rule examines every comment in the file, which makes it the heaviest of the layout
rules. It is still evaluated once per file, so the cost is negligible for typical source
files.
