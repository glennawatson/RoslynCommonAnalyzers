# SST1514 — Element documentation header should be preceded by a blank line

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | Yes (insert blank line) |

## Cause

An element's documentation header follows the preceding code without a blank line.

## Rule description

A documentation header is separated from the preceding element by a blank line. A
header that opens a block (directly after an opening brace) is exempt.

```csharp
// 👎 reported
private int _x;
/// <summary>Gets the value.</summary>
public int X => _x;

// 👍 blank line before the header
private int _x;

/// <summary>Gets the value.</summary>
public int X => _x;
```

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

The code fix inserts a blank line before the documentation header.
