# SST1515 — Single-line comment should be preceded by a blank line

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | Yes (insert blank line) |

## Cause

A standalone single-line comment follows code without a blank line between them.

## Rule description

A single-line comment is separated from the preceding code by a blank line, so it
reads as a heading for what follows. A comment that opens a block (directly after an
opening brace), follows a preprocessor directive (for example the first line inside an
`#if`/`#else` branch), or begins the file is exempt. Consecutive comment lines form one
block; the check applies to the first line of the block.

```csharp
// 👍 directive, not code — no preceding blank line required
#if NET8_0_OR_GREATER
// Use the span-based path.
var span = CollectionsMarshal.AsSpan(items);
#else
// Fall back to foreach.
foreach (var item in items)
#endif
```

```csharp
// 👎 reported
Setup();
// describes the next step
Work();

// 👍 blank line before the comment
Setup();

// describes the next step
Work();
```

The StyleSharp counterpart of StyleCop's `SA1515`.

## How to fix violations

The code fix inserts a blank line before the comment.

## Performance

This rule examines every comment in the file, which makes it the heaviest of the layout
rules. It is still evaluated once per file, so the cost is negligible for typical source
files.
