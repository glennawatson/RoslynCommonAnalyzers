# SST1516 — Elements should be separated by a blank line

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | Yes (insert blank line) |

## Cause

Two adjacent type or namespace members are not separated by a blank line.

## Rule description

Adjacent members are separated by a single blank line so each declaration is visually
distinct. A member's leading documentation header counts as part of the member.

```csharp
// 👎 reported
private void A()
{
}
private void B()
{
}

// 👍 separated
private void A()
{
}

private void B()
{
}
```

## How to fix violations

The code fix inserts a blank line above the second member (before its documentation
header, when present).
