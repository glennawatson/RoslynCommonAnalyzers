# SST1517 — Code should not contain blank lines at the start of the file

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | Yes (remove blank lines) |

## Cause

The file begins with one or more blank lines.

## Rule description

A file starts with code or a header comment, not a blank line.

```csharp
// 👎 reported — file begins with a blank line
⏎
namespace Example;

// 👍 file begins with content
namespace Example;
```

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

The code fix removes the blank line(s) at the start of the file.

## Performance

This rule looks at the file as a whole rather than a single declaration. It is still
evaluated once per file and the cost is negligible for normal source files — it is only
marginally heavier than the single-declaration rules.
