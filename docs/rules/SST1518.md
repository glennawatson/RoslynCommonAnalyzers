# SST1518 — File should end with a single newline

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | Yes (normalise file ending) |

## Cause

The file does not end with exactly one newline — either the final newline is missing
or there are trailing blank lines or whitespace after the last line of content.

## Rule description

A file ends with the last line of content followed by exactly one newline.

The StyleSharp counterpart of StyleCop's `SA1518`.

## How to fix violations

The code fix replaces the trailing whitespace at the end of the file with a single
newline.

## Performance

This rule looks at the file as a whole rather than a single declaration. It is still
evaluated once per file and the cost is negligible for normal source files — it is only
marginally heavier than the single-declaration rules.
