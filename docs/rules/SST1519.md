# SST1519 тАФ Braces should not be omitted from a multi-line child statement

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | Yes (add braces) |

## Cause

A control-flow statement (`if`, `else`, `for`, `foreach`, `while`, `do`, `using`,
`lock`, `fixed`) has an embedded child statement that spans multiple lines but omits
its braces.

## Rule description

A child statement that spans multiple lines is wrapped in braces so its extent is
unambiguous. A single-line child is not reported here; an `else if` chain is not an
unbraced child; and stacked scope statements тАФ `using (a) using (b) { тАж }` and the
same pattern with `lock` or `fixed` тАФ are not reported, because the inner statement's
own braced body already groups the block.

```csharp
// ЁЯСО reported
if (enabled)
    Configure()
        .Apply();

// ЁЯСН braces make the extent clear
if (enabled)
{
    Configure()
        .Apply();
}
```

## How to fix violations

The code fix wraps the multi-line child statement in braces.
