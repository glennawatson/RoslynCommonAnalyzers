# SST1519 — Braces should not be omitted from a multi-line child statement

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | Yes (add braces) |

## Cause

A control-flow statement (`if`, `else`, `for`, `foreach`, `while`, `do`, `using`,
`lock`, `fixed`) has an embedded child statement that spans multiple lines but omits
its braces.

## Rule description

A child statement that spans multiple lines is wrapped in braces so its extent is
unambiguous. A single-line child is left to the analyzer's the rule; an `else if` chain is not an
unbraced child.

```csharp
// 👎 reported
if (enabled)
    Configure()
        .Apply();

// 👍 braces make the extent clear
if (enabled)
{
    Configure()
        .Apply();
}
```

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

The code fix wraps the multi-line child statement in braces.
