# SST1520 — Braces should be used consistently

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | Yes (add braces to all clauses) |

## Cause

The clauses of an `if`/`else` chain use braces inconsistently — some clauses are
wrapped in braces and others are not.

## Rule description

Every clause of an `if`/`else` chain uses braces, or none does.

```csharp
// 👎 reported — the else clause omits braces
if (enabled)
{
    Start();
}
else
    Stop();

// 👍 consistent braces
if (enabled)
{
    Start();
}
else
{
    Stop();
}
```

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

The code fix adds braces to every clause of the chain.
