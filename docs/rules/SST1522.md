# SST1522 — Files should not be too long

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A file has more **code lines** than the configured maximum, which defaults to 500.

## Rule description

A file nobody can hold in their head is a file that grows by accretion, because finding the right
place to add something is harder than appending to the end. The limit is a prompt to split, not a
law of nature: a 520-line file is not a bug, it is a question — *is this still one thing?*

The diagnostic is reported once per file, on the file's first token.

### What counts as a code line

A line counts when at least one **token** sits on it. That means these are free:

- blank lines;
- comment-only lines and documentation headers;
- directive-only lines (`#region`, `#if`, `#pragma`).

A file does not become too long by being explained, so commenting is never penalised. A multi-line
string literal is a single token spanning several lines, and every one of those lines counts —
they are all lines the reader has to scroll past.

```csharp
public class Orders          // 1
{                            // 2
    // Explains the field.   // free
                             // free
    private int _count;      // 3
}                            // 4
```

## How to fix violations

There is **no code fix**. Splitting a file is a design decision — which types move, whether a
`partial` split reflects the design or hides the problem, what the new file is called — and none of that
is recoverable from the line count. The usual moves:

1. **Move a type out.** A file with several types usually wants to be several files.
2. **Extract a collaborator.** One type doing several jobs is the common cause; the length is the
   symptom.
3. **Raise the limit.** A generated-code-adjacent file, or a large table of constants, is genuinely
   long and genuinely fine. Generated code is already excluded automatically.

## Configuration

```ini
# Rule-specific
stylesharp.SST1522.max_file_lines = 500

# Project-wide
stylesharp.max_file_lines = 500
```

| Option | Values | Default | Effect |
| --- | --- | --- | --- |
| `max_file_lines` | positive integer | `500` | The most code lines a file may contain. |

An unset or unparsable value falls back to the default.

## Performance

The file's raw line count is an upper bound on its code lines, so a file inside the limit is
rejected on a single comparison and the token walk never runs. When it does run, it is one
allocation-free indexed pass with a monotonic cursor into the line table — no binary search per
token, and no iterator.
