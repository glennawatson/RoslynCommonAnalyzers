# SST1523 — Members should not be too long

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A member has more **code lines** than the configured maximum, which defaults to 60.

```csharp
public Report Build(Order order)   // SST1523: 'Build' is 94 lines, over the maximum of 60
{
    // ... ninety lines that validate, then price, then format ...
}
```

## Rule description

Length is a cruder measure than complexity. A long method of straight-line assignments is easier to
read than a short one with four nested loops, and the complexity rules already catch the second.
What length catches is the *third* thing: the method that is long because it does several things,
one after another, and would read better as several methods.

```csharp
public Report Build(Order order)
{
    Validate(order);
    var priced = Price(order);
    return Format(priced);
}
```

### What is measured

Methods, constructors, operators, conversion operators, accessors and local functions — each from
its first token to its last. The signature counts, and so do the braces: they are lines the reader
has to hold.

A declaration with **no body** is never measured, because it has nothing to split: an abstract or
`extern` method, an interface member, the defining half of a partial member, an auto-property
accessor.

Blank lines and comments do not count (see [SST1522](SST1522.md) for the exact definition), so
explaining a member never pushes it over.

### Nested functions are reported twice, deliberately

A 90-line method containing a 70-line local function reports both. The local function is too long on
its own terms, and the method that hosts it is still exactly as long on screen as it looks. Neither
diagnostic is a duplicate of the other.

## How to fix violations

There is **no code fix**. Where to cut a method is the whole question: which lines form a step,
what the step is called, what it takes and returns. An automatic split at line 60 would produce a
`BuildPart2` that means nothing.

1. **Name the steps.** A long method usually has paragraphs already, separated by blank lines and a
   comment. Those comments are method names waiting to happen.
2. **Extract the branch.** A long `if` or `switch` arm is a method that has not been extracted yet —
   see [SST1524](SST1524.md).
3. **Raise the limit.** A big `switch` over a wire format, or a long table-building method, is
   genuinely long and genuinely fine.

## Configuration

```ini
# Rule-specific
stylesharp.SST1523.max_member_lines = 60

# Project-wide
stylesharp.max_member_lines = 60
```

| Option | Values | Default | Effect |
| --- | --- | --- | --- |
| `max_member_lines` | positive integer | `60` | The most code lines a member may contain. |

An unset or unparsable value falls back to the default.

## Performance

The raw line span of the declaration is an upper bound on its code lines, so a member inside the
limit is rejected on two line-table lookups and never walks a token. Clean code never pays for the
count.
