# SST1524 — Switch sections should not be too long

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A switch section has more **code lines** than the configured maximum, which defaults to 20.

```csharp
switch (message.Kind)
{
    case MessageKind.Order:          // SST1524: this case is 34 lines, over the maximum of 20
        // ... thirty lines of parsing, validation and dispatch ...
        break;

    case MessageKind.Cancel:
        Cancel(message.Id);
        break;
}
```

## Rule description

A switch reads as a table: one row per case, each saying what happens. A case that runs to dozens of
lines stops being a row and becomes a method that has not been extracted yet — and the shape of the
switch, which is the thing the reader came for, is lost between them.

```csharp
switch (message.Kind)
{
    case MessageKind.Order:
        HandleOrder(message);
        break;

    case MessageKind.Cancel:
        Cancel(message.Id);
        break;
}
```

Now the switch says what it dispatches on and where each case goes, in eight lines, and the thirty
lines of order handling are somewhere with a name on them.

### What is measured

The section, from its first label to its last statement, counting only lines that carry code — so a
heavily commented case is not penalised for explaining itself (see [SST1522](SST1522.md) for the
definition). A section with several labels is measured once, and reported on the first of them.

Switch **expression** arms are not measured. An arm is one expression by construction; the language
already stops it from growing into a procedure.

## How to fix violations

There is **no code fix**. The fix is to extract a method, and what that method is called — and what
it takes — is exactly the judgement the analyzer cannot make.

## Configuration

```ini
# Rule-specific
stylesharp.SST1524.max_switch_section_lines = 20

# Project-wide
stylesharp.max_switch_section_lines = 20
```

| Option | Values | Default | Effect |
| --- | --- | --- | --- |
| `max_switch_section_lines` | positive integer | `20` | The most code lines a switch section may contain. |

An unset or unparsable value falls back to the default.

## Performance

The section's raw line span bounds its code lines, so a short section is rejected on two line-table
lookups and never walks a token.
