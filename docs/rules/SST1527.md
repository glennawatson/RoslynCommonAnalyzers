# SST1527 — A wrapped expression-body arrow should sit on the configured side of the line break

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | Yes |
| Default | Disabled |

## Cause

An expression-bodied member wraps its body onto its own line and the `=>` sits on a different side of the
break than the configured style requires.

```csharp
// Default style is 'after': the arrow trails the signature.
public int Total()
    => subtotal + shipping;   // SST1527: the '=>' leads the body line
```

## Rule description

An expression-bodied member whose body wraps onto its own line can keep the arrow trailing the signature
or lead the body line with it. The default trails the signature, so the eye reaches the arrow before
dropping to the expression.

```csharp
public int Total() =>
    subtotal + shipping;
```

Only an arrow with an adjacent line break is checked; a single-line expression body is never touched.

## How to fix violations

Move the line break to the configured side of the arrow. The code fix does this in place.

## Configuration

```ini
# Project-wide
stylesharp.arrow_token_new_line = after

# Rule-specific
stylesharp.SST1527.arrow_token_new_line = after
```

| Option | Values | Default | Effect |
| --- | --- | --- | --- |
| `arrow_token_new_line` | `after`, `before` | `after` | `after` trails the signature with the arrow; `before` leads the body line with it. |

An unset or unrecognised value falls back to the default.
