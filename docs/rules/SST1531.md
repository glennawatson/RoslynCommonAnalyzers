# SST1531 — A short initializer should be written on a single line

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | Yes |
| Default | Disabled |

## Cause

An object or collection initializer is split across several lines but is short enough to fit on one line
once collapsed.

```csharp
var point = new Point
{
    X = 1,
    Y = 2,
};   // SST1531
```

## Rule description

An object or collection initializer short enough to fit on one line reads better on one line: the
multi-line form spends four lines to say what one line says, and the extra lines carry no information.

```csharp
var point = new Point { X = 1, Y = 2 };
```

Only initializers that stay within the configured maximum line length once collapsed are reported, so a
genuinely long initializer keeps its lines. An initializer that carries a comment between its elements is
left alone, because collapsing it would lose the comment.

This targets **initializers** only. A collapsed statement block is instead expanded by the single-line
block rules ([SST1501](SST1501.md) / [SST1502](SST1502.md)) — the boundary between the two is deliberate.

## How to fix violations

Collapse the initializer onto a single line. The code fix rewrites each wrapped gap to its canonical
spacing.

## Configuration

The rule reuses the shared line-length maximum.

```ini
stylesharp.max_line_length = 120
```

| Option | Values | Default | Effect |
| --- | --- | --- | --- |
| `max_line_length` | positive integer | `120` | The longest a collapsed initializer line may be before the rule leaves the initializer wrapped. |
