# SST1532 — Files should use consistent line endings

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | Yes |
| Default | Disabled |

## Cause

A file's line endings vary within it, or differ from the project's configured style.

```text
internal class C\r\n   ← SST1532 when the configured style is 'lf'
{\r\n
}\r\n
```

## Rule description

Line endings that vary within a file, or differ from the project's chosen style, turn every diff into a
wall of changed lines and make a merge fight over whitespace nobody typed. One style is enforced per
file. The default is LF.

One diagnostic is raised per file, anchored on the first line whose ending is wrong.

## How to fix violations

Rewrite every line ending in the file to the configured newline. The code fix does this in one pass.

## Configuration

```ini
# Project-wide
stylesharp.line_ending = lf

# Rule-specific
stylesharp.SST1532.line_ending = lf
```

| Option | Values | Default | Effect |
| --- | --- | --- | --- |
| `line_ending` | `lf`, `crlf` | `lf` | `lf` requires line-feed endings; `crlf` requires carriage-return/line-feed endings. |

An unset or unrecognised value falls back to the default. There is one rule, not two opposing ones — the
configured value decides what is required.
