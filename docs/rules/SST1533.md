# SST1533 — A source file should declare at least one type

| Item | Value |
| --- | --- |
| Category | Layout |
| Severity | Warning |
| Code fix | No |
| Default | Disabled |

## Cause

A `.cs` file carries usings or comments but declares no namespace, type, or top-level statement.

```csharp
using System;
using System.Text;   // SST1533: nothing below declares anything
```

## Rule description

A file that carries usings or comments but declares no namespace, type, or top-level statement is a file
that has lost its contents to a move or a delete and left its header behind. It compiles, so nothing
complains, and it lingers as a dead entry in every file list.

A file with assembly-level attributes, or one that is genuinely empty, is left alone; generated files are
excluded from analysis.

## How to fix violations

There is **no code fix**. Either put the declaration the file was meant to hold back into it, or delete
the file — a choice the analyzer cannot make.
