# SST1600 — Elements should be documented

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning |
| Code fix | No |

## Cause

An externally visible (public or protected) type or member has no XML
documentation comment.

## Rule description

Public API carries XML documentation so generated docs and IntelliSense are
complete. Visibility is computed from syntax: a member is exposed when it and
every containing type are `public` or `protected`. Overrides, explicit interface
implementations, `partial` members, and members using `<inheritdoc>` are skipped.

```csharp
// 👎 reported
public class Widget { }

// 👍
/// <summary>A widget.</summary>
public class Widget { }
```

The StyleSharp counterpart of StyleCop's `SA1600`. Enum members are covered by
[SST1602](SST1602.md).
