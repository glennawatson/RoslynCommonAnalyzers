# SST1600 — Elements should be documented

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning |
| Code fix | No |

## Cause

An externally visible (public or protected) type, member, or field has no XML
documentation comment.

## Rule description

Documented API carries XML documentation so generated docs and IntelliSense are
complete. Visibility is computed from syntax (no semantic-model binding), using the
declaration's **effective accessibility** — its own accessibility narrowed by every
containing type. Overrides, explicit interface implementations, `partial` members
(see [SST1601](SST1601.md)), and members using `<inheritdoc>` are skipped.

```csharp
// 👎 reported
public class Widget { }

// 👍
/// <summary>A widget.</summary>
public class Widget { }
```

## Coverage scope

Which accessibilities are reported is configurable and defaults to **exposed and
internal elements are required, private elements are not.** Because an undocumented
`internal` type is reported by default, set
`stylesharp.document_internal_elements = false` to require only the public API
surface. The full option set (`document_exposed_elements`,
`document_internal_elements`, `document_private_elements`,
`document_private_fields`, `document_interfaces`)
is described in [Documentation coverage scope](../CONFIGURATION.md#documentation-coverage-scope);
the same options govern [SST1601](SST1601.md), [SST1602](SST1602.md), and
[SST1654](SST1654.md).

## Fields

Fields (and field-like events) follow the same coverage rules as other members:
public / protected fields are required by `document_exposed_elements`, internal
fields by `document_internal_elements`. Private fields are special — they are
governed by their own opt-in **`stylesharp.document_private_fields`** option
(default `false`), kept separate from the other coverage options, so
turning on `document_private_elements` does **not** start requiring documentation on
private fields. Set `stylesharp.document_private_fields = true` to require it:

```csharp
// With stylesharp.document_private_fields = true
public class Widget
{
    private int _count;   // 👎 reported (SST1600)

    /// <summary>The count.</summary>
    private int _total;   // 👍
}
```

A private `const` is treated as a private field and follows the same option, and
`private protected` is treated as private. Each declarator of a multi-declarator
field (`private int _a, _b;`) is reported separately. Compiler-generated backing
fields for auto-properties and field-like events have no source declaration, and
`[GeneratedCode]` members are excluded, so none of these are ever flagged.

Enum members are covered by [SST1602](SST1602.md).
