# SST1601 — Partial elements should be documented

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning |
| Code fix | No |

## Cause

A partial type or partial method that is in the documentation-coverage scope has no
documentation comment.

## Rule description

Partial types and methods carry XML documentation just like non-partial elements
([SST1600](SST1600.md) covers the non-partial case). The same coverage options
apply: exposed and internal partial elements are required by default, private ones
are not. See [Documentation coverage scope](../CONFIGURATION.md#documentation-coverage-scope)
to require only the public surface or to include private members.

## How to fix violations

Add a documentation comment to the partial element.
