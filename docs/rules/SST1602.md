# SST1602 — Enumeration items should be documented

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning |
| Code fix | No |

## Cause

A member of an externally visible enum has no documentation comment.

## Rule description

```csharp
/// <summary>Traffic light colors.</summary>
public enum Color
{
    /// <summary>Stop.</summary>
    Red,
    Green, // 👎 reported
}
```

Whether an enum's members are required follows the enum's effective accessibility
and the shared documentation-coverage options (internal enums are covered by
default; see [Documentation coverage scope](../CONFIGURATION.md#documentation-coverage-scope)).

The StyleSharp counterpart of the analyzer's `the rule`.
