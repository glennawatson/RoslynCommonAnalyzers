# SST1605 — Partial element documentation should have a summary

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning (disabled by default) |
| Code fix | No |

## Cause

A documented partial element's documentation has no `<summary>`.

## Rule description

Off by default, matching StyleCop — documenting one part of a partial declaration is
often enough. Enable it in `.editorconfig` to require a `<summary>` on every documented
partial element.

The StyleSharp counterpart of StyleCop's `SA1605`.

## How to fix violations

Add a `<summary>` element to the documentation.
