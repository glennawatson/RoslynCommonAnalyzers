# SST1609 — Property documentation should have a value

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning (disabled by default) |
| Code fix | No |

## Cause

A documented property's documentation has no `<value>` element.

## Rule description

Off by default, matching the analyzer. Enable it to require a `<value>` element describing
what the property holds.

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

Add a `<value>` element to the property documentation.
