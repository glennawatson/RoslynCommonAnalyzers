# SST1623 — Property summary should describe its accessors

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning |
| Code fix | Yes (prefixes the accessor phrase) |

## Cause

A property's `<summary>` does not begin with the phrase that matches its
accessors.

## Rule description

| Accessors | Expected start |
| --- | --- |
| get + set / init | `Gets or sets ` |
| get only | `Gets ` |
| set / init only | `Sets ` |

```csharp
// 👎 reported
/// <summary>The widget count.</summary>
public int Count { get; set; }

// 👍
/// <summary>Gets or sets the widget count.</summary>
public int Count { get; set; }
```

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

The code fix prepends the accessor phrase and lower-cases the first existing
word.
