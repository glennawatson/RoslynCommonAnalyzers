# SST1624 — Property summaries should omit restricted set accessors

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning |
| Code fix | Yes |
| Default | Disabled (opt-in) |

## Cause

A property with a restricted `set` or `init` accessor is documented as "Gets or sets".

```csharp
/// <summary>Gets or sets the value.</summary>
public int Value { get; private set; }
```

## Rule description

The externally visible contract is read-only, so its summary should begin with "Gets".
This rule mirrors StyleCop SA1624.

## How to fix violations

Replace "Gets or sets" with "Gets". The provided code fix does this automatically.
