# SST1625 — Element documentation should not be copy-pasted

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning |
| Code fix | No |

## Cause

Two elements in the same documentation comment have identical text — for example a
parameter description that repeats the summary verbatim.

## Rule description

Each documentation element describes its own element rather than repeating another's text.

```csharp
// 👎 reported
/// <summary>The widget count.</summary>
/// <param name="count">The widget count.</param>
public void SetCount(int count) { }

// 👍 distinct descriptions
/// <summary>Sets the number of widgets to display.</summary>
/// <param name="count">The number of widgets.</param>
public void SetCount(int count) { }
```

Inline references count toward the comparison, so two elements whose prose matches but
whose `cref`/`name` targets differ are **not** treated as copies:

```csharp
// 👍 same wording, different cref — not reported
/// <param name="writeNoType">Called when <see cref="Write(string)"/> is invoked.</param>
/// <param name="writeWithType">Called when <see cref="Write(string, int)"/> is invoked.</param>
```

The StyleSharp counterpart of StyleCop's `SA1625`.

## How to fix violations

Rewrite the duplicated text so each element describes its own purpose.
