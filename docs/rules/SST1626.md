# SST1626 — Single-line comments should not use documentation style slashes

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning |
| Code fix | No |

## Cause

A `///` documentation-style comment appears where it does not document a type or
member (for example inside a method body, before a statement).

## Rule description

`///` is reserved for documenting types, members, and local functions. A comment in any
other position uses `//`.

```csharp
// 👎 reported
public void M()
{
    /// not documentation
    Work();
}

// 👍 ordinary comment
public void M()
{
    // an ordinary comment
    Work();
}
```

The StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

Change the `///` comment to a `//` comment.

## Performance

This rule inspects documentation comments throughout the file. Such comments are sparse, so
the cost is negligible in practice.
