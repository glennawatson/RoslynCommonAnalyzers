# SST1627 — Documentation text must not be empty

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning |
| Code fix | No |
| Default | Disabled (opt-in) |

## Cause

A non-summary documentation section such as `<remarks>` or `<para>` contains no text.

## Rule description

Empty documentation sections add noise without information. Elements containing child
XML elements are not treated as empty. Empty summaries remain the responsibility of
SST1606. This rule mirrors StyleCop SA1627.

## How to fix violations

Add meaningful text or remove the empty element.
