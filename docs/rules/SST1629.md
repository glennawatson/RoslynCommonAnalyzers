# SST1629 — Documentation text should end with a period

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning |
| Code fix | Yes (adds a period) |

## Cause

The prose of a `<summary>`, `<returns>`, `<remarks>`, or `<value>` element does
not end with terminal punctuation (`.`, `!`, `?`, `:`, or `;`).

## Rule description

```csharp
// 👎 reported
/// <summary>Gets the widget count</summary>

// 👍
/// <summary>Gets the widget count.</summary>
```

Elements whose content ends with an inline element (for example `<see/>`) are
left alone to avoid false positives. The StyleSharp counterpart of StyleCop's
`SA1629`.

## How to fix violations

The code fix appends a period to the text.
