# SST1630 — Documentation text should contain whitespace

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning (disabled by default) |
| Code fix | No |

## Cause

A documentation summary is a single word with no whitespace.

## Rule description

Off by default, matching StyleCop. Enable it to require summaries to contain more than a
single word.

The StyleSharp counterpart of StyleCop's `SA1630`.

## How to fix violations

Expand the summary into a descriptive phrase.
