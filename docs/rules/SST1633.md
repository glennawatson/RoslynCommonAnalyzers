# SST1633 — File should have a header

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning |
| Code fix | Yes (inserts the header) |
| Config | `file_header_template` |

## Cause

A file does not begin with the header configured by `file_header_template`, and a
non-empty template is set.

## Rule description

When `file_header_template` is set in `.editorconfig`, every file must begin with
the rendered header. This is the same option the .NET SDK's `the rule` uses — but
SST1633 is a normal analyzer, so it runs in ordinary builds **without** turning on
`EnforceCodeStyleInBuild` (which is why teams often reach for the the analyzer rule
instead of the IDE rule).

When the template is `unset` or empty, the rule does nothing.

```ini
[*.cs]
file_header_template = Copyright (c) Contoso.\nLicensed under the MIT license.
```

```csharp
// Copyright (c) Contoso.
// Licensed under the MIT license.
namespace Contoso;
```

Each `\n` in the template starts a new `//` line, and `{fileName}` is substituted
with the file name. This is the StyleSharp counterpart of the analyzer's `the rule`.

## How to fix violations

The code fix inserts the rendered header at the top of the file.
