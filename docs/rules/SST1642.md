# SST1642 — Constructor summary should begin with the standard text

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning |
| Code fix | Yes (rewrites to the standard text) |

## Cause

An instance constructor's `<summary>` does not begin with the standard
"Initializes a new instance of the …" text.

## Rule description

```csharp
// 👎 reported
/// <summary>Creates a widget.</summary>
public Widget() { }

// 👍
/// <summary>Initializes a new instance of the <see cref="Widget"/> class.</summary>
public Widget() { }
```

An explicitly `private` constructor may instead use the "prevent instantiation"
wording:

```csharp
// 👍 also accepted for a private constructor
/// <summary>Prevents a default instance of the <see cref="Widget"/> class from being created.</summary>
private Widget() { }
```

Static constructors are exempt.

## How to fix violations

The code fix replaces the summary with the standard text referencing the
declaring type.
