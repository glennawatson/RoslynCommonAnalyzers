# SST1643 — Destructor summary should begin with the standard text

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning |
| Code fix | Yes (rewrites to the standard text) |

## Cause

A destructor's `<summary>` does not begin with the standard "Finalizes an
instance of the …" text.

## Rule description

```csharp
// 👍
/// <summary>Finalizes an instance of the <see cref="Widget"/> class.</summary>
~Widget() { }
```

Pairs with [SST1642](SST1642.md) (constructors). The StyleSharp counterpart of
the analyzer's `the rule`.

## How to fix violations

The code fix replaces the summary with the standard text referencing the type.
