# SST1644 — Documentation headers must not contain blank lines

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning |
| Code fix | No |
| Default | Disabled (opt-in) |

## Cause

An interior line of a single-line documentation comment contains only `///`.

## Rule description

Documentation prose remains contiguous. Comments containing `<code>` or `<example>`
elements are excluded because whitespace can be intentional. This rule mirrors the
deprecated StyleCop SA1644 and is therefore opt-in.

## How to fix violations

Remove the blank documentation line.
