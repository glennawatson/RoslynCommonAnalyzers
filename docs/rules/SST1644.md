# SST1644 — Documentation headers must not contain blank lines

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning |
| Code fix | No |
| Default | Disabled (opt-in) |

## Cause

An interior line of a single-line documentation comment contains only `///`.

## Rule description

Documentation prose remains contiguous. Blank lines that fall inside a `<code>` or
`<example>` element — wherever it appears, including nested in `<remarks>`/`<para>`
or inside a `<![CDATA[ ]]>` block — are excluded because whitespace can be
intentional there. Blank prose lines elsewhere in the same comment are still
reported. This rule is opt-in.

## How to fix violations

Remove the blank documentation line.
