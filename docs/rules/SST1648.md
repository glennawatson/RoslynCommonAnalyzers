# SST1648 — inheritdoc should be used with inheriting elements

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning |
| Code fix | No |

## Cause

An `<inheritdoc>` element is used on an element that has nothing to inherit documentation
from — it is not an override, not an interface implementation, and not a type with a
documented base type or interface.

## Rule description

`<inheritdoc>` only makes sense where documentation can actually be inherited.

```csharp
// 👎 reported — nothing to inherit from
/// <inheritdoc/>
public void Start() { }

// 👍 implements an interface member
/// <inheritdoc/>
public void Dispose() { }   // on a type implementing IDisposable
```

## How to fix violations

Replace `<inheritdoc>` with real documentation, or remove it if the element does not
inherit from a documented base.
