# SST1649 — File name should match the first type name

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning |
| Code fix | Yes (rename file to match type) |

## Cause

A file's name does not match the first type it declares.

## Rule description

A file is named after the first type it declares. Generic arity is ignored, and a
`.suffix` before the extension is allowed so partial files like `Widget.Logic.cs`
match a partial `Widget`.

```text
Widget.cs        -> public class Widget        👍
Widget.Logic.cs  -> public partial class Widget 👍
Other.cs         -> public class Widget        👎 reported
```

Note: SST1623's accessor phrasing also folds in the restricted-setter case
covered by SST1624 (a property with a restricted setter reads "Gets", not
"Gets or sets").

## How to fix violations

The code fix renames the file to match its first type, leaving the file contents
untouched. Generic types use the configured convention — `Widget{T}.cs` by default,
or `Widget`1.cs` when `stylesharp.file_naming_convention = metadata`. See
[CONFIGURATION](../CONFIGURATION.md). The rename is applied to every linked
target-framework copy at once. "Fix all" is not offered (a rename creates/removes
documents, which the batch fixer cannot carry).
