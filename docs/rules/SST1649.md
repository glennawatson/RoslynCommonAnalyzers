# SST1649 — File name should match the first type name

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning |
| Code fix | No |

## Cause

A file's name does not match the first type it declares.

## Rule description

A file is named after the first type it declares. Generic arity is ignored, and a
`.suffix` before the extension is allowed so partial files like `Widget.Logic.cs`
match a partial `Widget`.

```text
Widget.cs        -> public class Widget        👍
Widget.Logic.cs  -> public partial class Widget 👍
Other.cs         -> public class Widget        👎 reported
```

The StyleSharp counterpart of StyleCop's `SA1649`. Note: SST1623's accessor
phrasing also folds in StyleCop's SA1624 (a property with a restricted setter
reads "Gets", not "Gets or sets").
