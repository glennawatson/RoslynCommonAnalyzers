# SST1653 — Short documentation summaries should be on a single line

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning |
| Code fix | Yes (collapse to one line) |
| Default limit | 100 characters |

## Cause

A `<summary>` is spread across multiple lines even though its combined text is
short enough to fit on a single line (under the configured limit, 100 characters
by default).

## Rule description

For short summaries the one-line form is more compact and just as readable:

```csharp
// 👎 reported (short, but spread over three lines)
/// <summary>
/// Gets the widget count.
/// </summary>
public int Count { get; }

// 👍 single line
/// <summary>Gets the widget count.</summary>
public int Count { get; }
```

"Combined" length is the summary's visible text with runs of whitespace (and the
line breaks / `///` exteriors) collapsed to single spaces and the ends trimmed.
Empty summaries and summaries at or over the limit are left alone, so longer
descriptions keep their multi-line layout.

This is a StyleSharp-original rule (no the analyzer equivalent).

## Configuration

Set the limit in `.editorconfig` (CA-analyzer style — rule-specific key wins over
the general key):

```ini
[*.cs]
# default is 100
stylesharp.summary_single_line_max_length = 120
# or per-rule:
stylesharp.SST1653.summary_single_line_max_length = 120
```

## How to fix violations

The code fix rewrites the summary onto a single line, preserving its text.
