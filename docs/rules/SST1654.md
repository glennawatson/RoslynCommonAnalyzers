# SST1654 — Extension blocks should be documented

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning |
| Code fix | No |

## Cause

An externally visible C# 14 extension block has no `<summary>` (either no
documentation comment at all, or one that omits the summary).

## Rule description

An extension block adds members to its receiver type, so it forms part of the
public API surface and carries a `<summary>` like any other documented element:

```csharp
public static class ObservableExtensions
{
    // 👎 reported — no summary
    extension<TSource>(IObservable<TSource> source)
    {
        public bool IsCold => true;
    }

    /// <summary>Adds cold/hot helpers to an observable sequence.</summary>
    /// <typeparam name="TSource">The type of the elements in the source sequence.</typeparam>
    /// <param name="source">An observable sequence of elements to project.</param>
    // 👍 documented
    extension<TSource>(IObservable<TSource> source)
    {
        public bool IsCold => true;
    }
}
```

A block whose container is `internal`/`private`, or that uses `<inheritdoc>`, is
left alone. This is a StyleSharp-original rule (no StyleCop equivalent); it pairs
with [SST1655](SST1655.md), [SST1656](SST1656.md), and [SST1657](SST1657.md).

## How to fix violations

Add a `<summary>` describing the members the extension block introduces.
