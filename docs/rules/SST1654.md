# SST1654 — Extension blocks should be documented

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning |
| Code fix | No |

## Cause

A C# 14 extension block that is in the documentation-coverage scope has no
`<summary>` (either no documentation comment at all, or one that omits the summary).

## Rule description

An extension block adds members to its receiver type, so it forms part of its
container's API surface and carries a `<summary>` like any other documented element:

```csharp
public static class ObservableExtensions
{
    // 👎 reported — no summary
    extension<TSource>(IObservable<TSource> source)
    {
        public bool IsCold => true;
    }

    /// <summary>Adds cold/hot helpers to an observable sequence.</summary>
    /// <typeparam name="TSource">The type of the elements in the source sequence.</typeparam>
    /// <param name="source">An observable sequence of elements to project.</param>
    // 👍 documented
    extension<TSource>(IObservable<TSource> source)
    {
        public bool IsCold => true;
    }
}
```

A block's visibility comes from its container: an `internal` container is covered by
default (set `stylesharp.document_internal_elements = false` to require only public
extension blocks), a private-effective container is not, and a block using
`<inheritdoc>` is left alone. The shared
[coverage options](../CONFIGURATION.md#documentation-coverage-scope) apply. This is a
StyleSharp-original rule (no StyleCop equivalent); it pairs with
[SST1655](SST1655.md), [SST1656](SST1656.md), and [SST1657](SST1657.md).

## How to fix violations

Add a `<summary>` describing the members the extension block introduces.
