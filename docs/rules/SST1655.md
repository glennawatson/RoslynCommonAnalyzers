# SST1655 — Extension block parameters should be documented

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning |
| Code fix | Yes (insert a `<param>` stub) |

## Cause

A documented C# 14 extension block has a parameter with no matching `<param>`
element.

## Rule description

The receiver parameter of an extension block is documented with a `<param>`
element, just like a method parameter:

```csharp
/// <summary>Adds projection helpers to an observable sequence.</summary>
/// <typeparam name="TSource">The type of the elements in the source sequence.</typeparam>
/// <param name="source">An observable sequence of elements to project.</param>
extension<TSource>(IObservable<TSource> source)
{
    public bool IsCold => true;
}
```

The rule only runs once the block is documented; a wholly undocumented block is
reported by [SST1654](SST1654.md) instead. The extension-block counterpart of
[SST1611](SST1611.md).

## How to fix violations

The code fix inserts a `<param name="…"></param>` scaffold for the undocumented
parameter; fill in its description.
