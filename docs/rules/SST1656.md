# SST1656 — Extension block type parameters should be documented

| Item | Value |
| --- | --- |
| Category | Documentation |
| Severity | Warning |
| Code fix | Yes (insert a `<typeparam>` stub) |

## Cause

A documented C# 14 extension block has a generic type parameter with no matching
`<typeparam>` element.

## Rule description

Each type parameter of an extension block is documented with a `<typeparam>`
element:

```csharp
/// <summary>Adds projection helpers to an observable sequence.</summary>
/// <typeparam name="TSource">The type of the elements in the source sequence.</typeparam>
/// <param name="source">An observable sequence of elements to project.</param>
extension<TSource>(IObservable<TSource> source)
{
    public bool IsCold => true;
}
```

The rule only runs once the block is documented; a wholly undocumented block is
reported by [SST1654](SST1654.md) instead. The extension-block counterpart of
[SST1618](SST1618.md).

## How to fix violations

The code fix inserts a `<typeparam name="…"></typeparam>` scaffold for the
undocumented type parameter; fill in its description.
