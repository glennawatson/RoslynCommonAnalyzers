# SST1658 - Documentation should not repeat a word

**Category:** Documentation
**Severity:** Warning
**Code fix:** Yes (removes the repeated word)

A word typed twice in a row in documentation prose — "the the", "is is" — is
almost always an editing leftover. The analyzer compares neighbouring words
case-insensitively, so "The the" is reported too, and flags the second
occurrence.

Only prose is scanned. The contents of `<c>`, `<code>`, `<see>`, and
`<seealso>` elements are ignored, and any punctuation between two matching
words keeps them from being treated as a repeat ("The end. End of story." is
fine). A pair split across a documentation line break is still reported.

## Example

```csharp
/// <summary>Gets the the value.</summary> // SST1658 on the second "the"
public int Value { get; }
```

Use:

```csharp
/// <summary>Gets the value.</summary>
public int Value { get; }
```

The code fix removes the second occurrence together with the whitespace before
it; when the duplicate starts a new documentation line, the line break is kept
and the word plus one following space is removed instead.
