# SST1701 — Combine extension blocks for the same receiver type

| Item | Value |
| --- | --- |
| Category | Extensions |
| Severity | Warning |
| Code fix | No |
| Applies to | C# 14 extension blocks |

## Cause

A class declares more than one `extension` block for the same receiver type.

## Rule description

Keeping all extension members for one receiver type in a single block makes the surface area for
that type easy to find and avoids repeating the receiver declaration.

```csharp
public static class StringExtensions
{
    // 👎 reported — two blocks for the same receiver
    extension(string text)
    {
        public bool IsBlank => text.Trim().Length == 0;
    }

    extension(string other)
    {
        public int Words => other.Split(' ').Length;
    }

    // 👍 one block
    extension(string text)
    {
        public bool IsBlank => text.Trim().Length == 0;

        public int Words => text.Split(' ').Length;
    }
}
```

Receiver types are compared textually, so `string` and `System.String` are treated as distinct.

This rule has no the analyzer counterpart — it covers the new extension-member syntax introduced in
C# 14.

## How to fix violations

Move the members into the earlier extension block for that receiver type and delete the duplicate
block.
