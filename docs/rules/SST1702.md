# SST1702 — Extension blocks should be grouped together

| Item | Value |
| --- | --- |
| Category | Extensions |
| Severity | Warning |
| Code fix | No |
| Applies to | C# 14 extension blocks |

## Cause

A class declares an `extension` block that is separated from the type's other extension blocks by a non-extension member.

## Rule description

Keeping all of a type's extension blocks contiguous makes the type's extension surface easy to find
and read, and keeps unrelated members from breaking up the block run.

```csharp
public static class StringExtensions
{
    // 👎 reported — the second block is separated from the first
    extension(string text)
    {
        public bool IsBlank => text.Trim().Length == 0;
    }

    public static string Combine(string a, string b) => a + b;

    extension(int value)
    {
        public bool IsZero => value == 0;
    }
}
```

This rule covers the new extension-member syntax introduced in
C# 14.

## How to fix violations

Move the extension blocks so they sit next to one another, with other members before or after the run.
