# SST1704 — Extension container class names should end in 'Extensions' or 'Mixins'

| Item | Value |
| --- | --- |
| Category | Extensions |
| Severity | Warning |
| Code fix | Yes |
| Applies to | C# 14 extension blocks |

## Cause

A static class that declares one or more `extension` blocks is not named with an accepted `Extensions` or `Mixins` suffix.

## Rule description

Extension container classes should end with a recognizable suffix such as `Extensions` or `Mixins`
(`StringExtensions`, `EnumerableMixins`). The suffix makes the class's purpose obvious and keeps
extension containers easy to find.

```csharp
// 👎 reported
public static class StringStuff
{
    extension(string text) { /* … */ }
}

// 👍
public static class StringExtensions
{
    extension(string text) { /* … */ }
}

// 👍 also valid
public static class StringMixins
{
    extension(string text) { /* … */ }
}
```

This rule has no StyleCop counterpart — it covers the new extension-member syntax introduced in
C# 14.

## How to fix violations

Rename the class so its name ends in `Extensions` or `Mixins`.

When the SST1704 code fix is used, the preferred suffix can be configured via:

```ini
[*.cs]
stylesharp.extension_container_preferred_suffix = Extensions
# or
stylesharp.SST1704.preferred_suffix = Mixins
```
