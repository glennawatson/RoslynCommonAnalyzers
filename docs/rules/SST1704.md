# SST1704 — Extension container class names should end in 'Extensions'

| Item | Value |
| --- | --- |
| Category | Extensions |
| Severity | Warning |
| Code fix | No |
| Applies to | C# 14 extension blocks |

## Cause

A static class that declares one or more `extension` blocks is not named with an `Extensions` suffix.

## Rule description

The .NET design guidelines name a static class that holds extension members with an `Extensions`
suffix (`StringExtensions`, `EnumerableExtensions`). The suffix makes the class's purpose obvious and
keeps extension containers easy to find.

```csharp
// 👎 reported
public static class StringStuff
{
    extension(string text) { /* … */ }
}

// 👍
public static class StringExtensions
{
    extension(string text) { /* … */ }
}
```

This rule has no StyleCop counterpart — it covers the new extension-member syntax introduced in
C# 14.

## How to fix violations

Rename the class so its name ends in `Extensions`.
