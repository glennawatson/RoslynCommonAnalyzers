# SST1706 — Avoid declaring extension members on a broad receiver type

| Item | Value |
| --- | --- |
| Category | Extensions |
| Severity | Warning |
| Code fix | No |
| Applies to | C# 14 extension blocks |

## Cause

An `extension` block targets `object`, `System.Object`, or `dynamic`.

## Rule description

An extension on `object` or `dynamic` attaches its members to *every* type, cluttering completion
lists everywhere and inviting accidental use. Target the most specific type that makes sense instead.

```csharp
// 👎 reported — attaches to everything
public static class ObjectExtensions
{
    extension(object value)
    {
        public bool IsNull => value is null;
    }
}
```

Receiver types are matched textually, so the common spellings `object`, `System.Object`, and
`dynamic` are recognised.

This rule covers the new extension-member syntax introduced in
C# 14.

## How to fix violations

Target a specific receiver type, or use a plain static method rather than an extension.
