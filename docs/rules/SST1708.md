# SST1708 — Extension methods should use their receiver parameter

| Item | Value |
| --- | --- |
| Category | Extensions |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |
| Analyzer | `Sst1708UnusedExtensionReceiverAnalyzer` |

## Cause

A `this`-parameter extension method never reads its receiver anywhere in its body, so it gains nothing
from being an extension.

```csharp
public static class Ext
{
    // 👎 reported: 'text' is never read
    public static int Zero(this string text) => 0;

    // 👍
    public static int Length(this string text) => text.Length;
}
```

## Rule description

An extension method exists to add behaviour to its receiver. When the body never references the `this`
parameter, the method is a plain static helper wearing extension syntax: it clutters the receiver type's
completion list without depending on it.

The body is scanned only after the cheap gate confirms a `this`-parameter method with an actual body,
and the scan stops at the first read of the receiver name. A receiver named `_` is treated as a
deliberate discard and is exempt.

## How to fix violations

There is no automatic fix, because removing the `this` modifier would break every instance-style call
site (`value.Zero()`). Decide which shape you want:

- If the receiver is genuinely irrelevant, drop the `this` modifier and call it as a plain static method.
- If it should act on the receiver, use the parameter in the body.
