# SST1800 — Record classes should be sealed

| Item | Value |
| --- | --- |
| Category | Records |
| Severity | Warning |
| Code fix | No |
| Default | Disabled (opt-in) |

## Cause

A `record` class is declared without `sealed` or `abstract`.

## Rule description

A record class that is not part of an inheritance hierarchy should be `sealed` so
its compiler-generated equality (`Equals`, `GetHashCode`, the `EqualityContract`)
is final and cannot be subtly broken by a derived record. Records intended as a
base type should instead be `abstract`.

```csharp
// 👎 reported
public record Animal(string Name);

// 👍
public sealed record Animal(string Name);
public abstract record Shape;
```

This rule is **off by default** — record inheritance is a deliberate design
choice, and enabling sealing-by-default is a project preference. Enable it in
`.editorconfig`:

```ini
[*.cs]
dotnet_diagnostic.SST1800.severity = warning
```

## How to fix violations

Add the `sealed` modifier, or `abstract` if the record is designed to be a base
type.
