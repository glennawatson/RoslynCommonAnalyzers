# SST1801 — Positional record parameters should match the configured casing

| Item | Value |
| --- | --- |
| Category | Records |
| Severity | Warning |
| Code fix | No |
| Default | PascalCase |

## Cause

A positional parameter of a `record` or `record struct` does not match the
configured casing convention (PascalCase by default).

## Rule description

A positional record parameter is compiled into a public property, so it follows
the same casing as any other public member — PascalCase by default.

```csharp
// 👎 reported (default PascalCase)
public sealed record Point(int x, int y);

// 👍
public sealed record Point(int X, int Y);
```

## Configuration

The casing is configurable in `.editorconfig`:

```ini
[*.cs]
# pascal_case (default) or camel_case
stylesharp.record_parameter_naming = pascal_case
# rule-specific override wins over the general key:
stylesharp.SST1801.record_parameter_naming = pascal_case
```

## How to fix violations

Rename the positional parameter to match the convention. (No automatic code fix —
renaming a positional parameter also renames its generated property and every
deconstruction site, so it is left as a manual, semantic rename.)
