# SST1802 — Record properties should be init-only

| Item | Value |
| --- | --- |
| Category | Records |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

An instance property declared on a `record` or `record struct` exposes a `set`
accessor instead of `init`.

## Rule description

Records are designed around value semantics: two records are equal when their
members are equal. A mutable (`set`) property lets a record change after
construction, which undermines that contract — the hash code and equality of a
record can silently drift. Use `init` so the property can be set during object
initialization but is immutable afterward.

```csharp
// 👎 reported
public sealed record Person
{
    public string Name { get; set; }
}

// 👍
public sealed record Person
{
    public string Name { get; init; }
}
```

Static properties are not reported — they are not part of a record's value
identity.

## How to fix violations

Replace the `set` accessor with `init`. The provided code fix does this
automatically.
