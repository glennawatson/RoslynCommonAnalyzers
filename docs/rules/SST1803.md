# SST1803 — Record structs should be readonly

| Item | Value |
| --- | --- |
| Category | Records |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A `record struct` is declared without the `readonly` modifier.

## Rule description

A `record struct` models an immutable value, matching how records are intended to
be used. Declaring it `readonly record struct` makes that immutability explicit
and lets the compiler enforce it (and avoid defensive copies). A mutable record
struct is almost always a mistake.

```csharp
// 👎 reported
public record struct Point(int X, int Y);

// 👍
public readonly record struct Point(int X, int Y);
```

## How to fix violations

Add the `readonly` modifier. The provided code fix does this automatically.
