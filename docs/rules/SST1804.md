# SST1804 — Empty positional record bodies should be a semicolon

| Item | Value |
| --- | --- |
| Category | Records |
| Severity | Info |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `RecordAnalyzer` |

## Cause

A positional `record` or `record struct` declares an empty `{ }` body where a semicolon would say the
same thing.

```csharp
// 👎 reported
public sealed record Point(int X, int Y) { }

// 👍
public sealed record Point(int X, int Y);
```

## Rule description

A positional record whose body is empty adds nothing over the semicolon-terminated form. The rule fires
only when the record has a parameter list, has an empty body (`{ }` with no members), and thus a
semicolon can replace the braces without changing anything.

## How to fix violations

The fix replaces the empty `{ }` body with a semicolon, keeping the trailing trivia so surrounding
layout is preserved. Add members to the body if the record needs them; otherwise the semicolon form is
idiomatic.
