# SST2001 — Use ArgumentException.ThrowIfNullOrEmpty

| Item | Value |
| --- | --- |
| Category | Modernization |
| Severity | Warning |
| Code fix | Yes |
| Default | Disabled (opt-in) |

## Cause

A `string.IsNullOrEmpty` check throws an argument exception, where
`ArgumentException.ThrowIfNullOrEmpty` (introduced in .NET 7) would do the same in
one line.

```csharp
// 👎 reported (when enabled)
if (string.IsNullOrEmpty(value))
{
    throw new ArgumentException("Value required.", nameof(value));
}

// 👍
ArgumentException.ThrowIfNullOrEmpty(value);
```

## Rule description

`ArgumentException.ThrowIfNullOrEmpty(argument)` checks for null or empty and
throws with the parameter name inferred via `CallerArgumentExpression`.

**This rule is off by default.** Unlike [SST2000](SST2000.md), adopting the helper
can change observable behavior:

- the **message** changes — the helper supplies its own standard message, so any
  custom message on the original exception is dropped; and
- the **exception type** for the null case changes — the helper throws
  `ArgumentNullException` for null (and `ArgumentException` only for empty),
  whereas the original may have thrown `ArgumentException` for both.

Because that is a deliberate adoption choice, enable it explicitly in
`.editorconfig` when you want it:

```ini
[*.cs]
dotnet_diagnostic.SST2001.severity = warning
```

### Availability detection

The rule targets a **runtime API**, so it is gated on
`ArgumentException.ThrowIfNullOrEmpty` existing in the referenced framework
(detected once per compilation via `Compilation.GetTypeByMetadataName`). On
pre-.NET 7 targets it reports nothing.

## How to fix violations

Replace the guard with `ArgumentException.ThrowIfNullOrEmpty(value);`. The provided
code fix does this automatically.
