# SST2002 — Use ArgumentException.ThrowIfNullOrWhiteSpace

| Item | Value |
| --- | --- |
| Category | Modernization |
| Severity | Warning |
| Code fix | Yes |
| Default | Disabled (opt-in) |

## Cause

A `string.IsNullOrWhiteSpace` check throws an argument exception, where
`ArgumentException.ThrowIfNullOrWhiteSpace` (introduced in .NET 8) would do the
same in one line.

```csharp
// 👎 reported (when enabled)
if (string.IsNullOrWhiteSpace(value))
{
    throw new ArgumentException("Value required.", nameof(value));
}

// 👍
ArgumentException.ThrowIfNullOrWhiteSpace(value);
```

## Rule description

`ArgumentException.ThrowIfNullOrWhiteSpace(argument)` checks for null or
whitespace and throws with the parameter name inferred via
`CallerArgumentExpression`.

**This rule is off by default**, for the same reason as
[SST2001](SST2001.md): adopting the helper can change the thrown **message** (the
helper supplies its own) and the **exception type** for the null case (the helper
throws `ArgumentNullException` for null). Enable it explicitly when you want it:

```ini
[*.cs]
dotnet_diagnostic.SST2002.severity = warning
```

### Availability detection

The rule targets a **runtime API**, so it is gated on
`ArgumentException.ThrowIfNullOrWhiteSpace` existing in the referenced framework
(detected once per compilation via `Compilation.GetTypeByMetadataName`). On
pre-.NET 8 targets it reports nothing.

## How to fix violations

Replace the guard with `ArgumentException.ThrowIfNullOrWhiteSpace(value);`. The
provided code fix does this automatically.
