# SST2004 — Use ArgumentOutOfRangeException range helpers

| Item | Value |
| --- | --- |
| Category | Modernization |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled when the matching runtime helper exists |

## Cause

A simple comparison throws `ArgumentOutOfRangeException`.

```csharp
if (value < 0) throw new ArgumentOutOfRangeException(nameof(value));
```

## Rule description

.NET 8 provides helpers such as `ThrowIfNegative`, `ThrowIfGreaterThan`, and
`ThrowIfEqual`. The rule requires the exception's `nameof` argument to identify the
guarded operand and reports only when the exact helper exists.

## How to fix violations

Replace the conditional throw with the matching helper call. The provided code fix does
this automatically.
