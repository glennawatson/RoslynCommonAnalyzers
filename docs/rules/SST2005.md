# SST2005 — Use the 'is' type pattern instead of comparing an 'as' cast to null

| Item | Value |
| --- | --- |
| Category | Modernization |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `PatternMatchingAnalyzer` |

## Cause

Code casts with `as` and then compares the result to `null` to test a type. The `is` type
pattern expresses the same test directly.

```csharp
// 👎 reported
if ((value as string) != null)
{
}

if ((value as string) == null)
{
}

// 👍
if (value is string)
{
}

if (value is not string)
{
}
```

## Rule description

A comparison is reported when one side is the `null` literal and the other is an `as`
expression whose target is a reference type. `!= null` becomes `x is T`, and `== null` becomes
`x is not T`. The `as`/`null` shape is matched syntactically; the target is confirmed to be a
reference type semantically, so an `as` to a nullable value type (which would need a different
pattern) is left alone.

## How to fix violations

Use the `is` type pattern. The provided code fix rewrites the comparison as `x is T` or
`x is not T`.
