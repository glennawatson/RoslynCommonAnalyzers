# SST2006 — Use the 'is not' pattern instead of negating an 'is' check

| Item | Value |
| --- | --- |
| Category | Modernization |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |
| Analyzer | `PatternMatchingAnalyzer` |

## Cause

A type test is negated with `!`, where the `is not` pattern reads more directly.

```csharp
// 👎 reported
if (!(value is string))
{
}

// 👍
if (value is not string)
{
}
```

## Rule description

A logical-not is reported when it wraps a type test (`x is T`). Declaration patterns
(`x is T t`), which bind a name, are left alone because negating them is not a simple rewrite.
The check is purely syntactic.

## How to fix violations

Use the `is not` pattern. The provided code fix rewrites `!(x is T)` as `x is not T`.
