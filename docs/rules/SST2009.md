# SST2009 - Filter exceptions with a when clause

**Category:** Modernization
**Severity:** Warning
**Code fix:** Yes

A catch block that immediately tests a condition and rethrows on the losing branch is an
exception filter written by hand. Moving the condition into a `catch ... when` clause lets the
runtime decide whether to enter the handler *before* the stack is unwound, so exceptions the
handler was never going to keep skip it entirely and the original stack stays intact for outer
handlers and crash dumps.

Two shapes are reported: a catch whose only statement is an `if`/`else` where exactly one branch
is a bare `throw;`, and a catch that opens with `if (condition) throw;` followed by the real
handling code. Catches that already have a filter are left alone.

The condition must be side-effect-free — identifiers, member-access chains, literals, `is`
type/pattern tests, comparisons, and `!`/`&&`/`||` over those. Inside a `when` filter a thrown
exception is silently treated as `false` instead of propagating, so a condition that invokes a
method (or could otherwise run arbitrary code) is not reported: moving it could swallow an
exception the original code would have surfaced.

## Example

```csharp
void Process(bool canHandle)
{
    try
    {
        Work();
    }
    catch (InvalidOperationException ex)
    {
        if (canHandle) // SST2009
        {
            Log(ex);
        }
        else
        {
            throw;
        }
    }
}
```

Use:

```csharp
void Process(bool canHandle)
{
    try
    {
        Work();
    }
    catch (InvalidOperationException ex) when (canHandle)
    {
        Log(ex);
    }
}
```

The code fix keeps the surviving branch as the catch body. When the rethrow is on the winning
side of the test (`if (condition) throw;`), the filter becomes the negated condition; a simple
comparison is negated by inverting its operator (`==` becomes `!=`, `<` becomes `>=`, and so on).
