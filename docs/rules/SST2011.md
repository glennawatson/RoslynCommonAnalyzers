# SST2011 — Record instants in UTC

| Item | Value |
| --- | --- |
| Category | Modernization |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

An instant is recorded from the **local** clock: a `DateTime.Now` or `DateTimeOffset.Now` read that
is stored in a field or property, or returned.

```csharp
public sealed class AuditEntry
{
    private readonly DateTime _recorded = DateTime.Now;   // SST2011

    public DateTime Recorded => _recorded;
}
```

## Rule description

Local time is a presentation format, not a point in time. An instant stored as local time is
ambiguous twice a year — the hour that repeats when the clocks go back genuinely maps to two
different moments — and it means something different again the moment the process moves to another
machine. Record UTC; convert to local only where a human reads it.

```csharp
public sealed class AuditEntry
{
    private readonly DateTime _recorded = DateTime.UtcNow;

    public DateTime Recorded => _recorded;

    public string ForDisplay() => _recorded.ToLocalTime().ToString("f");
}
```

### What "recorded" means

The clock read has to *be* the value being kept — not part of an expression that consumes it. So
these are reported:

| Position | Example |
| --- | --- |
| A field initializer | `private DateTime _at = DateTime.Now;` |
| A property initializer | `public DateTime At { get; set; } = DateTime.Now;` |
| A store into a field or property | `_at = DateTime.Now;` |
| A returned expression | `return DateTime.Now;` |
| An expression-bodied member | `public DateTime At => DateTime.Now;` |

And these are clean, because the value never escapes the expression that read it:

```csharp
public string Greeting() => DateTime.Now.Hour < 12 ? "morning" : "afternoon";   // ok
public string Stamp() => DateTime.Now.ToString("t");                            // ok
```

Displaying local time is what local time is for. A store into a **local variable** is also
not reported: it does not outlive the statement it was written for.

### How this differs from SST1451

[SST1451](SST1451.md) asks whether a `DateTime` being **constructed** states its `DateTimeKind` —
`new DateTime(2024, 1, 1)` is `Unspecified`, and later conversions guess. This rule asks whether an
instant **read from the clock** came from the right clock.

They never overlap: SST1451 only looks at object creations and never at a property read; SST2011
only looks at property reads and never at an object creation. A file can trip both, on different
lines, and each is telling you something the other is not.

## How to fix violations

The code fix rewrites `.Now` to `.UtcNow`.

```diff
- private readonly DateTime _recorded = DateTime.Now;
+ private readonly DateTime _recorded = DateTime.UtcNow;
```

This **changes the recorded value** — that is what the rule is asking for. If the stored value is
later formatted for a user, add the `ToLocalTime()` at the point of display, which is where it
belongs. If some other code round-trips the value through a local-time format, fix that too; the
diagnostic has found a real bug, not a style problem.

The rewritten member access is bound speculatively before the fix is offered, so a `Now` on a type of
your own is never rewritten into a `UtcNow` that does not exist.
