# SST2013 — Merge an if that only wraps another if

| Item | Value |
| --- | --- |
| Category | Modernization |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

An `if` whose entire body is another `if`, with no `else` on either.

```csharp
if (order is not null)      // SST2013
{
    if (order.IsPaid)
    {
        Ship(order);
    }
}
```

## Rule description

Two nested `if`s with nothing between them and no `else` on either are **one condition written
across two levels of indentation**. Combining them with `&&` says the same thing — `&&`
short-circuits, so the second test still only runs when the first passed — and gives the reader back
a nesting level for something that earns it.

```csharp
if (order is not null && order.IsPaid)
{
    Ship(order);
}
```

### What is not reported

An `else` on **either** `if` stops the merge, because a merged condition can no longer tell the two
failure branches apart:

```csharp
if (a)
{
    if (b)
    {
        First();
    }
}
else            // runs when 'a' fails — not when 'a && b' fails
{
    Second();
}
```

Anything else in the outer block stops it too — the merge would move that statement under a
condition it was never guarded by:

```csharp
if (a)
{
    Log();      // not guarded by 'b'
    if (b)
    {
        Work();
    }
}
```

The `if` of an `else if` clause is reported like any other: `else if (b) { if (c) ... }` merges to
`else if (b && c)`.

## How to fix violations

The code fix merges the conditions.

```diff
- if (order is not null)
- {
-     if (order.IsPaid)
-     {
-         Ship(order);
-     }
- }
+ if (order is not null && order.IsPaid)
+ {
+     Ship(order);
+ }
```

### Parentheses

An operand is parenthesized when it binds **looser** than `&&` — an `||`, a `??`, a `?:`, an
assignment, or a `switch` expression — because dropping those parentheses would silently regroup the
condition:

```diff
- if (a || b)
- {
-     if (c || d)
-     {
+ if ((a || b) && (c || d))
+ {
```

Everything that binds **tighter** (`==`, `is`, `&`, `|`, `+`) is left exactly as it was written; no
noise parentheses are added.

### Comments

A comment that lived in the scaffolding the merge removes — after the outer condition, around the
braces, before the inner `if` — is carried above the merged statement rather than deleted:

```diff
  if (a)
  {
-     // b is the expensive one, so it is tested second.
-     if (b)
-     {
-         Work();
-     }
- }
+ // b is the expensive one, so it is tested second.
+ if (a && b)
+ {
+     Work();
+ }
```

A comment is somebody's explanation; a code fix does not get to decide it was wrong.
