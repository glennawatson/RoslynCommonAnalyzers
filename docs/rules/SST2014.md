# SST2014 — Avoid goto

| Item | Value |
| --- | --- |
| Category | Modernization |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A `goto` jumps to a label.

```csharp
foreach (var value in values)
{
    if (value < 0)
    {
        goto Failed;   // SST2014
    }
}

return Ok;

Failed:
return Error;
```

## Rule description

A `goto` means the reader can no longer work out how control reached a line by looking at the lines
above it. Every label is a place execution might arrive from somewhere else in the method, and
finding out where means reading the whole method — which is exactly the thing structured control
flow was invented to stop.

The structured forms say where control goes *in the same breath as why*:

```csharp
foreach (var value in values)
{
    if (value < 0)
    {
        return Error;
    }
}

return Ok;
```

`break`, `continue`, `return`, an extracted method with an early return, or a flag where none of
those fit — one of them is almost always what the `goto` was standing in for.

### `goto case` and `goto default` are not reported

Jumping between switch sections is the one place `goto` still earns its keep, because the language
offers no other word for it. Falling through from one case to the next, deliberately and explicitly,
is idiomatic C#:

```csharp
switch (state)
{
    case State.Retrying:
        Log("retrying");
        goto case State.Running;   // ok — not reported

    case State.Running:
        Run();
        break;

    default:
        goto default_handling;     // this one IS reported: it is a label, not a section
}
```

The exclusion is free rather than checked: C# gives the three forms three different syntax kinds, so
the analyzer that registers for `goto <label>` never sees a `goto case` or a `goto default` at all.

## How to fix violations

There is **no code fix**. Replacing a jump means restructuring the control flow around it, and which
structure was meant — a loop, an extracted method, an early return, a flag — is not something the
jump records. Guessing would be rewriting the method, not fixing it.
