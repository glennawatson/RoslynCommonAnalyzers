# SST2100 — Use an empty collection expression

| Item | Value |
| --- | --- |
| Category | CollectionExpressions |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled for C# 12 or later |

## Cause

An explicitly typed array or standard generic collection is created empty.

```csharp
int[] values = Array.Empty<int>();
```

## Rule description

C# 12 collection expressions provide the concise `[]` form. The rule requires a
supported explicit target type and excludes targetless `var` initializers and
`ImmutableArray<T>.Empty`.

## How to fix violations

Replace the creation with `[]`. The provided code fix does this automatically.
