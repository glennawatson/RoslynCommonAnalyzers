# SST2101 — Use an explicit collection expression

| Item | Value |
| --- | --- |
| Category | CollectionExpressions |
| Severity | Warning |
| Code fix | Yes |
| Default | Disabled (opt-in); C# 12 or later |

## Cause

A non-empty array or standard collection initializer has a supported explicit target.

```csharp
int[] values = new[] { 1, 2, 3 };
```

## Rule description

The equivalent collection expression is `[1, 2, 3]`. Dictionary-style multi-argument
elements and targetless `var` initializers are excluded. The rule is opt-in because
target typing can affect overload resolution.

## How to fix violations

Replace the creation and initializer braces with a collection expression. The provided
code fix does this automatically.
