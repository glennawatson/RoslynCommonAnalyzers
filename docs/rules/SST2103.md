# SST2103 - Use a collection expression for this factory call

A collection-builder factory call can usually be read more directly as the collection expression that
feeds the target. The rule keeps the check semantic: the target type must declare a matching collection
builder method before the call is reported.

## Example

```csharp
MyCollection<int> values = MyCollection.Create(1, 2, 3);
```

Use:

```csharp
MyCollection<int> values = [1, 2, 3];
```

## Notes

The analyzer requires C# 12 or later and an explicit target type. It handles narrow `Create` and
`CreateRange` shapes where the replacement can be built without changing the surrounding expression.
