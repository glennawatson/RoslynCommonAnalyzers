# SST2200 — Prefer the field keyword

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Warning |
| Code fix | Yes |
| Default | Disabled (opt-in); C# 14 or later |

## Cause

A private field is referenced only by one property's non-trivial accessors.

## Rule description

C# 14's `field` keyword removes the explicit backing field while preserving validation
or other accessor logic. Trivial properties are handled by SST1420 instead — a property
whose accessors only read and write one field, and a property with no accessor list, are
both left alone. Partial types, shared fields, static properties, and explicit interface
implementations are excluded.

The backing field is the first field the *containing type declares* that the property
references, so reading another type's field first does not hide the report:

```csharp
public int Value
{
    get => Other.Total + _value;          // _value is still the backing field
    set => _value = value < 0 ? 0 : value;
}
```

## How to fix violations

Replace backing-field references with `field` and remove the field declaration. The
provided code fix does this automatically, rewriting a `this.`-qualified access as a
whole — `this._value` becomes `field`, not `this.field`.
