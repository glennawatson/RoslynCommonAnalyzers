# SST2202 - Let the target type carry object creation

Object creation should avoid repeating a type already supplied by the target. The analyzer reports
only when the explicit target type and the constructed type are the same, and only for C# versions
that support target-typed `new`.

## Example

```csharp
Person person = new Person();
```

Use:

```csharp
Person person = new();
```

## Notes

This rule is deliberately conservative. It currently handles explicit local declarations and simple
assignments, and leaves `var`, arguments, and other context-sensitive forms alone.
