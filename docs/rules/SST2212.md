# SST2212 - Use UTF-8 literal bytes

Literal UTF-8 byte data should be written with the C# `u8` suffix when the target type can accept it.
This keeps the encoded bytes next to the source text and lets the compiler produce the literal data.

## Example

```csharp
ReadOnlySpan<byte> Header() => System.Text.Encoding.UTF8.GetBytes("GET");
```

Use:

```csharp
ReadOnlySpan<byte> Header() => "GET"u8;
```

For array targets, the code fix keeps the existing `byte[]` shape:

```csharp
byte[] Header() => "GET"u8.ToArray();
```

## Notes

The analyzer reports literal `Encoding.UTF8.GetBytes("...")` calls and byte-array initializers that
decode cleanly as UTF-8. Runtime strings, invalid byte sequences, and targets that cannot receive the
replacement are left alone.
