# SST2214 - Declare tuple parts directly

A tuple temporary that exists only so the next locals can copy its elements should be deconstructed at
the declaration site. This keeps the local names beside the expression that produced the tuple.

## Example

```csharp
var pair = GetPair();
var name = pair.Name;
var age = pair.Age;
```

Use:

```csharp
var (name, age) = GetPair();
```

## Notes

The analyzer currently reports the low-risk two-element shape where the tuple temporary is not used
after the copied locals. Wider tuple rewrites are left for future work so the code fix can stay simple
and allocation-light.
