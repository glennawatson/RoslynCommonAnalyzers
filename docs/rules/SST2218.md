# SST2218 - Let lambda parameter types come from the target

When a lambda is assigned to a delegate or expression-tree target, explicit parameter types can repeat
information already present in the target signature. Omitting them keeps the lambda focused on names and
behavior.

## Example

```csharp
Func<int, int, int> add = (int left, int right) => left + right;
```

Use:

```csharp
Func<int, int, int> add = (left, right) => left + right;
```

## Notes

The analyzer skips parameters with modifiers, attributes, or defaults. It also requires a resolved target
type before reporting, so ambiguous lambdas are left alone.
