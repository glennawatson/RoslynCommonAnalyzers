# SST2222 - Remove value overwritten before use

A local initializer or assignment that is immediately replaced before any read adds visual noise and can
hide the value that actually reaches later code. Removing the overwritten value leaves the meaningful
write in place. A postfix increment or decrement whose written value can never be read is the same defect
in operator form: `return value++;` steps a local that dies with the returning frame, and
`value = value++;` puts the original value straight back over the step.

## Example

```csharp
int value = 0;
value = input;
```

Use:

```csharp
int value;
value = input;
```

For a step whose result can never be read:

```csharp
int value = input;
return value++;
```

Use:

```csharp
int value = input;
return value;
```

## Notes

The analyzer only reports adjacent writes to the same local and only when the removed value is a literal,
`default`, or a local or parameter read. Values that may allocate, call code, or otherwise produce side
effects are not reported.

The increment and decrement forms are only reported for a non-`ref` local stepped with the built-in
operator, and stay silent whenever the step could still be observed or could throw: a `return` inside a
`try` block, a local captured by a lambda or local function, a local whose reference is handed out through
`ref`, `out`, `in`, or address-of, a local that belongs to an enclosing function, a `checked` context or a
compilation that checks overflow, and `decimal` values, whose arithmetic always throws on overflow.
