# SST2233 - Avoid LINQ calls on hot paths

Performance-sensitive code often pays too much for LINQ's iterator objects, delegate calls, and layered
abstractions. This opt-in rule flags in-memory `System.Linq.Enumerable` calls so hot-path projects can
choose explicit loops or purpose-built helpers instead.

## Example

```csharp
return values.Select(value => value + offset);
```

Prefer an explicit loop or a framework-specific fast path in code where this allocation and dispatch cost
matters.

## Notes

This rule is disabled by default. Enable it only in projects or folders where LINQ overhead is known to be
unwanted. The analyzer screens method names syntactically first, then verifies the target symbol is
`System.Linq.Enumerable`.

Set `stylesharp.avoid_linq_on_hot_path = true` in the same editorconfig scope as the diagnostic severity.
