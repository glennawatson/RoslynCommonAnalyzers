# SST2234 — Use the T? shorthand for nullable value types

**Category:** ModernSyntax
**Severity:** Warning
**Code fix:** Rewrites the spelling as `T?` (supports Fix All)

An explicit `Nullable<T>` (or `System.Nullable<T>`) spelling can use the `T?`
shorthand. The two forms are the same type; the shorthand is shorter and
matches how nullable annotations read everywhere else in modern C#.

Positions where the shorthand cannot be written are skipped: unbound generics
(`typeof(Nullable<>)`), `nameof` operands, member-access qualifiers, and
using-directive targets. The non-generic `Nullable` helper class is unrelated
and never reported.

## Example

```csharp
private Nullable<int> _value;        // SST2234: Replace 'Nullable<int>' with 'int?'
private int? _value;                 // ✓
public Type M() => typeof(Nullable<>); // ✓ unbound
```
