# SST2238 - Flatten nested property patterns

**Category:** ModernSyntax
**Severity:** Warning
**Code fix:** None

Nested property patterns can use extended property-pattern syntax when the
language version supports it. The flattened form keeps the tested member path in
one place and avoids an extra pair of braces.

## Example

```csharp
bool IsLocal(Person person)
{
    return person is { Address: { City: "Melbourne" } }; // SST2238
}
```

Use:

```csharp
bool IsLocal(Person person)
{
    return person is { Address.City: "Melbourne" };
}
```
