# SST2239 - Use method groups for forwarding lambdas

**Category:** ModernSyntax
**Severity:** Warning
**Code fix:** None

A lambda that only forwards its parameters to one method call can be written as a
method group. The method-group form removes a redundant parameter list and keeps
the target method as the visible operation.

The analyzer reports only straightforward forwarding lambdas where the argument
order matches the lambda parameter order.

## When the rule stays silent

A method group conversion only ever uses the target method's normal form, so the
rule does not fire when the lambda body relies on a call form the conversion
cannot reproduce:

```csharp
static void Log(params object[] args) { }
static void Log2(int value, int extra = 0) { }

Action<int> a = x => Log(x);   // no SST2239: Log packs x into object[]
Action<int> b = x => Log2(x);  // no SST2239: Log2 supplies extra's default
```

In both cases `Action<int> a = Log;` fails to compile with CS0123. Forwarding
lambdas converted to an expression tree are also skipped, because an expression
tree cannot hold a method group (CS0428).

## Example

```csharp
Func<int, int> transform = value => Normalize(value); // SST2239
```

Use:

```csharp
Func<int, int> transform = Normalize;
```
