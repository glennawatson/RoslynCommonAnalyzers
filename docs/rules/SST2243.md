# SST2243 - Use a raw string literal

**Category:** ModernSyntax
**Severity:** Warning
**Code fix:** Rewrites the literal as a raw string literal (supports Fix All)

A verbatim string literal full of doubled-quote escapes, or one spanning
multiple lines, reads with less escape noise as a raw string literal. The rule
only fires when the language version is C# 11 or later, since raw string
literals do not exist below that.

The rewrite keeps the string's value byte-identical: doubled quotes become
plain quotes, the delimiter grows one quote past the longest quote run in the
value (never below three), and multi-line content hangs between the delimiters
prefixed with the indentation of the line on which the literal starts.

Single-line verbatim literals whose value starts or ends with a quote stay
verbatim — a one-line raw string cannot begin or end its content with a quote
character. Multi-line literals containing a whitespace-only value line also
stay verbatim, because raw string indentation stripping could not keep such a
line's spaces intact. Interpolated literals, regular (non-verbatim) literals,
and literals inside preprocessor directives are never reported.

## Example

```csharp
string greeting = @"say ""hi"" now"; // SST2243

string block = @"line one
line two"; // SST2243
```

Use:

```csharp
string greeting = """say "hi" now""";

string block = """
line one
line two
""";
```
