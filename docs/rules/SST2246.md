# SST2246 — Express a same-value conditional chain as a switch

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Warning |
| Code fix | Yes (supports Fix All) |
| Default | Enabled; C# 8 or later |
| Analyzer | `Sst2246ChainedConditionalToSwitchAnalyzer` |

## Cause

A chain of nested conditional (`?:`) expressions compares the same value against a constant in every
test:

```csharp
public string Describe(int code)
    => code == 1 ? "one"
     : code == 2 ? "two"
     : code == 3 ? "three"
     : "other";
```

## Rule description

Each `?:` in the chain asks the same question — *what is `code`?* — and re-writes the value being
tested every time. Read top to bottom, the code is a lookup table with the table structure hidden
inside conditional syntax: to see the cases you have to peel one `code ==` off each line.

A switch expression is that same lookup table written as one:

```csharp
public string Describe(int code)
    => code switch
    {
        1 => "one",
        2 => "two",
        3 => "three",
        _ => "other"
    };
```

The tested value appears once, each case lines up against its result, and the trailing default has a
name (`_`) instead of being the last value left over from the final `?:`.

## What is reported

Only a chain where every test is the shape *value `==` constant* against the **same** value, and the
value is a plain read that cannot change between tests:

- The value is a simple local, parameter, or field — never a property or method call, whose read
  could have a side effect that a single switch would run only once.
- The value's type is one where `==` and a constant pattern agree: an enum, or an integral, `char`,
  or `string` type.
- Every test compares that value against a compile-time constant, and no constant repeats.
- The rewritten switch expression binds to the same type the chain already produced.

```csharp
value == Status.Open   ? "open"                       // reported: enum against named values
    : value == Status.Closed ? "closed" : "unknown";
```

## What is not reported

**A chain that tests different values.** A switch expression can only carry one governing value.

```csharp
x == 1 ? a : y == 2 ? b : c        // not reported — tests x, then y
```

**A test against something that is not a constant.** A switch arm pattern must be constant.

```csharp
value == first ? a : value == second ? b : c   // not reported — first/second are variables
```

**A single `?:`.** One test is already as clear as a two-arm switch, and reads better as a
conditional.

```csharp
value == 1 ? a : b                 // not reported
```

**A value whose read may have a side effect, or whose `==` need not match a constant pattern.** A
property read, a `double`, or a type with a user-defined `==` is left alone, because the switch could
mean something the chain did not.

```csharp
Count == 1 ? a : Count == 2 ? b : c   // not reported — Count is a property
d == 1.0 ? a : d == 2.0 ? b : c       // not reported — floating-point value
```

**A duplicated constant.** Repeating a constant would make a switch arm unreachable, so the chain is
left as written.

## How to fix violations

The code fix rewrites the chain as a switch expression, keeping each result and each constant exactly
as they were and adding a `_` arm for the chain's final value. The tested value is written once as the
governing expression.

## Language versions

Switch expressions arrived in C# 8, so the rule is silent on earlier language versions and the fix is
never offered there.

## Performance

The clean path is pure syntax: the outer conditional must test an identifier against a right-hand
operand with `==`, and its else branch must be another conditional testing that same identifier.
Only when that shape matches does the rule consult the semantic model — to confirm the tested value
is a side-effect-free variable of a supported type, that every test is a distinct constant, and that
the rewritten switch keeps the chain's type. Chains, which are rare, pay for that work; every other
conditional expression pays nothing.
