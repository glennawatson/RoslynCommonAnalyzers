# SST2247 — Deconstruct a value instead of copying its members

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Warning |
| Code fix | Yes (supports Fix All) |
| Default | Enabled |

## Cause

Consecutive local declarations each copy one member of the same value into its own local, and the
members read cover every position of that value in order.

```csharp
var a = pair.Item1;
var b = pair.Item2;
```

```csharp
var x = point.X;   // point exposes Deconstruct(out int x, out int y)
var y = point.Y;
```

## Rule description

Reading a value apart one member at a time repeats the value's name on every line and scatters the
parts down the block. When the reads cover the whole value in order, one deconstruction says the same
thing and keeps the parts beside the value they come from:

```csharp
var (a, b) = pair;
var (x, y) = point;
```

The value being read is left untouched — the fix only folds the member copies, so anything that used
`pair` or `point` afterwards still does.

The rule works for two kinds of value:

- **Tuples.** The reads must be the tuple's elements, one per position, in order. Both the positional
  spelling (`pair.Item1`) and named elements (`person.Name`) are recognised; the local names you gave
  the copies are kept.
- **`Deconstruct`-able values.** The value's type must expose a single public instance
  `Deconstruct(out …)` whose arity matches the number of reads, and each member read must line up by
  name with the same-position `out` parameter. That name match is what proves position 0's read is the
  value `Deconstruct` writes to its first parameter, and so on.

## What is not reported

**A partial or reordered read.** Folding is only safe when the reads cover every position in order.
A read that skips a position, or takes them out of order, is left alone because deconstruction would
assign the wrong parts.

```csharp
var a = triple.Item1;   // not reported — Item3 is never read
var b = triple.Item2;

var a = pair.Item2;     // not reported — a deconstruction would swap the two
var b = pair.Item1;
```

**A single read.** One `var a = pair.Item1;` has nothing to fold.

**A value that is not deconstructible.** If the source is neither a tuple nor a type with a matching
`Deconstruct`, or the member names do not line up with the `Deconstruct` parameters, there is no safe
mapping and nothing is reported.

```csharp
var a = box.First;      // not reported — Box has no Deconstruct
var b = box.Second;
```

**A source that is not a plain local or parameter.** The rule reads the value once in the rewrite, so
it only fires when the source is a side-effect-free local or parameter. Explicitly typed copies are
also left alone, because `var (a, b)` would infer the deconstructed types rather than keep the ones
you wrote.

**A value declared in the line just above.** When the value exists only to be taken apart on the very
next lines, that temporary is redundant and a different rule removes it outright; this rule stays
quiet there so the two never both fire.

```csharp
var pair = (1, 2);      // not reported here — the temporary itself is the thing to remove
var a = pair.Item1;
var b = pair.Item2;
```

## How to fix violations

The code fix replaces the run of copies with one deconstruction declaration, keeping the local names
and the surrounding trivia. The source value is not changed.

```csharp
// before
var name = person.Name;
var age = person.Age;

// after
var (name, age) = person;
```

## Language versions

Deconstruction declarations arrived in C# 7, so the rule is gated on C# 7 or later and stays silent
below it. Tuple sources depend on `System.ValueTuple`, which is present wherever tuple syntax compiles;
`Deconstruct` sources are resolved in the analysed compilation, so a value with no matching
`Deconstruct` is never reported.

## Performance

The clean path is pure syntax: each local declaration is checked for the `var name = source.member;`
shape, and the semantic model is consulted only once a run of two or more copies off the same
identifier is found. Tuple and `Deconstruct` resolution happens only on that rare match.
