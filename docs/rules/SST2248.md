# SST2248 — Combine comparisons into a pattern

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Warning |
| Code fix | Yes (supports Fix All) |
| Default | Enabled |

## Cause

Two comparisons of the same side-effect-free value against constants, joined by `&&` or `||`,
where a single `is`-pattern says the same thing:

```csharp
bool InDigitRange(int x) => x >= 0 && x <= 9;
bool IsWeekendStart(DayOfWeek day) => day == DayOfWeek.Saturday || day == DayOfWeek.Sunday;
bool OutOfBand(int n) => n < 0 || n > 100;
```

## Rule description

A range check, a membership check, and an out-of-band check each name the same value twice and
spell the relationship with an operator on both sides. A relational-and-logical pattern names the
value once and lets the shape of the test carry the rest:

```csharp
bool InDigitRange(int x) => x is >= 0 and <= 9;
bool IsWeekendStart(DayOfWeek day) => day is DayOfWeek.Saturday or DayOfWeek.Sunday;
bool OutOfBand(int n) => n is < 0 or > 100;
```

The three reported shapes are:

- a **bounded range** — a lower and an upper bound joined by `&&` (`x >= 0 && x <= 9`);
- a **membership set** — two equalities joined by `||` (`t == A || t == B`), or two inequalities
  joined by `&&` (`x != 3 && x != 5`);
- the **region outside a range** — a low bound and a high bound joined by `||`
  (`n < 0 || n > 100`).

`&&` becomes an `and` pattern and `||` becomes an `or` pattern, so the merged pattern matches for
exactly the same inputs as the original expression. A constant on the left of its operator
(`0 <= x`) is folded with the operator mirrored (`x is >= 0`), so the value always leads the pattern.

## What is not reported

**A subject that is anything but a plain variable read.** The subject has to be a bare local, field,
or parameter, so reading it once in the pattern is the same as reading it twice in the original.
A property, an indexer, a method call, or a member-access chain is left alone, because collapsing two
reads into one could change what the program observes:

```csharp
bool M(int x, int y) => x >= 0 && y <= 9;   // not reported — two different subjects
bool M() => Next() >= 0 && Next() <= 9;      // not reported — the subject is a call
bool M() => Count >= 0 && Count <= 9;        // not reported — the subject is a property
```

**A subject whose type is not integral, `char`, or an enum.** Floating-point, `decimal`, `string`,
`bool`, and nullable value types are skipped, so the rule never trips over a relational pattern the
type does not allow or a floating-point equality whose pattern form could read differently.

**A bound that is not a compile-time constant, or one that cannot become a pattern operand.** A bound
read from another variable stays an operator comparison, and a bound whose type does not convert into
the subject's type — a `long` literal against an `int` subject — is left alone so the fix cannot offer
a pattern that does not compile.

**A combination the compiler would prove always or never matches.** The dead `x == 1 && x == 2`, the
empty range `x >= 9 && x <= 0`, and the tautological `x >= 0 || x <= 9` are all skipped, so the fix
never introduces a "never matches" error or an "always matches" warning where the original operator
expression compiled quietly.

## How to fix violations

The code fix rewrites the two comparisons as one `is`-pattern, reusing the subject and both constants
from the original source and rebuilding only the operators and the `and`/`or` combinator:

```csharp
// before
if (code >= 0x20 && code <= 0x7E)
{
    Append((char)code);
}

// after
if (code is >= 0x20 and <= 0x7E)
{
    Append((char)code);
}
```

The surrounding trivia is carried across, so nothing around the expression moves.

## Language versions

Relational patterns and the `and`/`or` pattern combinators arrive in C# 9, so the rule and its fix
are gated on that language version and stay silent below it.

## Performance

The clean path is a syntax-kind test: unless a node is `&&`/`||` whose two operands are both
comparisons, the analyzer returns before it touches the semantic model. Only a candidate that
survives that test is bound to confirm the shared subject, the supported type, and the constant
bounds.
