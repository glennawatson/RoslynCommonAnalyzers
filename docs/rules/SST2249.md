# SST2249 — Use an interpolated string

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Warning |
| Code fix | Yes (supports Fix All) |
| Default | Enabled |

## Cause

A `string.Format` call with a literal format string, or a concatenation that splices values between
string literals, builds text that an interpolated string states in place.

```csharp
var a = string.Format("{0} of {1}", part, whole);
var b = "Hello " + name + "!";
```

## Rule description

A composite format string keeps the text and the values apart: the reader matches `{0}`, `{1}` in the
template against a trailing argument list, counting positions. A concatenation splits one sentence
across a chain of `+`, so the reader reassembles it left to right. An interpolated string puts each
value where it lands in the text, so the result reads in one pass.

```csharp
var a = $"{part} of {whole}";
var b = $"Hello {name}!";
```

Alignment and format specifiers carry over unchanged — `{0,5:X}` becomes `{value,5:X}` — and escaped
braces stay escaped.

## What is not reported

**A `string.Format` that passes an explicit format provider.** `string.Format(provider, "{0}", value)`
formats with the culture the caller chose; a plain interpolated string formats with the current
culture. Rewriting one to the other would silently change formatting for anyone whose current culture
differs from the supplied one, so the call is left alone.

```csharp
var s = string.Format(CultureInfo.InvariantCulture, "{0}", value);  // not reported
```

**A concatenation whose leading operands add as numbers.** In `1 + 2 + " items"` the `1 + 2` is
integer addition that happens before any string joins the chain, so the result is `"3 items"`.
Flattening the operands into holes would produce `"12 items"`. The rule only rewrites a chain where
every `+` is string concatenation.

```csharp
var s = 1 + 2 + " items";   // not reported — the leading additions are numeric
```

**A format the rule cannot reproduce faithfully.** A format string that is not a plain literal, whose
placeholders are not a gap-free `{0}..{n}` set matching the value count, that repeats a value (which
would evaluate it twice), or whose specifier holds characters a plain interpolated string cannot carry
is left alone. A single array handed to the `params` parameter is spread across the placeholders
rather than printed as one value, so it is left alone too.

```csharp
var s = string.Format("{0} {0}", value);       // not reported — value would be evaluated twice
var s = string.Format("{0} {2}", a, b, c);     // not reported — the set has a gap
var s = string.Format("{0} {1}", args);        // not reported — the array is spread, not printed
```

## How to fix violations

The code fix builds the interpolated string, reparses it, and speculatively binds it to confirm it
still compiles to the same `string` before offering the fix. Values become interpolation holes; string
literals become the surrounding text with braces, quotes, and backslashes escaped for a plain
interpolated string.

```csharp
// before
Log(string.Format("{0} of {1}", part, whole));

// after
Log($"{part} of {whole}");
```

## Language versions

Interpolated strings arrived in C# 6, so the rule reports nothing on an earlier language version. Both
a provider-free `string.Format` and a plain interpolated string format with the current culture, so the
rewrite does not depend on the target framework.

## Performance

The clean path is syntax only: a call is not a candidate unless it is a `Format` member access on a
string receiver with at least two arguments, and a `+` is not a candidate unless the chain mixes a
string literal with a value. Only then is the semantic model consulted to bind the call or prove the
concatenation is string concatenation.
