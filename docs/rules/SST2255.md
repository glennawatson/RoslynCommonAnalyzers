# SST2255 — Use string.IsNullOrEmpty

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Warning |
| Code fix | Yes (supports Fix All) |
| Default | Enabled |

## Cause

A value is tested for null or emptiness by hand, spelling out both halves of what
`string.IsNullOrEmpty` already names.

```csharp
if (name == null || name.Length == 0)
{
    return;
}
```

## Rule description

`string.IsNullOrEmpty(name)` is the null-or-empty test written once. The hand-rolled
disjunction reads the value twice, invites the short-circuit order to be got wrong (a
`.Length` read placed before the null check throws on null), and makes the reader
assemble the intent from two comparisons.

The rule reports three shapes, plus the negations:

```csharp
s == null || s.Length == 0        // becomes string.IsNullOrEmpty(s)
s == null || s == ""              // becomes string.IsNullOrEmpty(s)
s != null && s.Length != 0        // becomes !string.IsNullOrEmpty(s)
```

The value must be a side-effect-free `string` expression, and its two reads must be
syntactically identical, so folding them into one read changes nothing.

## What is not reported

**A length read before the null check.** `s.Length == 0 || s == null` reads `s.Length`
before proving `s` is non-null, so it throws where `string.IsNullOrEmpty(s)` would return
`true`. Rewriting it would change behaviour, so it is left alone.

```csharp
if (s.Length == 0 || s == null) { }   // not reported — different behaviour
```

**A non-string value.** `array == null || array.Length == 0` uses `Length` on something
that is not a `string`, where `string.IsNullOrEmpty` does not apply.

**A value with a side effect.** `Get() == null || Get().Length == 0` calls `Get()` twice;
the helper would call it once, so the rule stays quiet.

## How to fix violations

The fix replaces the whole expression with `string.IsNullOrEmpty(value)`, or its negation
for the `&&` form, keeping the value expression and the surrounding trivia.

```csharp
// before
if (name == null || name.Length == 0) { }

// after
if (string.IsNullOrEmpty(name)) { }
```

## Language versions

`string.IsNullOrEmpty` has been available since .NET Framework 2.0 and on every
`netstandard` target, so the fix is valid on every framework and language version.

## Performance

The rule registers on `||` and `&&` nodes and does a pure-syntax match before it ever
consults the semantic model; the type of the value is checked only once a matching shape
is found. The clean path allocates nothing.
