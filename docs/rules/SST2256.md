# SST2256 — Call extension methods in instance form

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Info |
| Code fix | Yes (supports Fix All) |
| Default | Enabled |

## Cause

An extension method is called in static form, passing the value it extends as the first
argument.

```csharp
var upper = Enumerable.Select(names, n => n.ToUpperInvariant());
```

## Rule description

An extension method exists so it can be called as though it were an instance method on its
first parameter. Calling it in static form — `Type.Extension(receiver, args)` — hides that
`receiver` is the value being extended and breaks the left-to-right reading of a call chain.

```csharp
var upper = names.Select(n => n.ToUpperInvariant());
```

The rule reports a static-form call only when the first argument binds to the extension's
`this` parameter and the instance form provably resolves to the same method. The fix builds
`receiver.Extension(args)`, binds it speculatively at the call site, and is offered only
when it resolves to the same method the static call resolved to.

## What is not reported

**A call already in instance form.** `names.Select(...)` is the target shape and is left
alone.

**A plain static method.** `int.Parse("3")` is not an extension method, so there is no
instance form to move to.

**A rewrite that would resolve differently.** If naming the receiver instance-style would
bind to a different method — or would not bind at all — the rule stays quiet rather than
offer a fix that changes meaning or does not compile.

## How to fix violations

The fix makes the first argument the receiver and keeps the remaining arguments in order.
A first argument whose precedence requires it is parenthesized.

```csharp
// before
var one = Enumerable.First(source, predicate);

// after
var one = source.First(predicate);
```

## Language versions

Extension methods have been in the language since C# 3.0. The rule adds no requirement of
its own beyond the extension method already resolving, so it applies wherever the static
call compiles.

## Performance

The rule registers on invocations, rejects anything that is not a `Type.Method(...)` shape
before binding, and only then resolves the method and speculatively binds the instance
form. The clean path allocates nothing.
