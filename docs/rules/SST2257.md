# SST2257 — Keep single-return lambdas expression-shaped

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Info |
| Code fix | Yes (supports Fix All) |
| Default | Enabled |

## Cause

A lambda's block body is a single `return expr;`.

```csharp
Func<int, int> next = x => { return x + 1; };
```

## Rule description

A block body that does nothing but return one expression wraps that expression in a pair of
braces, a `return` keyword, and a semicolon. The expression-bodied form says the same thing
without the ceremony.

```csharp
Func<int, int> next = x => x + 1;
```

The rewrite is pure syntax and never changes what the lambda computes.

## What is not reported

**A lambda that already has an expression body.** `x => x + 1` is the target shape.

**A block with more than one statement.** If the body declares a local, runs a statement, or
branches before returning, the block is doing real work and is left alone.

```csharp
Func<int, int> f = x =>
{
    var y = x + 1;   // not reported — more than one statement
    return y;
};
```

**A void lambda whose block is an expression statement.** Only a single value-returning
`return expr;` is reported; a `x => { DoWork(x); }` body is left alone.

## How to fix violations

The fix replaces the block body with the returned expression, keeping the arrow, the
expression, and the body's trailing trivia.

```csharp
// before
Func<string, int> length = s => { return s.Length; };

// after
Func<string, int> length = s => s.Length;
```

## Language versions

Expression-bodied lambdas have been available since lambdas were introduced in C# 3.0, so
the fix is valid on every language version.

## Performance

The rule is a pure-syntax check on lambda nodes — is the body a block of exactly one
value-returning statement — and never consults the semantic model. The clean path allocates
nothing.
