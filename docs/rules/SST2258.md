# SST2258 — Drop a redundant delegate wrapper

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Info |
| Code fix | Yes (supports Fix All) |
| Default | Enabled |

## Cause

A method group is wrapped in an explicit delegate creation the target type already supplies.

```csharp
Changed += new EventHandler(OnChanged);
```

## Rule description

Subscribing to an event or assigning a delegate with `new SomeDelegate(method)` spells out a
conversion the target type performs on its own. The bare method group converts to the same
delegate.

```csharp
Changed += OnChanged;
```

The rule reports the wrapper only when its argument is a method group, the created type is a
delegate, and the syntactic position accepts a method-group conversion. The fix drops the
wrapper and is offered only when the bare method group provably binds to the same delegate.

## What is not reported

**A wrapper around a lambda.** `new Action(() => { })` wraps a lambda, not a method group;
there is nothing to unwrap.

**A wrapper around an existing delegate value.** `new Action(existing)` copies a delegate
rather than converting a method group, so removing it could change meaning; it is left alone.

**A position that does not accept a straightforward method-group conversion.** A method group
has no type of its own, so it converts only where a delegate target is supplied directly (an
assignment, an initializer, a return, an arrow body, a cast). An operand of `+`/`-` delegate
combination needs an already-typed delegate, and an argument position can change overload
resolution when the wrapper is dropped, so both are left alone.

## How to fix violations

The fix replaces the delegate creation with the bare method group.

```csharp
// before
Timer.Elapsed += new ElapsedEventHandler(OnElapsed);

// after
Timer.Elapsed += OnElapsed;
```

## Language versions

Method-group-to-delegate conversion has been in the language since C# 2.0, so the fix is
valid on every language version.

## Performance

The rule registers on object creations, checks the single-argument shape syntactically, and
only then resolves the delegate type and classifies the conversion. The clean path allocates
nothing.
