# SST2259 — Remove a stray semicolon after a declaration

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Info |
| Code fix | Yes (supports Fix All) |
| Default | Enabled |

## Cause

A type declaration that already ends in a brace body carries a trailing semicolon.

```csharp
internal sealed class Marker
{
};
```

## Rule description

A type declaration may carry an optional trailing semicolon that the grammar permits but
nothing needs. The semicolon states an empty statement that does nothing and leaves the
reader wondering what it is for.

```csharp
internal sealed class Marker
{
}
```

The rule reports the trailing semicolon on `class`, `struct`, `interface`, `record`,
`record struct`, and `enum` declarations that have a brace body.

## What is not reported

**A declaration with no stray semicolon.** `class C { }` is the target shape.

**A positional record's required semicolon.** `record R(int X);` ends in a semicolon because
it has no brace body; that semicolon is required, not stray.

**A trailing semicolon after a method body.** `void M() { };` is a compiler error (CS1597)
rather than a stray-but-legal token. The compiler already reports it, so this rule stays
silent there and does not add a second diagnostic.

## How to fix violations

The fix removes the semicolon and moves its trailing trivia onto the closing brace so the
newline after the declaration survives.

```csharp
// before
internal enum Mode { On, Off };

// after
internal enum Mode { On, Off }
```

## Language versions

The optional trailing semicolon on a type declaration has been in the language since C# 1.0,
so the fix is valid on every language version.

## Performance

The rule registers on type declaration nodes and reads two tokens — is there a closing brace
and a trailing semicolon. It never consults the semantic model, and the clean path allocates
nothing.
