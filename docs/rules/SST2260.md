# SST2260 — Remove an 'as' cast to a type the value already has

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Info |
| Code fix | Yes (supports Fix All) |
| Default | Enabled |

## Cause

An `as` conversion targets the type the operand already has.

```csharp
string label = text as string;   // text is already a string
```

## Rule description

An `as` conversion to the operand's own type can never return null and never changes the
value or its type. It only adds a conversion the reader has to check.

```csharp
string label = text;
```

The rule reports an `as` expression whose operand's static type is exactly the target type.

## What is not reported

**A widening or narrowing conversion.** `o as string`, where `o` is `object`, is a real
runtime test and is left alone. `s as object`, where `s` is `string`, changes the expression's
type from `string` to `object`, which can change overload resolution, so it too is left alone.

The related rule for a redundant parenthesized cast such as `(int)anInt` owns the `(T)` form;
this rule covers the `as` form.

## How to fix violations

The fix drops the `as` and keeps the operand. The operand of an `as` expression always binds
at least as tightly as the `as` expression it replaces, so no parentheses are ever needed.

```csharp
// before
var length = (value as string).Length;

// after
var length = (value).Length;
```

## Language versions

The `as` operator has been in the language since C# 1.0, so removing a redundant one is valid
on every language version and every target framework.

## Performance

The rule registers on `as` expressions — relatively rare — and compares the operand's type
with the target type through the semantic model. The clean path allocates nothing.
