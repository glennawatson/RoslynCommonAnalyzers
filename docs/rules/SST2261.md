# SST2261 — Use the exclusive-or operator

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Info |
| Code fix | Yes (supports Fix All) |
| Default | Enabled |

## Cause

Exclusive-or is spelled out with a pair of mirror-image conjunctions.

```csharp
bool differ = (left && !right) || (!left && right);
```

## Rule description

`(x && !y) || (!x && y)` — and the bitwise `(x & !y) | (!x & y)` — is exclusive-or written
the long way. It reads each operand twice and asks the reader to confirm the two conjunctions
are mirror images. `x ^ y` says it once.

```csharp
bool differ = left ^ right;
```

The rule reports the long form only when `x` and `y` are side-effect-free boolean
expressions, because the long form reads each of them twice and `^` reads each once. Both
operands are required to be `bool`, so the rewrite keeps logical exclusive-or semantics.

## What is not reported

**Operands with a side effect.** `(P() && !Q()) || (!P() && Q())` calls `P()` and `Q()`
twice; `^` would call each once, so the rule stays quiet.

**A disjunction that is not a mirror image.** `(x && !y) || (!x && z)` does not test the same
two values in both conjunctions, so it is not exclusive-or and is left alone.

## How to fix violations

The fix replaces the whole expression with `x ^ y`, taking the two values from the original
conjunctions.

```csharp
// before
if ((a && !b) || (!a && b)) { }

// after
if (a ^ b) { }
```

## Language versions

The `^` operator has been in the language since C# 1.0, so the fix is valid on every language
version.

## Performance

The rule registers on `||` and `|` nodes and matches the shape by pure syntax; it consults
the semantic model only to confirm both operands are `bool`, and only once a matching shape
is found. The clean path allocates nothing.
