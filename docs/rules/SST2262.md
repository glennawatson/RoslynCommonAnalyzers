# SST2262 — Use a regular string literal

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Info |
| Code fix | Yes (supports Fix All) |
| Default | Enabled |

## Cause

A single-line raw string literal holds content that needs none of what raw syntax is for.

```csharp
private const string Greeting = """plain text""";
```

## Rule description

Raw string literals exist to hold content that a regular literal would have to escape —
quotes, backslashes, or line breaks. A single-line raw literal whose content has no quote and
no backslash buys nothing over a regular literal, which is shorter and needs no escapes
because the content has none to escape.

```csharp
private const string Greeting = "plain text";
```

This is the inverse of promoting a literal to raw syntax when doubled quotes or line breaks
are what make raw pay off.

## What is not reported

**Content that carries a quote.** `"""say "hi" there"""` avoids escaping a `"`, which is
exactly what raw syntax is for, so it is left alone.

**Content that carries a backslash.** `"""a\b"""` would need a `\\` in a regular literal, so
raw syntax is earning its keep.

**A regular literal.** `"plain text"` is already the target shape.

## How to fix violations

The fix replaces the raw literal with a regular literal carrying the same content.

```csharp
// before
var name = """widget""";

// after
var name = "widget";
```

## Language versions

The rule only ever sees raw string literals, which require C# 11, and it rewrites to a
regular literal that is valid on every language version and target framework.

## Performance

The rule registers on string-literal nodes, ignores anything that is not a single-line raw
literal, and scans the content once for a quote, a backslash, or a control character. It
never consults the semantic model, and the clean path allocates nothing.
