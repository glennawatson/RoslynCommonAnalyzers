# SST2263 — Hoist a loop's guard condition into its header

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Info |
| Code fix | Yes (supports Fix All) |
| Default | Enabled |

## Cause

An infinite loop guards its body with a condition that belongs in the loop header.

```csharp
while (true)
{
    if (queue.Count > 0)
    {
        Handle(queue.Dequeue());
    }
    else
    {
        break;
    }
}
```

## Rule description

`while (true)` and `for (;;)` spend the loop header on `true` and then re-derive the real
stopping condition inside the body. Moving that condition into the header puts the loop's exit
where a reader looks for it.

```csharp
while (queue.Count > 0)
{
    Handle(queue.Dequeue());
}
```

The rule reports two shapes:

- `if (cond) { work } else break;` as the whole body, which hoists `cond` into the header.
- a leading `if (!cond) break;` guard before the work, which hoists the negated guard into
  the header.

## What is not reported

**An empty guarded body.** `while (true) { if (cond) { } else break; }` guards nothing; that
degenerate shape is left to the rules that own empty branches.

**A loop that is not a condition-hoist shape.** A `while (true)` whose body simply runs and
breaks, with no guard condition to move, is left alone.

## How to fix violations

The fix rewrites the loop as `while (cond) { work }`. The guarded work becomes the loop body
and is re-indented to its new depth. A guard-then-break shape has its guard negated for the
header — a leading `!` is unwrapped where present.

```csharp
// before
for (;;)
{
    if (!reader.Read()) break;
    Process(reader);
}

// after
while (reader.Read())
{
    Process(reader);
}
```

## Language versions

`while` has been in the language since C# 1.0, so the rewrite is valid on every language
version and target framework.

## Performance

The rule registers on `while` and `for` nodes, matches the infinite-loop-with-guard shape by
pure syntax, and never consults the semantic model. The clean path allocates nothing.
