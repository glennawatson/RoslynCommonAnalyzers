# SST2266 — Inline a local that is read once

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Warning |
| Code fix | Yes (supports Fix All) |
| Default | Disabled (opt-in) |

## Cause

A local is assigned once and read exactly once, and its initializer can be inlined into that one use.

```csharp
var name = _value;
return name;
```

## Rule description

A local that is used a single time asks the reader to carry a name from its declaration to one downstream use.
Inlining the initializer removes the hop.

```csharp
return _value;
```

Whether an intermediate local reads better than the inlined expression is a house-style call — a well-named
temporary is often kept on purpose — so the rule ships **disabled**. Enable it in `.editorconfig` for a
codebase that prefers to inline single-use locals.

The rule only reports when inlining is provably behaviour-preserving:

- **The initializer is side-effect-free.** It reads state without invoking a method, allocating, awaiting, or
  mutating — a literal, an identifier, a member access, a cast, or an operator expression over those. A local
  initialized by a call (`var x = Compute();`) is left alone, because moving the call would move its effect.
- **The single read immediately follows the declaration.** With nothing between the declaration and the use,
  no intervening statement can change the value the initializer would read.
- **Nothing side-effecting runs before the read in that statement.** If the use statement evaluates a call or
  assignment before reaching the read — `return Side() + value;` — the initializer is not moved there, because
  that would reorder the side effect relative to the read.
- **The local is a plain, uncaptured read.** It is not captured by a lambda or local function, not aliased by
  `ref`/`out`/`in`, and not written after its declaration.

## What is not reported

```csharp
var value = Compute();          // not reported — the initializer has side effects
return value;

var value = a;                  // not reported — read more than once
return value + value;

var value = a;                  // not reported — the use is not adjacent
DoSomething();
return value;

var value = a;                  // not reported — captured by a lambda
return () => value;
```

## How to fix violations

The code fix replaces the reference with the initializer — wrapped in parentheses when its precedence could
otherwise change — and removes the declaration statement.

```csharp
// before
var sum = a + b;
return sum * 2;

// after
return (a + b) * 2;
```

## Enabling the rule

The rule is off by default. Turn it on in `.editorconfig`:

```ini
[*.cs]
dotnet_diagnostic.SST2266.severity = warning
```

## Performance

The rule matches the declaration shape and checks the initializer's purity syntactically before it resolves the
local or scans for its references, so an ordinary declaration pays almost nothing.
