# SST2267 — Write infinite loops in one consistent style

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Warning |
| Code fix | Yes (supports Fix All) |
| Default | Disabled (opt-in) |

## Cause

An infinite loop is written in the form the codebase did not choose: `for (;;)` when the configured style is
`while`, or `while (true)` when it is `for`.

```csharp
for (;;)
{
    Pump();
}
```

## Rule description

A loop meant to run forever has two idiomatic spellings — `for (;;)` and `while (true)` — and a codebase that
mixes them makes the reader pause on the difference where there is none. This rule normalizes both to the one
style the project picks. The two forms compile identically, so the choice is a pure preference and the rule
ships **disabled**.

```csharp
while (true)
{
    Pump();
}
```

Only the two canonical infinite loops are candidates: a `for` with no declaration, initializer, condition, or
incrementor, and a `while` whose condition is the literal `true`. A `for` that still carries a condition or a
clause is not an infinite loop and is not touched.

## Configuration

```ini
# Rule-specific
stylesharp.SST2267.infinite_loop_style = while

# Project-wide
stylesharp.infinite_loop_style = while
```

| Option | Values | Default | Effect |
| --- | --- | --- | --- |
| `infinite_loop_style` | `while`, `for` | `while` | The form every infinite loop is normalized to. |

An unset or unrecognized value falls back to `while`.

## How to fix violations

The code fix rewrites the loop into the configured form, reusing the loop's own keyword trivia, parentheses,
and body so nothing else moves.

```csharp
// with stylesharp.infinite_loop_style = while
for (;;) { Pump(); }   // becomes:  while (true) { Pump(); }

// with stylesharp.infinite_loop_style = for
while (true) { Pump(); }   // becomes:  for (;;) { Pump(); }
```

## Enabling the rule

The rule is off by default. Turn it on in `.editorconfig`:

```ini
[*.cs]
dotnet_diagnostic.SST2267.severity = warning
```

## Performance

The option is read only after a loop is matched as one of the two infinite-loop shapes, so a source with no
infinite loops pays nothing. The check is pure syntax and never binds a symbol.
