# SST2268 — Keep object-creation parentheses consistent

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Warning |
| Code fix | Yes (supports Fix All) |
| Default | Disabled (opt-in) |

## Cause

An object creation that carries an initializer uses the empty argument parentheses in the form the codebase
did not choose: `new T() { ... }` when the configured style is `omit`, or `new T { ... }` when it is `include`.

```csharp
var point = new Point() { X = 1, Y = 2 };
```

## Rule description

When an object creation has an initializer, the empty `()` before it is optional — `new T() { ... }` and
`new T { ... }` construct the same object. A codebase that mixes the two spellings has no reason to; this rule
normalizes to the one it picks. The choice is a pure preference, so the rule ships **disabled**.

```csharp
var point = new Point { X = 1, Y = 2 };
```

Only a creation that already has an initializer and whose argument list is empty (or absent) is a candidate.
A creation that passes real constructor arguments — `new T(4) { ... }` — cannot drop its parentheses and is
never touched.

## Configuration

```ini
# Rule-specific
stylesharp.SST2268.object_creation_parentheses = omit

# Project-wide
stylesharp.object_creation_parentheses = omit
```

| Option | Values | Default | Effect |
| --- | --- | --- | --- |
| `object_creation_parentheses` | `omit`, `include` | `omit` | Whether an initializer creation keeps its empty `()`. |

An unset or unrecognized value falls back to `omit`.

## How to fix violations

The code fix adds or removes the empty parentheses, keeping the type, the initializer, and the surrounding
trivia.

```csharp
// with stylesharp.object_creation_parentheses = omit
new Point() { X = 1 };   // becomes:  new Point { X = 1 };

// with stylesharp.object_creation_parentheses = include
new Point { X = 1 };     // becomes:  new Point() { X = 1 };
```

## Enabling the rule

The rule is off by default. Turn it on in `.editorconfig`:

```ini
[*.cs]
dotnet_diagnostic.SST2268.severity = warning
```

## Performance

The candidate shape — an object creation with an initializer and no real arguments — is matched purely
syntactically, and the option is read only for a candidate. The rule never binds a symbol.
