# SST2269 — Keep conditional-condition parentheses consistent

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Warning |
| Code fix | Yes (supports Fix All) |
| Default | Disabled (opt-in) |

## Cause

The condition of a conditional (`?:`) expression carries parentheses in the form the codebase did not choose: a
parenthesized single token when the configured style is `omit_when_single_token`, or an unparenthesized
condition when it is `include`.

```csharp
var label = (ready) ? "go" : "wait";
```

## Rule description

The condition of a `?:` expression is sometimes wrapped in parentheses and sometimes not. When the condition is
a single identifier or literal, the parentheses add nothing.

```csharp
var label = ready ? "go" : "wait";
```

This rule normalizes to the one style the project picks. The choice is a pure preference, so the rule ships
**disabled**.

Under the default, only a condition that is a single identifier or literal wrapped in parentheses is reported.
A parenthesized larger expression — `(a && b) ? ... : ...` — keeps its parentheses, because there they can
group and clarify.

## Configuration

```ini
# Rule-specific
stylesharp.SST2269.conditional_condition_parentheses = omit_when_single_token

# Project-wide
stylesharp.conditional_condition_parentheses = omit_when_single_token
```

| Option | Values | Default | Effect |
| --- | --- | --- | --- |
| `conditional_condition_parentheses` | `omit_when_single_token`, `include` | `omit_when_single_token` | Whether a `?:` condition is parenthesized. |

Under `omit_when_single_token`, parentheses around a single identifier or literal are removed and everything
else is left alone. Under `include`, a condition without parentheses is wrapped. An unset or unrecognized value
falls back to `omit_when_single_token`.

## How to fix violations

The code fix unwraps or wraps the condition, keeping the condition expression and the surrounding trivia.

```csharp
// with stylesharp.conditional_condition_parentheses = omit_when_single_token
(ready) ? a : b;   // becomes:  ready ? a : b;

// with stylesharp.conditional_condition_parentheses = include
ready ? a : b;     // becomes:  (ready) ? a : b;
```

## Enabling the rule

The rule is off by default. Turn it on in `.editorconfig`:

```ini
[*.cs]
dotnet_diagnostic.SST2269.severity = warning
```

## Performance

The rule matches the condition's shape syntactically and reads the option only for a candidate condition. It
never binds a symbol.
