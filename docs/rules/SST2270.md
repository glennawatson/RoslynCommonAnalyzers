# SST2270 — Keep array-creation element types consistent

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Warning |
| Code fix | Yes (supports Fix All) |
| Default | Disabled (opt-in) |

## Cause

An array creation names its element type in the form the codebase did not choose: an implicit `new[] { ... }`
when the configured style is `explicit`, or an explicit `new T[] { ... }` when it is `implicit` or
`implicit_when_obvious`.

```csharp
var values = new int[] { 1, 2, 3 };
```

## Rule description

An array creation with an initializer can name its element type or let the compiler infer it. This rule
normalizes to the one form the project picks. The choice is a pure preference, so the rule ships **disabled**.

```csharp
var values = new[] { 1, 2, 3 };
```

This rule governs **only** the explicit-versus-implicit element-type axis — it never suggests a collection
expression (`[ ... ]`), which is a separate direction owned by another rule.

Only a single-dimensional array with an initializer and no explicit size is convertible. Safety is a
speculative bind: the rewritten form is bound at the creation's position, and the diagnostic is raised only
when it resolves to the identical array type, so an array whose elements have no single best type —
`new object[] { 1, "text" }` — is left alone rather than reported with a rewrite that would not compile.

## Configuration

```ini
# Rule-specific
stylesharp.SST2270.array_creation_type_style = implicit_when_obvious

# Project-wide
stylesharp.array_creation_type_style = implicit_when_obvious
```

| Option | Values | Default | Effect |
| --- | --- | --- | --- |
| `array_creation_type_style` | `explicit`, `implicit`, `implicit_when_obvious` | `implicit_when_obvious` | Which element-type form arrays are normalized to. |

- `explicit` — every implicit `new[] { ... }` gains its element type.
- `implicit` — every convertible explicit `new T[] { ... }` drops its element type.
- `implicit_when_obvious` — an explicit array drops its element type only when the elements make the type
  obvious (each is a literal, a cast, or an object/array creation); other explicit arrays, and all implicit
  arrays, are left alone.

An unset or unrecognized value falls back to `implicit_when_obvious`.

## How to fix violations

The code fix names or drops the element type, keeping the initializer and surrounding trivia. The rewrite is
re-bound before it is offered.

```csharp
// with stylesharp.array_creation_type_style = explicit
new[] { 1, 2 };        // becomes:  new int[] { 1, 2 };

// with stylesharp.array_creation_type_style = implicit (or implicit_when_obvious)
new int[] { 1, 2 };    // becomes:  new[] { 1, 2 };
```

## Enabling the rule

The rule is off by default. Turn it on in `.editorconfig`:

```ini
[*.cs]
dotnet_diagnostic.SST2270.severity = warning
```

## Performance

The convertibility of an explicit array — single dimension, initializer, no size — is checked syntactically,
and the option is read before any bind. The speculative bind runs only for a candidate that could actually be
rewritten.
