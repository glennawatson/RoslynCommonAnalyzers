# SST2272 — Keep Flags-enum values in one consistent style

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Warning |
| Code fix | Yes (supports Fix All) |
| Default | Disabled (opt-in) |

## Cause

A single-bit member of a `[Flags]` enum is written in the form the codebase did not choose: a decimal literal
when the configured style is `shift`, or a `1 << n` shift when it is `decimal`.

```csharp
[Flags]
public enum FileAccess
{
    None = 0,
    Read = 1,
    Write = 2,
    Execute = 4,
}
```

## Rule description

The single-flag members of a `[Flags]` enum can be written as decimal values or as bit shifts. A codebase that
mixes them obscures which members are single flags and which are combinations. This rule normalizes the
single-bit members to the one form the project picks. The choice is a pure preference, so the rule ships
**disabled**.

```csharp
[Flags]
public enum FileAccess
{
    None = 0,
    Read = 1 << 0,
    Write = 1 << 1,
    Execute = 1 << 2,
}
```

Only a member whose constant value is a single set bit — a power of two, up to `1 << 30` — is normalized. A
combined value such as `3`, a zero, and a member with no explicit value are all left alone, so only the true
single flags are touched. The enum must carry the `[Flags]` attribute.

## Configuration

```ini
# Rule-specific
stylesharp.SST2272.enum_flag_value_style = shift

# Project-wide
stylesharp.enum_flag_value_style = shift
```

| Option | Values | Default | Effect |
| --- | --- | --- | --- |
| `enum_flag_value_style` | `shift`, `decimal` | `shift` | The form single-bit flag values are normalized to. |

An unset or unrecognized value falls back to `shift`.

## How to fix violations

The code fix rewrites the member value into the configured form, keeping the value's trivia.

```csharp
// with stylesharp.enum_flag_value_style = shift
Execute = 4;        // becomes:  Execute = 1 << 2;

// with stylesharp.enum_flag_value_style = decimal
Execute = 1 << 2;   // becomes:  Execute = 4;
```

## Enabling the rule

The rule is off by default. Turn it on in `.editorconfig`:

```ini
[*.cs]
dotnet_diagnostic.SST2272.severity = warning
```

## Performance

The rule resolves the `[Flags]` attribute and each member's constant only for an enum that matched the
candidate shape, and the value form (decimal versus shift) is distinguished syntactically first.
