# SST2273 — Prefer a guard clause over a trailing wrapping if

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Warning |
| Code fix | Yes (supports Fix All) |
| Default | Disabled (opt-in) |

## Cause

A method, constructor, accessor, local function, or `for`/`foreach`/`while` loop does its real work last,
wrapped in a single trailing `if (cond) { work }` with no `else`.

```csharp
public void Save(Order order)
{
    if (order.IsValid)
    {
        Normalize(order);
        Persist(order);
    }
}
```

## Rule description

Wrapping the body in a trailing `if` nests the real work one level deeper than it needs to be. Inverting the
condition into an early-exit guard states the exit up front and lets the body run at the outer level.

```csharp
public void Save(Order order)
{
    if (!order.IsValid)
        return;
    Normalize(order);
    Persist(order);
}
```

Inside a loop the guard exits the current iteration with `continue`:

```csharp
foreach (var order in orders)
{
    if (!order.IsValid)
        continue;
    Normalize(order);
    Persist(order);
}
```

Whether a guard clause reads better than a wrapping `if` is a house-style call, so the rule ships **disabled**.
Enable it in `.editorconfig` for a codebase that writes its exits as guards.

The rule reports only when inverting is provably behaviour-preserving:

- **The `if` is the last statement of the body and has no `else`.** Nothing runs after it, so lifting the work
  out cannot reorder anything, and there is no alternate branch whose flow the inversion would drop. Statements
  *before* the `if` stay exactly where they are, ahead of the guard.
- **The guard's exit can actually be expressed.** A `void` member, an async non-generic `Task`/`ValueTask`
  member, a constructor, and a value-free accessor (`set`, `init`, `add`, `remove`) all end with a bare
  `return;`; a loop body exits its iteration with `continue;`. A member that must produce a value on every
  path — a value-returning method, an operator, a `get` accessor, an iterator — is left alone, because no bare
  `return;` fits it.
- **The wrapped work is worth flattening.** The body must hold at least `min_wrapped_statements` statements
  (see below), so a one-line `if (x) Do();` is left as it is.

Only a direct member or loop body is treated as an exit boundary. A block nested inside an `if`, `try`,
`using`, or `lock` is not: a jump there would leave the whole member, not just that block, so those are not
reported.

## What is not reported

```csharp
if (ready)                       // not reported — has an else branch
{
    Work();
}
else
{
    Other();
}

if (ready)                       // not reported — wraps a single statement (below the threshold)
{
    Work();
}

if (ready)                       // not reported — not the last statement
{
    Work();
    More();
}
Trailing();
```

## How to fix violations

The code fix negates the condition, heads the body with `if (!cond) return;` (or `continue;` inside a loop),
and lifts the previously wrapped work to the outer block, re-indenting it. A leading `!` is stripped and an
equality comparison is flipped to its opposite operator; a relational or combined condition is wrapped in
`!(...)`, because flipping `<` to `>=` would disagree on `NaN` operands.

```csharp
// before
public void Save(Order order)
{
    if (order.IsValid)
    {
        Normalize(order);
        Persist(order);
    }
}

// after
public void Save(Order order)
{
    if (!order.IsValid)
        return;
    Normalize(order);
    Persist(order);
}
```

## Enabling the rule

The rule is off by default. Turn it on in `.editorconfig`:

```ini
[*.cs]
dotnet_diagnostic.SST2273.severity = warning
```

## Options

| Option | Default | Meaning |
| --- | --- | --- |
| `stylesharp.SST2273.min_wrapped_statements` | `2` | The fewest statements a trailing `if` must wrap before it is reported. Set it to `1` to flag every wrapping `if`, or higher to only flag larger bodies. |

The general form `stylesharp.min_wrapped_statements` sets the same value for the whole project; the
rule-specific key wins where both are present. An unset, non-numeric, or non-positive value keeps the default.

```ini
[*.cs]
dotnet_diagnostic.SST2273.severity = warning
stylesharp.SST2273.min_wrapped_statements = 3
```

## Performance

The whole match is syntactic — no `else`, last statement, an exit-boundary owner, and enough wrapped
statements — so a clean `if` rejects before any `.editorconfig` option is read and without binding a single
symbol. The rule is disabled by default, so a project that has not enabled it pays nothing at all.
