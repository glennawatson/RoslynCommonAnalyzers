# SST2274 — Convert an as assignment and null check to an is pattern

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Warning |
| Code fix | Yes (supports Fix All) |
| Default | Enabled |

## Cause

A local is assigned from an `as` conversion and then tested for null, splitting one type test across a
declaration and a separate null check.

```csharp
public int Measure(object value)
{
    var text = value as string;
    if (text != null)
    {
        return text.Length;
    }

    return 0;
}
```

## Rule description

`var s = o as T;` followed by a null check on `s` is a type test written in two parts: the conversion that
captures the value and the null check that decides whether the conversion succeeded. An `is` declaration
pattern says both at once — it tests the type and binds the value in a single expression — so the local no
longer has to exist just to be null-checked.

Two shapes are recognised, distinguished by which way the null check points.

**Guarded-use shape.** When a non-null check guards a block that uses the value, the declaration folds into a
positive pattern and the guarded body is left untouched:

```csharp
public int Measure(object value)
{
    if (value is string text)
    {
        return text.Length;
    }

    return 0;
}
```

The non-null check can be written as `text != null`, `text is not null`, or `text is object` — all three mean
the same thing here.

**Early-exit shape.** When a null check guards an early exit, the declaration folds into a negated pattern and
the later uses of the value stay exactly as they were, now reached only when the pattern matched:

```csharp
// before
var text = value as string;
if (text == null)
{
    return 0;
}

return text.Length;

// after
if (value is not string text)
{
    return 0;
}

return text.Length;
```

The null check can be written as `text == null` or `text is null`.

## What is not reported

The rule stays silent whenever folding the local into a pattern variable would change behaviour:

```csharp
var s = value as int?;           // not reported — a nullable value type; 'value is int s' would bind a
if (s != null) { /* ... */ }     //                non-null int, a different type

var s = value as string;         // not reported — the local is reassigned
if (s != null) { s = ""; Use(s); }

var s = value as string;         // not reported — the null check is not the next statement
Log();
if (s != null) { /* ... */ }

var s = value as string;         // not reported — the guarded-use value is read after the if, where the
if (s != null) { Use(s); }       //                pattern variable would not be definitely assigned
return s == null ? 0 : 1;

var s = Fetch() as string;       // not reported — the 'as' operand has side effects, so it cannot be
if (s != null) { Use(s); }       //                re-read at the pattern

var s = value as string;         // not reported — the early-exit guard body reads the local, which the
if (s == null) { Log(s); return; } //              negated pattern leaves unassigned there
return s.Length;

var s = value as string;         // not reported — the guard body can fall through, so a later use of the
if (s == null) { Log(); }        //                pattern variable would not be assigned
return s == null ? 0 : s.Length;
```

A compound condition such as `s != null && s.Length > 0` is also left alone, as is a local declared with a
base type of the `as` target (`object s = value as string;`), because narrowing it to the target type could
resolve a later use to a different overload.

## Relationship to other rules

This rule owns only the **assigned-local** flow — an `as` result stored in a local and then null-checked.
Neighbouring shapes belong to other rules:

- **SST2005** handles the inline `(o as T) != null` comparison with no local at all.
- **SST2007** handles an `is` type check followed by a cast to a local (`if (o is T) { var t = (T)o; ... }`).
- **SST2018** handles a null check placed beside an `is` pattern that already excludes null.

## How to fix violations

The code fix removes the declaration and rewrites the `if` condition to `o is T s` (guarded-use shape) or
`o is not T s` (early-exit shape). The guarded body and every later use of the value are left unchanged; the
`as` operand is re-read in the pattern. Before offering the fix, the analyzer confirms the target is a
reference type, the local's type matches the pattern variable's, the local is never reassigned, every use of
the local still resolves against the pattern variable, and the rewritten test still binds.

## Performance

The clean path is a pure syntax check — a single-declarator local whose initializer is an `as` conversion and
whose immediately following statement is an `else`-less `if` that null-checks that local — so an ordinary
local declaration is rejected before any symbol is bound. Only a matched candidate pays for the semantic work
that proves the rewrite safe.
