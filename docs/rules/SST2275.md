# SST2275 — Keep a single-statement method expression-shaped

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Info |
| Code fix | Yes (supports Fix All) |
| Default | Enabled |

## Cause

A method's block body does nothing but return one expression, or run one call.

```csharp
public int Area()
{
    return width * height;
}

public void Log(string line)
{
    _sink.Write(line);
}
```

## Rule description

A block that wraps a single `return expr;` — or, for a `void` method, a single expression statement —
spends a pair of braces (and a `return` keyword) on one expression. The expression-bodied form says the
same thing with none of the ceremony.

```csharp
public int Area() => width * height;

public void Log(string line) => _sink.Write(line);
```

The rewrite is pure syntax and never changes what the method computes. Because the collapsed body no longer
lives inside a block, the fix also settles any single-line-block warning (SST1501/SST1502) on the same
member — the expression-body form supersedes it.

The rule ships enabled at **Info**: it is a gentle nudge rather than a build-breaking warning, matching how
the analyzers already lean toward expression bodies.

## What is not reported

**A method that already has an expression body.** `int Area() => width * height;` is the target shape.

**A block with more than one statement.** A body that declares a local, branches, or runs several
statements is doing real work and is left alone.

**A `void` method whose only statement is a bare `return;`.** There is no expression to carry, so nothing
is reported.

**A single-statement body that carries a comment.** A comment inside the braces would be lost by the
collapse, so the method is left as it is.

## How to fix violations

The fix replaces the block body with the returned or evaluated expression, pulls the arrow up beside the
signature, and keeps the member's trailing trivia.

```csharp
// before
public string Describe()
{
    return $"{Name} ({Id})";
}

// after
public string Describe() => $"{Name} ({Id})";
```

## Language versions

Expression-bodied methods are available from C# 6, so the rule stays silent below that language version.

## Performance

The rule is a pure-syntax check on method nodes — is the body a block of exactly one returning or expression
statement — and never consults the semantic model. The clean path allocates nothing.
