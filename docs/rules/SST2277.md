# SST2277 — Keep a single-return operator expression-shaped

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Warning |
| Code fix | Yes (supports Fix All) |
| Default | Disabled (opt-in) |

## Cause

An operator's block body is a single `return expr;`.

```csharp
public static Vector operator +(Vector a, Vector b)
{
    return new Vector(a.X + b.X, a.Y + b.Y);
}
```

## Rule description

An operator always produces a value, so a body that is one `return expr;` can use an expression body.

```csharp
public static Vector operator +(Vector a, Vector b) => new Vector(a.X + b.X, a.Y + b.Y);
```

Whether operators read better on one line is a house-style call, so the rule ships **disabled**. Enable it
in `.editorconfig` for a codebase that prefers it.

The rewrite is pure syntax and never changes what the operator computes. Because the collapsed body no
longer lives inside a block, the fix also settles any single-line-block warning (SST1501/SST1502) on the
same operator — the expression-body form supersedes it.

## What is not reported

**An operator that already has an expression body.**

**A block with more than one statement.**

**A single-statement body that carries a comment**, which the collapse would lose.

## How to fix violations

The fix replaces the block body with the returned expression, pulls the arrow up beside the parameter list,
and keeps the operator's trailing trivia.

```csharp
// before
public static bool operator ==(Money a, Money b)
{
    return a.Amount == b.Amount;
}

// after
public static bool operator ==(Money a, Money b) => a.Amount == b.Amount;
```

## Enabling the rule

```ini
[*.cs]
dotnet_diagnostic.SST2277.severity = warning
```

## Language versions

Expression-bodied operators are available from C# 6, so the rule stays silent below that language version.

## Performance

The rule is a pure-syntax check on operator nodes — is the body a block of exactly one `return expr;` — and
never consults the semantic model. It is disabled by default, so a project that has not enabled it pays
nothing at all.
