# SST2278 — Keep a single-return conversion operator expression-shaped

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Warning |
| Code fix | Yes (supports Fix All) |
| Default | Disabled (opt-in) |

## Cause

A conversion operator's block body is a single `return expr;`.

```csharp
public static implicit operator int(Money money)
{
    return money.Amount;
}
```

## Rule description

A conversion operator always produces a value, so a body that is one `return expr;` can use an expression
body.

```csharp
public static implicit operator int(Money money) => money.Amount;
```

Like the arithmetic-operator rule it ships **disabled**, because whether a conversion reads better on one
line is a preference. Enable it in `.editorconfig` for a codebase that prefers it.

The rewrite is pure syntax and never changes the conversion. Because the collapsed body no longer lives
inside a block, the fix also settles any single-line-block warning (SST1501/SST1502) on the same operator —
the expression-body form supersedes it.

## What is not reported

**A conversion operator that already has an expression body.**

**A block with more than one statement.**

**A single-statement body that carries a comment**, which the collapse would lose.

## How to fix violations

The fix replaces the block body with the returned expression, pulls the arrow up beside the parameter list,
and keeps the operator's trailing trivia.

```csharp
// before
public static explicit operator string(Token token)
{
    return token.Text;
}

// after
public static explicit operator string(Token token) => token.Text;
```

## Enabling the rule

```ini
[*.cs]
dotnet_diagnostic.SST2278.severity = warning
```

## Language versions

Expression-bodied conversion operators are available from C# 6, so the rule stays silent below that
language version.

## Performance

The rule is a pure-syntax check on conversion-operator nodes — is the body a block of exactly one
`return expr;` — and never consults the semantic model. It is disabled by default, so a project that has not
enabled it pays nothing at all.
