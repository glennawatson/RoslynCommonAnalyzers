# SST2279 — Keep a get-only property expression-shaped

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Info |
| Code fix | Yes (supports Fix All) |
| Default | Enabled |

## Cause

A property has a single `get` accessor whose block body is one `return expr;`.

```csharp
public int Count
{
    get { return _items.Length; }
}
```

## Rule description

When the whole property is a lone `get` that returns one value, it can be written as a whole-member
expression body, dropping both the accessor block and the accessor keyword.

```csharp
public int Count => _items.Length;
```

The rewrite is pure syntax and never changes what the property returns. Because the collapsed getter no
longer lives inside a block, the fix also settles any single-line-block warning (SST1501/SST1502) on the
accessor — the whole-member expression-body form supersedes it.

The rule ships enabled at **Info**: a gentle nudge rather than a build-breaking warning.

## What is not reported

**A property with a setter, an `init` accessor, or more than one accessor.** Those keep their per-accessor
shape; the whole-member form only fits a lone `get`. Simplifying an individual accessor block is covered by
SST2219.

**A getter that already has an expression body.** `int Count => _items.Length;` — and `get => _items.Length;`
— are already compact.

**A `get` accessor carrying its own attributes or modifiers.** The whole-member form has nowhere to put
them, so the property is left alone.

**A getter whose body carries a comment**, which the collapse would lose.

## How to fix violations

The fix replaces the accessor list with the returned expression, pulls the arrow up beside the property
name, and keeps the property's trailing trivia.

```csharp
// before
public string Display
{
    get { return $"{First} {Last}"; }
}

// after
public string Display => $"{First} {Last}";
```

## Language versions

Expression-bodied properties are available from C# 6, so the rule stays silent below that language version.

## Performance

The rule is a pure-syntax check on property nodes — a single block-bodied `get` returning one value — and
never consults the semantic model. The clean path allocates nothing.
