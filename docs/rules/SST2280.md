# SST2280 — Keep a get-only indexer expression-shaped

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Info |
| Code fix | Yes (supports Fix All) |
| Default | Enabled |

## Cause

An indexer has a single `get` accessor whose block body is one `return expr;`.

```csharp
public int this[int index]
{
    get { return _items[index]; }
}
```

## Rule description

When the whole indexer is a lone `get` that returns one value, it can be written as a whole-member
expression body, dropping both the accessor block and the accessor keyword.

```csharp
public int this[int index] => _items[index];
```

The rewrite is pure syntax and never changes what the indexer returns. Because the collapsed getter no
longer lives inside a block, the fix also settles any single-line-block warning (SST1501/SST1502) on the
accessor — the whole-member expression-body form supersedes it.

The rule ships enabled at **Info**: a gentle nudge rather than a build-breaking warning.

## What is not reported

**An indexer with a setter, an `init` accessor, or more than one accessor.** Those keep their per-accessor
shape; the whole-member form only fits a lone `get`. Simplifying an individual accessor block is covered by
SST2219.

**A getter that already has an expression body.**

**A `get` accessor carrying its own attributes or modifiers.** The whole-member form has nowhere to put
them, so the indexer is left alone.

**A getter whose body carries a comment**, which the collapse would lose.

## How to fix violations

The fix replaces the accessor list with the returned expression, pulls the arrow up beside the parameter
list, and keeps the indexer's trailing trivia.

```csharp
// before
public char this[int index]
{
    get { return _text[index]; }
}

// after
public char this[int index] => _text[index];
```

## Language versions

Expression-bodied indexers are available from C# 6, so the rule stays silent below that language version.

## Performance

The rule is a pure-syntax check on indexer nodes — a single block-bodied `get` returning one value — and
never consults the semantic model. The clean path allocates nothing.
