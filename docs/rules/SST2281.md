# SST2281 — Keep a single-statement local function expression-shaped

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Info |
| Code fix | Yes (supports Fix All) |
| Default | Enabled |

## Cause

A local function's block body does nothing but return one expression, or run one call.

```csharp
int Double(int n)
{
    return n * 2;
}

void Add(int n)
{
    sink.Add(n);
}
```

## Rule description

A block that wraps a single `return expr;` — or, for a `void` local function, a single expression
statement — can use an expression body.

```csharp
int Double(int n) => n * 2;

void Add(int n) => sink.Add(n);
```

The rewrite is pure syntax and never changes what the local function computes. Because the collapsed body no
longer lives inside a block, the fix also settles any single-line-block warning (SST1501/SST1502) on the
same local function — the expression-body form supersedes it.

The rule ships enabled at **Info**: a gentle nudge rather than a build-breaking warning.

## What is not reported

**A local function that already has an expression body.**

**A block with more than one statement.**

**A `void` local function whose only statement is a bare `return;`.** There is no expression to carry.

**A single-statement body that carries a comment**, which the collapse would lose.

## How to fix violations

The fix replaces the block body with the returned or evaluated expression, pulls the arrow up beside the
signature, and keeps the local function's trailing trivia.

```csharp
// before
bool IsEven(int n)
{
    return n % 2 == 0;
}

// after
bool IsEven(int n) => n % 2 == 0;
```

## Language versions

Local functions and their expression bodies are available from C# 7, so the rule stays silent below that
language version.

## Performance

The rule is a pure-syntax check on local-function nodes — is the body a block of exactly one returning or
expression statement — and never consults the semantic model. The clean path allocates nothing.
