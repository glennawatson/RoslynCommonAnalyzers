# SST2282 — Use an is-null pattern instead of ReferenceEquals against null

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Warning |
| Code fix | Yes (supports Fix All) |
| Default | Enabled |

## Cause

A call to `object.ReferenceEquals` compares a value against the `null` literal, in either argument order.

```csharp
if (ReferenceEquals(value, null))
{
    return "missing";
}

// or, negated:
if (!ReferenceEquals(value, null))
{
    Use(value);
}
```

## Rule description

`ReferenceEquals(value, null)` states a null check as a static method call: the reader has to recognize the
method, read past the receiver-less call, and notice which argument is `null`. A null pattern says the same
thing left to right and needs no method at all.

```csharp
if (value is null)
{
    return "missing";
}

if (value is not null)
{
    Use(value);
}
```

The rewrite is reported only when it is provably equivalent:

- **The call binds to `System.Object.ReferenceEquals.`** A same-named static method declared elsewhere is left
  alone — only the framework method has the reference-comparison meaning the pattern reproduces.
- **Exactly one argument is the `null` literal.** The other argument is the value that carries into the
  pattern. `ReferenceEquals(null, null)` has no value to test and is not reported.
- **The non-null operand is a reference type or an unconstrained type parameter.** For those, `is null` is
  legal and means a reference null check, exactly as `ReferenceEquals` does. A value-type operand is left
  alone: `ReferenceEquals` boxes it, so the two forms are not the same operation, and that boxing is a
  separate concern.
- **The pattern the rewrite needs is available.** `value is null` needs the constant null pattern (C# 7); the
  negated `value is not null` needs the `not` pattern (C# 9).

## What is not reported

```csharp
ReferenceEquals(number, null);          // not reported — 'number' is a value type (boxed by the call)
ReferenceEquals(null, null);            // not reported — no value to test
Helpers.ReferenceEquals(value, null);   // not reported — not System.Object.ReferenceEquals
value == null;                          // not reported — an equality operator, owned elsewhere
```

## How to fix violations

Replace the call with `value is null`, or the negated call with `value is not null`. The code fix does this
and parenthesizes the operand only when it is not already a primary expression.

## Notes

The plain form is offered from C# 7 and the negated form from C# 9. The rule stays silent when the operand is
any value type — including a nullable value type — because rewriting a boxing comparison is out of scope here.
