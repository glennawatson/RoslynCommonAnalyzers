# SST2283 — Fold a preceding null guard into the assigned value

| Item | Value |
| --- | --- |
| Category | ModernSyntax |
| Severity | Warning |
| Code fix | Yes (supports Fix All) |
| Default | Enabled |

## Cause

A null guard whose one statement throws is immediately followed by assigning the guarded value to a field,
property, or local.

```csharp
public Connection(Endpoint endpoint)
{
    if (endpoint is null)
    {
        throw new InvalidOperationException("An endpoint is required.");
    }

    _endpoint = endpoint;
}
```

## Rule description

The guard and the assignment describe one contract — "store this value, but only if it is present" — split
across two statements. A throw expression folds them into the assignment, so the value and the exit that
guards it sit together and the value is written once.

```csharp
public Connection(Endpoint endpoint)
{
    _endpoint = endpoint ?? throw new InvalidOperationException("An endpoint is required.");
}
```

The fold is reported only when it is provably equivalent:

- **The guard is an `else`-less `if` whose one statement throws an expression.** A `throw new …` carries a
  value that becomes the right operand of `??`. A guard body with more than the single throw, or a bare
  rethrow, is left alone.
- **The condition is a built-in null check on the guarded value** — `x is null`, `x == null`, or `null == x`.
  A user-defined `==` is not the reference null check `??` performs, so it is left alone.
- **The guarded value is a side-effect-free local or parameter of a reference type.** It is read in the guard
  and again in the assignment, so folding the two reads into one must not change what runs; a reference type
  keeps the coalescing throw legal and its result identical.
- **The next statement assigns the guarded value to a safe target.** The target is a simple name or a `this`
  member, so it has no receiver of its own to evaluate before the throw. `obj.Member = x` is left alone,
  because evaluating `obj` ahead of the guard could change which exception is thrown.
- **Nothing sits between the guard and the assignment**, and throw expressions are available (C# 7).

## What is not reported

```csharp
// not reported — an argument-null guard whose throw is better expressed as a runtime null-check helper
if (value is null)
    throw new ArgumentNullException(nameof(value));
_value = value;

// not reported — the guarded value is returned, not assigned
if (value is null)
    throw new InvalidOperationException();
return value;

// not reported — a statement runs between the guard and the assignment
if (value is null)
    throw new InvalidOperationException();
Log(value);
_value = value;

// not reported — the assignment target has its own receiver
if (value is null)
    throw new InvalidOperationException();
other.Field = value;
```

## How to fix violations

The code fix removes the guard and moves its throw onto the assigned value as `target = value ?? throw …;`.

## Notes

This rule covers the guard-then-assignment shape only. A guard whose value is returned, a value assigned and
then checked, and an argument-null guard that a runtime null-check helper replaces are each handled by the
rule that owns that shape, so no two of them report the same guard.
