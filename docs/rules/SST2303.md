# SST2303 — Flags enums should declare bit values

| Item | Value |
| --- | --- |
| Category | Design |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

An enum is marked `[Flags]` but its members are not distinct bit values.

```csharp
[Flags]
public enum Status
{
    None,
    Active,
    Pending,
    Done,
}
```

Nobody wrote `Done = 3`. The compiler counted to it — and 3 is `Active | Pending`.

## Rule description

`[Flags]` is a promise to everyone who reads the enum, and to the runtime: these members combine with
bitwise or. `ToString()` prints a combined value as a comma-separated list. `HasFlag` tests a bit mask.
Both only work when each member owns its own bit.

With the enum above, `Status.Done.ToString()` returns `"Active, Pending"`. `value.HasFlag(Status.Done)`
returns true for a value that is `Active | Pending` and was never `Done`. And `Status.Done | Status.Active`
is `Done`, which is nonsense in both directions. The values are wrong, the printing is wrong, the tests
are wrong, and nothing anywhere failed to compile.

### What passes

A member is fine when it is one of three things:

- **Zero.** `None = 0` is the empty set, and it is expected — a flags enum without one has no way to say
  "no flags". It is never reported.
- **A single bit** — a power of two.
- **A declared combination**: a member with an **explicit initializer** whose value is made up entirely of
  bits the enum already declares. Both of these are the attribute working as intended, and neither is
  reported:

```csharp
[Flags]
public enum Access
{
    None = 0,
    Read = 1,
    Write = 2,
    ReadWrite = Read | Write,   // a combination, written as one
    Execute = 4,
    All = 7,                    // a combination, written as the literal it adds up to
}
```

### What is reported

Anything else. Concretely, the two shapes that reach it:

- **A member the compiler numbered.** A value that is neither zero nor a single bit and has **no explicit
  initializer** was arrived at by counting, not by design — that is the `Status` enum above, and it is the
  bug this rule exists for. It is what separates a deliberate `= 3` from an accidental one: by value alone
  they are identical, and only the initializer says which one a human meant.
- **A value carrying a bit no member declares.** `Mixed = 5` in an enum that only declares 1 and 2 can
  never be a combination of anything, however it was written, so it is reported whether the initializer is
  there or not.

The value is read as the bit pattern it actually stores, whatever the underlying type: an unsigned enum's
high bit is a bit like any other, and `Everything = -1` on a signed enum is every bit there is, which is
not one bit, and is reported.

An enum **without** the attribute makes no promise about combining, so its numbering is its own business
and it is never looked at. And an enum whose counted values happen to land on bits — `None`, `Read`,
`Write` numbering 0, 1, 2 — is a valid flags enum however it was arrived at, and is not reported: the rule
reports facts, not luck.

## How to fix violations

There is **no code fix**, deliberately. Renumbering an enum is a **breaking change** to everything that
has already stored one of its values: rows in a database, JSON on disk, a message on a queue, a setting in
a registry, an argument in a caller compiled against the old assembly (enum constants are inlined at the
call site, so a recompile of *your* assembly does not fix theirs). An analyzer cannot know which of your
values have escaped, and a fix that silently rewrote `Done = 3` to `Done = 4` would corrupt every one of
them that had.

Do it by hand, deliberately:

```csharp
[Flags]
public enum Status
{
    None = 0,
    Active = 1,
    Pending = 2,
    Done = 4,
}
```

Or — if the members were never meant to combine, and `Status` is one of `Active`, `Pending`, `Done` and
nothing else — **remove the `[Flags]` attribute**. That is just as often the right answer, and it is not a
breaking change at all.

## Configuration

None.

## Performance

The clean path is a type-kind comparison: everything that is not an enum is rejected on it, and an enum
without the attribute is rejected on one scan of its attribute list. Only a flags enum reads its members,
and it reads them twice — once to build the mask of declared bits, once to measure each value against it —
over an array Roslyn already had.
