# SST2318 -- Members should not have identical bodies

| Item | Value |
| --- | --- |
| Category | Design |
| Severity | Warning |
| Code fix | No |
| Default | Disabled |

## Cause

Two methods declared in the same type have bodies that are identical token for token.

```csharp
public sealed class Totals
{
    public int WithTax(int a, int b)
    {
        var subtotal = a + b;
        return subtotal + (subtotal / 10);
    }

    public int WithDiscount(int a, int b)   // SST2318: same body as WithTax
    {
        var subtotal = a + b;
        return subtotal + (subtotal / 10);
    }
}
```

## Rule description

Two methods in one type with the same body are almost always a copy-paste where the second method was
meant to do something different and never got changed. `WithDiscount` above silently computes the tax, and
nothing about the call site reveals it. The rule reports the second of the matching pair.

This rule is disabled by default, because identical bodies are sometimes intentional, and it narrows to the
duplicates most likely to be a mistake:

- Only ordinary methods with a real body are considered. Abstract, extern, and unimplemented partial
  declarations have no body and never qualify.
- A trivial body is ignored: a block of fewer than two statements, or an expression body that is just a
  literal, a name, a member access, a `throw`, `default`, or `null`. A one-line forwarding method and a
  shared `throw new NotImplementedException();` are expected to repeat.
- Bodies are compared by their normalized token stream, so whitespace and comments do not matter.

If the shared body is deliberate, have one method call the other so the logic lives in one place.

## How to fix violations

Decide what the second method was supposed to do and write that body, or, when the duplication is intended,
call one method from the other instead of repeating its statements.

## Performance

The clean path is a single count of the type's methods that have a non-trivial body. The token key and the
tracking dictionary are built only when a type has at least two such methods, which most types never reach.
