# SST2320 -- An interface inherits the same member from two interfaces

| Item | Value |
| --- | --- |
| Category | Design |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

An interface derives from two base interfaces that each declare a member of the same name and signature,
and the two are separate declarations rather than one member reached through a shared base.

```csharp
public interface ILeft  { string Name { get; } }
public interface IRight { string Name { get; } }

public interface IBoth : ILeft, IRight { }   // SST2320: Name comes from both ILeft and IRight
```

## Rule description

`IBoth` itself compiles. The failure lands on the consumer:

```csharp
string Read(IBoth both) => both.Name;   // error CS0229: ambiguity between 'ILeft.Name' and 'IRight.Name'
```

Because `ILeft.Name` and `IRight.Name` are two different members, the compiler cannot choose one when the
member is accessed through `IBoth`, so every caller gets a hard ambiguity error. The author of `IBoth` never
sees it. This rule moves the failure back to the author, where it can be fixed once.

Two cases that look similar are not reported:

- A diamond where both members trace to a single shared base interface. There is only one member, reached by
  two paths, and no ambiguity.
- An interface that re-declares the member itself with `new`. That re-declaration is the fix, so flagging it
  would be perverse.

## How to fix violations

Re-declare the member on the derived interface with `new`, choosing the one the interface should expose:

```csharp
public interface IBoth : ILeft, IRight
{
    new string Name { get; }
}
```

This makes `IBoth.Name` a single member again, and every caller resolves it without ambiguity. There is no
code fix, because only the author knows which of the two members is the intended one.

## Performance

The rule runs only for an interface that declares at least two direct base interfaces, so nearly every type
pays nothing. For those interfaces it walks the base members once, grouping by signature, and reports a
signature only when two distinct declarations share it.
