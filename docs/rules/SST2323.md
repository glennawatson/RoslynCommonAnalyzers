# SST2323 - Express an all-abstract class as an interface

| Item | Value |
| --- | --- |
| Category | Design |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A non-static class is `abstract`, extends nothing but `object`, and every member it declares is a public
abstract member - no fields, no implemented member, and no hand-written constructor.

```csharp
public abstract class Shape // SST2323
{
    public abstract double Area();

    public abstract void Draw();
}
```

## Rule description

`abstract` on a class carries two promises: *you cannot instantiate me*, and *you must supply these
members*. A class that declares nothing but abstract members and holds no state keeps only the second
promise. It asks its derived types for exactly what an interface asks for, and offers none of the things a
base class is actually for - shared fields, a constructor that runs, a method with a body to inherit.

The cost of writing that contract as a class instead of an interface is the single base type a class is
allowed. A type can implement any number of interfaces but derive from only one class, so anything that
inherits this one has spent its only base slot on a contract. The same members on an interface leave that
slot open: a type can satisfy several such contracts at once, and a reader sees at a glance that this is a
contract rather than a base to build on.

Declaring it as an interface says plainly what the type is:

```csharp
public interface IShape
{
    double Area();

    void Draw();
}
```

## Language version

An interface can carry the full range of members an abstract class may hold - member accessibility
modifiers and default implementations from C# 8, and static abstract members from C# 11 - only from those
versions onward. The suggestion is only sound where the language can express on an interface what the class
expresses, so the rule is **gated on C# 8**: below it, the whole rule registers nothing and no diagnostic is
raised, because a project on an older language version could not act on the advice.

## What is reported

- A non-static `abstract class` that extends only `object`, declares at least one abstract member, and
  declares nothing but public abstract members. A nested class of that same shape is reported in its own
  right.

## What is not reported

- **A class with any state or implementation.** A field, a hand-written constructor, an implemented method
  or property, or a nested type each means the type is more than a contract, and the class stays:

  ```csharp
  public abstract class Shape // clean: _sides is state an interface cannot hold
  {
      private int _sides;

      public abstract double Area();
  }
  ```

- **An abstract member that is not public.** A `protected` or `internal abstract` member is a base-class
  idiom - the template-method shape where the base calls what the derived type fills in - that an interface
  would not reproduce.
- **An abstract class that declares nothing abstract.** That is the opposite shape: a base class wearing a
  contract's clothes, reported by its own maintainability rule, never by this one. The two rules can never
  fire on the same type.
- **A class that extends another class.** An interface cannot inherit a class, so a real base type rules the
  conversion out.
- **A `record`.** The compiler writes its equality contract, and an abstract record base is how a closed
  hierarchy is spelled.
- **A `static` class**, which is abstract and sealed in metadata but is not an abstract class; a concrete
  class; and an `interface`, which is already the shape the rule suggests.
- **Any type on a language version below C# 8.**

## How to fix

There is **no code fix**. Turning a class into an interface is a breaking, non-local change: every derived
type has to switch from `:` inheritance to interface implementation, callers that name the type as a base
may need to change, and the decision to give up single-inheritance semantics belongs to the author. The
diagnostic marks the type; the conversion is yours to make.

## Configuration

None.

## Performance

The rule runs as a symbol action, so it costs nothing per syntax node and sees a partial type once rather
than once per part. It is registered only when the compilation's language version is C# 8 or later, so an
older project pays a single check at compilation start and nothing more. The filter is four symbol flags -
class, abstract, not static, not a record - and a single base-type check before any member is enumerated, so
the member scan runs only for the narrow set of types that could qualify, and it stops at the first member
that is not a public abstract one.
