# SST2332 — Make a construction-only property get-only

| Item | Value |
| --- | --- |
| Category | Design |
| Severity | Warning |
| Code fix | Yes (remove the private setter) |
| Default | Enabled |

## Cause

An auto-property declared `get; private set;` is only ever written inside a constructor or the property
initializer, never after construction.

```csharp
public class Counter
{
    public int Value { get; private set; }   // SST2332

    public Counter(int value) => Value = value;
}
```

## Rule description

A private setter says the value can change after construction. When the only writes are in a constructor, that
is not true, and the setter invites a future change that quietly mutates what everyone assumed was immutable.
Collapsing the property to a get-only `get;` states what is actually the case — set once, then fixed — and the
compiler enforces it: a later stray assignment stops compiling instead of silently succeeding.

The rule proves there is no post-construction write before it fires. It scans the whole declaring type for any
assignment to the property — including a compound assignment, an increment, a `ref`/`out` argument, and an
object-initializer target — and any such write that is not inside a constructor of that same type (including
one reached through a lambda or a local function) leaves the property alone.

## What is reported

- A `get; private set;` auto-property whose only writes are in a constructor of the declaring type or in the
  property initializer. Reported on the property name.

## What is not reported

- A property written after construction — in a method, a lambda, a local function, or an object initializer.
- A property whose setter is `init` or public: neither is this rule's concern.
- A property with an accessor body (not an auto-property).
- A property on a `partial` type, because a write could sit in a part this analysis cannot see.

## How to fix violations

The code fix removes the private setter, leaving a get-only auto-property. Any initializer is kept:

```diff
 public class Counter
 {
-    public int Value { get; private set; }
+    public int Value { get; }

     public Counter(int value) => Value = value;
 }
```

A get-only auto-property can still be assigned in a constructor and through an initializer, so the constructor
write keeps compiling.

## Performance

The clean path is a syntactic prepass for the exact `get; private set;` auto-property shape, so an ordinary
property never reaches the type-wide write scan.
