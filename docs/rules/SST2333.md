# SST2333 — Provide the non-generic form of a generic comparison contract

| Item | Value |
| --- | --- |
| Category | Design |
| Severity | Warning |
| Code fix | Yes (add the non-generic member) |
| Default | Disabled (opt-in) |

## Cause

A publicly visible type implements a generic comparison or equality contract but not its non-generic
counterpart:

| Generic contract | Non-generic counterpart |
| --- | --- |
| `System.IComparable<T>` | `System.IComparable` |
| `System.Collections.Generic.IComparer<T>` | `System.Collections.IComparer` |
| `System.Collections.Generic.IEqualityComparer<T>` | `System.Collections.IEqualityComparer` |
| `System.IEquatable<T>` | an override of `object.Equals(object)` |

```csharp
using System;

public class Money : IComparable<Money>   // SST2333: no IComparable
{
    public int CompareTo(Money other) => 0;
}
```

## Rule description

Plenty of runtime code still reaches for the non-generic contracts — a non-generic collection, a sort through
`IComparer`, a hashtable asking a value for `object.Equals`. A type that implements only the generic form is
invisible to those paths: they fall back to reference identity or an ordering the type never intended, and the
mistake never surfaces as an error. Supplying the non-generic member as a type-checked forward to the generic
one closes the gap without duplicating the logic.

Many modern types deliberately omit the non-generic contracts, so this rule is **disabled by default** and
opt-in through `.editorconfig`. It resolves each non-generic counterpart in the compilation and stays silent
when it is absent, so the fix it offers always has a target.

## Opting in

```ini
[*.cs]
dotnet_diagnostic.SST2333.severity = warning
```

## What is reported

- An externally visible class or struct that implements one of the generic contracts above but not its
  non-generic counterpart. A type missing several counterparts is reported once for each.

## What is not reported

- A type that already implements the non-generic counterpart (for `IEquatable<T>`, one that already overrides
  `object.Equals(object)`).
- A record, whose equality members the compiler already generates.
- An `internal` or otherwise non-externally-visible type: inside the assembly every caller is rebuilt
  together.

## How to fix violations

The code fix adds the non-generic interface (where the counterpart is an interface) and the member, forwarding
to the generic implementation after a type check:

```diff
 using System;

-public class Money : IComparable<Money>
+public class Money : IComparable<Money>, IComparable
 {
     public int CompareTo(Money other) => 0;
+
+    int IComparable.CompareTo(object obj) => obj is Money other ? ((IComparable<Money>)this).CompareTo(other) : throw new InvalidCastException();
 }
```

For `IEquatable<T>` the fix adds an `object.Equals(object)` override that forwards to the generic `Equals`.
The generated member is a working skeleton; tighten the null and mismatch handling to suit the type.

## Performance

The contracts are resolved once per compilation, and the rule does nothing on a framework that has none of the
generic forms.
