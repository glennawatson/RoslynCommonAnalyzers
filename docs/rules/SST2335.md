# SST2335 — Declare the same 'static' modifier on every part of a partial type

| Item | Value |
| --- | --- |
| Category | Design |
| Severity | Warning |
| Code fix | Yes (add `static` to the part that omits it) |
| Default | Disabled (opt-in) |

## Cause

One part of a partial class is declared `static` while another part omits the modifier.

```csharp
static partial class Widget
{
}

partial class Widget   // SST2335: this part omits 'static'
{
}
```

## Rule description

When one part of a partial class says `static` and another does not, the type still compiles as static — the
part that names `static` decides it. But a reader looking at the part without the keyword sees an ordinary
class and can waste time trying to add instance state or a constructor to it. Repeating `static` on every part
makes each one tell the truth on its own.

This is a house-style consistency nudge, not a defect, so it is **disabled by default** and opt-in through
`.editorconfig`.

## Opting in

```ini
[*.cs]
dotnet_diagnostic.SST2335.severity = warning
```

## What is reported

- Each part of a partial class that omits `static` when at least one other part declares it. Every omitting
  part is reported, on its own type name.

## What is not reported

- A partial class whose parts all declare `static`.
- A partial class whose parts all omit `static`.
- A non-partial type, or any type that is not a class.

## How to fix violations

The code fix adds `static` to the part that omits it (Fix All updates every such part):

```diff
 static partial class Widget
 {
 }

-partial class Widget
+static partial class Widget
 {
 }
```

## Performance

The clean path is a type-kind check followed by a declaration count, so a non-partial class costs almost
nothing; only a partial class with more than one part is examined for the modifier.
