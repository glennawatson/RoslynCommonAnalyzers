# SST2412 — A for loop should step its counter toward the bound

| Item | Value |
| --- | --- |
| Category | Correctness |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

The counter's step and the side of the comparison disagree: an ascending step is paired with a "greater than"
test, or a descending step with a "less than" test.

```csharp
for (var i = 0; i > 10; i++) // SST2412
{
    Console.WriteLine(i);
}
```

## Rule description

Each iteration moves the counter further from the value that would end the loop, so it spins forever, or the
condition is already false and the body never runs. The classic form is the reverse-loop typo — an index meant
to walk backwards from the end, written with the forward comparison:

```csharp
for (var i = arr.Length - 1; i < 0; i--) // SST2412
```

The bound may sit on either side of the comparison (`10 < i` reads the same as `i > 10`). The rule stays silent
where the direction is not provably wrong:

- A **not-equal** condition (`i != n`) does not pick a side, so it is not reported.
- A **non-constant step** (`i += delta`) could move either way at runtime.
- A **body that also re-steps the counter** may correct the direction, so the loop is left alone.

## How to fix violations

Flip the comparison operator so the counter moves toward the bound.

```csharp
for (var i = 0; i <= 10; i++)
{
    Console.WriteLine(i);
}
```

The code fix flips the operator: `i > 10` becomes `i <= 10`, and the reverse-index `i < 0` becomes `i >= 0`.
Nothing else in the loop changes.

## Performance

The check is part of the single loop walk. An ordinary stepping loop is rejected once its step and comparison
side are seen to agree, before the counter's start or bound is evaluated.
