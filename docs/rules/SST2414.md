# SST2414 — Two branches should not share an implementation

| Item | Value |
| --- | --- |
| Category | Correctness |
| Severity | Warning |
| Code fix | Yes (switch statements) |
| Default | Enabled |

## Cause

Two arms of one `if`/`else if` chain, or two sections of one `switch`, run a token-identical body while other
arms differ.

```csharp
switch (x)
{
    case 1:
        return A();
    case 2:
        return B();
    case 3: // SST2414
        return A();
}
```

## Rule description

The two conditions are distinguished at the top and then handled the same way, which usually means a
copy-and-paste left one arm holding the value that belonged to the other — `case 3` was meant to return
something of its own. The rule reports the second of any two branches whose bodies match while at least one
other branch differs.

When **every** branch matches, the condition decides nothing at all; that is the separate exhaustive-duplication
rule (SST1476), and it takes precedence here. The rule also stays silent where the sharing is legitimate or the
comparison is unsafe:

- A case that shares its body with the **`default` section** is a normal "these values, plus everything else"
  grouping.
- A `switch` that a **`goto case` / `goto default`** could jump into is left alone, because a jump can make two
  identical-looking sections behave differently.
- **Single-statement `if` branches** are below the reporting threshold; the shared body must be substantial.
- **Discard arms** of a `switch` expression are skipped for the same reason as `default`.

## How to fix violations

Where the duplication is intended, the two cases belong together. A `switch` merges them into one section, an
`if` chain merges side-effect-free conditions with `||`, and a `switch` expression combines patterns with `or`:

```csharp
switch (x)
{
    case 1:
    case 3:
        return A();
    case 2:
        return B();
}
```

The code fix applies to the `switch` statement form: it stacks the duplicated section's labels onto the earlier
one. The `if`-chain and `switch`-expression forms are report-only — merging conditions or patterns there is a
judgement the author must make, since one of the two branches may instead have been meant to differ.

## Performance

Every construct is analyzed in a single walk. The structural body comparison is trivia-insensitive and bails on
the first difference; the two-arm scan compares span lengths before tokens, and only runs after the
exhaustive-duplication check has declined the construct.
