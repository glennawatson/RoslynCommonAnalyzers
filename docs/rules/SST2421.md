# SST2421 — A readonly field of an unconstrained type parameter should not be mutated

| Item | Value |
| --- | --- |
| Category | Correctness |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A `readonly` field whose type is a type parameter with no reference-type constraint is written through — a
property or field is assigned on it, or a mutating member is called on it.

```csharp
public sealed class Holder<T>
    where T : IPoint
{
    private readonly T _p;

    public void M() => _p.X = 5; // SST2421
}
```

## Rule description

When the type argument is a struct, reading a `readonly` field yields a defensive copy, so the write changes the
copy and is silently discarded; the field keeps its old value. Nothing warns, because the same code is correct
when the type argument is a class — which is exactly why the defect hides.

The three repairs each mean something different, so the author must choose:

- **Constrain the parameter to a reference type** (`where T : class, ...`) if the field is shared state that all
  callers should see change.
- **Drop `readonly`** if the value is meant to change in place.
- **Give the type a value-returning API** so the update produces a new value instead of mutating one.

A reference-constrained type parameter, an interface-typed field (which holds a reference, not a struct copy),
and a call to an `object` member that cannot mutate the value are all left alone.

## How to fix violations

Pick the repair that matches the intent. For shared state, constrain the parameter so the field holds a
reference:

```csharp
public sealed class Holder<T>
    where T : class, IPoint
{
    private readonly T _p;

    public void M() => _p.X = 5;
}
```

There is **no code fix**. The three repairs change the type's contract in incompatible ways, and only the author
knows which was intended.

## Performance

The clean path resolves nothing: it rejects every write whose target does not root at a plain field reference
before binding, then bails unless that field is a `readonly` field of an unconstrained (or struct-constrained)
type parameter.
