# SST2422 — A property's getter and setter should use the same field

| Item | Value |
| --- | --- |
| Category | Correctness |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A property's getter returns one instance field and its setter assigns `value` to a different one.

```csharp
public int Width // SST2422
{
    get => _height;
    set => _width = value;
}
```

## Rule description

What is written can never be read back through the property, and what is read is whatever some other code last
put in the getter's field. The property does not round-trip.

This is a proof, not a guess about names: the getter's single field read and the setter's single
assignment-from-`value` are resolved to two field symbols, and reported only when they differ. The setter may do
more than assign — validate, raise a change notification — and the getter may be any body that reduces to a
single field read; the rule reports only when exactly one field is assigned from `value` and exactly one,
different, field is returned.

A property that round-trips one field, a change-notification setter over a single field, and a computed getter
(`get => _width * _height;`, which does not reduce to one field read) are all clean. An auto-property, an
expression-bodied read-only property, and an interface property are never bound.

## How to fix violations

Point one accessor at the other's field so the property round-trips.

```csharp
public int Width
{
    get => _width;
    set => _width = value;
}
```

The code fix points the getter at the field the setter writes.

## Performance

The rule registers on property declarations and never binds an auto-property, an expression-bodied read-only
property, or an interface property. It resolves the two field symbols only after the getter and setter are each
found to reduce to a single field.
