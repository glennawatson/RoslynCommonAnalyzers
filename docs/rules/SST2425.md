# SST2425 — Forward the optional argument to the base call

| Item | Value |
| --- | --- |
| Category | Correctness |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

An override takes an optional parameter and then calls the method it overrides without passing that
parameter on.

```csharp
public class Derived : Base
{
    public override void Go(int a, string mode = "fast")
    {
        base.Go(a); // SST2425: mode is not forwarded
    }
}
```

## Rule description

The base call fills in the base's own default for the missing argument, so whatever value the caller of the
override supplied is thrown away before the base ever sees it:

```csharp
new Derived().Go(1, "slow"); // "slow" is discarded; base.Go runs with "fast"
```

This is easy to write when the forwarding call is short and the omission is invisible. Primary constructors
and `with`-style forwarding make it easier still.

## How to fix violations

Pass the parameter through to the base call so the caller's value reaches it:

```csharp
public override void Go(int a, string mode = "fast")
{
    base.Go(a, mode);
}
```

There is no code fix, because forwarding the argument changes what the base call does, and the analyzer
should not alter run-time behaviour on its own.

## Performance

The clean path is a syntactic prepass: only a `base.Name(...)` invocation inside a method reaches the
semantic model. The invocation is then confirmed to target the enclosing method's overridden method, and the
argument-to-parameter mapping identifies the dropped forward.
