# SST2426 — An override should keep the base's params modifier

| Item | Value |
| --- | --- |
| Category | Correctness |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

An `override` declares a `params` modifier that disagrees with the method it overrides: adding one the base
lacks, or dropping one the base has.

```csharp
public class Base
{
    public virtual void Log(string first, params int[] rest) { }
}

public class Derived : Base
{
    public override void Log(string first, int[] rest) { } // SST2426
}
```

## Rule description

Whether a call expands its trailing arguments into an array is decided entirely by the declaration being
overridden. An override's own `params` modifier has no effect at all:

- An override that adds `params` the base lacks does not let callers of the derived type pass an expanded
  argument list.
- An override that drops `params` the base has does not stop them.

The modifier is inert either way. The only thing a mismatch does is tell a reader something untrue about how
callers of the type behave.

## How to fix violations

Make the override's modifier match the base. The code fix adds the `params` modifier when the base has it,
and removes it when the base does not. Because the modifier is inert, the fix is behaviour-preserving — it
only makes the declaration honest.

```csharp
public override void Log(string first, params int[] rest) { }
```

## Performance

The clean path is a single modifier check: a method with no `override` keyword is rejected before the
semantic model is touched. The `params` modifier is read from the override's own syntax, because the symbol
inherits the base's value and would hide the mismatch.
