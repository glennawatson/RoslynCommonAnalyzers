# SST2431 — ToString() should not return null

| Item | Value |
| --- | --- |
| Category | Correctness |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

An overridden `ToString` can hand back `null` — written as `null`, `null!`, a parenthesised or cast null, or
a null branch of a conditional.

```csharp
public override string ToString() => null!; // SST2431
```

## Rule description

`ToString` is called from string interpolation, concatenation, `Console` output, logging, and every debugger
display. None of those callers check for `null` first, so when one comes back the failure surfaces far from
the `ToString` that produced it — usually as a `NullReferenceException` inside framework code the caller cannot
see, with a stack trace that points nowhere useful.

Returning `string.Empty` for the empty case keeps every one of those call sites working. Only a return that is
*syntactically* null is reported, so the rule never needs the semantic model:

```csharp
public override string ToString() => null;                 // SST2431
public override string ToString() { return null; }         // SST2431
public override string ToString() => Full ? Name : null;   // SST2431 on the null branch
```

A null produced by a nested lambda or local function inside `ToString` does not return from `ToString`, so it
is left alone:

```csharp
public override string ToString()
{
    Func<string> factory = () => null; // not reported: returns from the lambda
    return factory() ?? "unnamed";
}
```

## How to fix violations

Return `string.Empty` where the value would have been null.

```csharp
public override string ToString() => string.Empty;
```

The code fix replaces the reported null — including a `null!` or a single conditional branch — with
`string.Empty`, keeping the rest of the expression as it was. Where a real string is available, prefer it; the
fix picks the safe empty default because that is the one an analyzer can always supply.

## Performance

The clean path is three token checks: the method must be named `ToString`, carry `override`, and take no
parameters. Every other method is rejected before anything is allocated, and detection never touches the
semantic model.
