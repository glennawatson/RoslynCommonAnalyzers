# SST2432 — Do not call GetType() on a value that is already a Type

| Item | Value |
| --- | --- |
| Category | Correctness |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

`GetType()` is called on an expression whose type is already `System.Type` (or a type derived from it).

```csharp
public string Describe(Type type) => type.GetType().Name; // SST2432
```

## Rule description

`GetType()` returns the runtime type of the object it is called on. When the object is itself a `Type`, the
result is not the type that object describes — it is the runtime type of the reflection object, which is the
internal `RuntimeType`. So `type.GetType().Name` is the string `"RuntimeType"` for every input, and
`type.GetType()` is never the type the caller meant.

The mistake is silent: it compiles, runs, and returns a `Type`, just always the wrong one. It shows up most
often in reflection and source-generator support code, where a variable already holds a `Type` and an extra
`.GetType()` slips in.

## How to fix violations

Use the value directly.

```csharp
public string Describe(Type type) => type.Name;
```

The code fix removes the `.GetType()` call.

## Performance

The clean path is a syntax check: only `receiver.GetType()` with no arguments reaches the semantic model.
`System.Type` is resolved once per compilation and the whole rule is gated on it.
