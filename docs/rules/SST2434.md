# SST2434 — Avoid array covariance

| Item | Value |
| --- | --- |
| Category | Correctness |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A reference-type array is implicitly converted to an array of one of its element type's base types.

```csharp
object[] items = new string[3]; // SST2434
```

## Rule description

An array of a reference type converts implicitly to an array of any base type of its element, so a `string[]`
can be handed over as an `object[]`. The conversion compiles with no warning, but it changes what a write
through the reference costs and whether it is safe. The runtime now checks every store against the array's real
element type and throws `ArrayTypeMismatchException` on a mismatch:

```csharp
object[] items = new string[3];
items[0] = 42; // compiles; throws ArrayTypeMismatchException at run time
```

The failure lands at the write, far from the widening that allowed it, and only on the paths that actually
store an incompatible value — so it can sit untriggered until production.

Only that exact shape is reported: an implicit reference conversion whose operand and result are both arrays,
whose element types differ, and whose source element is a reference type. Converting to `IReadOnlyList<T>`,
`IReadOnlyCollection<T>`, `IEnumerable<T>`, or `ReadOnlySpan<T>` is not a conversion to an array type and is
never reported; neither is a value-type element array (`int[]`), which has no reference conversion, nor an
array passed to a `params` parameter.

## How to fix violations

If the receiver only reads the elements, give it a read-only view, which does not allow covariant stores:

```csharp
IReadOnlyList<object> items = new string[3];
```

`ReadOnlySpan<T>` is another read-only option where the target framework has one. If the receiver needs to
write, keep the array typed as its concrete element type so no widening happens:

```csharp
string[] items = new string[3];
```

There is **no code fix**: which replacement is right depends on whether the receiver reads or writes, and only
the author knows that.

## Performance

The rule is an operation action over conversions, and the guard chain rejects a conversion the moment any link
fails — reference conversion, operand an array, result an array, element types differ, source element a
reference type. Only that shape reaches the report. `ReadOnlySpan<T>` is resolved once at compilation start,
and the message mentions it only where it exists.
