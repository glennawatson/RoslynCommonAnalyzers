# SST2435 — Do not use a base value equality as an equality fast path

| Item | Value |
| --- | --- |
| Category | Correctness |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

Inside an `Equals` override, `base.Equals(obj)` is used as an early-out fast path — an
`if (base.Equals(obj)) return true;` guard or a `base.Equals(obj) || ...` short-circuit — when the base
class overrides `Equals` with value semantics.

```csharp
public class Derived : Base
{
    public int Z { get; set; }

    public override bool Equals(object obj)
    {
        if (base.Equals(obj)) // SST2435
        {
            return true;
        }

        return obj is Derived d && d.Z == Z;
    }
}
```

## Rule description

The `if (fast check) return true;` idiom is the reference-equality shortcut, and it is only valid when the
base call really is reference equality — that is, when the base is `object`.

When the base class overrides `Equals` with value semantics, `base.Equals(obj)` returns true whenever the
base fields match. Returning `true` early then reports two instances as equal even when the derived fields
differ, because the derived comparison never runs. Two `Derived` values whose base fields match but whose
`Z` differ compare equal.

The same defect appears as a `base.Equals(obj) || <derived comparison>` short-circuit.

The reference-equality shortcut against an `object` base is left alone; that case belongs to SST1447, which
reports a base equality delegated as the whole result.

## How to fix violations

Combine the base and derived comparisons with `&&`, so both must hold:

```csharp
public override bool Equals(object obj)
    => base.Equals(obj) && obj is Derived d && d.Z == Z;
```

There is no code fix; rewriting an early-out into a combined comparison is a change the author should make
deliberately.

## Performance

The shape is settled syntactically — a `base.Equals` call with one argument, inside an `Equals` method, used
as an early-return-true guard or the left side of `||` — before any bind. Only then is the call bound, to
confirm the base implementation is a real value-equality override.
