# SST2436 — Do not raise an event with a null sender or null args

| Item | Value |
| --- | --- |
| Category | Correctness |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

An instance event is raised with a null sender or null event args.

```csharp
Changed?.Invoke(null, e);    // SST2436 on null
Changed?.Invoke(this, null); // SST2436 on null
```

## Rule description

The two-parameter event pattern hands every subscriber a sender and an event-args object. A handler that reads
either — `((Widget)sender).Name`, `e.PropertyName` — throws a `NullReferenceException` when the value is null,
and the stack trace points at the subscriber rather than the code that raised the event, so the cause is hard
to find and easy to blame on the wrong component.

Pass `this` as the sender of an instance event and `EventArgs.Empty` (or a real event-args instance) as the
args:

```csharp
Changed?.Invoke(this, EventArgs.Empty);
```

The rule reports both the null sender and the null args, in the conditional (`Changed?.Invoke(...)`) and plain
(`Changed.Invoke(...)`) forms, and sees through a `null!` suppression. It leaves alone the cases that are
correct rather than wrong:

- a **static** event, where a null sender is right because there is no instance to name;
- `EventArgs.Empty` as the args, which is the fix, not the bug;
- a delegate that is not the `(object sender, EventArgs args)` shape.

## How to fix violations

A null sender becomes `this`; null args become `EventArgs.Empty`.

```csharp
Changed?.Invoke(this, EventArgs.Empty);
```

The code fix replaces the reported null. It offers the args fix only when the delegate's event-args parameter
is exactly `EventArgs`: a custom args type may have no parameterless construction, and a fix that produced code
which did not compile would be worse than none. In that case, pass a real instance of the args type yourself.

## Performance

The prepass ends analysis for almost every call before the semantic model is touched: the invocation must have
exactly two arguments, name `Invoke`, and pass a syntactic null literal. Only then does the rule bind, confirm
the target is an event-handler `Invoke` whose receiver is an event, and check the exemptions. `System.EventArgs`
is resolved once at compilation start.
