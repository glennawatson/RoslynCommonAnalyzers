# SST2441 - A message template placeholder should name a property

| Item | Value |
| --- | --- |
| Category | Correctness |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A balanced placeholder is empty, whitespace, or holds text that is not a property name.

```csharp
logger.LogInformation("received {} bytes", count);       // SST2441
logger.LogInformation("{Item Count} items", count);      // SST2441
```

## Rule description

A message template placeholder must contain a property name so its value can be captured under that name. A
placeholder that is empty, whitespace, or contains a character a name cannot - a space inside it, punctuation -
captures nothing: the value is dropped from the structured payload and the braces render as literal text in the
message.

The name is read after any leading capturing operator is removed and any alignment or format suffix is set
aside, so these are all accepted:

```csharp
logger.LogInformation("{Count:N0} {Count,5} {@order} {$id}", a, b, c, d);
```

A placeholder whose name is all digits is positional, and is left to the concern that owns positional
placeholders. An unbalanced or stray brace is a separate defect the compiler already reports and is not covered
here.

## How to fix violations

Give the placeholder a property name, or escape the braces as `{{` and `}}` if they were meant to be literal
text.

```csharp
logger.LogInformation("received {Count} bytes", count);
```

There is no code fix, because the intended name cannot be known.

## Performance

The template is parsed in one index scan, only after a syntactic filter passes. The placeholder array is sized
to the count and filled only when the template holds a placeholder, so a message with no placeholders allocates
nothing.
