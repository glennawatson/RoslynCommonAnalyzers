# SST2444 — A regular expression pattern should be valid

| Item | Value |
| --- | --- |
| Category | Correctness |
| Severity | Warning |
| Refactoring | Yes (convert to a source-generated regular expression) |
| Default | Enabled |

## Cause

A regular expression is built from a constant pattern that does not parse.

```csharp
public static readonly Regex Token = new Regex("[a-z"); // SST2444: Unterminated [] set.
```

## Rule description

A regular expression built from a constant pattern that does not parse throws the moment the pattern is
first used, not at build time. A stray bracket, a reversed quantifier, or a backreference to a group the
pattern never defines therefore ships and fails in production on the first input that reaches it.

Rather than reimplement the grammar — which drifts from the real one as options and syntax are added —
the pattern is validated by constructing the real engine with the real options the call passes, and the
engine's own error is reported. Two option bits are handled before construction:

- the compile option is stripped, because it does not change whether the pattern parses, and constructing
  with it would spin up code generation on every keystroke;
- the non-backtracking option is kept, because it legitimately rejects backreferences and lookarounds, so
  a pattern that is valid in the default engine but not under that option is still reported.

The engine never runs a match, so there is no catastrophic backtracking to trigger, and every result is
cached per compilation.

## What is reported

The pattern argument of a construction (`new Regex(...)`) or a static query
(`IsMatch`, `Match`, `Matches`, `Replace`, `Split`, `Count`, `EnumerateMatches`, `EnumerateSplits`), when
the pattern is a constant string. A non-constant pattern, a same-named method on another type, and an
instance query that carries no pattern are never reported.

## How to fix violations

Correct the pattern so it parses. Where the framework supports it (`.NET 7` and C# 11 or later), moving the
pattern to a source-generated regular expression is better still: the pattern is validated by the compiler,
and the matcher is generated ahead of time rather than parsed at run time.

The **refactoring** converts a valid single-literal `new Regex("literal")` into a `[GeneratedRegex]` partial
method and replaces the construction with a call to it:

```csharp
public partial class C
{
    public Regex Build() => TokenRegex();

    [GeneratedRegex("[a-z]+")]
    private static partial Regex TokenRegex();
}
```

It is offered only where the generated-regex attribute resolves and the language version supports it, and
only for a single-literal construction inside a non-generic, non-nested class, struct, or record — the shape
the partial method can be added to without a wider rewrite. A construction that carries options or a timeout,
or a pattern that does not parse, is not converted.

## Performance

The clean path is a token compare and a scan for a string-literal argument; nothing binds until a call names
the engine or one of its static query methods and passes a literal. The engine type is resolved once per
compilation, and validation results are cached against the pattern and its effective options.
