# SST2448 — Delegate subtraction has order-dependent results

| Item | Value |
| --- | --- |
| Category | Correctness |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A combined or opaque delegate is removed with `-` or `-=`, so whether the handlers actually come off
depends on the order they were combined in.

```csharp
Action handlers = first + second;
source -= handlers;            // SST2448: removed only if first, second sit as one contiguous run
source -= first + second;      // SST2448: same combination, inline
var combined = a - b;          // SST2448: composing delegates with '-' is order-dependent
```

## Rule description

Delegate subtraction removes the right-hand invocation list from the left-hand one **only when that
list appears in it as one contiguous run**, and when it appears more than once only the *last* run is
removed. Subtracting a combined delegate therefore either succeeds or silently leaves every handler in
place, and which one happens depends on the order the handlers were added — a fact the call site
cannot see.

Two shapes are reported:

- A binary `a - b` between delegate values. Composing delegates with `-` is rare and its result is
  order-dependent.
- A `target -= value` where the removed `value` is itself a combination: an inline `first + second`,
  a variable this member visibly built with `+` or `+=`, or an opaque delegate-producing expression
  (a call result, a conditional, a coalesce) whose composition the call site cannot see.

### What is not reported

- **The mirror of a single subscription** — a method group, a stored handler read from a field,
  property, parameter, element, or conditional access, a `new EventHandler(Handler)` delegate
  creation, or a cast around any of these. Removing exactly what one `+=` added is well defined.
- **A lambda or anonymous method.** Removing one takes off nothing at all — a different defect, not an
  ordering problem.
- **A combination built elsewhere.** Only a combination visible in the same member is tracked; a field
  combined in another method is left alone.

## How to fix violations

Remove handlers one at a time so each subtraction targets a single, unambiguous invocation list, or
track subscriptions in a collection instead of a combined delegate:

```csharp
source -= first;
source -= second;
```

There is **no code fix**: splitting a combination back into single removals, or moving to a different
storage model, is a change to how the code manages its subscriptions that only the author can make.

## Performance

The clean path stays syntactic. Literals, unary operands, and lambdas are rejected without binding; an
identifier is bound only after a same-member combination of that name has been found on syntax; and
only add/coalesce shapes among binary right-hand sides ever reach the semantic model.
