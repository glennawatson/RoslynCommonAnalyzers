# SST2452 — A pure method should return a value

| Item | Value |
| --- | --- |
| Category | Correctness |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

The pure-contract attribute (`System.Diagnostics.Contracts.PureAttribute`) is applied to a method that
returns nothing a caller could observe — `void`, a bare `Task`, or a bare `ValueTask`.

```csharp
[Pure]
public void Recompute() => _cache = Build(); // SST2452
```

## Rule description

Marking a method `[Pure]` promises it changes no visible state, so calling it is useful *only* for the
value it returns. A `void` method has no value to return, which leaves two possibilities and both are
bugs: either the method really does have side effects, and the attribute misleads every reader and tool
that trusts it, or it truly has none and every call to it is dead code.

A bare `Task` or `ValueTask` is the same contradiction one step removed — awaiting the completion of a
computation that changed nothing and produced nothing is not observable either. `Task<T>` and
`ValueTask<T>` carry a result and stay silent.

### What is not reported

- **A method with a result** — any non-`void` return other than a bare `Task`/`ValueTask`, including
  `Task<T>` and `ValueTask<T>`.
- **A method with an `out` or `ref` parameter.** Its result is observed through that parameter, which
  is how a pure `Deconstruct` produces its values.
- **A same-named attribute from another namespace.** Only the contracts `PureAttribute` is meant; a
  `Pure` attribute a different library declares carries its own meaning and is left alone.

## How to fix violations

Return the computed value, or remove the attribute when the method is genuinely a command:

```csharp
[Pure]
public State Recompute() => Build();
```

The code fix removes the `[Pure]` attribute — the safe, mechanical repair. Turning a command into a
function by giving it a return value is a design change the diagnostic leaves to you.

## Performance

The clean path binds nothing. The return shape and the attribute's simple name are matched on syntax
first; only a full syntactic match binds the attribute — which keeps a same-named attribute from
another namespace silent — and, for the awaitable shapes, the return type, so a user type merely named
`Task` counts as a real result.
