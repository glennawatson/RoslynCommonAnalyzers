## Cause

`System.ComponentModel.DefaultValueAttribute` is applied to a method or record primary-constructor parameter.

```csharp
using System.ComponentModel;

public static void Connect([DefaultValue(5)] int retries) // SST2460
{
}
```

## Rule description

`DefaultValueAttribute` tells designers and serializers what a property's normal value is. On a parameter it
is inert: it does not make the parameter optional, and the compiler never consults it when a call site omits
the argument. A caller must still pass the value, so the attribute records an intention the language does
nothing with.

Paired with `[Optional]` the mistake is silent. The parameter becomes optional, the call compiles with the
argument omitted, and the callee receives `default(T)` â€” not the value in the attribute â€” because the compiler
reads `[Optional]` but ignores `[DefaultValue]`.

```csharp
using System.ComponentModel;
using System.Runtime.InteropServices;

// The caller can omit timeout, and then timeout is 0, not 30.
public static void Connect([Optional, DefaultValue(30)] int timeout)
{
}
```

An attribute that a record positional parameter retargets to the generated property or field with
`[property:]` or `[field:]` reaches a home that really does read it, and is not reported.

## How to fix violations

Write the default into the parameter's declaration:

```csharp
public static void Connect(int retries = 5)
{
}
```

Or, for the interop shape, use `System.Runtime.InteropServices.DefaultParameterValueAttribute` (paired with
`[Optional]`), which the compiler does honour:

```csharp
using System.Runtime.InteropServices;

public static void Connect([Optional, DefaultParameterValue(30)] int timeout)
{
}
```

The code fix swaps `[DefaultValue]` for `[DefaultParameterValue]`, keeping the argument and the suffix spelling
and qualifying the name when `System.Runtime.InteropServices` is not imported. It is offered only for the
single-value form whose value can be the parameter's default; the two-argument converter form
(`[DefaultValue(typeof(T), "...")]`) and a value the parameter type cannot hold are left for the author.

## Performance

The clean path is a syntax check: a parameter with no attribute lists returns before the semantic model is
touched, and only an attribute spelled `DefaultValue`/`DefaultValueAttribute` that lands on the parameter is
bound. `DefaultValueAttribute` is resolved once per compilation and the whole rule is gated on it.
