# SST2468 — A partial method with no implementation is silently discarded

| Item | Value |
| --- | --- |
| Category | Correctness |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A classic partial method — a `partial void` defining declaration with no accessibility modifier, no return
value, and no `out` parameters — has no implementing declaration anywhere in the compilation.

```csharp
public partial class C
{
    partial void OnChanged(); // SST2468: never implemented

    public void Set() => OnChanged(); // erased by the compiler; nothing runs
}
```

## Rule description

A classic partial method is not required to have a body. When no implementing declaration is written, the
C# compiler silently deletes the declaration and rewrites away every call to it — no error, no warning. A
hook that was declared and then never implemented is almost always a mistake: the author expected the call
to run something, but because the body was never written each call is erased and does nothing. The bug is
invisible at the call site, which still reads as if it invokes real work.

## What this rule reports

Only the classic form, declared and left unimplemented:

```csharp
partial void OnChanged();              // reported
static partial void OnChanged();       // reported
partial void OnChanged(int value);     // reported
partial void OnChanged(ref int value); // reported
```

An implemented classic partial method is correct and is never reported:

```csharp
partial void OnChanged();
partial void OnChanged() { } // the implementation makes the call run
```

## What this rule leaves to the compiler

A partial method that carries an accessibility modifier, returns a value, or takes an `out` parameter is an
*extended* partial method: the language requires it to have an implementation, and the compiler already
reports the missing body as an error. Those shapes are the compiler's to enforce and are never reported here.

| shape | why it is not reported |
| --- | --- |
| `private partial void OnChanged();` (no body) | a missing implementation is already a compiler error |
| `private partial int Compute();` (no body) | a value-returning partial must be implemented |
| `private partial void TryGet(out int value);` (no body) | a partial with an `out` parameter must be implemented |
| an implemented classic partial | the implementation makes the call run |
| an ordinary method | not a partial method |

## How to fix violations

Either write the implementing declaration so the hook runs, or delete the declaration and its call sites if
the hook is not wanted. There is no code fix, because removing the declaration also requires removing every
call to it, and adding a body requires knowing what it should do.

## Performance

The clean path is a syntactic prepass over each method declaration: it must be a bare signature terminated
by a semicolon, return `void`, take no `out` parameter, and carry the `partial` modifier with none of the
modifiers that make a partial method extended. Only a declaration that survives that prepass binds, and it is
reported only when the semantic model confirms it is a partial definition with no implementation part.
