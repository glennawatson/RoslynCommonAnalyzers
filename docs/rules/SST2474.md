# SST2474 — A part-creation-policy attribute is meaningless without an export

| Item | Value |
| --- | --- |
| Category | Correctness |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A part-creation-policy attribute is applied to a type that has no export. The attribute only tells the
container how to share instances of an exported part, so on a type the container never creates it governs
nothing.

```csharp
using System.ComponentModel.Composition;

[PartCreationPolicy(CreationPolicy.Shared)] // SST2474: no [Export], so the policy governs nothing
public class Orphan { }
```

## Rule description

A creation-policy attribute is a directive to the container about the lifetime of the instances it
creates for an export. If the type is not exported, the container never creates it, and the attribute has
no effect — it reads as though the type participates in composition when it does not. The attribute is
usually left over from an export that was removed, or it signals a missing export that was meant to be
there. Either way the code says one thing and does another.

Both flavors are covered:

- `System.ComponentModel.Composition` — `[PartCreationPolicy(...)]` with no `[Export]`.
- `System.Composition` — `[Shared]` with no `[Export]`.

```csharp
using System.Composition;

[Shared] // SST2474: a sharing boundary with nothing exported to share
public class Orphan { }
```

## What this rule reports

Only a creation-policy or shared attribute on a type that carries no MEF export. These are never
reported:

| shape | why it is fine |
| --- | --- |
| `[Export(typeof(IService))] [PartCreationPolicy(CreationPolicy.Shared)] class Service` | the policy governs a real export |
| `[Export] [Shared] class Service` | the sharing boundary applies to an exported part |
| `[PartCreationPolicy(...)]` on a type in a project that references no MEF assembly | the rule is gated off |

## How to fix violations

Remove the attribute, or add the export it was meant to accompany:

```csharp
// remove the orphaned policy
public class Orphan { }

// or add the export the policy belongs to
[Export(typeof(IService))]
[PartCreationPolicy(CreationPolicy.Shared)]
public class Service : IService { }
```

## Performance

The rule is gated at compilation start on a MEF marker attribute resolving. On the hot path each
attribute is rejected on its written simple name before any binding; only an attribute named like a
creation policy is bound, and the type's export is checked only once that binding confirms the marker.
