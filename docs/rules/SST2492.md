# SST2492 — Do not throw for null on a parameter the signature allows to be null

| Item | Value |
| --- | --- |
| Category | Correctness |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A null guard rejects a value the signature says is allowed to be null: a nullable-annotated reference
parameter, or an optional parameter that defaults to null.

```csharp
public void Configure(Options? options) // null is a legal argument
{
    ArgumentNullException.ThrowIfNull(options); // SST2492 — but the signature invited null
}
```

## Rule description

A signature is a promise to callers. `Options?` and `Options options = null` both say the same thing:
null is a legal argument. A guard that throws when the argument is null breaks that promise at runtime —
for exactly the value the signature invited. One of the two is wrong, and the diagnostic asks which.

The two guard shapes it recognizes are the hand-written throw and the runtime helper:

```csharp
if (options is null) throw new ArgumentNullException(nameof(options)); // reported
ArgumentNullException.ThrowIfNull(options);                            // reported
```

### What is reported

- A null-throwing guard — `if (p is null)`/`if (p == null)` throwing `ArgumentNullException`, or
  `ArgumentNullException.ThrowIfNull(p)` — where `p` is a parameter whose contract permits null:
  - a nullable-annotated reference parameter (`T?`) or unconstrained type parameter, or
  - an optional parameter whose default value is null.

### What is not reported

- A guard on a non-nullable reference parameter. The signature and the guard agree there.
- A guard on a local, a field, or any expression that is not a parameter. The contradiction is between a
  guard and a *signature*, so only parameters are examined.

## How to fix violations

Decide what the parameter's contract should be.

If null is not actually allowed, tighten the signature so callers are warned at compile time and the
guard becomes redundant:

```csharp
public void Configure(Options options) // drop the '?' (or the '= null')
{
}
```

If null is allowed, remove the throw and handle the null value.

There is **no code fix**. The two repairs point in opposite directions — one changes the contract for
every caller, the other changes what this method does with null — and choosing between them is the
decision the diagnostic is asking for.

## Performance

The rule registers on `if` statements and invocations. Both guard shapes are matched syntactically — the
hand-written form with the shared guard-pattern matcher, the helper form by method name and shape —
before any binding. Only a matched guard binds: the checked expression to a parameter symbol, then that
parameter's nullability contract.
