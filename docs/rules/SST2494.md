# SST2494 — Null-coalescing over a constant null always takes the right operand

| Item | Value |
| --- | --- |
| Category | Correctness |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

The left operand of a `??` is a compile-time constant null, so the coalescing can never keep it and the
right operand is taken every time.

```csharp
const string? Fallback = null;
var name = Fallback ?? "anonymous"; // SST2494 — always "anonymous"
```

## Rule description

`??` reads as a fallback: keep the left unless it is null, otherwise use the right. When the left is a
compile-time constant null — the null literal, a `default` of a reference or nullable type, or a constant
whose value is null — the "unless" can never fire. The operator guards nothing, and the expression is
equivalent to its right operand. Usually the left operand is the mistake: a constant that was meant to
hold a real value, or a stray `default`.

The fix folds the expression to its right operand:

```csharp
var name = "anonymous";
```

If the left was supposed to be a live value, the real repair is to give it one.

### What is reported

- `a ?? b` where `a` is a compile-time constant null: the null literal, `default`/`default(T)` of a
  reference or nullable type, or a constant whose value is null (including through a cast or
  parentheses).

### What is not reported

- Any `??` whose left operand is not a compile-time constant — a variable, a field, a call — even if it
  happens to be null at run time. Whether such a value is null is a flow question, not a constant one.
- `a ??= b`: its left operand must be an assignable location, which a compile-time constant null can
  never be, so the shape cannot occur in code that compiles.

## How to fix violations

The fix replaces the whole `a ?? b` with `b`, the operand it always evaluated to. Because the right
operand was already evaluated on every execution, folding to it changes nothing about behavior.

## Performance

The rule registers on the coalescing expression. The clean path rejects on syntax: a left operand whose
shape cannot be a constant — a call, a `new`, an index — is dropped without binding. Only a
constant-shaped left operand is asked for its constant value.
