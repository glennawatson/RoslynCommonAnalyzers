# SST2495 — A flags combination should not repeat bits another operand already sets

| Item | Value |
| --- | --- |
| Category | Correctness |
| Severity | Warning |
| Code fix | Yes |
| Default | Enabled |

## Cause

A `|` combination of `[Flags]` enum members includes an operand whose bits are all already set by another
operand in the same expression, so it contributes nothing.

```csharp
[Flags]
enum Access { Read = 1, Write = 2, ReadWrite = Read | Write }

var access = Access.ReadWrite | Access.Read; // SST2495 — Read adds nothing
```

## Rule description

Every operand of the combination has a compile-time constant value, so the redundancy is proven, not
guessed: `Access.Read` sets bit 0, and `Access.ReadWrite` already sets bit 0, so or-ing `Read` in leaves
the result identical. The repeated operand is usually a symptom — a composite member listed beside a flag
it already contains, or a copy-paste of the wrong member — and the intent was a different flag.

Removing the operand that adds nothing keeps the value the same and makes the combination say what it
computes:

```csharp
var access = Access.ReadWrite;
```

If a different flag was meant, that is the real fix.

### What is reported

- Each operand of a `|` combination over a `[Flags]` enum whose constant bits are a subset of another
  operand's constant bits in the same expression. When two operands set the same bits, only the later one
  is reported, so removing the reported operands leaves the combined value unchanged.

### What is not reported

- A combination of operands that each add bits of their own — an ordinary flag set.
- An operand whose value is not a compile-time constant: the subset relationship cannot be proven, so it
  is left alone.
- A `|` on an enum that is not `[Flags]`, which is a different mistake, and bitwise work on integers.

## How to fix violations

The fix removes the redundant operand together with its adjacent `|`, replacing that operand's `|`
operation with whichever side survives. The remaining operands compute the same value.

## Performance

Only the outermost operation of a `|` chain is examined, and the clean path binds one operand at most: a
result type that is not a `[Flags]` enum settles the whole expression after a single bind. Only a
combination that clears that has its operands' constant values read.
