# SST2496 — Do not dispose a value an enclosing using already disposes

| Item | Value |
| --- | --- |
| Category | Correctness |
| Severity | Info |
| Code fix | Yes |
| Default | Enabled |

## Cause

A local governed by a `using` statement or a `using` declaration is disposed again by an explicit
`Dispose()` or `Close()` call inside the same scope.

```csharp
using var stream = File.OpenRead(path);
Read(stream);
stream.Dispose(); // SST2496 — the using already disposes it
```

## Rule description

A `using` disposes the value when its scope ends, so an explicit `Dispose()` or `Close()` on the same
local runs a second disposal. Many types tolerate a repeat disposal, but the contract does not require
it, and the double call muddies ownership: it reads as though the explicit call is responsible when the
`using` already is. Removing the explicit call leaves disposal to the `using`, where it runs exactly
once and the ownership is clear:

```csharp
using var stream = File.OpenRead(path);
Read(stream);
```

Reported as a suggestion because a repeat disposal is usually harmless — the problem is the muddled
ownership, not a crash.

### What is reported

- A parameterless `Dispose()` or `Close()` on a local whose declaration carries the `using` keyword —
  either a `using` declaration or a `using (...)` statement — inside that using's scope.

### What is not reported

- Disposal of a plain local that no `using` governs: the explicit call is the only disposal there.
- Disposal woven into a larger expression rather than made as its own statement.

## How to fix violations

The fix removes the redundant call statement, leaving the `using` to dispose the value. Only a call that
is its own expression statement is removed; a disposal woven into a larger expression is left for the
author.

## Performance

The rule registers on invocations. The clean path is syntactic: only a parameterless `Dispose`/`Close`
member call on a bare identifier is considered, and everything else is dropped before binding. A
candidate binds its receiver once and reports only when it is a local whose declaration carries the
`using` keyword.
