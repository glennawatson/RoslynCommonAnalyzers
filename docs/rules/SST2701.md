# SST2701 — A [JSInvokable] method must be public

| Item | Value |
| --- | --- |
| Category | Frameworks |
| Severity | Warning |
| Code fix | Yes — makes the method public (supports Fix All) |
| Default | Enabled |

## Cause

A method is annotated `[JSInvokable]` but is not `public`.

```csharp
using Microsoft.JSInterop;

public class Interop
{
    [JSInvokable]
    private void OnResize() { }
}
```

## Rule description

JavaScript interop calls a .NET method by name, over the interop boundary, through reflection. That lookup
only finds a **public** method. A method that is private, internal, or protected still compiles and still
carries the `[JSInvokable]` attribute, so the code reads as if JavaScript can call it — but at runtime the call
fails to resolve the moment it is made. The failure is deferred to the browser, not surfaced at build time,
which is exactly the kind of gap this rule closes: the attribute advertises an entry point that the
accessibility contradicts.

### What is reported

An ordinary method carrying `[JSInvokable]` whose declared accessibility is anything other than `public` —
`private` (including the implicit default), `internal`, `protected`, `protected internal`, or
`private protected`. Static and instance methods are treated the same, because both are reached the same way.

### What is not reported

- A `public` method with `[JSInvokable]` — it is reachable, which is the whole point.
- A non-public method **without** `[JSInvokable]` — it makes no interop promise.
- Anything in a project that does not reference the JavaScript-interop assembly. The rule resolves the
  `[JSInvokable]` attribute type in the compilation and stays completely silent when it is absent, so a
  look-alike attribute of the same name in another namespace is never mistaken for it.

## How to fix

Make the method `public`. The code fix does this and supports Fix All across a document, project, or solution.

```csharp
using Microsoft.JSInterop;

public class Interop
{
    [JSInvokable]
    public void OnResize() { }
}
```

A method that explicitly implements an interface carries no accessibility modifier to change, so no fix is
offered there — `[JSInvokable]` belongs on an ordinary public method, not an explicit implementation.

## Configuration

None.
