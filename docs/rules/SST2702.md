# SST2702 ‚Äî A [SupplyParameterFromQuery] property must be a query-bindable type

| Item | Value |
| --- | --- |
| Category | Frameworks |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A property is annotated `[SupplyParameterFromQuery]` but its type is one the framework cannot bind from the
query string.

```csharp
using Microsoft.AspNetCore.Components;

public class SearchPage
{
    [SupplyParameterFromQuery]
    public SortOrder Sort { get; set; }   // an enum ‚Äî not bindable
}

public enum SortOrder { Ascending, Descending }
```

## Rule description

A property bound from the query string is filled by parsing text into the property's type. The framework can
only parse a fixed set of types; a property of any other type compiles cleanly and then **throws at runtime**
the moment the component is navigated to and the framework tries to bind the value. Because the failure only
appears on the request that carries that query parameter, it is easy to ship.

The supported set is:

- `bool`
- the numeric primitives: `byte`, `sbyte`, `short`, `ushort`, `int`, `uint`, `long`, `ulong`, `float`,
  `double`, `decimal`
- `Guid`
- `string`
- `DateTime`, `DateTimeOffset`, `DateOnly`, `TimeOnly`
- the `Nullable<>` form of any of the above (for example `int?`)
- a single-dimension array of any of the above (for example `int[]` or `string[]`)

Each supported type is resolved against the compilation, so on a target framework that predates `DateOnly`,
`TimeOnly`, or `DateTimeOffset` those simply never count as supported ‚Äî the rule never points at a type the
project cannot use.

### What is reported

A `[SupplyParameterFromQuery]` property whose type ‚Äî after unwrapping a single-dimension array and a
`Nullable<>` wrapper ‚Äî is outside the supported set: an enum, a custom type, `char`, `TimeSpan`, `Uri`, a
multi-dimensional array (`int[,]`), a jagged array (`int[][]`), and so on. The property need not be public; the
framework binds it by reflection regardless of accessibility, so accessibility is not part of the check.

### What is not reported

- A property whose type is in the supported set, or its nullable or single-dimension array form.
- A property without the attribute.
- A property whose type does not resolve ‚Äî a broken type is the compiler's to flag, not this rule's.
- Anything in a project that does not reference the components assembly; the rule resolves the attribute type
  and stays silent when it is absent.

## How to fix

Expose the value as a supported type and convert it yourself where you use it.

```csharp
using Microsoft.AspNetCore.Components;

public class SearchPage
{
    [SupplyParameterFromQuery]
    public string Sort { get; set; }

    private SortOrder Order =>
        Enum.TryParse<SortOrder>(Sort, ignoreCase: true, out var value) ? value : SortOrder.Ascending;
}
```
