# SST2705 — A bound model member should be marked required or made nullable

| Item | Value |
| --- | --- |
| Category | Frameworks |
| Severity | Warning |
| Code fix | No |
| Default | Disabled (opt-in) |

## Cause

A model bound from the body of an `[ApiController]` action exposes a public settable member whose type is a
non-nullable value type, with no required marker.

```csharp
public class OrderRequest
{
    public int Quantity { get; set; } // SST2705
}

[ApiController]
public class OrdersController : ControllerBase
{
    [HttpPost]
    public IActionResult Create(OrderRequest request) => Ok();
}
```

## Rule description

Model binding fills a body-bound model from the incoming payload. A public settable property or public field whose
type is a non-nullable value type — `int`, `bool`, `decimal`, an enum, `Guid`, and so on — cannot represent
"absent": when the request omits the field, binding silently leaves it at its default (`0`, `false`, the zero
enum member), and the action cannot tell a supplied default from a missing value. This is the **under-posting**
defect — a client that forgets a field, or a malicious client that drops one, gets default behaviour instead of a
validation error.

Two shapes are already safe and are not reported:

```csharp
public int? Quantity { get; set; }   // nullable — absence is observable as null
[Required] public int Quantity { get; set; } // an omitted value fails validation
```

`[BindRequired]` is honoured the same way as `[Required]`.

Because a default value is frequently *exactly* what an API intends (an optional `int Page` that defaults to `0`,
a `bool` flag that defaults to `false`), this shape produces real false positives. **The rule is therefore
disabled by default** and must be opted into:

```ini
# .editorconfig
dotnet_diagnostic.SST2705.severity = warning
```

### Scope

To keep precision high when it is enabled, the rule is tightly scoped:

- Only members of a **source-declared model class** bound directly as an action parameter are considered — a model
  from a referenced assembly, which you cannot edit, is never reported.
- Only a parameter bound from the request body counts: an explicit `[FromBody]` parameter, or (under
  `[ApiController]`) a complex-type parameter that infers body binding. A parameter routed elsewhere with
  `[FromQuery]`, `[FromRoute]`, `[FromForm]`, `[FromHeader]`, or `[FromServices]` is out of scope, as is a
  simple-type parameter and a collection.
- Only public members with a public setter (or a public, non-`readonly`, non-`const` field) are reported.

### Availability detection

The rule is gated on `Microsoft.AspNetCore.Mvc.ApiControllerAttribute` and
`Microsoft.AspNetCore.Mvc.ControllerBase` resolving in the referenced framework (detected once per compilation). A
non-web project reports nothing.

## How to fix violations

Pick the option that matches the member's meaning:

```csharp
// The field is mandatory — require it, so an omission is a 400 rather than a silent default.
[Required] public int Quantity { get; set; }

// The field is optional and its absence is meaningful — make it nullable and handle null.
public int? Quantity { get; set; }
```

**There is no automatic code fix**, because choosing between "require it" and "make it nullable" is a decision
about the API contract, not a mechanical rewrite.
