# SST2708 - Unsubscribe a lifecycle event subscription

| Item | Value |
| --- | --- |
| Category | Frameworks |
| Severity | Warning |
| Code fix | No |
| Default | Enabled |

## Cause

A component hooks a .NET event with `+=` inside a render-lifecycle method (`OnInitialized`,
`OnParametersSet`, `OnAfterRender` and their async forms) and never removes that subscription.

```csharp
using System;
using Microsoft.AspNetCore.Components;

public class Clock : ComponentBase
{
    private readonly TimeService _time = new();

    protected override void OnInitialized()
    {
        _time.Tick += OnTick; // SST2708
    }

    private void OnTick(object? sender, EventArgs e) => StateHasChanged();
}
```

## Rule description

Subscribing to an event stores a delegate that closes over the component in the event source. The
source - an injected service, a static, a singleton - usually outlives the component, so that delegate
roots the whole component graph for as long as the source lives. Nothing collects the component: on a
server-rendered circuit every render or navigation leaves one more instance behind, and memory climbs
for the life of the source.

Only a `source.Event += handler` whose receiver is not `this`/`base` is considered - subscribing to
your own event roots nothing external - and the left side must bind to an event, so a plain
delegate-field `+=` is left alone. A subscription is treated as handled when a matching
`source.Event -= handler` for the same event appears anywhere in the component, which is where an
unsubscribe in `Dispose`/`DisposeAsync` clears it.

This is the complement to the disposable-ownership rules (SST2315, SST2316): those watch a resource a
type creates and never disposes; this watches the event handoff those rules never see.

## How to fix violations

Remove the subscription with a matching `-=` for the same event, normally from `Dispose` or
`DisposeAsync`, and make the component implement the matching interface so the removal actually runs.

```csharp
using System;
using Microsoft.AspNetCore.Components;

public class Clock : ComponentBase, IDisposable
{
    private readonly TimeService _time = new();

    protected override void OnInitialized()
    {
        _time.Tick += OnTick;
    }

    public void Dispose()
    {
        _time.Tick -= OnTick;
    }

    private void OnTick(object? sender, EventArgs e) => StateHasChanged();
}
```

## When not to use the rule

If the event source is guaranteed to have exactly the component's lifetime - and is collected together
with it - the subscription cannot outlive the component, so the diagnostic is a false positive there.
Unsubscribing anyway is still the safer habit.
