# SST2711 — A synchronous component lifecycle override must not be async void

| Item | Value |
| --- | --- |
| Category | Frameworks |
| Severity | Warning |
| Code fix | Yes — overrides the Task-returning twin instead (supports Fix All) |
| Default | Enabled |

## Cause

A component overrides one of the synchronous lifecycle methods — `OnInitialized`, `OnParametersSet`, or
`OnAfterRender` — and declares it `async void`.

```csharp
using System.Threading.Tasks;
using Microsoft.AspNetCore.Components;

public class Counter : ComponentBase
{
    protected override async void OnInitialized() => await LoadAsync();

    private static Task LoadAsync() => Task.CompletedTask;
}
```

## Rule description

The component runtime calls a synchronous lifecycle method and moves straight on — there is no `Task` for it to
await. Declaring one of them `async void` turns it into fire-and-forget: control returns to the runtime before
the awaited work finishes, and any exception thrown after the first `await` has nowhere to surface. On an
interactive server circuit that unobserved exception tears the circuit down and drops the whole session, so a
transient failure in initialization becomes a lost connection.

Each of these methods has an `...Async` twin that returns `Task` and that the runtime **does** await. Overriding
the twin sequences the work into the component's lifecycle and lets its failures be observed and handled.

### What is reported

An `override` of `OnInitialized`, `OnParametersSet`, or `OnAfterRender` that is declared `async void`, where the
overridden method is the one defined on `ComponentBase`. The override chain is checked, so a same-named method on
an unrelated base type is never mistaken for the framework hook.

### What is not reported

- An override of the `...Async` twin (`OnInitializedAsync` and the rest) — the correct, awaited shape.
- A plain synchronous `override void OnInitialized()` — it makes no `async` promise.
- An `async void` method that is not one of the three lifecycle hooks, or is not an override of `ComponentBase`.
- Anything in a project that does not reference the components assembly. The rule resolves `ComponentBase` in the
  compilation and stays completely silent when it is absent.

## How to fix

Override the `...Async` twin and return `Task`. The code fix renames the method to the twin and changes the
return type to `Task`, leaving the body unchanged. It supports Fix All across a document, project, or solution.

```csharp
using System.Threading.Tasks;
using Microsoft.AspNetCore.Components;

public class Counter : ComponentBase
{
    protected override async Task OnInitializedAsync() => await LoadAsync();

    private static Task LoadAsync() => Task.CompletedTask;
}
```

No fix is offered when the type already declares a member with the twin name, because the rename would collide;
resolve the duplicate first.

## Configuration

None.
