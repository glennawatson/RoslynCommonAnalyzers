# SST2712 — An injected or cascading component property must have a setter

| Item | Value |
| --- | --- |
| Category | Frameworks |
| Severity | Warning |
| Code fix | Yes — adds a `private set;` to the auto-property (supports Fix All) |
| Default | Enabled |

## Cause

A property marked `[Inject]` or `[CascadingParameter]` has no setter — it is get-only or expression-bodied.

```csharp
using Microsoft.AspNetCore.Components;

public class Widget
{
    [Inject]
    public IClock Clock { get; }
}
```

## Rule description

The runtime fills an `[Inject]` or `[CascadingParameter]` property by assigning it through reflection over the
component's settable properties. A property with no setter — get-only, or expression-bodied — is invisible to
that pass, so the runtime never assigns it and it keeps its default value. The member reads as if it is wired up,
but it is `null` at first use, and the component throws a `NullReferenceException` the moment it touches the
dependency. Because the property compiles and the attribute is present, nothing flags the gap until the code
runs.

### What is reported

An instance property carrying `[Inject]` or `[CascadingParameter]` that has no setter: a get-only auto-property
or an expression-bodied property. Static properties and indexers are not reported, because the runtime does not
bind them.

### What is not reported

- A property with any setter, including `private set` and `init` — the runtime can assign it.
- A property without either marker — it makes no injection promise.
- Anything in a project that references neither the injection nor the cascading attribute. The rule resolves both
  attribute types in the compilation and stays completely silent when neither is present, so a look-alike
  attribute of the same name in another namespace is never mistaken for one.

Note that `[Inject]` on a *field* is a separate compiler error and is not this rule's concern; the setter-less
property is the gap this rule closes.

## How to fix

Give the property a setter. A `private set` keeps it encapsulated while still letting the runtime bind it. The
code fix adds a `private set;` to a get-only auto-property and supports Fix All across a document, project, or
solution.

```csharp
using Microsoft.AspNetCore.Components;

public class Widget
{
    [Inject]
    public IClock Clock { get; private set; }
}
```

No fix is offered for an expression-bodied property: it has no accessor list to extend and no backing field to
assign, so turning it into a settable property is a design change to make by hand.

## Configuration

None.
